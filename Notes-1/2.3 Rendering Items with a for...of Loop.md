# 7. Reading Items with a `for...of` Loop

## 7.1 What You Will Learn

A `for...of` loop gives direct access to the values inside an iterable collection such as an array.

Basic syntax:

```js
for (const product of products) {
  console.log(product);
}
```

Unlike a standard `for` loop, you do not manually manage:

* The starting index.
* The loop condition.
* The counter increment.

JavaScript handles those details for you.

---

# 7.2 Basic Syntax

```js
for (const item of items) {
  console.log(item);
}
```

General structure:

```js
for (const currentValue of collection) {
  // Use currentValue
}
```

Example:

```js
const itemNames = [
  "Laptop",
  "Phone",
  "Keyboard",
];

for (const itemName of itemNames) {
  console.log(itemName);
}
```

Expected output:

```text
Laptop
Phone
Keyboard
```

---

# 7.3 Syntax Breakdown

Consider:

```js
for (const product of products) {
  console.log(product.name);
}
```

## `products`

The array being looped over.

```js
const products = [
  {
    id: 1,
    name: "Laptop",
  },
];
```

## `product`

A variable that represents the current item during each repetition.

First repetition:

```js
product = products[0];
```

Second repetition:

```js
product = products[1];
```

## `of`

The `of` keyword means:

```text
Give me each value from this collection.
```

## Loop body

```js
console.log(product.name);
```

This code runs once for every product.

---

# 7.4 Why `const` Is Used

Inside:

```js
for (const product of products)
```

the variable `product` is newly assigned during each iteration.

You normally do not reassign the variable itself.

Therefore, `const` is appropriate.

Correct:

```js
for (const product of products) {
  console.log(product.name);
}
```

You can still modify the object referenced by `product`:

```js
for (const product of products) {
  product.quantity += 1;
}
```

However, doing this inside a render function would mutate state and should usually be avoided.

---

# 7.5 `for...of` Gives Values Directly

Standard `for` loop:

```js
for (
  let index = 0;
  index < products.length;
  index++
) {
  const product = products[index];

  console.log(product.name);
}
```

`for...of` loop:

```js
for (const product of products) {
  console.log(product.name);
}
```

Both read the same values.

The second version is shorter because the current item is provided directly.

---

# 7.6 `for...of` Does Not Give the Index Automatically

This loop provides only the value:

```js
for (const product of products) {
  console.log(product);
}
```

If you need a display number, create a counter.

```js
let displayNumber = 1;

for (const product of products) {
  console.log(
    `${displayNumber}. ${product.name}`
  );

  displayNumber++;
}
```

Expected output:

```text
1. Laptop
2. Phone
3. Keyboard
```

---

# 7.7 Using `entries()` for Index and Value

Arrays provide an `entries()` method.

```js
for (const [index, product] of products.entries()) {
  console.log(index);
  console.log(product.name);
}
```

Expected output:

```text
0
Laptop
1
Phone
2
Keyboard
```

You can display user-friendly numbering:

```js
for (const [index, product] of products.entries()) {
  console.log(
    `${index + 1}. ${product.name}`
  );
}
```

This lesson focuses on plain loops, so `entries()` is introduced only as a useful helper.

# 7.8 Rendering into the DOM

HTML:

```html
<ul id="product-list"></ul>
```

```js
const productList =
  document.getElementById("product-list");

for (const product of products) {
  const listItem =
    document.createElement("li");

  listItem.textContent = product.name;

  productList.appendChild(listItem);
}
```

Expected DOM:

```text
Laptop
Phone
Keyboard
```

# 7.9 Reusable Render Function

```js
function renderProducts() {
  productList.innerHTML = "";

  for (const product of products) {
    const listItem =
      document.createElement("li");

    listItem.textContent = product.name;

    productList.appendChild(listItem);
  }
}
```

Call it:

```js
renderProducts();
```

# 7.10 Empty-State Handling

```js
function renderProducts() {
  productList.innerHTML = "";

  if (products.length === 0) {
    const emptyMessage =
      document.createElement("li");

    emptyMessage.textContent =
      "No products available.";

    productList.appendChild(emptyMessage);

    return;
  }

  for (const product of products) {
    const listItem =
      document.createElement("li");

    listItem.textContent = product.name;

    productList.appendChild(listItem);
  }
}
```

Expected output for an empty array:

```text
No products available.
```

---

# 7.11 Adding Display Numbering

Use a separate counter:

```js
let displayNumber = 1;

for (const product of products) {
  const listItem =
    document.createElement("li");

  listItem.textContent =
    `${displayNumber}. ${product.name}`;

  productList.appendChild(listItem);

  displayNumber++;
}
```

The display counter is not the product ID.

Example:

```js
{
  id: 205,
  name: "Phone"
}
```

May display as:

```text
2. Phone
```

while its stable ID remains:

```text
205
```

---

# 7.12 Storing Stable IDs

Inside the loop:

```js
listItem.dataset.id = product.id;
```

Generated HTML:

```html
<li data-id="205">
  2. Phone
</li>
```

Later, the ID helps identify the correct product during Update and Delete operations.

# 7.13 Adding CSS Classes

```js
listItem.classList.add("product-item");
```

Generated HTML:

```html
<li class="product-item"></li>
```

Conditional class:

```js
if (product.quantity === 0) {
  listItem.classList.add("out-of-stock");
}
```

Generated HTML:

```html
<li class="product-item out-of-stock"></li>
```

# 7.14 Conditional Rendering

```js
for (const product of products) {
  const status =
    document.createElement("p");

  if (product.quantity === 0) {
    status.textContent = "Out of stock";
  } else {
    status.textContent = "In stock";
  }
}
```

The condition affects the DOM output.

The product object remains unchanged.

# 7.15 Skipping Items with `continue`

State:

```js
const products = [
  {
    id: 1,
    name: "Laptop",
    isActive: true,
  },
  {
    id: 2,
    name: "Phone",
    isActive: false,
  },
  {
    id: 3,
    name: "Keyboard",
    isActive: true,
  },
];
```

Loop:

```js
for (const product of products) {
  if (!product.isActive) {
    continue;
  }

  console.log(product.name);
}
```

Expected output:

```text
Laptop
Keyboard
```

`continue` skips the remaining code for the current item and moves to the next item.

---

# 7.16 Stopping Early with `break`

```js
for (const product of products) {
  if (product.id === selectedId) {
    console.log(product);

    break;
  }
}
```

This stops after the matching product is found.

This pattern will later be used for Update operations.

# 7.17 `for...of` Versus `for...in`

These two loops are different.

## `for...of`

Used to access values.

```js
for (const product of products) {
  console.log(product.name);
}
```

## `for...in`

Used to access property keys or array indexes.

```js
for (const index in products) {
  console.log(index);
}
```

For:

```js
const products = [
  "Laptop",
  "Phone",
];
```

`for...in` outputs:

```text
0
1
```

`for...of` outputs:

```text
Laptop
Phone
```

> Use `for...of` when you want array values. Do not confuse it with `for...in`.

---

# 7.18 Why `for...in` Is Usually Avoided for Arrays

`for...in` is designed mainly for enumerable property keys.

```js
for (const key in object) {
  console.log(key);
}
```

For arrays, it returns string-like index keys.

This can create confusion.

Prefer:

```js
for (const item of items)
```

when you need values.

Prefer a standard `for` loop when you need precise numeric index control.

# 7.21 Comparing All Three Plain Loops

## Standard `for`

```js
for (
  let index = 0;
  index < products.length;
  index++
) {
  const product = products[index];

  console.log(product.name);
}
```

### Strengths

* Direct index access.
* Easy reverse iteration.
* Easy custom step size.
* Precise control.

### Weaknesses

* More syntax.
* Easier to make index mistakes.

---

## `while`

```js
let index = 0;

while (index < products.length) {
  const product = products[index];

  console.log(product.name);

  index++;
}
```

### Strengths

* Flexible condition control.
* Useful when repetition count is not known in advance.

### Weaknesses

* Easy to forget the counter update.
* Higher infinite-loop risk.

---

## `for...of`

```js
for (const product of products) {
  console.log(product.name);
}
```

### Strengths

* Very readable.
* Direct value access.
* Less loop-management code.
* Supports `break` and `continue`.

### Weaknesses

* No automatic numeric index.
* Less convenient for custom stepping or reverse iteration.

---

# 7.22 Which Plain Loop Should You Use?

Use a standard `for` loop when:

* You need the index.
* You need reverse iteration.
* You need a custom increment.
* You need exact loop control.

Use a `while` loop when:

* The loop depends mainly on a condition.
* The number of repetitions is not known in advance.
* Counter logic is unusual.

Use `for...of` when:

* You only need array values.
* Readability matters.
* You want to use `break` or `continue`.
* You do not need detailed index control.

# 7.23 Common Beginner Mistakes

## Mistake 1: Using `for...in`

Incorrect for direct values:

```js
for (const product in products) {
  console.log(product.name);
}
```

`product` is an index key, not the object.

Correct:

```js
for (const product of products) {
  console.log(product.name);
}
```

---

## Mistake 2: Expecting an Automatic Index

Incorrect:

```js
for (const product of products) {
  console.log(index);
}
```

No `index` variable exists unless you create one.

---

## Mistake 3: Forgetting Display Counter Increment

```js
let number = 1;

for (const product of products) {
  console.log(
    `${number}. ${product.name}`
  );
}
```

Every product displays as number `1`.

---

## Mistake 4: Mutating During Render

```js
for (const product of products) {
  product.quantity += 1;
}
```

This changes state.

---

## Mistake 5: Storing Display Number as ID

Incorrect:

```js
button.dataset.id = displayNumber;
```

## Key Takeaways

* `for...of` gives direct access to array values.
* It does not require manual initialization, condition, or increment logic.
* The loop variable represents the current item.
* `for...of` is often very readable for array rendering.
* It does not provide the index automatically.
* A separate counter or `entries()` can provide numbering.
* `for...of` supports `break` and `continue`.
* `for...in` returns keys or indexes, not array values.
* Stable IDs should come from item objects, not display numbers.
* Render functions should clear containers before rebuilding them.
* Empty arrays should display a helpful message.
* Rendering should read state without mutating it.
* `for...of` is often a good plain-loop choice when direct value access is all you need.
