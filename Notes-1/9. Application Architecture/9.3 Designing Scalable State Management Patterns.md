# 1. The Problem: Growing Complexity

## Example

```js
let state = [
  {
    id: 1,
    name: "Laptop",
    category: {
      id: 10,
      name: "Electronics"
    }
  }
];
```

---

## Problem

* Nested data ❌
* Hard to update ❌
* Duplicate category objects ❌

---

# 2. Normalized State

## Concept Explanation

Instead of nesting:

```text
Flatten your data structure
```

---

## Bad Structure

```js
products = [
  {
    id: 1,
    category: { id: 10, name: "Electronics" }
  }
];
```

---

## Good Structure

```js
const state = {
  products: [
    { id: 1, categoryId: 10 }
  ],
  categories: [
    { id: 10, name: "Electronics" }
  ]
};
```

---

## Benefits

* No duplication
* Easier updates
* Faster lookups
* Scalable

---

# 3. Derived State

## Concept

Derived state = calculated data

---

## Example

```js
const totalValue = products.reduce(
  (sum, p) => sum + p.price * p.quantity,
  0
);
```

---

## Rule

> Never store derived values in state

---

# 4. State Shape Design

## Good Design

```js
const state = {
  products: [],
  ui: {
    search: "",
    filter: "all",
    sort: "price"
  }
};
```

---

## Why

* Separate data from UI
* Easier persistence
* Easier debugging

---

# 5. Immutable Updates at Scale

## Concept

Never mutate large state directly.

---

## Example

```js
state = {
  ...state,
  products: state.products.map(p =>
    p.id === id ? { ...p, name: "New" } : p
  )
};
```

---

# 6. Three Complete Examples

## Example 1: Flat Data

```js
const products = [
  { id: 1, name: "Laptop" }
];
```

---

## Example 2: Normalized Data

```js
const state = {
  products: [{ id: 1, categoryId: 10 }],
  categories: [{ id: 10, name: "Electronics" }]
};
```

---

## Example 3: Full Application

```js
function getProductWithCategory(product) {
  const category = state.categories.find(
    c => c.id === product.categoryId
  );

  return {
    ...product,
    categoryName: category ? category.name : "Unknown"
  };
}
```

---

# 7. Scaling CRUD Operations

## Concept

Use centralized functions:

```js
function updateProduct(id, updates) {
  state = {
    ...state,
    products: state.products.map(p =>
      p.id === id ? { ...p, ...updates } : p
    )
  };
}
```

---

# 8. Avoiding Common Pitfalls

* Deep nesting
* Duplicate data
* Mutating state
* Storing derived values
* Mixing UI with data

---

# 9. Best Practices

* Normalize data
* Keep state flat
* Separate UI state
* Use derived calculations
* Keep updates immutable
* Structure state clearly
