# 1. The Core Problem: State Growth

## What You Will Learn

Why simple state structures break as apps grow.

---

## Example

```js
let products = [
  {
    id: 1,
    name: "Laptop",
    category: {
      id: 10,
      name: "Electronics"
    }
  }
];
```

---

## Problems

```text
❌ Nested data (hard to update)
❌ Duplicate categories
❌ Complex updates
❌ Poor performance
```

---

# 2. Nested vs Flat State

## Concept Explanation

Nested state:

```js
product.category.name = "New Name"; // complex update
```

Flat state:

```js
product.categoryId = 10;
```

---

## Why Flat State Wins

```text
✔ Easier updates
✔ No duplication
✔ Faster lookups
✔ Scalable
```

---

# 3. Normalizing State

## Concept

Split related data into separate collections.

---

## Bad Structure

```js
products = [
  {
    id: 1,
    category: { id: 10, name: "Electronics" }
  }
];
```

---

## Good Structure

```js
const state = {
  products: [
    { id: 1, categoryId: 10 }
  ],
  categories: [
    { id: 10, name: "Electronics" }
  ]
};
```

---

## Why This Works

```text
One source of truth for categories
No duplication
Easy updates
```

---

# 4. Derived State (Deep Understanding)

## Concept

Derived state = calculated from base state.

---

## Example

```js
const totalValue = products.reduce(
  (sum, p) => sum + p.price * p.quantity,
  0
);
```

---

## Rule

```text
DO NOT store derived values in state
```

---

## Why?

```text
Stored value → becomes outdated ❌
Derived value → always correct ✅
```

---

# 5. Designing State Shape

## Concept

State should be predictable and structured.

---

## Example

```js
const state = {
  products: [],
  categories: [],
  ui: {
    search: "",
    filter: "all",
    sort: "price"
  }
};
```

---

## Why Separate UI State?

```text
UI state changes often
Data state persists
```

---

# 6. Immutable Updates at Scale

## Concept

Never mutate large state directly.

---

## Example

```js
state = {
  ...state,
  products: state.products.map(p =>
    p.id === id
      ? { ...p, name: "Updated" }
      : p
  )
};
```

---

## Why This Matters

```text
Predictable updates
No side effects
Safe history tracking
```

---

# 7. Three Complete Examples

## Example 1 (Basic Flat Data)

```js
const products = [
  { id: 1, name: "Laptop" }
];
```

---

## Example 2 (Normalized State)

```js
const state = {
  products: [{ id: 1, categoryId: 10 }],
  categories: [{ id: 10, name: "Electronics" }]
};
```

---

## Example 3 (Full Application Logic)

```js
function getProductWithCategory(product) {
  const category = state.categories.find(
    c => c.id === product.categoryId
  );

  return {
    ...product,
    categoryName: category ? category.name : "Unknown"
  };
}
```

---

## Flow

```text
Base state → derived data → render → UI
```

---

# 8. Efficient Lookups

## Problem

```js
find() → O(n)
```

---

## Solution: Indexing

```js
const categoryMap = {};

state.categories.forEach(c => {
  categoryMap[c.id] = c;
});
```

---

## Usage

```js
const category = categoryMap[product.categoryId];
```
