# 1. The Core Problem: Duplicate Logic

## What You Will Learn

Why writing multiple similar functions is dangerous.

---

## Bad Example

```js
function updateProductName(id, name) {
  products = products.map(p =>
    p.id === id ? { ...p, name } : p
  );
}

function updateProductPrice(id, price) {
  products = products.map(p =>
    p.id === id ? { ...p, price } : p
  );
}
```

---

## Problems

```text
❌ Duplicate logic
❌ Hard to maintain
❌ Not scalable
❌ Error-prone
```

# 2. The Solution: Reusable Functions

## Concept Explanation

Instead of:

Many specific functions

Use:

```text
One flexible function
```

# 3. Designing a Generic Update Function

## Concept

We pass:

* ID
* Updates object

---

## Implementation

```js
function updateItemById(id, updates) {
  products = products.map(item =>
    item.id === id
      ? { ...item, ...updates }
      : item
  );
}
```

---

## Why This Works

```text
updates = { name: "Tablet" }
updates = { price: 30000 }
updates = { quantity: 10 }
```

Same function → many use cases.

---

## Example Usage

```js
updateItemById(1, { name: "Tablet" });
updateItemById(1, { price: 30000 });
updateItemById(1, { quantity: 5 });
```

---

# 4. Designing a Generic Delete Function

## Implementation

```js
function deleteItemById(id) {
  products = products.filter(item => item.id !== id);
}
```

---

## Explanation

* Keeps all items except the one with matching ID

---

# 5. Designing a Generic Find Function

## Implementation

```js
function findItemById(id) {
  return products.find(item => item.id === id);
}
```

---

## Return Value

```text
Item → if found
undefined → if not found
```

---

# 6. Designing a Generic Create Function

## Implementation

```js
function createItem(data) {
  const newItem = {
    id: Date.now(),
    ...data
  };

  products = [...products, newItem];

  return newItem;
}
```

---

## Why Return Value Matters

```js
const item = createItem(data);
```

You can:

* Use it immediately
* Display confirmation
* Track actions

# 7. Three Complete Examples

## Example 1 (Basic Numbers)

```js
let numbers = [1,2,3];

function deleteNumber(n) {
  numbers = numbers.filter(x => x !== n);
}

deleteNumber(2);

console.log(numbers);
```

### Output

```text
[1,3]
```

# 8. Making Functions Fully Reusable

## Problem

Current functions depend on `products`.

---

## Solution

Pass array as parameter:

```js
function deleteById(list, id) {
  return list.filter(item => item.id !== id);
}
```

---

## Usage

```js
products = deleteById(products, 1);
tasks = deleteById(tasks, 2);
```

---

# 9. Handling Missing Items

## Problem

Item may not exist.

---

## Solution

```js
function updateItemById(id, updates) {
  let found = false;

  products = products.map(item => {
    if (item.id === id) {
      found = true;
      return { ...item, ...updates };
    }
    return item;
  });

  if (!found) {
    console.warn("Item not found");
  }
}
```
