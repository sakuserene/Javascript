# 11. Completing the Create Operation

## 11.1 What You Will Learn

The Create operation is complete only when all required steps happen in the correct order.

A complete browser-based Create flow is:

```text
User enters values
        ↓
User submits form
        ↓
Submit event handler runs
        ↓
Default form behavior is prevented
        ↓
Values are read
        ↓
Values are normalized
        ↓
Values are validated
        ↓
Unique ID is generated
        ↓
New object is created
        ↓
Object is added to state
        ↓
Form is reset
        ↓
Render function runs
        ↓
DOM displays current state
```

Missing any of these steps may create bugs.

---

# 11.2 The Complete Create Sequence

A reliable sequence is:

```js
function handleProductSubmit(event) {
  event.preventDefault();

  clearError();

  const formData = readProductForm();

  const validationResult =
    validateProductForm(formData);

  if (!validationResult.isValid) {
    showError(validationResult.message);
    focusField(validationResult.field);
    return;
  }

  const newProduct = {
    id: generateProductId(),
    ...formData,
  };

  products.push(newProduct);

  resetProductForm();

  renderProducts();
}
```

This structure separates responsibilities.

# 11.3 Responsibility 1: Reading Form Data

Create a function:

```js
function readProductForm() {
  return {
    name:
      productNameInput.value.trim(),

    priceText:
      productPriceInput.value.trim(),

    quantityText:
      productQuantityInput.value.trim(),

    category:
      productCategorySelect.value,

    isActive:
      productActiveInput.checked,
  };
}
```

This function only reads data.

It does not:

* Validate.
* Add to state.
* Render.
* Reset the form.

---

# 11.4 Responsibility 2: Validating Data

```js
function validateProductForm(data) {
  if (data.name === "") {
    return {
      isValid: false,
      message:
        "Product name is required.",
      field: "name",
    };
  }

  if (data.priceText === "") {
    return {
      isValid: false,
      message:
        "Price is required.",
      field: "price",
    };
  }

  const price =
    Number(data.priceText);

  if (
    Number.isNaN(price) ||
    !Number.isFinite(price) ||
    price < 0
  ) {
    return {
      isValid: false,
      message:
        "Price must be a non-negative number.",
      field: "price",
    };
  }

  const quantity =
    Number(data.quantityText);

  if (
    data.quantityText === "" ||
    !Number.isInteger(quantity) ||
    quantity < 0
  ) {
    return {
      isValid: false,
      message:
        "Quantity must be a non-negative whole number.",
      field: "quantity",
    };
  }

  if (data.category === "") {
    return {
      isValid: false,
      message:
        "Please select a category.",
      field: "category",
    };
  }

  return {
    isValid: true,
    message: "",
    field: null,
    normalizedData: {
      name: data.name,
      price,
      quantity,
      category: data.category,
      isActive: data.isActive,
    },
  };
}
```

This function returns:

* Whether the data is valid.
* An error message.
* The field responsible.
* Normalized values when valid.

```js
function addProduct(product) {
  products.push(product);
```

```js

addProduct({
```

# 11.8 Responsibility 6: Resetting the Form

```js
function resetProductForm() {
  productForm.reset();

  productActiveInput.checked = true;

  productNameInput.focus();
}
```

This affects only form UI state.

It does not clear `products`.

# 11.9 Responsibility 7: Rendering

```js
function renderProducts() {
  productList.innerHTML = "";

  productCount.textContent =
    `Total products: ${products.length}`;

  if (products.length === 0) {
    renderEmptyState();
    return;
  }

  for (
    let index = 0;
    index < products.length;
    index++
  ) {
    renderProductCard(
      products[index],
      index
    );
  }
}
```

This function displays current state.

It does not create new products.

# 11.10 Rendering One Product Card

```js
function renderProductCard(
  product,
  index
) {
  const card =
    document.createElement("li");

  const name =
    document.createElement("h2");

  const price =
    document.createElement("p");

  const quantity =
    document.createElement("p");

  const category =
    document.createElement("p");

  const status =
    document.createElement("p");

  card.classList.add("product-card");
  card.dataset.id = product.id;

  name.textContent =
    `${index + 1}. ${product.name}`;

  price.textContent =
    `Price: KSh ${product.price.toLocaleString()}`;

  quantity.textContent =
    `Quantity: ${product.quantity}`;

  category.textContent =
    `Category: ${product.category}`;

  status.textContent =
    product.isActive
      ? "Status: Active"
      : "Status: Inactive";

  card.append(
    name,
    price,
    quantity,
    category,
    status
  );

  productList.appendChild(card);
}
```

The card is generated entirely from state.

# 11.12 Complete Submit Handler

```js
function handleProductSubmit(event) {
  event.preventDefault();

  clearError();

  const rawData =
    readProductForm();

  const validationResult =
    validateProductForm(rawData);

  if (!validationResult.isValid) {
    showError(
      validationResult.message
    );

    focusInvalidField(
      validationResult.field
    );

    return;
  }

  const product =
    createProduct(
      validationResult.normalizedData
    );

  addProduct(product);

  resetProductForm();

  renderProducts();
}
```

Attach it:

```js
productForm.addEventListener(
  "submit",
  handleProductSubmit
);
```

---

# 11.13 Why Rendering Happens Last

Correct order:

```js
addProduct(product);
resetProductForm();
renderProducts();
```

The render function must read the updated state.

If rendering happens first:

```js
renderProducts();
addProduct(product);
```

the new item is not included in that render.

---

# 11.14 Why the Container Must Be Cleared

```js
productList.innerHTML = "";
```

Without this line:

```js
renderProducts();
renderProducts();
```

duplicates all displayed cards.

The state may contain three products, but the DOM may display six cards.

Clearing ensures the DOM is rebuilt from the current state.

---

# 11.15 Derived Product Count

Do not store a separate count:

```js
let productCountValue = 0;
```

Prefer:

```js
products.length
```

Rendering:

```js
productCount.textContent =
  `Total products: ${products.length}`;
```

This prevents count synchronization bugs.

# 11.16 Preventing Accidental Double Submission

A user may click the submit button repeatedly.

For synchronous code, this usually occurs quickly but may still create duplicates.

You can temporarily disable the button.

HTML:

```html
<button
  id="submit-button"
  type="submit"
>
  Add Product
</button>
```

JavaScript:

```js
const submitButton =
  document.getElementById(
    "submit-button"
  );
```

During valid creation:

```js
submitButton.disabled = true;

addProduct(product);
resetProductForm();
renderProducts();

submitButton.disabled = false;
```

For synchronous operations, the disabled state may be extremely brief.

It becomes more important during API requests.

# 11.17 Preventing Duplicate Names

A separate validation rule may reject duplicate names.

Using a plain loop:

```js
function productNameExists(name) {
  for (
    let index = 0;
    index < products.length;
    index++
  ) {
    const existingName =
      products[index].name
        .trim()
        .toLowerCase();

    if (
      existingName ===
      name.trim().toLowerCase()
    ) {
      return true;
    }
  }

  return false;
}
```

Usage:

```js
if (productNameExists(data.name)) {
  return {
    isValid: false,
    message:
      "A product with this name already exists.",
    field: "name",
  };
}
```

This is optional depending on application rules.

# 11.18 Case-Insensitive Duplicate Checking

These names may represent the same product:

```text
Laptop
laptop
 LAPTOP
```

Normalize both values:

```js
name.trim().toLowerCase()
```

Example:

```js
"LAPTOP".trim().toLowerCase();
```

Result:

```text
laptop
```

# 11.19 Avoiding Duplicate Rendering

Duplicate products in state and duplicate DOM rendering are different problems.

## Duplicate state records

```js
[
  { id: 1, name: "Laptop" },
  { id: 2, name: "Laptop" },
]
```

This may be allowed or rejected depending on rules.

## Duplicate DOM rendering

State contains one product, but the page shows it several times because the container was not cleared.

These problems require different fixes.

# 11.20 Mutable Create Flow

```js
products.push(product);
```

The existing array is modified.

Complete flow:

```js
const product =
  createProduct(validatedData);

products.push(product);

renderProducts();
```

---

# 11.21 Immutable Create Flow

```js
products = [
  ...products,
  product,
];
```

The state variable receives a new array.

This requires:

```js
let products = [];
```

not:

```js
const products = [];
```

Both approaches should still call:

```js
renderProducts();
```

---

# 11.22 Object References During Create

Consider:

```js
const product = {
  id: 1,
  name: "Laptop",
};

products.push(product);
```

The array stores a reference to the object.

Later:

```js
product.name =
  "Gaming Laptop";
```

The object inside the array also appears changed.

```js
console.log(
  products[0].name
);
```

Output:

```text
Gaming Laptop
```

This happens because both references point to the same object.

---

# 11.23 Creating Fresh Objects

Each submission should create a new object.

Correct:

```js
const product = {
  id: generateProductId(),
  name,
};
```

inside the submit flow.

Incorrect:

```js
const product = {};

function handleSubmit() {
  product.id =
    generateProductId();

  product.name =
    productNameInput.value;

  products.push(product);
}
```

Every array entry may refer to the same object.

Later changes can affect all entries unexpectedly.

---

# 11.24 Shared-Reference Bug

Incorrect:

```js
const reusableProduct = {};

function addProduct(name) {
  reusableProduct.id =
    generateProductId();

  reusableProduct.name = name;

  products.push(reusableProduct);
}

addProduct("Laptop");
addProduct("Phone");
```

Both entries reference the same object.

Final state may appear as:

```js
[
  {
    id: 2,
    name: "Phone",
  },
  {
    id: 2,
    name: "Phone",
  },
]
```

Create a fresh object every time.

# 11.26 Three Complete Examples

## Example 1: Complete Create Without the DOM

### Problem

Validate product data, generate an ID, create a fresh object, and add it to state.

### Input Data

```js
{
  name: "Laptop",
  price: 50000,
  quantity: 2
}
```

### Complete Code

```js
let products = [];
let nextProductId = 1;

function generateProductId() {
  const id = nextProductId;

  nextProductId++;

  return id;
}

function isValidProductData(data) {
  if (
    typeof data.name !== "string" ||
    data.name.trim() === ""
  ) {
    return false;
  }

  if (
    typeof data.price !== "number" ||
    !Number.isFinite(data.price) ||
    data.price < 0
  ) {
    return false;
  }

  if (
    !Number.isInteger(data.quantity) ||
    data.quantity < 0
  ) {
    return false;
  }

  return true;
}

function createProduct(data) {
  if (!isValidProductData(data)) {
    return null;
  }

  return {
    id: generateProductId(),
    name: data.name.trim(),
    price: data.price,
    quantity: data.quantity,
  };
}

function addProduct(data) {
  const product =
    createProduct(data);

  if (product === null) {
    console.log(
      "Product was not created."
    );

    return false;
  }

  products.push(product);

  return true;
}

addProduct({
  name: "Laptop",
  price: 50000,
  quantity: 2,
});

console.log(products);
```

### Expected Console Output

```text
[
  {
    id: 1,
    name: "Laptop",
    price: 50000,
    quantity: 2
  }
]
```

### Final State

```js
[
  {
    id: 1,
    name: "Laptop",
    price: 50000,
    quantity: 2,
  },
]
```

### Common Mistakes

* Generating the ID before validation.
* Returning an invalid object.
* Reusing the same object for every product.
* Incrementing the ID after failed creation.

### Small Modification Challenge

Try adding:

```js
{
  name: "",
  price: -20,
  quantity: 2.5
}
```

Confirm that state remains unchanged.

## Example 2: Create and Render a Shopping Item

### Problem

Create shopping-list items from a form and render them.

The item structure is:

```js
{
  id,
  name,
  purchased
}
```

### Complete Code

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8" />

  <meta
    name="viewport"
    content="width=device-width, initial-scale=1.0"
  />

  <title>Shopping List Create</title>
</head>

<body>
  <h1>Shopping List</h1>

  <form id="shopping-form">
    <label for="item-name">
      Item Name
    </label>

    <input
      id="item-name"
      type="text"
      autocomplete="off"
    />

    <button type="submit">
      Add Item
    </button>
  </form>

  <p id="error-message"></p>
  <p id="item-count"></p>

  <ul id="item-list"></ul>

  <script>
    let items = [];
    let nextItemId = 1;

    const shoppingForm =
      document.getElementById(
        "shopping-form"
      );

    const itemNameInput =
      document.getElementById(
        "item-name"
      );

    const errorMessage =
      document.getElementById(
        "error-message"
      );

    const itemCount =
      document.getElementById(
        "item-count"
      );

    const itemList =
      document.getElementById(
        "item-list"
      );

    function showError(message) {
      errorMessage.textContent =
        message;
    }

    function clearError() {
      errorMessage.textContent = "";
    }

    function generateItemId() {
      const id = nextItemId;

      nextItemId++;

      return id;
    }

    function renderItems() {
      itemList.innerHTML = "";

      itemCount.textContent =
        `Total items: ${items.length}`;

      if (items.length === 0) {
        const emptyMessage =
          document.createElement("li");

        emptyMessage.textContent =
          "No shopping items available.";

        itemList.appendChild(
          emptyMessage
        );

        return;
      }

      for (
        let index = 0;
        index < items.length;
        index++
      ) {
        const item = items[index];

        const listItem =
          document.createElement("li");

        listItem.dataset.id =
          item.id;

        listItem.textContent =
          `${index + 1}. ${item.name} — ` +
          (
            item.purchased
              ? "Purchased"
              : "Not purchased"
          );

        itemList.appendChild(
          listItem
        );
      }
    }

    shoppingForm.addEventListener(
      "submit",
      function (event) {
        event.preventDefault();

        clearError();

        const name =
          itemNameInput.value.trim();

        if (name === "") {
          showError(
            "Item name is required."
          );

          itemNameInput.focus();
          return;
        }

        const newItem = {
          id: generateItemId(),
          name,
          purchased: false,
        };

        items.push(newItem);

        shoppingForm.reset();
        itemNameInput.focus();

        renderItems();
      }
    );

    renderItems();
  </script>
</body>
</html>
```

### Expected Initial DOM

```text
Shopping List

Total items: 0

No shopping items available.
```

### Expected DOM After Adding `Milk`

```text
Shopping List

Total items: 1

1. Milk — Not purchased
```

### Final State

```js
[
  {
    id: 1,
    name: "Milk",
    purchased: false,
  },
]
```

### Common Mistakes

* Forgetting `purchased: false`.
* Rendering before `push()`.
* Resetting the item ID.
* Clearing state instead of the form.

### Small Modification Challenge

Reject duplicate shopping-item names using a standard `for` loop.

```html
        "product-count"
      );

    const productList =
      document.getElementById(
        "product-list"
      );

    function showError(message) {
      errorMessage.textContent =
        message;
    }

    function clearError() {
      errorMessage.textContent = "";
    }

    function focusInvalidField(field) {
      if (field === "name") {
        productNameInput.focus();
      } else if (field === "price") {
        productPriceInput.focus();
      } else if (
        field === "quantity"
      ) {
        productQuantityInput.focus();
      } else if (
        field === "category"
      ) {
        productCategorySelect.focus();
      }
    }

    function readProductForm() {
      return {
        name:
          productNameInput.value.trim(),

        priceText:
          productPriceInput.value.trim(),

        quantityText:
          productQuantityInput.value.trim(),

        category:
          productCategorySelect.value,

        isActive:
          productActiveInput.checked,
      };
    }

    function productNameExists(name) {
      const normalizedName =
        name.toLowerCase();

      for (
        let index = 0;
        index < products.length;
        index++
      ) {
        const existingName =
          products[index].name
            .trim()
            .toLowerCase();

        if (
          existingName ===
          normalizedName
        ) {
          return true;
        }
      }

      return false;
    }

    function validateProductForm(data) {
      if (data.name === "") {
        return {
          isValid: false,
          message:
            "Product name is required.",
          field: "name",
        };
      }

      if (data.name.length < 2) {
        return {
          isValid: false,
          message:
            "Product name must contain at least 2 characters.",
          field: "name",
        };
      }

      if (
        productNameExists(data.name)
      ) {
        return {
          isValid: false,
          message:
            "A product with this name already exists.",
          field: "name",
        };
      }

      if (data.priceText === "") {
        return {
          isValid: false,
          message:
            "Price is required.",
          field: "price",
        };
      }

      const price =
        Number(data.priceText);

      if (
        Number.isNaN(price) ||
        !Number.isFinite(price) ||
        price < 0
      ) {
        return {
          isValid: false,
          message:
            "Price must be a non-negative number.",
          field: "price",
        };
      }

      if (
        data.quantityText === ""
      ) {
        return {
          isValid: false,
          message:
            "Quantity is required.",
          field: "quantity",
        };
      }

      const quantity =
        Number(data.quantityText);

      if (
        !Number.isInteger(quantity) ||
        quantity < 0
      ) {
        return {
          isValid: false,
          message:
            "Quantity must be a non-negative whole number.",
          field: "quantity",
        };
      }

      if (data.category === "") {
        return {
          isValid: false,
          message:
            "Please select a category.",
          field: "category",
        };
      }

      return {
        isValid: true,
        message: "",
        field: null,
        normalizedData: {
          name: data.name,
          price,
          quantity,
          category: data.category,
          isActive: data.isActive,
        },
      };
    }

    function generateProductId() {
      const id = nextProductId;

      nextProductId++;

      return id;
    }

    function createProduct(data) {
      return {
        id: generateProductId(),
        name: data.name,
        price: data.price,
        quantity: data.quantity,
        category: data.category,
        isActive: data.isActive,
      };
    }

    function addProduct(product) {
      products.push(product);
    }

    function resetProductForm() {
      productForm.reset();

      productActiveInput.checked =
        true;

      productNameInput.focus();
    }

    function renderEmptyState() {
      const emptyState =
        document.createElement("li");

      emptyState.classList.add(
        "empty-state"
      );

      emptyState.textContent =
        "No products available.";

      productList.appendChild(
        emptyState
      );
    }

    function renderProductCard(
      product,
      index
    ) {
      const card =
        document.createElement("li");

      const name =
        document.createElement("h2");

      const price =
        document.createElement("p");

      const quantity =
        document.createElement("p");

      const category =
        document.createElement("p");

      const status =
        document.createElement("p");

      card.classList.add(
        "product-card"
      );

      card.dataset.id = product.id;

      name.textContent =
        `${index + 1}. ${product.name}`;

      price.textContent =
        `Price: KSh ${product.price.toLocaleString()}`;

      quantity.textContent =
        `Quantity: ${product.quantity}`;

      category.textContent =
        `Category: ${product.category}`;

      status.textContent =
        product.isActive
          ? "Status: Active"
          : "Status: Inactive";

      card.append(
        name,
        price,
        quantity,
        category,
        status
      );

      productList.appendChild(card);
    }

    function renderProducts() {
      productList.innerHTML = "";

      productCount.textContent =
        `Total products: ${products.length}`;

      if (products.length === 0) {
        renderEmptyState();
        return;
      }

      for (
        let index = 0;
        index < products.length;
        index++
      ) {
        renderProductCard(
          products[index],
          index
        );
      }
    }

    function handleProductSubmit(
      event
    ) {
      event.preventDefault();

      clearError();

      const rawData =
        readProductForm();

      const validationResult =
        validateProductForm(rawData);

      if (
        !validationResult.isValid
      ) {
        showError(
          validationResult.message
        );

        focusInvalidField(
          validationResult.field
        );

        return;
      }

      submitButton.disabled = true;

      const product =
        createProduct(
          validationResult.normalizedData
        );

      addProduct(product);

      resetProductForm();
      renderProducts();

      submitButton.disabled = false;
    }

    productForm.addEventListener(
      "submit",
      handleProductSubmit
    );

    renderProducts();
  </script>
</body>
</html>
```

## Key Takeaways

* A complete Create operation includes reading, validation, ID generation, state mutation, reset, and rendering.
* The order of operations matters.
* State should change before the DOM is rendered.
* Validation must happen before state mutation.
* Each created item must be a fresh object.
* ID generation should happen only after validation.
* The form should reset only after successful creation.
* The product count should be derived from state.
* The render container must be cleared before rebuilding.
* Duplicate state records and duplicate DOM rendering are different problems.
* Functions should have one clear responsibility.
* The event handler should coordinate the workflow.
* The completed Create architecture prepares the application for Update and Delete.
