# 1. What `map()` Does

## What You Will Learn

`map()` visits every item in an array and returns a new array containing the callback's returned values.

It is useful when you want to:

* Transform every array value.
* Create a new array of names.
* Convert prices.
* Add calculated properties.
* Update one object immutably.
* Preserve the original array structure.

In CRUD applications, `map()` is especially useful for immutable Update operations.

---

## Concept Explanation

Suppose you have:

```js
const numbers = [
  1,
  2,
  3,
];
```

You want:

```js
[
  2,
  4,
  6,
]
```

Using `map()`:

```js
const doubledNumbers =
  numbers.map(
    (number) =>
      number * 2
  );
```

The original array remains:

```js
[
  1,
  2,
  3,
]
```

The new array is:

```js
[
  2,
  4,
  6,
]
```

---

## Why It Matters for CRUD

Suppose you need to update product ID `2`.

You want to:

1. Visit every product.
2. Replace the matching product with a new object.
3. Keep every non-matching product unchanged.
4. Return a complete new array.
5. Assign the new array to the state variable.

That is a natural use of `map()`.

---

## Common Beginner Mistakes

* Forgetting that `map()` returns a new array.
* Calling `map()` without assigning its result.
* Forgetting to return from the callback.
* Returning only the updated property instead of a product object.
* Replacing every object accidentally.
* Forgetting to preserve unchanged properties.
* Confusing `map()` with `forEach()`.
* Mutating the object inside `map()` and calling it immutable.


```js

```



# 5. Plain Loop Versus `map()`

## Using a Plain Loop

```js
const updatedProducts = [];

for (
  let index = 0;
  index < products.length;
  index++
) {
  const product =
    products[index];

  if (
    product.id === selectedId
  ) {
    updatedProducts.push({
      ...product,
      name: updatedName,
    });
  } else {
    updatedProducts.push(
      product
    );
  }
}

products =
  updatedProducts;
```

## Using `map()`

```js
products =
  products.map(
    (product) =>
      product.id === selectedId
        ? {
            ...product,
            name: updatedName,
          }
        : product
  );
```

---

## What Both Versions Do

Both:

* Visit every product.
* Compare IDs.
* Create a replacement object for the match.
* Preserve non-matching products.
* Create a new array.
* Reassign the main state variable.

---

## Which Is Easier to Read?

For a straightforward immutable transformation, `map()` is usually easier to read.

The purpose is clear:

```text
Map every old product into the product that should exist in the new state.
```

---

## Which Gives More Control?

A plain loop provides more manual control.

Use it when:

* You need unusual iteration.
* You need several external values.
* The transformation is very complex.
* You need to stop early.
* You need custom index movement.

However, immutable updates normally require visiting the complete array, so `map()` is a strong fit.

# 6. `map()` Versus `forEach()`

## `forEach()`

```js
products.forEach(
  (product) => {
    console.log(
      product.name
    );
  }
);
```

Used for side effects.

Returns:

```js
undefined
```

---

## `map()`

```js
const names =
  products.map(
    (product) =>
      product.name
  );
```

Used for transformation.

Returns:

```js
[
  "Laptop",
  "Phone",
  "Keyboard",
]
```

---

## Key Difference

Use `forEach()` when you want to do something for every item.

Use `map()` when you need a new array containing transformed values.

# 7. Mutable Update

A mutable update changes an existing object.

```js
const product =
  products.find(
    (product) =>
      product.id === selectedId
  );

if (
  product !== undefined
) {
  product.name =
    "Updated Laptop";
}
```

The original object reference is modified.

The array reference remains the same.

---

# 8. Immutable Update

An immutable update creates:

* A new array.
* A new object for the matching product.
* Existing object references for unchanged products.

```js
products =
  products.map(
    (product) =>
      product.id === selectedId
        ? {
            ...product,
            name: "Updated Laptop",
          }
        : product
  );
```

---

# 9. Mutable Versus Immutable Comparison

## Mutable

```js
selectedProduct.name =
  updatedName;
```

Changes the existing object.

## Immutable

```js
products =
  products.map(
    (product) =>
      product.id === selectedId
        ? {
            ...product,
            name: updatedName,
          }
        : product
  );
```

Creates a new object and new array.

---

## Mutable Advantages

* Simple.
* Less code.
* May use less memory.
* Appropriate in some plain JavaScript applications.

---

## Immutable Advantages

* Previous array remains unchanged.
* State changes are easier to track.
* Works well with state-management systems.
* Helps with undo and redo.
* Reduces hidden side effects.
* Fits React and similar libraries.
* Makes reference comparisons possible.

---

Suppose:

```js
  products;
```

```js
  products
```

# 12. Updating Several Properties

```js
products =
```

```js

```

and not overridden.

---

# 13. Spread Order Matters

Correct:

```js
{
  ...product,
  name: updatedName,
}
```

The new name overrides the old name.

Incorrect:

```js
{
  name: updatedName,
  ...product,
}
```

The old product name is spread afterward and overwrites `updatedName`.

---

# 14. Preserving the Stable ID

```js
{
  ...product,
  name: updatedName,
}
```

copies:

```js
id
price
quantity
```

and every other existing property.

The ID remains unchanged.

Do not generate a new ID during Update.

---

# 15. Forgetting to Reassign the Result

Incorrect:

```js
products.map(
  (product) =>
    product.id === selectedId
      ? {
          ...product,
          name: updatedName,
        }
      : product
);
```

The new array is created and discarded.

The `products` state remains unchanged.

Correct:

```js
products =
  products.map(
    (product) =>
      product.id === selectedId
        ? {
            ...product,
            name: updatedName,
          }
        : product
  );
```

# 16. Missing Return Inside `map()`

Incorrect:

```js
products =
  products.map(
    (product) => {
      if (
        product.id ===
        selectedId
      ) {
        {
          ...product,
          name: updatedName,
        };
      } else {
        product;
      }
    }
  );
```

The callback returns:

```js
undefined
```

for every item.

Result:

```js
[
  undefined,
  undefined,
  undefined,
]
```

---

## Correct with Explicit Returns

```js
products =
  products.map(
    (product) => {
      if (
        product.id ===
        selectedId
      ) {
        return {
          ...product,
          name: updatedName,
        };
      }

      return product;
    }
  );
```

# 17. Updating a Product with a Reusable Function

```js
function updateProductById(
  productId,
  updates
) {
  let productWasFound =
    false;

  const updatedProducts =
    products.map(
      (product) => {
        if (
          product.id !==
          productId
        ) {
          return product;
        }

        productWasFound = true;

        return {
          ...product,
          ...updates,
          id: product.id,
        };
      }
    );

  if (!productWasFound) {
    return {
      success: false,
      updatedProduct: null,
      message:
        "Product not found.",
    };
  }

  products =
    updatedProducts;

  const updatedProduct =
    products.find(
      (product) =>
        product.id ===
        productId
    );

  return {
    success: true,
    updatedProduct,
    message:
      "Product updated.",
  };
}
```

---

# 18. Why Protect the ID?

This code:

```js
{
  ...product,
  ...updates,
  id: product.id,
}
```

ensures that even if `updates` contains another ID, the original ID wins.

Example dangerous input:

```js
{
  id: 999,
  name: "Smartphone",
}
```

Final result still preserves:

```js
id: product.id
```

# 19. Simpler Update Function

When the update fields are controlled:

```js
function updateProductById(
  productId,
  data
) {
  const productExists =
    products.some(
      (product) =>
        product.id ===
        productId
    );

  if (!productExists) {
    return null;
  }

  products =
    products.map(
      (product) =>
        product.id ===
        productId
          ? {
              ...product,
              name: data.name,
              price: data.price,
              quantity:
                data.quantity,
            }
          : product
    );

  return products.find(
    (product) =>
      product.id ===
      productId
  );
}
```

`some()` will be taught in detail later.

# 20. Handling a Missing Product

`map()` does not automatically tell you whether an update happened.

If no ID matches:

```js
products =
  products.map(
    (product) =>
      product.id === 999
        ? {
            ...product,
            name: "Updated",
          }
        : product
  );
```

The new array contains the same product objects.

No error is thrown.

---

## Option 1: Check with `find()`

```js
const existingProduct =
  products.find(
    (product) =>
      product.id === productId
  );

if (
  existingProduct === undefined
) {
  return null;
}

products =
  products.map(
    // Update.
  );
```

---

## Option 2: Track a Match

```js
let wasUpdated = false;

products =
  products.map(
    (product) => {
      if (
        product.id !== productId
      ) {
        return product;
      }

      wasUpdated = true;

      return {
        ...product,
        name: updatedName,
      };
    }
  );
```

```js
function updateProductById(
  productId,
  data
) {
  const existingProduct =
    products.find(
      (product) =>
        product.id === productId

  if (
    existingProduct ===
    undefined

    return null;
  }

  products =
    products.map(
      (product) =>
        product.id === productId
          ? {
              ...product,
              name: data.name,
              price: data.price,
              quantity:
                data.quantity,
            }

    (product) =>

}
```

# 25. Updating the DOM

`map()` changes JavaScript state only after reassignment.

It does not automatically update HTML.

You must call:

```js
renderProducts();
```

after the update.

---

# 26. Updating Only the DOM Is Incorrect

Incorrect:

```js
productNameElement.textContent =
  updatedName;
```

This changes what the user sees but leaves state unchanged.

A later render restores the old product name.

Correct:

```js
products =
  products.map(
    // Update state.
  );

renderProducts();
```

---

# 27. Updating State Without Rendering Is Also Incomplete

```js
products =
  products.map(
    // Update state.
  );
```

The array is correct, but the page still shows old values.

Always synchronize the DOM after successful state changes.

```html
      document.getElementById(
        "cancel-button"
      );

    const message =
      document.getElementById(
        "message"
      );

    const productList =
      document.getElementById(
        "product-list"
      );

    function renderProducts() {
      productList.innerHTML = "";

      products.forEach(
        (product) => {
          const card =
            document.createElement(
              "li"
            );

          const details =
            document.createElement(
              "span"
            );

          const editButton =
            document.createElement(
              "button"
            );

          card.classList.add(
            "product-card"
          );

          if (
            product.id ===
            editingProductId
          ) {
            card.classList.add(
              "currently-editing"
            );
          }

          details.textContent =
            `${product.name} — ` +
            `KSh ${product.price.toLocaleString()}`;

          editButton.type =
            "button";

          editButton.textContent =
            "Edit";

          editButton.dataset.action =
            "edit";

          editButton.dataset.id =
            product.id;

          card.append(
            details,
            editButton
          );

          productList.appendChild(
            card
          );
        }
      );
    }

    function resetForm() {
      editingProductId = null;

      form.reset();

      formTitle.textContent =
        "Add Product";

      submitButton.textContent =
        "Add Product";

      cancelButton.hidden =
        true;
    }

    function startEditing(
      productId
    ) {
      const product =
        products.find(
          (product) =>
            product.id ===
            productId
        );

      if (
        product === undefined
      ) {
        message.textContent =
          "Product not found.";

        return;
      }

      editingProductId =
        product.id;

      nameInput.value =
        product.name;

      priceInput.value =
        product.price;

      formTitle.textContent =
        "Edit Product";

      submitButton.textContent =
        "Update Product";

      cancelButton.hidden =
        false;

      renderProducts();
    }

    function updateProduct(
      productId,
      name,
      price
    ) {
      const existingProduct =
        products.find(
          (product) =>
            product.id ===
            productId
        );

      if (
        existingProduct ===
        undefined
      ) {
        return null;
      }

      products =
        products.map(
          (product) =>
            product.id ===
            productId
              ? {
                  ...product,
                  name,
                  price,
                }
              : product
        );

      return products.find(
        (product) =>
          product.id ===
          productId
      );
    }

    productList.addEventListener(
      "click",
      function (event) {
        const editButton =
          event.target.closest(
            '[data-action="edit"]'
          );

        if (
          editButton === null
        ) {
          return;
        }

        const productId =
          Number(
            editButton.dataset.id
          );

        startEditing(
          productId
        );
      }
    );

    form.addEventListener(
      "submit",
      function (event) {
        event.preventDefault();

        message.textContent = "";

        if (
          editingProductId ===
          null
        ) {
          message.textContent =
            "Create is not included in this focused example.";

          return;
        }

        const name =
          nameInput.value.trim();

        const price =
          Number(
            priceInput.value
          );

        if (
          name === "" ||
          !Number.isFinite(price) ||
          price < 0
        ) {
          message.textContent =
            "Enter valid product values.";

          return;
        }

        const updatedProduct =
          updateProduct(
            editingProductId,
            name,
            price
          );

        if (
          updatedProduct === null
        ) {
          message.textContent =
            "Product no longer exists.";

          resetForm();
          renderProducts();

          return;
        }

        const updatedName =
          updatedProduct.name;

        resetForm();
        renderProducts();

        message.textContent =
          `${updatedName} was updated.`;
      }
    );

    cancelButton.addEventListener(
      "click",
      function () {
        resetForm();
        renderProducts();

        message.textContent =
          "Editing canceled.";
      }
    );

    renderProducts();
  </script>
</body>
</html>
```

# 29. Common Mistakes

## Mistake 1: Using `map()` Without Its Result

```js
products.map(
  callback
);
```

The returned array is discarded.

---

## Mistake 2: Missing Return

```js
products.map(
  (product) => {
    product.name;
  }
);
```

Result:

```js
[
  undefined,
  undefined,
]
```

---

## Mistake 3: Returning Only One Property

```js
products =
  products.map(
    (product) =>
      product.name
  );
```

State becomes an array of strings.

---

## Mistake 4: Forgetting Unchanged Products

```js
products =
  products.map(
    (product) => {
      if (
        product.id === id
      ) {
        return {
          ...product,
          name,
        };
      }
    }
  );
```

Non-matching positions become `undefined`.

---

## Mistake 5: Wrong Spread Order

```js
{
  name: updatedName,
  ...product,
}
```

The old name overwrites the new value.

---

## Mistake 6: Directly Mutating Inside `map()`

```js
products =
  products.map(
    (product) => {
      if (
        product.id === id
      ) {
        product.name =
          updatedName;
      }

      return product;
    }
  );
```

A new array is created, but the product object is mutated.

This is not a fully immutable update.

---

## Mistake 7: Creating a New ID

Update must preserve identity.

---

## Mistake 8: Assuming Missing IDs Cause Errors

`map()` completes normally even when nothing matches.

Handle missing products deliberately.

---

## Mistake 9: Updating the DOM Only

State remains stale.

---

## Mistake 10: Forgetting to Render

The DOM remains stale.

# 30. Best Practices

* Use `map()` when producing a new array.
* Reassign the returned array to state.
* Return one value for every old item.
* Return unchanged objects for non-matches.
* Create a new object only for changed items.
* Spread the old object before new property values.
* Preserve stable IDs.
* Validate data before updating.
* Check whether the target exists.
* Re-render after state replacement.
* Avoid mutating objects inside `map()`.
* Use `forEach()` for side effects.
* Use `find()` when only one object is needed.
* Use `filter()` for immutable deletion.

## Key Takeaways

* `map()` creates a new array from callback return values.
* It visits every array element.
* It is appropriate for transformations and immutable Update.
* Every callback execution must return a value.
* The matching product should be replaced with a new object.
* Unchanged products should be returned as they are.
* The object spread operator preserves existing properties.
* New property values must appear after the spread to override old values.
* Stable IDs must remain unchanged.
* The result of `map()` must be assigned to the state variable.
* A missing ID does not automatically produce an error.
* Updating JavaScript state does not automatically update the DOM.
* Call the render function after a successful state update.
