# Why Compare Both Approaches?

Many beginners learn array methods and think they are "magic."

They are **not**.

Every array method eventually performs a loop internally.

For example:

```js
items.find(...)
```

internally performs something conceptually similar to:

```js
for (...) {
    ...
}
```

Understanding plain loops helps you understand how array methods work.

Understanding array methods helps you write cleaner JavaScript.

Professional developers normally know **both**.

---

# How Array Methods Work Internally

Suppose we have:

```js
const products = [
    {
        id: 1,
        name: "Laptop",
    },
    {
        id: 2,
        name: "Phone",
    },
];
```

When you write:

```js
products.find(
    product => product.id === 2
);
```

JavaScript internally performs something similar to:

```js
let foundProduct;

for (let index = 0; index < products.length; index++) {

    if (products[index].id === 2) {

        foundProduct = products[index];

        break;

    }

}
```

The difference is simply that JavaScript performs the loop for you.

# CRUD Comparison Overview

| CRUD Operation  | Plain Loop                 | Modern Method |
| --------------- | -------------------------- | ------------- |
| Display         | `for`, `while`, `for...of` | `forEach()`   |
| Find Item       | `for`                      | `find()`      |
| Find Position   | `for`                      | `findIndex()` |
| Update          | `for`                      | `map()`       |
| Delete          | `splice()`                 | `filter()`    |
| Duplicate Check | `for`                      | `some()`      |
| Validate All    | `for`                      | `every()`     |
| Totals          | `for`                      | `reduce()`    |

# 1. Reading Items

## Using a Plain Loop

```js
for (let index = 0; index < products.length; index++) {

    console.log(products[index].name);

}
```

---

## Using `forEach()`

```js
products.forEach((product) => {

    console.log(product.name);

});
```

---

## Comparison

### Plain Loop

Advantages

* Full control
* Can skip items
* Can break early
* Familiar

Disadvantages

* More code
* Easier to introduce bugs

---

### `forEach()`

Advantages

* Very readable
* Short
* Common in professional code

Disadvantages

* Cannot break early
* Cannot return a value

---

Recommendation

Use

```js
forEach()
```

for rendering.

# 2. Finding One Item

## Plain Loop

```js
let selectedProduct = null;

for (let index = 0; index < products.length; index++) {

    if (products[index].id === selectedId) {

        selectedProduct = products[index];

        break;

    }

}
```

---

## Modern Method

```js
const selectedProduct =
products.find(

    product =>
        product.id === selectedId

);
```

---

## Comparison

Plain Loop

Advantages

* Full control
* Easy to customize

Disadvantages

* More typing

---

`find()`

Advantages

* Very expressive
* Short
* Easier to read

Disadvantages

* Developers must understand callbacks

---

Recommendation

Use

```js
find()
```

for locating one object.

---

# 3. Finding an Index

## Plain Loop

```js
let foundIndex = -1;

for (let index = 0; index < products.length; index++) {

    if (products[index].id === selectedId) {

        foundIndex = index;

        break;

    }

}
```

---

## Modern Method

```js
const foundIndex =
products.findIndex(

    product =>
        product.id === selectedId

);
```

---

## Comparison

`findIndex()` clearly communicates:

> "I want the position."

Professional code usually prefers it.

---

# 4. Updating Items

## Plain Loop (Mutable)

```js
for (let index = 0; index < products.length; index++) {

    if (products[index].id === selectedId) {

        products[index].price = 60000;

        break;

    }

}
```

---

## Array Method (Immutable)

```js
products = products.map((product) =>

    product.id === selectedId

        ? {

            ...product,

            price: 60000,

        }

        : product

);
```

---

## Comparison

### Mutable

Changes the existing object.

Pros

* Faster
* Less memory

Cons

* Harder to track changes

---

### Immutable

Creates a new object.

Pros

* Predictable
* Easier debugging
* Used in React, Redux, Vuex, NgRx

Cons

* Slightly more memory

---

Recommendation

Use immutable updates whenever possible.

---

# 5. Deleting Items

## Plain Loop

```js
for (let index = 0; index < products.length; index++) {

    if (products[index].id === selectedId) {

        products.splice(index, 1);

        break;

    }

}
```

---

## Array Method

```js
products = products.filter(

    product =>

        product.id !== selectedId

);
```

---

## Comparison

### Plain Loop

Uses

```js
splice()
```

Mutates the array.

---

### `filter()`

Creates a new array.

Preferred in modern applications.

---

# 6. Checking for Duplicates

## Plain Loop

```js
let duplicateFound = false;

for (const product of products) {

    if (product.name === newName) {

        duplicateFound = true;

        break;

    }

}
```

---

## Array Method

```js
const duplicateFound =
products.some(

    product =>

        product.name === newName

);
```

---

Comparison

`some()`

reads almost like English.

---

# 7. Validating Every Product

## Plain Loop

```js
let valid = true;

for (const product of products) {

    if (product.price < 0) {

        valid = false;

        break;

    }

}
```

---

## Array Method

```js
const valid =
products.every(

    product =>

        product.price >= 0

);
```

---

Recommendation

Use

```js
every()
```

for validation.

---

# 8. Calculating Totals

## Plain Loop

```js
let total = 0;

for (const product of products) {

    total +=
        product.price *
        product.quantity;

}
```

---

## Array Method

```js
const total =

products.reduce(

    (sum, product) =>

        sum +
        product.price *
        product.quantity,

    0

);
```

---

Recommendation

Use

```js
reduce()
```

for derived values.

---

# Performance Comparison

For small and medium applications:

Performance differences are negligible.

Choose readability.

For very large datasets:

Sometimes plain loops can be slightly faster.

However,

maintainability is usually more important.

---

# Readability Comparison

Suppose another developer opens your project.

Which is easier?

Plain loop:

```js
let found;

for (...) {

    ...

}
```

or

```js
const found =
products.find(...);
```

Most developers immediately understand

```js
find()
```

because it clearly states the intention.

---

# When Should You Use Plain Loops?

Plain loops are still useful when:

* You need several break conditions.
* You need nested loops.
* You need maximum control.
* You are writing algorithms.
* You are teaching beginners.
* You are optimizing performance.

---

# When Should You Use Array Methods?

Use array methods when:

* Rendering
* Searching
* Updating
* Filtering
* Validation
* Totals
* CRUD applications
* Production web applications

# Common Mistakes

## Mistake 1

Using

```js
map()
```

without returning a value.

Incorrect

```js
products.map(product => {

    product.price = 100;

});
```

---

Correct

```js
products.map(product => {

    return {

        ...product,

        price: 100,

    };

});
```

---

## Mistake 2

Using

```js
forEach()
```

to build a new array.

Use

```js
map()
```

instead.

---

## Mistake 3

Using

```js
filter()
```

to find one item.

Use

```js
find()
```

---

## Mistake 4

Using

```js
find()
```

to delete.

Use

```js
filter()
```

---

## Mistake 5

Forgetting to assign the new array.

Incorrect

```js
products.filter(...);
```

Correct

```js
products =
products.filter(...);
```

# Best Practices

* Learn plain loops first.
* Learn array methods next.
* Understand what each method returns.
* Choose methods based on intent.
* Keep state immutable where practical.
* Write readable code.
* Keep render functions small.
* Separate business logic from rendering.
