# 1. What `filter()` Does

## What You Will Learn

`filter()` visits every element in an array and creates a new array containing only the elements whose callback returns `true`.

It answers this question for every item:

```text
Should this item remain in the new array?
```

It is useful for:

* Removing one item immutably.
* Showing only completed tasks.
* Showing only products in stock.
* Searching by product name.
* Selecting records in a category.
* Removing invalid values.
* Building a temporary visible list.

---

## Concept Explanation

Suppose you have:

```js
const numbers = [
  1,
  2,
  3,
  4,
];
```

You want only even numbers.

```js
const evenNumbers =
  numbers.filter(
    (number) =>
      number % 2 === 0
  );
```

The callback returns:

```text
1 → false
2 → true
3 → false
4 → true
```

The new array becomes:

```js
[
  2,
  4,
]
```

The original array remains unchanged.

---

## Why It Matters for Delete

Suppose state contains:

```js
let products = [
  {
    id: 1,
    name: "Laptop",
  },
  {
    id: 2,
    name: "Phone",
  },
  {
    id: 3,
    name: "Keyboard",
  },
];
```

To delete ID `2`, keep every product whose ID is not `2`.

```js
products =
  products.filter(
    (product) =>
      product.id !== 2
  );
```

New state:

```js
[
  {
    id: 1,
    name: "Laptop",
  },
  {
    id: 3,
    name: "Keyboard",
  },
]
```

---

## Common Beginner Mistakes

* Using `===` when deletion requires `!==`.
* Forgetting to assign the returned array.
* Forgetting to return a boolean.
* Expecting `filter()` to return one object.
* Expecting `filter()` to mutate the original array.
* Using `filter()` when only one item needs to be found.
* Replacing main state while performing a temporary search.
* Assuming a missing ID produces an error.
* Forgetting to re-render after deletion.


# 4. The Callback Must Return a Boolean-Like Result

Example:

```js
products.filter(
  (product) =>
    product.quantity > 0
);
```

For every product:

```text
quantity > 0
```

returns:

```js
true
```

or:

```js
false
```

`true` means keep the item.

`false` means exclude it.

# 5. `filter()` Returns a New Array

```js
const originalProducts = [
  {
    id: 1,
  },
  {
    id: 2,
  },
];

const filteredProducts =
  originalProducts.filter(
    (product) =>
      product.id !== 2
  );
```

Comparison:

```js
originalProducts ===
  filteredProducts
```

returns:

```text
false
```

The arrays are different objects.

---

## Original State

```js
[
  {
    id: 1,
  },
  {
    id: 2,
  },
]
```

## Filtered Result

```js
[
  {
    id: 1,
  },
]
```

The original array remains unchanged until the state variable is reassigned.

# 6. Plain Loop Versus `filter()`

## Using a Plain Loop

```js
const remainingProducts = [];

for (
  let index = 0;
  index < products.length;
  index++
) {
  const product =
    products[index];

  if (
    product.id !==
    selectedProductId
  ) {
    remainingProducts.push(
      product
    );
  }
}

products =
  remainingProducts;
```

## Using `filter()`

```js
products =
  products.filter(
    (product) =>
      product.id !==
      selectedProductId
  );
```

---

## What Both Versions Do

Both:

* Visit every product.
* Check each ID.
* Exclude the selected product.
* Preserve every non-matching product.
* Produce a new array.
* Require state reassignment.

---

## Which Is Easier to Read?

For a simple keep-or-remove condition, `filter()` is usually clearer.

```text
Keep every product whose ID is not the deleted ID.
```

---

## Which Gives More Control?

A standard loop offers more control when:

* You need several output arrays.
* You need custom index movement.
* You need detailed logging.
* You need complex side effects.
* You need to stop before reaching the end.

However, `filter()` always examines the full array because it must determine every item that belongs in the result.

# 7. Mutable Delete with `splice()`

```js
const productIndex =
  products.findIndex(
    (product) =>
      product.id === productId
  );

if (
  productIndex !== -1
) {
  products.splice(
    productIndex,
    1
  );
}
```

This changes the existing array.

Array reference:

```js
oldProducts === products
```

remains:

```text
true
```

---

# 8. Immutable Delete with `filter()`

```js
products =
  products.filter(
    (product) =>
      product.id !== productId
  );
```

This creates a new array.

Array reference:

```js
oldProducts === products
```

becomes:

```text
false
```

---

# 9. `splice()` Versus `filter()`

| Feature                | `splice()`            | `filter()`                   |
| ---------------------- | --------------------- | ---------------------------- |
| Mutates original array | Yes                   | No                           |
| Returns                | Removed items         | New filtered array           |
| Needs an index         | Yes                   | No                           |
| Common CRUD use        | Mutable Delete        | Immutable Delete             |
| Missing item result    | Must handle `-1`      | New array remains equivalent |
| Reads naturally by ID  | Requires lookup first | Yes                          |
| Creates new array      | No                    | Yes                          |

# 10. The Most Important Delete Condition

To delete the selected product, keep products whose IDs do not match.

Correct:

```js
product.id !==
selectedProductId
```

Incorrect:

```js
product.id ===
selectedProductId
```

The incorrect version keeps only the product you intended to delete.

# 12. Forgetting to Reassign the Result

Incorrect:

```js
products.filter(
  (product) =>
    product.id !== productId
);
```

A new array is created and discarded.

State remains unchanged.

Correct:

```js
products =
  products.filter(
    (product) =>
      product.id !== productId

```

# 13. Missing Return with Braces

Incorrect:

```js
products =
  products.filter(
    (product) => {
      product.id !==
        productId;
    }
  );
```

The callback returns:

```js
undefined
```

for every product.

Since `undefined` is falsy, every product is excluded.

Result:

```js
[]
```

---

## Correct with Explicit Return

```js
products =
  products.filter(

      return (
        product.id !==
        productId

    }
  );
```

## Correct with Concise Syntax

```js
products =
  products.filter(

      product.id !== productId
```

# 14. Handling a Missing Product

Suppose:

```js
productId === 999;
```

and no product has that ID.

```js
products =
  products.filter(
    (product) =>
      product.id !== 999

```

Every product returns `true`.

No error is thrown.

# 15. Detecting Whether Deletion Happened

Compare lengths:

```js
const previousLength =
  products.length;

const remainingProducts =
  products.filter(
    (product) =>
      product.id !== productId
  );

if (
  remainingProducts.length ===
  previousLength
) {
  return {
    success: false,
    deletedProduct: null,
    message:
      "Product not found.",
  };
}

products =
  remainingProducts;
```

If the length decreases, at least one product was removed.

# 16. Finding the Deleted Product First

A clearer CRUD flow often uses `find()` before `filter()`.

```js
const productToDelete =
  products.find(
    (product) =>
      product.id === productId
  );

if (
  productToDelete === undefined
) {
  return {
    success: false,
    deletedProduct: null,
    message:
      "Product not found.",
  };
}

products =
  products.filter(
    (product) =>
      product.id !== productId
  );
```

This gives access to the deleted product for:

* Confirmation.
* Feedback.
* Logging.
* Undo support.

---

# 17. Reusable Immutable Delete Function

```js
function deleteProductById(
  productId
) {
  const productToDelete =
    products.find(
      (product) =>
        product.id === productId
    );

  if (
    productToDelete ===
    undefined
  ) {
    return {
      success: false,
      deletedProduct: null,
      message:
        "The selected product no longer exists.",
    };
  }

  products =
    products.filter(
      (product) =>
        product.id !== productId
    );

  return {
    success: true,
    deletedProduct:
      productToDelete,
    message:
      "Product deleted successfully.",
  };
}
```

---

# 18. Why Use `find()` Before `filter()`?

`filter()` gives only the new array.

It does not directly return the removed object.

By finding first, the application can say:

```text
Phone was deleted successfully.
```

It can also store the removed object for Undo.

---

# 19. Confirmation Before Immutable Deletion

```js
function handleDeleteRequest(
  productId
) {
  clearMessages();

  const product =
    products.find(
      (product) =>
        product.id === productId
    );

  if (
    product === undefined
  ) {
    showError(
      "The selected product no longer exists."
    );

    renderProducts();

    return false;
  }

  const shouldDelete =
    window.confirm(
      `Are you sure you want to delete ${product.name}?`
    );

  if (!shouldDelete) {
    showInfo(
      `${product.name} was not deleted.`
    );

    return false;
  }

  const result =
    deleteProductById(
      productId
    );

  if (!result.success) {
    showError(
      result.message
    );

    renderProducts();

    return false;
  }

  if (
    editingProductId ===
    result.deletedProduct.id
  ) {
    resetFormToCreateMode();
  }

  renderProducts();

  showSuccess(
    `${result.deletedProduct.name} was deleted successfully.`
  );

  return true;
}
```

# 21. Rendering After `filter()`

`filter()` updates JavaScript state only after reassignment.

It does not remove DOM elements automatically.

Call:

```js
renderProducts();
```

after successful deletion.

This updates:

* Product cards.
* Visible numbering.
* Product totals.
* Inventory value.
* Out-of-stock count.
* Search results.
* Empty state.
* Action buttons.

# 23. Object References in the Filtered Array

`filter()` creates a new array, but it does not clone the kept objects.

```js
const originalProducts =
  products;

products =
  products.filter(
    (product) =>
      product.id !== 2
  );
```

For a kept object:

```js
originalProducts[0] ===
  products[0]
```

may return:

```text
true
```

The array is new, but retained objects usually keep their original references.

---

# 24. `filter()` Is a Shallow Collection Operation

`filter()` decides which array elements remain.

It does not:

* Clone objects.
* Deep-copy nested data.
* Modify matching objects.
* Automatically protect objects from mutation.

It creates a new array containing references to the selected values.

# 30. `filter()` Versus `some()`

## `filter()`

Returns a new array:

```js
const duplicates =
  products.filter(
    condition
  );
```

## `some()`

Returns a boolean:

```js
const duplicateExists =
  products.some(
    condition
  );
```

When you only need to know whether at least one match exists, `some()` is more appropriate.

# 31. Exactly Three Complete Examples

## Example 1: Basic Filtering Without the DOM

### Problem

Create a new array containing numbers greater than `10`.

### Input Data

```js
const numbers = [
  4,
  12,
  7,
  20,
];
```

### Complete Code

```js
const numbers = [
  4,
  12,
  7,
  20,
];

const largeNumbers =
  numbers.filter(
    (number) =>
      number > 10
  );

console.log(
  largeNumbers
);

```

1. `filter()` checks `4`.
2. `4 > 10` is false, so `4` is excluded.
3. It checks `12`.
4. `12 > 10` is true, so `12` is kept.
5. It checks `7`.
6. `7 > 10` is false.
7. It checks `20`.
8. `20 > 10` is true.
9. A new array is returned.

```text
[
  12,
  20
]

[
  4,
  12,
  7,
  20
]
```

### Final State

Original array:

```js
[
  4,
  12,
  7,
  20,
]
```

Filtered result:

```js
[
  12,
  20,
]
```

### Common Mistakes

* Expecting the original array to change.
* Forgetting to assign the result.
* Reversing the comparison.
* Using `find()` when all matches are required.

Keep only numbers less than or equal to `10`.

```html
      );

    const productList =
      document.getElementById(
        "product-list"
      );

    function renderProducts() {
      productList.innerHTML = "";

      productCount.textContent =
        `Total products: ${products.length}`;

      if (products.length === 0) {
        const emptyState =
          document.createElement(
            "li"
          );

        emptyState.classList.add(
          "empty-state"
        );

        emptyState.textContent =
          "No products available.";

        productList.appendChild(
          emptyState
        );

        return;
      }

      products.forEach(
        (product) => {
          const card =
            document.createElement(
              "li"
            );

          const name =
            document.createElement(
              "span"
            );

          const deleteButton =
            document.createElement(
              "button"
            );

          card.classList.add(
            "product-card"
          );

          name.textContent =
            product.name;

          deleteButton.type =
            "button";

          deleteButton.textContent =
            "Delete";

          deleteButton.dataset.action =
            "delete";

          deleteButton.dataset.id =
            product.id;

          card.append(
            name,
            deleteButton
          );

          productList.appendChild(
            card
          );
        }
      );
    }

    function deleteProductById(
      productId
    ) {
      const productToDelete =
        products.find(
          (product) =>
            product.id ===
            productId
        );

      if (
        productToDelete ===
        undefined
      ) {
        return {
          success: false,
          deletedProduct: null,
        };
      }

      products =
        products.filter(
          (product) =>
            product.id !==
            productId
        );

      return {
        success: true,
        deletedProduct:
          productToDelete,
      };
    }

    productList.addEventListener(
      "click",
      function (event) {
        const deleteButton =
          event.target.closest(
            '[data-action="delete"]'
          );

        if (
          deleteButton === null
        ) {
          return;
        }

        const productId =
          Number(
            deleteButton.dataset.id
          );

        if (
          !Number.isInteger(
            productId
          ) ||
          productId <= 0
        ) {
          message.textContent =
            "Invalid product ID.";

          return;
        }

        const product =
          products.find(
            (product) =>
              product.id ===
              productId
          );

        if (
          product === undefined
        ) {
          message.textContent =
            "Product not found.";

          renderProducts();

          return;
        }

        const confirmed =
          window.confirm(
            `Delete ${product.name}?`
          );

        if (!confirmed) {
          message.textContent =
            `${product.name} was not deleted.`;

          return;
        }

        const result =
          deleteProductById(
            productId
          );

        if (!result.success) {
          message.textContent =
            "Product could not be deleted.";

          renderProducts();

          return;
        }

        renderProducts();

        message.textContent =
          `${result.deletedProduct.name} was deleted.`;
      }
    );

    renderProducts();
  </script>
</body>
</html>
```

# 32. Common Mistakes

## Mistake 1: Reversed Delete Condition

```js
products =
  products.filter(
    (product) =>
      product.id === productId
  );
```

This keeps the selected item and removes the others.

---

## Mistake 2: Result Is Discarded

```js
products.filter(
  condition
);
```

State remains unchanged.

---

## Mistake 3: Missing Callback Return

```js
products =
  products.filter(
    (product) => {
      product.id !== id;
    }
  );
```

Every item is removed.

---

## Mistake 4: Expecting One Object

```js
const deletedProduct =
  products.filter(
    condition
  );
```

The result is always an array.

---

## Mistake 5: Expecting the Original Array to Change

`filter()` does not mutate the original array.

---

## Mistake 6: Replacing State During Search

Temporary filters should use a separate variable.

---

## Mistake 7: Assuming Missing IDs Throw Errors

A missing ID leaves all products in the result.

---

## Mistake 8: Forgetting Confirmation

Destructive actions may occur accidentally.

---

## Mistake 9: Forgetting to Reset Matching Edit Mode

The form may keep editing a missing object.

---

## Mistake 10: Forgetting to Re-render

The DOM remains stale after state replacement.

# 33. Best Practices

* Use `filter()` when producing a subset of an array.
* Think in terms of which items should remain.
* Use `!==` for deleting one matching ID.
* Reassign the returned array to state.
* Handle missing targets before deletion.
* Use `find()` when the removed object is needed.
* Confirm destructive actions before filtering state.
* Reset related UI state after deletion.
* Re-render after state replacement.
* Preserve stable IDs for remaining objects.
* Use a temporary result variable for search and view filters.
* Do not mutate objects inside the filter callback.
* Use `some()` when only a boolean is required.
* Use `find()` when only one object is required.

## Key Takeaways

* `filter()` returns a new array containing items whose callback returns `true`.
* It performs internal iteration over the entire array.
* The callback describes which items should remain.
* Immutable deletion keeps every item whose ID does not match the deleted ID.
* The returned array must be assigned to the state variable.
* `filter()` does not mutate the original array.
* A missing ID leaves the resulting contents unchanged.
* Use `find()` first when you need the deleted object or confirmation name.
* Reset edit mode when its product is removed.
* Re-render after state replacement.
* Search filters should normally use a temporary array rather than replace main state.
* Retained objects usually keep their existing references.
* Use `find()` for one match.
* Use `some()` when only a boolean existence check is needed.
* The next subtopic will use `some()` to detect duplicate names and validate whether at least one matching item exists.
