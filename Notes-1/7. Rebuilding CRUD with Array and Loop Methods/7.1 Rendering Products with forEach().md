# 1. What `forEach()` Does

## What You Will Learn

`forEach()` runs a callback function once for every item in an array.

It is commonly used when you want to perform an action for each item.

Examples include:

* Printing values.
* Rendering DOM elements.
* Adding event-related attributes.
* Updating external counters.
* Logging array contents.

It connects directly to CRUD because the Read operation often requires displaying every object in state.

---

## Concept Explanation

Suppose you have:

```js
const products = [
  "Laptop",
  "Phone",
  "Keyboard",
];
```

You want to print each product.

A plain loop works:

```js
for (
  let index = 0;
  index < products.length;
  index++
) {
  console.log(
    products[index]
  );
}
```

`forEach()` provides another way:

```js
products.forEach(
  (product) => {
    console.log(product);
  }
);
```

The array still gets traversed one item at a time.

`forEach()` is not magic.

Internally, JavaScript performs repeated processing similar to a loop.

---

## Why It Matters

`forEach()` is widely used in modern JavaScript because it:

* Reduces manual loop setup.
* Avoids manually incrementing an index.
* Makes the current item directly available.
* Often reads more clearly for simple repeated actions.

For rendering, it communicates:

```text
For each product, create and display one product card.
```

---

## Common Beginner Mistakes

* Expecting `forEach()` to create a new array.
* Forgetting that it returns `undefined`.
* Trying to use `break`.
* Forgetting to clear the DOM container.
* Mutating state during rendering.
* Using `innerHTML` with untrusted user input.

# 4. Plain Loop Versus `forEach()`

## Using a Plain Loop

```js
for (
  let index = 0;
  index < products.length;
  index++
) {
  const product =
    products[index];

  console.log(
    product.name
  );
}
```

## Using `forEach()`

```js
products.forEach(
  (product) => {
    console.log(
      product.name
    );
  }
);
```

---

## What Both Versions Do

Both versions:

* Visit every product.
* Access one product at a time.
* Can perform DOM rendering.
* Leave the array unchanged unless the callback mutates it.
* Run from the first item to the last item.

---

## Which Is Easier to Read?

For a straightforward operation on every item, `forEach()` is often easier to read.

```js
products.forEach(
  (product) => {
    renderProduct(product);
  }
);
```

This closely matches natural language:

```text
For each product, render the product.
```

---

## Which Gives More Control?

A standard `for` loop gives more control.

It allows:

* `break`.
* `continue`.
* Custom increments.
* Reverse iteration.
* Partial-array traversal.
* Changing the loop index manually.

---

## Which Is Common in Modern JavaScript?

`forEach()` is common when:

* Every item should be processed.
* No new array is needed.
* Early termination is unnecessary.
* The code performs a side effect such as DOM rendering.

---

## When a Plain Loop Is Still Better

Use a plain loop when:

* You need `break`.
* You need to stop after a match.
* You need custom index movement.
* You need to loop backwards.
* You need maximum control.

---

# 5. `forEach()` and Internal Looping

This code:

```js
products.forEach(
  (product) => {
    console.log(
      product.name
    );
  }
);
```

can be understood conceptually as:

```js
for (
  let index = 0;
  index < products.length;
  index++
) {
  const product =
    products[index];

  console.log(
    product.name
  );
}
```

The exact JavaScript engine implementation is more complex, but this mental model is useful.

> `forEach()` does not eliminate looping. It packages the looping behavior inside an array method.

---

# 6. `forEach()` Is Good for Side Effects

A side effect is an observable action performed outside the callback's local return value.

Examples:

```js
console.log(
  product.name
);
```

```js
productList.appendChild(
  card
);
```

```js
totalQuantity +=
  product.quantity;
```

Rendering the DOM is a side effect because it changes the webpage.

`forEach()` is therefore appropriate for rendering.

---

# 7. `forEach()` Does Not Transform the Array

Incorrect expectation:

```js
const names =
  products.forEach(
    (product) => {
      return product.name;
    }
  );
```

`names` becomes:

```js
undefined
```

`forEach()` ignores callback return values.

To create a new transformed array, use:

```js
map()
```

which will be taught later.

Changing the loop method does not change the rendering rules.

Without clearing:

```text

Phone
```

# 11. Empty-State Handling

`forEach()` does nothing when the array is empty.

Example:

```js
const products = [];

products.forEach(
  (product) => {
    console.log(
      product.name
    );
  }
);
```

No callback runs.

Therefore, empty state must be handled separately.

```js
if (products.length === 0) {
  renderEmptyState();
  return;
}
```

# 12. Reusable `renderProductCard()`

```js
function renderProductCard(
  product,
  index
) {
  const card =
    document.createElement("li");

  const name =
    document.createElement("h3");

  const price =
    document.createElement("p");

  const quantity =
    document.createElement("p");

  const status =
    document.createElement("p");

  card.classList.add(
    "product-card"
  );

  card.dataset.id =
    product.id;

  name.textContent =
    `${index + 1}. ${product.name}`;

  price.textContent =
    `Price: KSh ${product.price.toLocaleString()}`;

  quantity.textContent =
    `Quantity: ${product.quantity}`;

  status.textContent =
    product.quantity === 0
      ? "Status: Out of stock"
      : "Status: In stock";

  card.append(
    name,
    price,
    quantity,
    status
  );

  productList.appendChild(
    card
  );
}
```

The rendering function is independent of whether the caller uses:

* A standard `for` loop.
* `forEach()`.
* Another iteration method.

# 16. `forEach()` and Object References

The callback receives each array element.

For an array of objects, `product` refers to the same object stored in the array.

Example:

```js
products.forEach(
  (product) => {
    product.quantity++;
  }
);
```

This mutates every product.

Afterward, the original array contains changed quantities.

Therefore:

> Do not mutate product objects inside a render callback unless mutation is intentionally part of the operation.

Incorrect rendering:

```js
products.forEach(
  (product) => {
    product.quantity++;

    renderProductCard(
      product
    );
  }
);
```

Correct rendering:

```js
products.forEach(
  (product, index) => {
    renderProductCard(
      product,
      index
    );
  }
);
```

---

# 17. `forEach()` Cannot Use `break`

Incorrect:

```js
products.forEach(
  (product) => {
    if (product.id === 2) {
      break;
    }
  }
);
```

This causes a syntax error.

Use:

* A standard loop.
* `find()`.
* `findIndex()`.
* `some()`.

depending on the goal.

---

# 18. Does `return` Stop `forEach()`?

Inside a callback:

```js
products.forEach(
  (product) => {
    if (
      product.quantity === 0
    ) {
      return;
    }

    console.log(
      product.name
    );
  }
);
```

The `return` exits only the current callback execution.

It behaves somewhat like skipping the current item.

It does not stop the entire `forEach()`.

## Mistake 9: Using `map()` Only for Rendering

Although `map()` can create HTML-related values, `forEach()` is more appropriate when no transformed array is required.

---

## Mistake 10: Using the Index as Identity

The callback index may change after:

* Deletion.
* Sorting.
* Filtering.

Use the stable product ID for Edit and Delete.

---

# 21. Best Practices

* Use `forEach()` when every item must be processed.
* Use meaningful callback names such as `product`.
* Include the index only when needed.
* Keep card creation in a separate function.
* Clear the list before rendering.
* Handle the empty state before `forEach()`.
* Use stable IDs in `dataset`.
* Use `textContent` for user-controlled values.
* Keep rendering free from state mutation.
* Attach delegated listeners once.
* Do not use `forEach()` when you need early termination.
* Use `map()` when you need a transformed array.

## Key Takeaways

* `forEach()` runs a callback once for every array element.
* It performs internal looping rather than removing the need for loops.
* The callback receives the current item and optionally its index.
* `forEach()` is appropriate for side effects such as DOM rendering.
* It returns `undefined`.
* It does not create a transformed array.
* It cannot be stopped with `break`.
* A callback `return` skips only the current callback execution.
* Rendering rules remain the same: clear, handle empty state, build elements, and append.
* Stable IDs must be used for CRUD actions.
* Rendering must not accidentally mutate state.
* The next subtopic will use `find()` to locate one product by ID.
