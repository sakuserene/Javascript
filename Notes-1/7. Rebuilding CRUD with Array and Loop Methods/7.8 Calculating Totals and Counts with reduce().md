# 1. What `reduce()` Does

## What You Will Learn

`reduce()` processes every item in an array and combines them into one final result.

That result may be:

* A number.
* A string.
* An object.
* Another array.
* A grouped data structure.
* A summary.

Common uses include:

* Adding numbers.
* Calculating totals.
* Counting items.
* Building summary objects.
* Grouping items by category.
* Creating lookup objects.
* Finding minimum or maximum values.

---

## Concept Explanation

Suppose:

```js
const numbers = [
  10,
  20,
  30,
];
```

You want:

```text
60
```

Using a plain loop:

```js
let total = 0;

for (
  let index = 0;
  index < numbers.length;
  index++
) {
  total +=
    numbers[index];
}
```

Using `reduce()`:

```js
const total =
  numbers.reduce(
    (
      runningTotal,
      number
    ) =>
      runningTotal +
      number,
    0
  );
```

`reduce()` carries a running result from one callback execution to the next.

---

## Why It Matters for CRUD

CRUD applications frequently display derived information.

Examples:

```text
Total number of products
Total units in stock
Total inventory value
Total completed tasks
Total expenses
Total shopping-cart value
```

These values should usually be calculated from the main state rather than stored independently.

`reduce()` is useful for producing this derived state.

---

## Common Beginner Mistakes

* Forgetting the initial value.
* Confusing the accumulator with the current item.
* Returning the current item instead of the updated accumulator.
* Forgetting to return anything.
* Using the wrong initial accumulator type.
* Mutating product objects during calculation.
* Storing totals separately and letting them become stale.
* Writing overly complex `reduce()` callbacks.

# 4. Understanding the Accumulator

The accumulator is the value carried forward during reduction.

Example:

```js
const numbers = [
  10,
  20,
  30,
];
```

```js
const total =
  numbers.reduce(
    (
      accumulator,
      number
    ) =>
      accumulator +
      number,
    0
  );
```

Process:

```text
Initial accumulator: 0

First item:
0 + 10 = 10

Second item:
10 + 20 = 30

Third item:
30 + 30 = 60

Final result:
60
```

```js
numbers.reduce(

  0

```

The initial value determines:

* The accumulator's starting value.
* Often the accumulator's expected type.
* How empty arrays behave.

For totals, use:

```js
0
```

For text, you may use:

```js
""
```

For an object summary:

```js
{
  total: 0,
  count: 0,
}
```

For grouping:

```js
{}
```

For building an array:

```js
[]
```

# 6. Plain Loop Versus `reduce()`

## Using a Plain Loop

```js
let totalUnits = 0;

for (
  let index = 0;
  index < products.length;
  index++
) {
  totalUnits +=
    products[index].quantity;
}
```

## Using `reduce()`

```js
const totalUnits =
  products.reduce(
    (
      total,
      product
    ) =>
      total +
      product.quantity,
    0
  );
```

---

## What Both Versions Do

Both:

* Visit every product.
* Read each quantity.
* Maintain a running total.
* Return or produce one final number.
* Leave product state unchanged.

---

## Which Is Easier to Read?

For a simple total, `reduce()` can be concise and expressive.

```js
const total =
  products.reduce(
    (sum, product) =>
      sum + product.quantity,
    0
  );
```

---

## Which Gives More Control?

A plain loop may be clearer when:

* Several conditions are involved.
* Debugging each step is important.
* The reducer becomes deeply nested.
* Several unrelated side effects occur.
* The accumulator structure is difficult to understand.
* The learner is still mastering the underlying process.

> `reduce()` should improve clarity, not merely shorten code.

---

# 7. `reduce()` Performs Internal Iteration

This:

```js
const totalUnits =
  products.reduce(
    (
      total,
      product
    ) =>
      total +
      product.quantity,
    0
  );
```

can be understood as:

```js
let totalUnits = 0;

for (
  let index = 0;
  index < products.length;
  index++
) {
  totalUnits =
    totalUnits +
    products[index].quantity;
}
```

`reduce()` packages the accumulator loop into an array method.

# 10. Counting Items with `reduce()`

Count out-of-stock products:

```js
const outOfStockCount =
  products.reduce(
    (
      count,
      product
    ) =>
      product.quantity === 0
        ? count + 1
        : count,
    0
  );
```

---

## Syntax Explanation

```js
product.quantity === 0
  ? count + 1
  : count
```

When out of stock:

```js
count + 1
```

Otherwise:

```js
count
```

# 11. Counting Completed Tasks

```js
const completedTaskCount =
  tasks.reduce(
    (
      count,
      task
    ) =>
      task.completed
        ? count + 1
        : count,
    0
  );
```

```js
const incompleteTaskCount =
  tasks.reduce(
    (
      count,
      task
    ) =>
      !task.completed

        : count,
    0

```

# 12. Total Products Does Not Require `reduce()`

To count every product:

```js
products.length
```

is clearer than:

```js
products.reduce(
  (count) =>
    count + 1,
  0
);
```

Although both work, prefer the simplest tool.

Use:

```js
products.length
```

for total records.

Use `reduce()` when the count depends on a condition.

# 13. Building a Summary Object

Instead of running several separate reductions, you can calculate several values in one pass.

```js
const inventorySummary =
  products.reduce(
    (
      summary,
      product
    ) => {
      summary.totalUnits +=
        product.quantity;

      summary.totalInventoryValue +=
        product.price *
        product.quantity;

      if (
        product.quantity === 0
      ) {
        summary.outOfStockCount++;
      }

      return summary;
    },
    {
      totalUnits: 0,
      totalInventoryValue: 0,
      outOfStockCount: 0,
    }
  );
```

---

# 14. Summary Result

For the sample products:

```js
{
  totalUnits: 7,
  totalInventoryValue: 225000,
  outOfStockCount: 1
}
```

Add total products separately:

```js
const fullSummary = {
  totalProducts:
    products.length,
  ...inventorySummary,
};
```

---

# 15. Mutable Accumulator Object

The previous example mutates the accumulator object:

```js
summary.totalUnits +=
  product.quantity;
```

This is usually acceptable because the accumulator is local to the reduction.

It does not mutate product state.

---

## Important Difference

This mutates the accumulator:

```js
summary.totalUnits +=
  product.quantity;
```

This would mutate application state:

```js
product.quantity++;
```

The first can be intentional.

The second is inappropriate during summary calculation.

---

# 16. Immutable Accumulator Alternative

```js
const inventorySummary =
  products.reduce(
    (
      summary,
      product
    ) => ({
      totalUnits:
        summary.totalUnits +
        product.quantity,

      totalInventoryValue:
        summary.totalInventoryValue +
        product.price *
          product.quantity,

      outOfStockCount:
        summary.outOfStockCount +
        (
          product.quantity === 0
            ? 1
            : 0
        ),
    }),
    {
      totalUnits: 0,
      totalInventoryValue: 0,
      outOfStockCount: 0,
    }
  );
```

This creates a new summary object for every callback execution.

It is more immutable but may be harder for beginners to read.

# 18. Missing the Initial Value

This works for some non-empty numeric arrays:

```js
const total =
  numbers.reduce(
    (
      accumulator,
      number
    ) =>
      accumulator +
      number
  );
```

JavaScript uses the first array item as the initial accumulator.

However, this can create confusion and errors.

---

## Empty Array Problem

```js
const numbers = [];

numbers.reduce(
  (
    total,
    number
  ) =>
    total + number
);
```

This throws an error because there is no first item.

Using:

```js
0
```

solves the problem.

```js
const total =
  numbers.reduce(
    (
      total,
      number
    ) =>
      total + number,
    0
  );
```

Result:

```text
0
```

---

# 19. Best Practice: Usually Provide an Initial Value

For beginner-friendly and reliable code:

```js
const total =
  values.reduce(
    reducer,
    0
  );
```

Benefits:

* Handles empty arrays.
* Makes the accumulator type clear.
* Makes behavior predictable.
* Reduces type-related mistakes.

Incorrect:

```js

  products.reduce(
    (
      total,
      product
```

```js
      product.quantity,
```


# 28. Derived State

Values such as:

```text
Total units
Inventory value
Completed task count
Out-of-stock count
Average price
```

are derived state.

They are calculated from the main state.

Prefer:

```js
const totalUnits =
  products.reduce(...);
```

instead of separately maintaining:

```js
let totalUnits = 7;
```

A separately stored total can become stale after Create, Update, or Delete.

---

# 29. Rendering a Summary with `reduce()`

```js
function calculateInventorySummary() {
  return products.reduce(
    (
      summary,
      product
    ) => {
      summary.totalUnits +=
        product.quantity;

      summary.totalInventoryValue +=
        product.price *
        product.quantity;

      if (
        product.quantity === 0
      ) {
        summary.outOfStockCount++;
      }

      return summary;
    },
    {
      totalUnits: 0,
      totalInventoryValue: 0,
      outOfStockCount: 0,
    }
  );
}
```

Render:

```js
function renderSummary() {
  const summary =
    calculateInventorySummary();

  totalProductCount.textContent =
    products.length;

  totalUnitCount.textContent =
    summary.totalUnits;

  totalInventoryValue.textContent =
    `KSh ${summary.totalInventoryValue.toLocaleString()}`;

  outOfStockCount.textContent =
    summary.outOfStockCount;
}
```

# 30. Recalculate After Every State Change

After Create:

```js
products = [
  ...products,
  newProduct,
];

renderProducts();
```

After Update:

```js
products =
  products.map(...);

renderProducts();
```

After Delete:

```js
products =
  products.filter(...);

renderProducts();
```

If `renderProducts()` calls `renderSummary()`, all derived totals stay synchronized.

# 31. `reduce()` Should Not Change Products

Incorrect:

```js
const total =
  products.reduce(
    (
      total,
      product
    ) => {
      product.quantity++;

      return (
        total +
        product.quantity
      );
    },
    0
  );
```

This changes inventory state while calculating a total.

Correct:

```js
const total =
  products.reduce(
    (
      total,
      product
    ) =>
      total +
      product.quantity,
    0
  );
```

# 32. `reduce()` Versus Other Array Methods

| Goal                           | Appropriate Method |
| ------------------------------ | ------------------ |
| Render every product           | `forEach()`        |
| Transform every product        | `map()`            |
| Find one product               | `find()`           |
| Find one position              | `findIndex()`      |
| Keep matching products         | `filter()`         |
| Check whether one match exists | `some()`           |
| Check whether all items pass   | `every()`          |
| Combine items into one result  | `reduce()`         |

## Example 1: Basic Sum Without the DOM

### Problem

Calculate the sum of all numbers.

### Input Data

```js
const numbers = [
  10,
  20,
  30,
];
```

### Complete Code

```js
const numbers = [
  10,
  20,
  30,
];

const total =
  numbers.reduce(
    (
      runningTotal,
      number
    ) =>
      runningTotal +
      number,
    0
  );

console.log(total);
```

### Step-by-Step Explanation

1. The accumulator starts at `0`.
2. The first number is `10`.
3. `0 + 10` returns `10`.
4. The accumulator becomes `10`.
5. The second number is `20`.
6. `10 + 20` returns `30`.
7. The third number is `30`.
8. `30 + 30` returns `60`.
9. `reduce()` returns `60`.

### Expected Console Output

```text
60
```

### Final State

```js
[
  10,
  20,
  30,
]
```

The array remains unchanged.

### Common Mistakes

* Forgetting the initial value.
* Forgetting to return the new total.
* Returning only the current number.
* Using `""` instead of `0`.

### Problem

Render products and calculate summary values from current state.

### Complete Code

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8" />

  <meta
    name="viewport"
    content="width=device-width, initial-scale=1.0"
  />

  <title>
    Inventory Totals with reduce
  </title>

  <style>
    body {
      max-width: 900px;
      margin: 0 auto;
      padding: 24px;
      font-family: Arial, sans-serif;
      line-height: 1.5;
    }

    .summary-grid {
      display: grid;
      grid-template-columns:
        repeat(
          auto-fit,
          minmax(160px, 1fr)
        );
      gap: 12px;
      margin-bottom: 20px;
    }

    .summary-card,
    .product-card {
      padding: 14px;
      border: 1px solid #cccccc;
      border-radius: 8px;
    }

    #product-list {
      display: grid;
      gap: 12px;
      padding: 0;
      list-style: none;
    }
  </style>
</head>

<body>
  <h1>Product Inventory</h1>

  <section class="summary-grid">
    <article class="summary-card">
      <h2>Total Products</h2>
      <p id="total-products">0</p>
    </article>

    <article class="summary-card">
      <h2>Total Units</h2>
      <p id="total-units">0</p>
    </article>

    <article class="summary-card">
      <h2>Inventory Value</h2>
      <p id="inventory-value">
        KSh 0
      </p>
    </article>

    <article class="summary-card">
      <h2>Out of Stock</h2>
      <p id="out-of-stock">0</p>
    </article>
  </section>

  <ul id="product-list"></ul>

  <script>
    const products = [
      {
        id: 1,
        name: "Laptop",
        price: 50000,
        quantity: 2,
      },
      {
        id: 2,
        name: "Phone",
        price: 25000,
        quantity: 5,
      },
      {
        id: 3,
        name: "Keyboard",
        price: 3000,
        quantity: 0,
      },
    ];

    const totalProductsElement =
      document.getElementById(
        "total-products"
      );

    const totalUnitsElement =
      document.getElementById(
        "total-units"
      );

    const inventoryValueElement =
      document.getElementById(
        "inventory-value"
      );

    const outOfStockElement =
      document.getElementById(
        "out-of-stock"
      );

    const productList =
      document.getElementById(
        "product-list"
      );

    function calculateSummary() {
      return products.reduce(
        (
          summary,
          product
        ) => {
          summary.totalUnits +=
            product.quantity;

          summary.inventoryValue +=
            product.price *
            product.quantity;

          if (
            product.quantity ===
            0
          ) {
            summary.outOfStock++;
          }

          return summary;
        },
        {
          totalUnits: 0,
          inventoryValue: 0,
          outOfStock: 0,
        }
      );
    }

    function renderSummary() {
      const summary =
        calculateSummary();

      totalProductsElement
        .textContent =
        products.length;

      totalUnitsElement
        .textContent =
        summary.totalUnits;

      inventoryValueElement
        .textContent =
        `KSh ${summary.inventoryValue.toLocaleString()}`;

      outOfStockElement
        .textContent =
        summary.outOfStock;
    }

    function renderProducts() {
      productList.innerHTML = "";

      products.forEach(
        (product, index) => {
          const card =
            document.createElement(
              "li"
            );

          const name =
            document.createElement(
              "h2"
            );

          const details =
            document.createElement(
              "p"
            );

          card.classList.add(
            "product-card"
          );

          name.textContent =
            `${index + 1}. ${product.name}`;

          details.textContent =
            `KSh ${product.price.toLocaleString()} × ` +
            `${product.quantity}`;

          card.append(
            name,
            details
          );

          productList.appendChild(
            card
          );
        }
      );

      renderSummary();
    }

    renderProducts();
  </script>
</body>
</html>
```

# 34. Common Mistakes

## Mistake 1: Forgetting the Initial Value

This can break on empty arrays or create confusing accumulator types.

---

## Mistake 2: Missing Return

```js
products.reduce(
  (
    total,
    product
  ) => {
    total +
      product.quantity;
  },
  0
);
```

---

## Mistake 3: Returning Only the Current Value

```js
return product.quantity;
```

The previous total is lost.

---

## Mistake 4: Wrong Initial Type

```js
""
```

causes string concatenation during numeric addition.

---

## Mistake 5: Resetting the Accumulator

Incorrect:

```js
products.reduce(
  (
    total,
    product
  ) => {
    total = 0;

    return (
      total +
      product.quantity
    );
  },
  0
);
```

---

## Mistake 6: Mutating Product State

Summary calculation should be read-only.

---

## Mistake 7: Using `reduce()` for Simple Length

Prefer:

```js
products.length
```

---

## Mistake 8: Overcomplicating the Reducer

Several clear reductions can be better than one unreadable reduction.

---

## Mistake 9: Storing Derived Totals Permanently

Totals can become stale after CRUD changes.

---

## Mistake 10: Forgetting to Re-render Totals

State changes but displayed summaries remain outdated.

---

# 35. Best Practices

* Use `reduce()` when combining an array into one result.
* Provide an explicit initial value.
* Name the accumulator according to its purpose.
* Return the accumulator on every callback execution.
* Use numeric initial values for numeric totals.
* Use object accumulators for related summaries.
* Keep reducers free from product-state mutation.
* Prefer `length` for simple total-item counts.
* Keep reducers readable.
* Extract complex reducer logic into a named function.
* Treat totals and summaries as derived state.
* Recalculate summaries after state changes.
* Use a plain loop when it communicates the logic more clearly.

## Key Takeaways

* `reduce()` combines an array into one final result.
* The accumulator stores the result built so far.
* The initial value defines the accumulator's starting value and often its type.
* Numeric totals usually begin with `0`.
* Object summaries begin with an object containing initial fields.
* The callback must return the next accumulator.
* Providing an initial value makes empty arrays safe.
* `reduce()` can calculate totals, counts, groups, averages, and summaries.
* Product state should not be mutated during reduction.
* Totals should normally be derived from the current state.
* Recalculate summaries after Create, Update, and Delete.
* Use `products.length` for a simple total product count.
* Use a plain loop when it produces clearer code.
