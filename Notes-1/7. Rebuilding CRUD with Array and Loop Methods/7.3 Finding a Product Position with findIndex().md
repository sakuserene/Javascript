# 1. What `findIndex()` Does

## What You Will Learn

`findIndex()` searches an array and returns the position of the first element that satisfies a condition.

It is useful when you need:

```text
The matching item's array index
```

rather than:

* The matching object.
* A boolean.
* All matching objects.
* A transformed array.

In CRUD applications, `findIndex()` is commonly useful for:

* Mutable deletion with `splice()`.
* Replacing one array entry.
* Inserting before or after a selected item.
* Moving an item.
* Working with neighboring items.

---

## Concept Explanation

Suppose state contains:

```js
const products = [
  {
    id: 101,
    name: "Laptop",
  },
  {
    id: 205,
    name: "Phone",
  },
  {
    id: 309,
    name: "Keyboard",
  },
];
```

You need the current position of product ID:

```js
205
```

Expected result:

```text
1
```

The ID is:

```text
205
```

The index is:

```text
1
```

These values are not interchangeable.

---

## Why It Matters

Many mutable array operations use positions.

For example:

```js
products.splice(
  selectedIndex,
  1
);
```

Before calling `splice()`, the application must know the selected product's current index.

`findIndex()` expresses that goal clearly:

```text
Find the position of the first product whose ID matches the selected ID.
```

---

## Common Beginner Mistakes

* Assuming the returned index is the product ID.
* Assuming the returned value is the product object.
* Forgetting that a missing result is `-1`.
* Passing `-1` directly to `splice()`.
* Rejecting index `0` with a falsy check.
* Comparing numeric IDs with string dataset values.
* Forgetting `return` inside a callback with braces.
* Storing an index for too long.

```js
true
```

```js
product
```

# 4. Plain Loop Versus `findIndex()`

## Using a Plain Loop

```js
let productIndex = -1;

for (
  let index = 0;
  index < products.length;
  index++
) {
  if (
    products[index].id ===
    productId
  ) {
    productIndex = index;
    break;
  }
}
```

## Using `findIndex()`

```js
const productIndex =
  products.findIndex(
    (product) =>
      product.id === productId
  );
```

---

## What Both Versions Do

Both:

* Search from the beginning.
* Compare one product at a time.
* Stop after the first match.
* Return or store the matching position.
* Produce a missing value when no match exists.
* Leave the array unchanged during lookup.

---

## Which Is Easier to Read?

For a simple condition, `findIndex()` is usually easier to read.

```js
const index =
  products.findIndex(
    (product) =>
      product.id === id
  );
```

The intention is immediately visible.

---

## Which Gives More Control?

A plain loop is more flexible when you need:

* Reverse searching.
* Custom increments.
* Search ranges.
* Multiple return values.
* Logging at every step.
* Unusual stopping rules.

---

# 5. `findIndex()` Performs Internal Iteration

This:

```js
const productIndex =
  products.findIndex(
    (product) =>
      product.id === productId
  );
```

can be understood as:

```js
let productIndex = -1;

for (
  let index = 0;
  index < products.length;
  index++
) {
  if (
    products[index].id ===
    productId
  ) {
    productIndex = index;
    break;
  }
}
```

`findIndex()` packages the repeated search into an array method.

## `find()`

```js
      product.id === productId
  );
```

Returns:

```js
{
  id: 205,
  name: "Phone",

```

or:

```js
undefined
```

---

## `findIndex()`

```js
const productIndex =
  products.findIndex(
    (product) =>
      product.id === productId
  );
```

Returns:

```text
1
```

or:

```text
-1
```

---

## Use `find()` When

You need:

* Product properties.
* Product details.
* A name for confirmation.
* Values for an edit form.
* Direct mutable access to the object.

---

## Use `findIndex()` When

You need:

* A position for `splice()`.
* A position for replacement.
* A neighboring element.
* Insertion before or after a product.
* Mutable array-position operations.

# 7. ID Versus Index

State:

```js
const products = [
  {
    id: 101,
    name: "Laptop",
  },
  {
    id: 205,
    name: "Phone",
  },
  {
    id: 309,
    name: "Keyboard",
  },
];
```

| Product  | Stable ID | Current Index |
| -------- | --------: | ------------: |
| Laptop   |     `101` |           `0` |
| Phone    |     `205` |           `1` |
| Keyboard |     `309` |           `2` |

Do not write:

```js
products.splice(
  productId,
  1
);
```

When:

```js
productId === 205
```

JavaScript tries to delete at index `205`.

---

# 8. Indexes Can Change

Initial state:

```js
[
  {
    id: 101,
    name: "Laptop",
  },
  {
    id: 205,
    name: "Phone",
  },
  {
    id: 309,
    name: "Keyboard",
  },
]
```

Phone index:

```text
1
```

After Laptop is deleted:

```js
[
  {
    id: 205,
    name: "Phone",
  },
  {
    id: 309,
    name: "Keyboard",
  },
]
```

Phone index:

```text
0
```

Phone ID remains:

```text
205
```

Therefore:

> Store stable IDs for user actions and calculate the current index when needed.

---

# 9. Handling `-1`

Incorrect:

```js
const productIndex =
  products.findIndex(
    (product) =>
      product.id === 999
  );

products.splice(
  productIndex,
  1
);
```

This becomes:

```js
products.splice(-1, 1);
```

and removes the last product.

Correct:

```js
if (productIndex === -1) {
  showError(
    "Product not found."
  );

  return;
}

products.splice(
  productIndex,
  1
);
```

---

# 10. Why `if (!productIndex)` Is Wrong

Consider:

```js
const productIndex = 0;
```

This is a valid result for the first product.

But:

```js
!productIndex
```

is:

```js
true
```

because zero is falsy.

Incorrect:

```js
if (!productIndex) {
  showError(
    "Product not found."
  );
}
```

Correct:

```js
if (
  productIndex === -1
) {
  showError(
    "Product not found."
  );
}
```

---

# 11. Missing `return` Bug

Incorrect:

```js
const productIndex =
  products.findIndex(
    (product) => {
      product.id ===
        productId;
    }
  );
```

The callback returns `undefined` for every element.

Result:

```text
-1
```

Correct:

```js
const productIndex =
  products.findIndex(
    (product) => {
      return (
        product.id ===
        productId
      );
    }
  );
```

or:

```js
const productIndex =
  products.findIndex(
    (product) =>
      product.id === productId
  );
```

# 12. Deleting with `findIndex()` and `splice()`

```js
function deleteProductById(
  productId
) {
  const productIndex =
    products.findIndex(
      (product) =>
        product.id ===
        productId
    );

  if (
    productIndex === -1
  ) {
    return null;
  }

  const removedProducts =
    products.splice(
      productIndex,
      1
    );

  return removedProducts[0];
}
```

---

# 13. Full Mutable Delete Flow

```js
function handleDeleteRequest(
  productId
) {
  const product =
    products.find(
      (product) =>
        product.id ===
        productId
    );

  if (
    product === undefined
  ) {
    showError(
      "Product not found."
    );

    return;
  }

  const confirmed =
    window.confirm(
      `Delete ${product.name}?`
    );

  if (!confirmed) {
    return;
  }

  const productIndex =
    products.findIndex(
      (product) =>
        product.id ===
        productId
    );

  if (
    productIndex === -1
  ) {
    showError(
      "Product no longer exists."
    );

    return;
  }

  products.splice(
    productIndex,
    1
  );

  renderProducts();
}
```

`find()` retrieves the object.

`findIndex()` retrieves its position.

# 14. Replacing an Array Entry

Suppose you want to replace one full product object.

```js
const productIndex =
  products.findIndex(
    (product) =>
      product.id ===
      updatedProduct.id
  );

if (
  productIndex !== -1
) {
  products[productIndex] =
    updatedProduct;
}
```

This replaces the object at that position.

---

# 15. Preserve the Stable ID During Replacement

Suppose form data contains:

```js
{
  name: "Smartphone",
  price: 28000,
  quantity: 7,
}
```

Create the replacement:

```js
const replacementProduct = {
  id: editingProductId,
  name: data.name,
  price: data.price,
  quantity: data.quantity,
};
```

Then:

```js
products[productIndex] =
  replacementProduct;
```

The ID remains stable.

---

# 16. Direct Property Update Versus Replacement

## Direct Mutation

```js
products[productIndex].name =
  data.name;

products[productIndex].price =
  data.price;
```

The same object remains in the array.

---

## Object Replacement

```js
products[productIndex] = {
  ...products[productIndex],
  name: data.name,
  price: data.price,
};
```

A new object replaces the old object at the same index.

The array itself is still mutated because one position is assigned.

---

# 17. When `findIndex()` Is Unnecessary

For an immutable Update using `map()`:

```js
products =
  products.map(
    (product) =>
      product.id === productId
        ? {
            ...product,
            name: newName,
          }
        : product
  );
```

No position is required.

For immutable Delete using `filter()`:

```js
products =
  products.filter(
    (product) =>
      product.id !== productId
  );
```

Again, no index is required.

`findIndex()` is most useful when an operation specifically needs a position.

# 19. Validate the Dataset ID

```js
    let products = [
      {
        id: 101,
```

# 21. Common Mistakes

## Mistake 1: Expecting the Object

```js
const product =
  products.findIndex(
    condition
  );
```

The result is a number, not a product.

---

## Mistake 2: Expecting `undefined`

A missing result is:

```js
-1
```

not:

```js
undefined
```

---

## Mistake 3: Passing `-1` to `splice()`

This removes the last product.

---

## Mistake 4: Treating Index Zero as Missing

```js
if (!index)
```

is incorrect.

---

## Mistake 5: Using the ID as the Index

IDs and positions are different values.

---

## Mistake 6: Missing Callback Return

The search always produces `-1`.

---

## Mistake 7: Storing Long-Lived Indexes

Indexes can change when the array changes.

---

## Mistake 8: Using `findIndex()` When Only Existence Is Needed

Use `some()` for a boolean result.

---

## Mistake 9: Using `findIndex()` for Immutable Deletion

`filter()` will be clearer when replacing the whole array.

---

## Mistake 10: Searching the Filtered DOM Position

Always locate the product in the main state using its stable ID.

---

# 22. Best Practices

* Use `findIndex()` only when a position is actually required.

## Key Takeaways

* `findIndex()` returns the position of the first element that satisfies a condition.
* It returns `-1` when no match exists.
* Index `0` is valid and must not be treated as missing.
* IDs identify objects; indexes identify current positions.
* Indexes may change while stable IDs remain the same.
* `findIndex()` performs internal iteration and stops at the first match.
* It is useful for `splice()`, replacement, insertion, and movement.
* Always check for `-1` before using the result.
* Never pass a stable ID directly to `splice()`.
* `find()` returns the object; `findIndex()` returns its position.
* Immutable `map()` and `filter()` operations often do not need an index.
* The next subtopic will update state immutably using `map()`.
