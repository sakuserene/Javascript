# 18. Removing an Item with `splice()`

## 18.1 What You Will Learn

The Delete operation changes the JavaScript state by removing an object.

With plain mutable arrays, a common approach is:

```js
products.splice(index, 1);
```

This means:

```text
Start at the specified index.
Remove exactly one item.
```

The complete basic flow is:

```text
Read selected ID
        ↓
Find its current index
        ↓
Check that index is not -1
        ↓
Remove one item with splice()
        ↓
Re-render the DOM
```

---

# 18.2 Basic `splice()` Syntax

```js
array.splice(startIndex, deleteCount);
```

For deleting one product:

```js
products.splice(selectedIndex, 1);
```

## `selectedIndex`

The position where deletion begins.

## `1`

The number of items to remove.

---

# 18.3 Simple Example

```js
const items = [
  "Laptop",
  "Phone",
  "Keyboard",
];

items.splice(1, 1);

console.log(items);
```

Expected output:

```text
[
  "Laptop",
  "Keyboard"
]
```

The item at index `1` was removed.

---

# 18.4 `splice()` Mutates the Original Array

Before:

```js
const items = [
  "Laptop",
  "Phone",
];
```

After:

```js
items.splice(1, 1);
```

The same array becomes:

```js
[
  "Laptop"
]
```

`splice()` does not merely return a changed copy.

It changes the original array.

This is called mutation.

---

# 18.5 What `splice()` Returns

`splice()` returns an array containing the removed items.

```js
const removedItems =
  products.splice(1, 1);
```

If Phone was removed:

```js
removedItems
```

contains:

```js
[
  {
    id: 205,
    name: "Phone",
  },
]
```

Even when one item is removed, the return value is still an array.

---

# 18.6 Accessing the Removed Object

```js
const removedItems =
  products.splice(
    selectedIndex,
    1
  );

const removedProduct =
  removedItems[0];
```

Now:

```js
removedProduct.name
```

may be used in a success message.

```js
showSuccess(
  `${removedProduct.name} was deleted.`
);
```

# 18.7 Delete by ID Safely

```js
function deleteProductById(
  productId
) {
  const productIndex =
    findProductIndexById(
      productId
    );

  if (productIndex === -1) {
    return null;
  }

  const removedItems =
    products.splice(
      productIndex,
      1
    );

  return removedItems[0];
}
```

Usage:

```js
const deletedProduct =
  deleteProductById(205);
```

Success result:

```js
{
  id: 205,
  name: "Phone",
}
```

Failure result:

```text
null
```

---

# 18.8 Why Check for `-1` First?

Incorrect:

```js
const index =
  findProductIndexById(999);

products.splice(index, 1);
```

If the result is:

```text
-1
```

then:

```js
products.splice(-1, 1);
```

removes the last item.

Correct:

```js
if (index === -1) {
  showError(
    "Product not found."
  );

  return;
}

products.splice(index, 1);
```

# 18.9 Deleting the First Item

State:

```js
[
  "Laptop",
  "Phone",
  "Keyboard"
]
```

Delete index `0`:

```js
items.splice(0, 1);
```

Result:

```js
[
  "Phone",
  "Keyboard"
]
```

Remember that index `0` is valid.

Do not test it with:

```js
if (!index)
```

---

# 18.10 Deleting the Last Item

State:

```js
[
  "Laptop",
  "Phone",
  "Keyboard"
]
```

Last index:

```js
items.length - 1;
```

Delete:

```js
items.splice(
  items.length - 1,
  1
);
```

Result:

```js
[
  "Laptop",
  "Phone"
]
```

---

# 18.11 Deleting Multiple Items

`splice()` can remove more than one item.

```js
items.splice(1, 2);
```

This means:

```text
Start at index 1.
Remove 2 items.
```

For CRUD deletion of one selected item, normally use:

```js
splice(index, 1);
```

Using a larger number may remove unintended records.

---

# 18.12 What Happens with Delete Count Zero?

```js
items.splice(1, 0);
```

This removes nothing.

Final array remains unchanged.

For deletion, the second argument must usually be:

```text
1
```

---

# 18.13 What Happens If Delete Count Is Missing?

```js
items.splice(1);
```

This removes everything from index `1` to the end.

Example:

```js
[
  "Laptop",
  "Phone",
  "Keyboard",
]
```

After:

```js
items.splice(1);
```

state becomes:

```js
[
  "Laptop"
]
```

> **Warning:** Always include the delete count when removing one CRUD item.

# 18.14 Complete Plain-Loop Deletion

```js
function findProductIndexById(
  productId
) {
  for (
    let index = 0;
    index < products.length;
    index++
  ) {
    if (
      products[index].id ===
      productId
    ) {
      return index;
    }
  }

  return -1;
}

function deleteProductById(
  productId
) {
  const productIndex =
    findProductIndexById(
      productId
    );

  if (productIndex === -1) {
    return null;
  }

  const removedProducts =
    products.splice(
      productIndex,
      1
    );

  return removedProducts[0];
}
```

---

# 18.15 Re-rendering After Deletion

State changes first:

```js
products.splice(index, 1);
```

Then:

```js
renderProducts();
```

Correct order:

```text
Remove from state
        ↓
Render current state
```

Incorrect:

```js
renderProducts();

products.splice(index, 1);
```

The deleted product remains visible until another render occurs.

---

# 18.16 Product Count After Deletion

The count should be derived from:

```js
products.length
```

Inside rendering:

```js
productCount.textContent =
  `Total products: ${products.length}`;
```

Before deletion:

```text
Total products: 3
```

After deletion:

```text
Total products: 2
```

No separate count mutation is needed.

---

# 18.17 Empty State After Deleting the Last Item

Suppose state contains:

```js
[
  {
    id: 101,
    name: "Laptop",
  },
]
```

Delete Laptop.

State becomes:

```js
[]
```

The render function should display:

```text
No products available.
```

Example:

```js
if (products.length === 0) {
  renderEmptyState();
  return;
}
```

# 18.18 Delete Buttons and Event Delegation

Each product card:

```js
deleteButton.dataset.action =
  "delete";

deleteButton.dataset.id =
  product.id;
```

Parent listener:

```js
productList.addEventListener(
  "click",
  function (event) {
    const deleteButton =
      event.target.closest(
        '[data-action="delete"]'
      );

    if (deleteButton === null) {
      return;
    }

    const productId =
      Number(
        deleteButton.dataset.id
      );

    deleteProductById(
      productId
    );
  }
);
```

Validation and confirmation will be improved in the next subtopic.

# 18.19 Deleting While an Item Is Being Edited

Suppose:

```js
editingProductId = 205;
```

Then product `205` is deleted.

The form still appears to edit an item that no longer exists.

A safe rule:

```js
if (
  editingProductId ===
  productId
) {
  resetFormToCreateMode();
}
```

This prevents stale edit mode.

---

# 18.20 Deleting Another Item During Edit Mode

Suppose:

```js
editingProductId = 205;
```

but product `309` is deleted.

The active edit can remain.

```js
if (
  editingProductId ===
  deletedProduct.id
) {
  resetFormToCreateMode();
}
```

Only reset when the deleted item is the item currently being edited.

# 18.21 Delete Success Message

```js
const deletedProduct =
  deleteProductById(
    productId
  );

if (deletedProduct === null) {
  showError(
    "The selected product was not found."
  );

  return;
}

renderProducts();

showSuccess(
  `${deletedProduct.name} was deleted successfully.`
);
```

Because `splice()` returns the removed item, the name remains available after deletion.

---

# 18.22 Keep State as the Source of Truth

Do not delete only the card:

```js
card.remove();
```

This changes the DOM but leaves the object in state.

A later render brings the product back.

Incorrect flow:

```text
Remove DOM card only
        ↓
State still contains product
        ↓
Later render restores product
```

Correct flow:

```text
Remove product from state
        ↓
Render state
        ↓
DOM no longer contains product
```

---

# 18.23 Why Direct DOM Removal Is Not Enough

Suppose:

```js
products
```

still contains Phone.

The code removes the visible card:

```js
phoneCard.remove();
```

The page looks correct temporarily.

Then:

```js
renderProducts();
```

reads the array and displays Phone again.

The array must remain the source of truth.

---

# 18.24 Mutable Deletion

`splice()` mutates:

```js
products
```

Before:

```js
const originalReference =
  products;
```

After deletion:

```js
products.splice(1, 1);
```

This is still true:

```js
originalReference === products;
```

The array reference remains the same.

Its contents changed.

---

# 18.25 Immutable Deletion Preview

Later, you will use:

```js
products =
  products.filter(
    (product) =>
      product.id !== productId
  );
```

This creates a new array.

For now, use:

```js
splice()
```

so the index-based deletion process is fully understood.

## Example 2: Delete Product by ID

### Problem

Find the product index with a standard loop and remove the product with `splice()`.

### Input Data

```js
let products = [
  {
    id: 101,
    name: "Laptop",
  },
  {
    id: 205,
    name: "Phone",
  },
  {
    id: 309,
    name: "Keyboard",
  },
];
```

### Complete Code

```js
let products = [
  {
    id: 101,
    name: "Laptop",
  },
  {
    id: 205,
    name: "Phone",
  },
  {
    id: 309,
    name: "Keyboard",
  },
];

function findProductIndexById(
  productId
) {
  for (
    let index = 0;
    index < products.length;
    index++
  ) {
    if (
      products[index].id ===
      productId
    ) {
      return index;
    }
  }

  return -1;
}

function deleteProductById(
  productId
) {
  const index =
    findProductIndexById(
      productId
    );

  if (index === -1) {
    return null;
  }

  const removedProducts =
    products.splice(
      index,
      1
    );

  return removedProducts[0];
}

const deletedProduct =
  deleteProductById(205);

console.log(deletedProduct);
console.log(products);
```

### Expected Console Output

```text
{
  id: 205,
  name: "Phone"
}

[
  {
    id: 101,
    name: "Laptop"
  },
  {
    id: 309,
    name: "Keyboard"
  }
]
```

### Final State

Phone is no longer present.

### Common Mistakes

* Passing product ID directly to `splice()`.
* Forgetting the `-1` check.
* Returning `removedProducts` when the caller expects one object.

### Small Modification Challenge

Search for ID `999` and confirm that:

* The result is `null`.
* State remains unchanged.

### Problem

Delete products using dynamically rendered buttons.

### Complete Code

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8" />

  <meta
    name="viewport"
    content="width=device-width, initial-scale=1.0"

    }

    .product-card {
      margin-bottom: 12px;
      padding: 14px;
      border: 1px solid #cccccc;
      border-radius: 6px;
    }

    button {
      padding: 8px 12px;
      cursor: pointer;
    }
  </style>
</head>

<body>
  <h1>Product Inventory</h1>

  <p
    id="error-message"
    role="alert"
  ></p>

  <p
    id="success-message"
    role="status"
  ></p>

  <p id="product-count"></p>

  <ul id="product-list"></ul>

  <script>
    let products = [
      {
        id: 101,
        name: "Laptop",
        price: 50000,
      },
      {
        id: 205,
        name: "Phone",
        price: 25000,
      },
      {
        id: 309,
        name: "Keyboard",
        price: 3000,
      },
    ];

    const errorMessage =
      document.getElementById(
        "error-message"
      );

    const successMessage =
      document.getElementById(
        "success-message"
      );

    const productCount =
      document.getElementById(
        "product-count"
      );

    const productList =
      document.getElementById(
        "product-list"
      );

    function showError(message) {
      errorMessage.textContent =
        message;
    }

    function clearError() {
      errorMessage.textContent = "";
    }

    function showSuccess(message) {
      successMessage.textContent =
        message;
    }

    function clearSuccess() {
      successMessage.textContent = "";
    }

    function findProductIndexById(
      productId
    ) {
      for (
        let index = 0;
        index < products.length;
        index++
      ) {
        if (
          products[index].id ===
          productId
        ) {
          return index;
        }
      }

      return -1;
    }

    function deleteProductById(
      productId
    ) {
      const productIndex =
        findProductIndexById(
          productId
        );

      if (
        productIndex === -1
      ) {
        return null;
      }

      const removedProducts =
        products.splice(
          productIndex,
          1
        );

      return removedProducts[0];
    }

      productCount.textContent =

      if (products.length === 0) {

          document.createElement("li");

        emptyState.textContent =
          "No products available.";

        productList.appendChild(
          emptyState
        );

        return;

      for (
        let index = 0;
        index < products.length;
        index++
      ) {
        const product =
          products[index];

        const card =
          document.createElement("li");

        const name =
          document.createElement("h2");

        const price =
          document.createElement("p");

        const deleteButton =
          document.createElement("button");

        card.classList.add(
          "product-card"
        );

        name.textContent =
          product.name;

        price.textContent =
          `KSh ${product.price.toLocaleString()}`;

        deleteButton.type =
          "button";

        deleteButton.textContent =
          "Delete";

        deleteButton.dataset.action =
          "delete";

        deleteButton.dataset.id =
          product.id;

        card.append(
          name,
          price,
          deleteButton
        );

        productList.appendChild(
          card
        );
      }
    }

    productList.addEventListener(
      "click",
      function (event) {
        const deleteButton =
          event.target.closest(
            '[data-action="delete"]'
          );

        if (
          deleteButton === null
        ) {
          return;
        }

        clearError();
        clearSuccess();

        const productId =
          Number(
            deleteButton.dataset.id
          );

        if (
          !Number.isInteger(
            productId
          ) ||
          productId <= 0

          showError(
            "Invalid product ID."
          );

          return;
        }

        const deletedProduct =
          deleteProductById(
            productId
          );

        if (
          deletedProduct === null
        ) {
          showError(
            "The selected product was not found."
          );

          return;
        }

        renderProducts();

        showSuccess(
          `${deletedProduct.name} was deleted successfully.`
        );
      }
    );

    renderProducts();

</html>
```

# 18.28 Delete Flow with Edit-Mode Safety

```js
function handleDeleteProduct(
  productId
) {
  clearError();
  clearSuccess();

  const deletedProduct =
    deleteProductById(
      productId
    );

  if (deletedProduct === null) {
    showError(
      "Product not found."
    );

    return false;
  }

  if (
    editingProductId ===
    deletedProduct.id
  ) {
    resetFormToCreateMode();
  }

  renderProducts();

  showSuccess(
    `${deletedProduct.name} was deleted.`
  );

  return true;
}
```

# 18.29 Deleting and Stable IDs

Deleting product ID `205` should not change other IDs.

Before:

```js
[
  { id: 101 },
  { id: 205 },
  { id: 309 },
]
```

After:

```js
[
  { id: 101 },
  { id: 309 },
]
```

Do not renumber IDs to:

```js
[
  { id: 1 },
  { id: 2 },
]
```

The remaining objects keep their original identity.

# 18.30 Common Beginner Mistakes

## Mistake 1: Passing the ID to `splice()`

```js
products.splice(
  productId,
  1
);
```

Find the index first.

---

## Mistake 2: Forgetting the Delete Count

```js
products.splice(index);
```

This deletes every item from that index onward.

---

## Mistake 3: Passing `-1`

```js
products.splice(-1, 1);
```

This deletes the last item.

---

## Mistake 4: Treating Index Zero as Missing

```js
if (!productIndex)
```

The first item has index `0`.

Use:

```js
if (
  productIndex === -1
)
```

---

## Mistake 5: Removing Only the DOM Card

State remains unchanged.

---

## Mistake 6: Forgetting to Re-render

The deleted object disappears from state but remains visible.

---

## Mistake 7: Regenerating Remaining IDs

IDs should remain stable.

---

## Mistake 8: Resetting Edit Mode for Every Deletion

Only reset when the deleted item is currently being edited.

---

## Mistake 9: Expecting `splice()` to Return the Updated Array

It returns removed items.

---

## Mistake 10: Deleting Before Confirmation

The next lesson will add user confirmation before mutation.

## Key Takeaways

* `splice(index, 1)` removes one item from an array.
* `splice()` mutates the existing array.
* It returns an array containing removed items.
* A stable ID must be converted into a current index before deletion.
* Always check for `-1`.
* `splice(-1, 1)` removes the final item.
* Omitting the delete count can remove several items.
* Delete state first, then re-render.
* Removing only the DOM card leaves state outdated.
* Counts and empty-state messages should be derived from current state.
* Remaining item IDs should never be renumbered.
* The next lesson will add deletion confirmation and stronger missing-item handling.
