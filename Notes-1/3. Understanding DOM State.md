# 3. Understanding DOM State

## 3.1 What You Will Learn

This subtopic explains:

* What the DOM represents.
* How JavaScript accesses HTML.
* What DOM state means.
* How displayed content changes.
* Why displayed HTML and JavaScript state are different.
* How DOM manipulation connects to CRUD applications.

### Why This Matters

Suppose JavaScript contains this state:

```js
let products = [
  {
    id: 1,
    name: "Laptop",
  },
];
```

The product exists in JavaScript memory.

However, the user cannot see it until JavaScript creates or updates HTML elements.

The DOM is what allows the application to display the product.

CRUD applications therefore involve two important areas:

1. JavaScript state.
2. The DOM.

The state stores the information.

The DOM displays the information.

---

## 3.2 What Is the DOM?

DOM stands for:

```text
Document Object Model
```

The browser reads HTML and converts it into a structure of JavaScript-accessible objects.

Consider this HTML:

```html
<body>
  <h1>Product Inventory</h1>
  <p>Welcome to the inventory system.</p>
</body>
```

The browser represents these elements as a document structure.

```text
document
└── html
    └── body
        ├── h1
        └── p
```

JavaScript can use the DOM to:

* Find elements.
* Read their content.
* Change their content.
* Create new elements.
* Remove elements.
* Add CSS classes.
* Listen for user actions.
* Read form values.

---

## 3.3 HTML Versus the DOM

HTML is the source code written in the file.

```html
<h1>Products</h1>
```

The DOM is the browser's live representation of that HTML.

JavaScript can change the DOM after the page loads:

```js
const heading = document.querySelector("h1");

heading.textContent = "Product Inventory";
```

The original HTML file may still contain:

```html
<h1>Products</h1>
```

But the user now sees:

```text
Product Inventory
```

This means the live DOM can change without rewriting the original HTML file.

---

## 3.4 What Is DOM State?

DOM state means:

> The current condition and content of the elements displayed by the browser.

For example:

```html
<p id="message">No products available.</p>
```

Current DOM state:

```text
No products available.
```

JavaScript changes it:

```js
const message = document.getElementById("message");

message.textContent = "3 products available.";
```

New DOM state:

```text
3 products available.
```

The displayed page has changed.

That visible condition is DOM state.

---

## 3.5 Examples of DOM State

DOM state can include:

* The text displayed in a paragraph.
* The value inside an input.
* Whether a button is disabled.
* Whether an element is hidden.
* Which CSS classes an element has.
* The number of list items displayed.
* Whether an error message is visible.
* Whether a form is in create mode or edit mode.

### Text state

```html
<p id="status">Loading...</p>
```

### Input state

```html
<input id="product-name" value="Laptop" />
```

### Button state

```html
<button disabled>Save</button>
```

### Class state

```html
<li class="completed">Study JavaScript</li>
```

All of these describe the current condition of DOM elements.

---

# 3.6 JavaScript State Versus DOM State

Consider this JavaScript state:

```js
let products = [
  {
    id: 1,
    name: "Laptop",
  },
];
```

Consider this HTML:

```html
<ul id="product-list">
  <li>Phone</li>
</ul>
```

JavaScript state says:

```text
Laptop
```

DOM state says:

```text
Phone
```

The two are inconsistent.

This is a problem because:

* The application stores one value.
* The user sees another value.
* Future updates may behave unpredictably.
* Deleting or editing may target the wrong item.

A reliable application should keep the DOM synchronized with the JavaScript state.

---

## 3.7 The Source of Truth

The JavaScript array should normally be treated as the source of truth.

```js
let products = [
  {
    id: 1,
    name: "Laptop",
  },
];
```

The DOM should be built from this state.

Expected DOM:

```text
Products

1. Laptop
```

The application should not rely on manually reading the displayed list every time it needs product information.

Instead:

```text
State determines what the DOM should display.
```

The correct direction is:

```text
JavaScript state
      ↓
Render function
      ↓
DOM output
```

---

# 3.8 Selecting DOM Elements

Before JavaScript can read or change an element, it must select it.

Common selection methods include:

* `document.getElementById()`
* `document.querySelector()`
* `document.querySelectorAll()`

---

## `document.getElementById()`

This selects one element by its `id`.

HTML:

```html
<h1 id="page-title">Products</h1>
```

JavaScript:

```js
const pageTitle = document.getElementById("page-title");

console.log(pageTitle);
```

The string passed to `getElementById()` does not include `#`.

Correct:

```js
document.getElementById("page-title");
```

Incorrect:

```js
document.getElementById("#page-title");
```

---

## `document.querySelector()`

This selects the first element matching a CSS selector.

HTML:

```html
<p class="message">First message</p>
<p class="message">Second message</p>
```

JavaScript:

```js
const firstMessage = document.querySelector(".message");

console.log(firstMessage.textContent);
```

Expected output:

```text
First message
```

Examples:

```js
document.querySelector("#page-title");
document.querySelector(".message");
document.querySelector("button");
document.querySelector("form input");
```

---

## `document.querySelectorAll()`

This selects all matching elements.

HTML:

```html
<li class="item">Laptop</li>
<li class="item">Phone</li>
<li class="item">Keyboard</li>
```

JavaScript:

```js
const itemElements = document.querySelectorAll(".item");

console.log(itemElements.length);
```

Expected output:

```text
3
```

`querySelectorAll()` returns a `NodeList`.

You can access elements by index:

```js
console.log(itemElements[0].textContent);
```

Expected output:

```text
Laptop
```


# 3.10 Changing DOM Content

HTML:

```html
<p id="message">No items available.</p>
```

JavaScript:

```js
const message = document.getElementById("message");

message.textContent = "One item available.";
```

Expected DOM output:

```text
One item available.
```

The paragraph object remains the same element.

Only its displayed text changes.

# 3.11 Creating DOM Elements

JavaScript can create HTML elements dynamically.

```js
const listItem = document.createElement("li");
```

This creates a new `<li>` element in memory.

At this point, it is not visible.

Add text:

```js
listItem.textContent = "Laptop";
```

The element now represents:

```html
<li>Laptop</li>
```

However, it is still not attached to the document.

---

# 3.12 Appending DOM Elements

HTML:

```html
<ul id="product-list"></ul>
```

JavaScript:

```js
const productList =
  document.getElementById("product-list");

const listItem = document.createElement("li");

listItem.textContent = "Laptop";

productList.appendChild(listItem);
```

Expected DOM:

```html
<ul id="product-list">
  <li>Laptop</li>
</ul>
```

Expected browser output:

```text
Laptop
```

---

## `appendChild()` Versus `append()`

Both can add content to a parent element.

### `appendChild()`

```js
productList.appendChild(listItem);
```

It appends one DOM node.

### `append()`

```js
productList.append(listItem);
```

It can append nodes or text.

```js
productList.append("No products available.");
```

For most beginner CRUD examples, either can be used.

This course will often use `appendChild()` when appending one created element because its purpose is clear.

---

# 3.13 Adding CSS Classes

HTML:

```html
<ul id="task-list"></ul>
```

JavaScript:

```js
const taskList = document.getElementById("task-list");

const taskItem = document.createElement("li");

taskItem.textContent = "Study JavaScript";

taskItem.classList.add("task");

taskList.appendChild(taskItem);
```

Result:

```html
<li class="task">Study JavaScript</li>
```

You can add multiple classes:

```js
taskItem.classList.add("task", "high-priority");
```

You can also remove a class:

```js
taskItem.classList.remove("high-priority");
```

Or toggle a class:

```js
taskItem.classList.toggle("completed");
```

# 3.14 Using `dataset`

HTML elements can store custom data attributes.

HTML:

```html
<button data-id="25">Edit</button>
```

JavaScript:

```js
const editButton = document.querySelector("button");

console.log(editButton.dataset.id);
```

Expected output:

```text
25
```

> Values read from `dataset` are strings.

Therefore:

```js
console.log(typeof editButton.dataset.id);
```

Expected output:

```text
string
```

Convert it when comparing it with numeric IDs:

```js
const selectedId = Number(editButton.dataset.id);
```

Later, `dataset` will help identify which item should be edited or deleted.

# 3.15 DOM Events

An event is something that happens in the browser.

Examples include:

* A button click.
* A form submission.
* Typing into an input.
* Selecting an option.
* Checking a checkbox.
* Moving the mouse.
* Pressing a keyboard key.

JavaScript listens for events using `addEventListener()`.

HTML:

```html
<button id="show-message-button">
  Show Message
</button>
```

JavaScript:

```js
const showMessageButton = document.getElementById(
  "show-message-button"
);

showMessageButton.addEventListener("click", function () {
  console.log("The button was clicked.");
});
```

When the user clicks the button, the event handler runs.

---

# 3.16 The Event Object

An event handler can receive an event object.

```js
button.addEventListener("click", function (event) {
  console.log(event);
});
```

The event object contains information about what happened.

Useful properties include:

```js
event.target;
event.currentTarget;
event.type;
```

For example:

```js
button.addEventListener("click", function (event) {
  console.log(event.type);
});
```

Expected output:

```text
click
```

---

# 3.17 Form Submit Events

HTML:

```html
<form id="product-form">
  <input id="product-name" type="text" />
  <button type="submit">Add Product</button>
</form>
```

JavaScript:

```js
const productForm =
  document.getElementById("product-form");

productForm.addEventListener("submit", function (event) {
  event.preventDefault();

  console.log("Form submitted.");
});
```

A form should normally listen for the `submit` event rather than only the button's `click` event.

Why?

A form can be submitted by:

* Clicking the submit button.
* Pressing Enter inside an input.
* Calling a form submission method.

Listening to `submit` handles the form behavior more completely.

---

# 3.18 Why `preventDefault()` Is Needed

By default, an HTML form attempts to submit and reload or navigate the page.

```js
event.preventDefault();
```

This prevents that default browser behavior.

Example:

```js
productForm.addEventListener("submit", function (event) {
  event.preventDefault();

  const productName = productNameInput.value;

  console.log(productName);
});
```

Without `preventDefault()`, the page may reload before the application can preserve and display its JavaScript state.

---

# 3.19 JavaScript State and DOM State Are Separate

Consider this state:

```js
let productName = "Laptop";
```

HTML:

```html
<p id="product-name">Phone</p>
```

The JavaScript state contains:

```text
Laptop
```

The DOM contains:

```text
Phone
```

Changing the variable:

```js
productName = "Keyboard";
```

does not automatically update this paragraph.

Changing the paragraph:

```js
document.getElementById(
  "product-name"
).textContent = "Monitor";
```

does not automatically update the variable.

The variable still contains:

```text
Keyboard
```

The paragraph displays:

```text
Monitor
```

They are separate until JavaScript explicitly synchronizes them.

---

# 3.20 Why Changing State Alone Does Not Change the DOM

Example:

```html
<p id="product-name">No product</p>

<script>
  let product = {
    id: 1,
    name: "Laptop",
  };

  product.name = "Gaming Laptop";
</script>
```

Expected browser output:

```text
No product
```

Final JavaScript state:

```js
{
  id: 1,
  name: "Gaming Laptop"
}
```

The browser paragraph remains unchanged because no DOM update was performed.

To update the paragraph:

```js
const productNameElement =
  document.getElementById("product-name");

productNameElement.textContent = product.name;
```

---

# 3.21 Why Changing the DOM Alone Does Not Change State

Example:

```html
<p id="product-name">Laptop</p>

<script>
  let product = {
    id: 1,
    name: "Laptop",
  };

  const productNameElement =
    document.getElementById("product-name");

  productNameElement.textContent = "Gaming Laptop";

  console.log(product.name);
</script>
```

Expected browser output:

```text
Gaming Laptop
```

Expected console output:

```text
Laptop
```

The displayed DOM changed.

The JavaScript object did not.

This is why CRUD logic should update state first.

---

# 3.22 `textContent` Versus `innerHTML`

## Using `textContent`

```js
element.textContent = userInput;
```

The browser treats the input as plain text.

Suppose:

```js
const userInput = "<strong>Laptop</strong>";
```

Using:

```js
element.textContent = userInput;
```

The user sees:

```text
<strong>Laptop</strong>
```

The tags are displayed as text.

---

## Using `innerHTML`

```js
element.innerHTML = userInput;
```

The browser interprets the input as HTML.

The user may see:

```text
Laptop
```

in bold.

This can be dangerous when the value comes from a user.

A malicious user could enter content containing harmful HTML or script-like behavior.

> **Security warning:** Never insert untrusted user input directly with `innerHTML`. Use `textContent` unless you intentionally need trusted HTML.

---

# 3.23 Disabling and Enabling Buttons

HTML:

```html
<button id="save-button">Save</button>
```

JavaScript:

```js
const saveButton =
  document.getElementById("save-button");

saveButton.disabled = true;
```

The button becomes disabled.

Enable it again:

```js
saveButton.disabled = false;
```

This can be used when:

* A form is invalid.
* An operation is in progress.
* No item has been selected.
* The user is not allowed to perform the action.

# 3.25 Three Complete Examples

## Example 1: Reading and Changing DOM Text

### Problem

Select a paragraph, read its current text, and change the displayed value.

### Input HTML

```html
<p id="status-message">No products available.</p>
```

### Complete Code

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8" />

  <meta
    name="viewport"
    content="width=device-width, initial-scale=1.0"
  />

  <title>DOM Text Example</title>
</head>

<body>
  <p id="status-message">
    No products available.
  </p>

  <script>
    const statusMessage =
      document.getElementById("status-message");

    console.log(statusMessage.textContent.trim());

    statusMessage.textContent =
      "One product available.";

    console.log(statusMessage.textContent);
  </script>
</body>
</html>
```

### Step-by-Step Explanation

#### Step 1: Select the paragraph

```js
const statusMessage =
  document.getElementById("status-message");
```

`statusMessage` now references the paragraph element.

#### Step 2: Read the text

```js
console.log(statusMessage.textContent.trim());
```

Expected output:

```text
No products available.
```

#### Step 3: Change the text

```js
statusMessage.textContent =
  "One product available.";
```

The visible DOM changes.

### Expected Console Output

```text
No products available.
One product available.
```

### Expected DOM Output

```text
One product available.
```

### Final JavaScript State

No application array or object was changed.

The only stored JavaScript reference is:

```js
statusMessage
```

### Final DOM State

```html
<p id="status-message">
  One product available.
</p>
```

### Common Mistakes

#### Misspelling the ID

Incorrect:

```js
document.getElementById("status-mesage");
```

This returns `null`.

#### Trying to change `textContent` on `null`

```js
const statusMessage =
  document.getElementById("wrong-id");

statusMessage.textContent = "Updated";
```

This causes an error because no element was found.

#### Including `#` in `getElementById()`

Incorrect:

```js
document.getElementById("#status-message");
```

Correct:

```js
document.getElementById("status-message");
```

Add a button that changes the message to:

```text
Products loaded successfully.
```

only when the button is clicked.

## Example 2: Reading Form Input

### Problem

Read a product name entered by the user and display a validation message.

### Expected Initial DOM

```text
Product Name: [____________]

[Show Product]

No product selected.
```

### Complete Code

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8" />

  <meta
    name="viewport"
    content="width=device-width, initial-scale=1.0"
  />

  <title>Read Input Example</title>
</head>

<body>
  <form id="product-form">
    <label for="product-name">
      Product Name:
    </label>

    <input
      id="product-name"
      type="text"
      autocomplete="off"
    />

    <button type="submit">
      Show Product
    </button>
  </form>

  <p id="result-message">
    No product selected.
  </p>

  <script>
    const productForm =
      document.getElementById("product-form");

    const productNameInput =
      document.getElementById("product-name");

    const resultMessage =
      document.getElementById("result-message");

    productForm.addEventListener(
      "submit",
      function (event) {
        event.preventDefault();

        const productName =
          productNameInput.value.trim();

        if (productName === "") {
          resultMessage.textContent =
            "Please enter a product name.";

          return;
        }

        resultMessage.textContent =
          `Selected product: ${productName}`;
      }
    );
  </script>
</body>
</html>
```

### Step-by-Step Explanation

#### Step 1: Listen for submission

```js
productForm.addEventListener(
  "submit",
  function (event) {
    // Handle the form.
  }
);
```

#### Step 2: Prevent page reload

```js
event.preventDefault();
```

#### Step 3: Read the input

```js
const productName =
  productNameInput.value.trim();
```

#### Step 4: Validate

```js
if (productName === "") {
  resultMessage.textContent =
    "Please enter a product name.";

  return;
}
```

#### Step 5: Display the result

```js
resultMessage.textContent =
  `Selected product: ${productName}`;
```

### Expected Output for Blank Input

```text
Please enter a product name.
```

### Expected Output for `Laptop`

```text
Selected product: Laptop
```

### Final JavaScript State

The local variable contains:

```js
productName = "Laptop";
```

It exists only during that submission handler execution.

No persistent items array has been created yet.

### Final DOM State

```text
Selected product: Laptop
```

### Common Mistakes

#### Forgetting `.value`

Incorrect:

```js
const productName = productNameInput;
```

This stores the element, not the entered text.

Correct:

```js
const productName = productNameInput.value;
```

#### Forgetting `.trim()`

A user could enter only spaces.

```text
"     "
```

Without trimming, that value may incorrectly pass validation.

#### Listening only for a button click

Listening to the form's `submit` event also supports pressing Enter.

### Small Modification Challenge

After successful submission:

1. Clear the input.
2. Move focus back to the input.
3. Change the message to include the number of characters in the name.

Example:

```text
Selected product: Laptop — 6 characters
```

## Example 3: Creating and Displaying DOM Elements

### Problem

Use JavaScript to create product list elements and display them in the browser.

This example uses JavaScript data, but it does not yet include complete CRUD.

### Input Data

```js
const products = [
  {
    id: 1,
    name: "Laptop",
  },
  {
    id: 2,
    name: "Phone",
  },
  {
    id: 3,
    name: "Keyboard",
  },
];
```

### Complete Code

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8" />

  <meta
    name="viewport"
    content="width=device-width, initial-scale=1.0"
  />

  <title>DOM Product List</title>

  <style>
    body {
      max-width: 600px;
      margin: 0 auto;
      padding: 24px;
      font-family: Arial, sans-serif;
    }

    .product {
      margin-bottom: 8px;
      padding: 10px;
      border: 1px solid #cccccc;
      border-radius: 4px;
    }
  </style>
</head>

<body>
  <h1>Products</h1>

  <ul id="product-list"></ul>

  <script>
    const products = [
      {
        id: 1,
        name: "Laptop",
      },
      {
        id: 2,
        name: "Phone",
      },
      {
        id: 3,
        name: "Keyboard",
      },
    ];

    const productList =
      document.getElementById("product-list");

    for (
      let index = 0;
      index < products.length;
      index++
    ) {
      const product = products[index];

      const listItem =
        document.createElement("li");

      listItem.classList.add("product");

      listItem.dataset.id = product.id;

      listItem.textContent =
        `${product.id}. ${product.name}`;

      productList.appendChild(listItem);
    }
  </script>
</body>
</html>
```

### Step-by-Step Explanation

#### Step 1: Store the data

```js
const products = [
  {
    id: 1,
    name: "Laptop",
  },
];
```

#### Step 2: Select the list

```js
const productList =
  document.getElementById("product-list");
```

#### Step 3: Loop through the products

```js
for (
  let index = 0;
  index < products.length;
  index++
) {
  const product = products[index];
}
```

#### Step 4: Create an `<li>`

```js
const listItem =
  document.createElement("li");
```

#### Step 5: Add a class

```js
listItem.classList.add("product");
```

#### Step 6: Store the product ID in the DOM

```js
listItem.dataset.id = product.id;
```

The browser creates:

```html
<li data-id="1"></li>
```

#### Step 7: Add safe text

```js
listItem.textContent =
  `${product.id}. ${product.name}`;
```

#### Step 8: Append the element

```js
productList.appendChild(listItem);
```

```text

2. Phone
3. Keyboard
```

### Expected Generated HTML

```html

  <li class="product" data-id="1">
    1. Laptop
  </li>

  <li class="product" data-id="2">
    2. Phone
  </li>

  <li class="product" data-id="3">
    3. Keyboard
  </li>
```

```js
  {
    id: 1,
    name: "Laptop",

    name: "Phone",
  },
```

### Common Mistakes

#### Creating elements without appending them

```js
const listItem =
  document.createElement("li");

listItem.textContent = "Laptop";
```

The item exists in memory but is not visible.

#### Appending to the wrong parent

```js
document.body.appendChild(listItem);
```

This displays the item outside the intended list.

#### Using the array index as a permanent ID

```js
listItem.dataset.id = index;
```

The index can change when items are deleted or reordered.

Prefer:

```js
listItem.dataset.id = product.id;
```

#### Using `innerHTML` unnecessarily

```js
listItem.innerHTML = product.name;
```

Use:

```js
listItem.textContent = product.name;
```

Consider:


Represents the current webpage.

### `getElementById()`

Searches for an element with the specified ID.

### `"item-list"`

The ID value being searched for.

The matching DOM element, or `null` if none exists.

Consider:

```js
const listItem =
  document.createElement("li");
```

### `createElement()`

Creates a new DOM element.

### `"li"`

The type of element to create.

### Returned value

A new `<li>` object that is not yet visible.

---

Consider:

```js
listItem.textContent = item.name;
```

### `listItem`

The DOM element being changed.

### `textContent`

The text displayed inside the element.

### `item.name`

The JavaScript value to display.

---

Consider:

```js
itemList.appendChild(listItem);
```

### `itemList`

The parent element.

### `appendChild()`

Adds a child node to the parent.

### `listItem`

The child being added.

The selected DOM element.

### `addEventListener()`

Registers an event handler.

### `"click"`

The event being observed.

### Callback function

Runs when the event occurs.

### `event`

Contains information about the browser event.

# 3.27 Common Beginner Mistakes

## Mistake 1: Selecting Before the Element Exists

Incorrect placement:

```html
<script>
  const button =
    document.getElementById("save-button");
</script>

<button id="save-button">Save</button>
```

At the time the script runs, the button may not exist yet.

Better:

```html
<button id="save-button">Save</button>

<script>
  const button =
    document.getElementById("save-button");
</script>
```

Or use the `defer` attribute:

```html
<script src="app.js" defer></script>
```

---

## Mistake 2: Forgetting to Check for `null`

```js
const element =
  document.getElementById("missing-element");

element.textContent = "Hello";
```

This fails because `element` is `null`.

A defensive check:

```js
if (element !== null) {
  element.textContent = "Hello";
}
```

---

## Mistake 3: Reading Input Text with `textContent`

Incorrect:

```js
const name =
  productNameInput.textContent;
```

Correct:

```js
const name =
  productNameInput.value;
```

---

## Mistake 4: Reading Paragraph Text with `.value`

Incorrect:

```js
const message =
  paragraph.value;
```

Correct:

```js
const message =
  paragraph.textContent;
```

---

## Mistake 5: Creating Elements but Not Appending Them

```js
const item =
  document.createElement("li");

item.textContent = "Laptop";
```

Nothing appears until:

```js
itemList.appendChild(item);
```

---

## Mistake 6: Changing Only the DOM

```js
listItem.textContent = "Updated Laptop";
```

This does not update the product object.

---

## Mistake 7: Changing Only State

```js
product.name = "Updated Laptop";
```

The user will not see the new value until the DOM is rendered again.

---

## Mistake 8: Using `innerHTML` for User Input

Unsafe:

```js
listItem.innerHTML =
  productNameInput.value;
```

Safer:

```js
listItem.textContent =
  productNameInput.value;
```

---

## Mistake 9: Adding the Same Event Listener Repeatedly

```js
function render() {
  button.addEventListener("click", handleClick);
}
```

If `render()` runs many times, the same event may trigger multiple handlers.

Later, you will learn how event delegation prevents this problem for dynamic lists.

---

## Mistake 10: Forgetting `preventDefault()`

```js
form.addEventListener("submit", function (event) {
  // Missing event.preventDefault()
});
```

The page may reload.

## Key Takeaways

* The DOM is the browser's live object representation of the webpage.
* DOM state describes what the user currently sees and how elements are currently configured.
* JavaScript can select, read, create, change, and remove DOM elements.
* `getElementById()` selects an element by ID.
* `querySelector()` selects the first matching element.
* `querySelectorAll()` selects all matching elements.
* Input values are read with `.value`.
* Normal element text is read and changed with `.textContent`.
* `createElement()` creates a new DOM element.
* `appendChild()` attaches a created element to the page.
* `classList` manages CSS classes.
* `dataset` stores custom data attributes.
* Form submissions should usually use the `submit` event.
* `preventDefault()` prevents normal page reloading.
* JavaScript state and DOM state are separate.
* Changing one does not automatically change the other.
* Application state should normally be the source of truth.
* The DOM should be rendered from the current state.
* Untrusted user input should be inserted with `textContent`, not `innerHTML`.
