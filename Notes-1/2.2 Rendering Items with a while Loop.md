# 6. Reading Items with a `while` Loop

## 6.1 What You Will Learn

A `while` loop repeats code while a condition remains true.

The general form is:

```js
while (condition) {
  // Repeated code
}
```

For array rendering, you normally need:

1. A counter variable.
2. A condition.
3. The repeated logic.
4. A manual counter update.

Example:

```js
let index = 0;

while (index < products.length) {
  console.log(products[index]);

  index++;
}
```

Unlike a standard `for` loop, the initialization and update are written outside and inside the loop separately.

---

# 6.2 Basic `while` Loop Syntax

```js
let index = 0;

while (index < products.length) {
  const product = products[index];

  console.log(product.name);

  index++;
}
```

This has four important parts.

---

## Counter Declaration

```js
let index = 0;
```

The counter must be declared before the loop.

Arrays begin at index `0`.

---

## Condition

```js
index < products.length
```

The loop continues while the current index is valid.

---

## Loop Body

```js
const product = products[index];

console.log(product.name);
```

This accesses and uses the current item.

---

## Counter Update

```js
index++;
```

The counter must change manually.

If you forget this line, the loop may never stop.

---

State:

```js
const products = [
  "Laptop",

```

Loop:

```js

```




```js

```

New value:

```text
1
```

---

### Second Repetition

Condition:

```js
1 < 3
```

Output:

```text
Phone
```

Counter becomes:

```text
2
```

---

### Third Repetition

Condition:

```js
2 < 3
```

Output:

```text
Keyboard
```

Counter becomes:

```text
3
```

---

### Final Check

Condition:

```js
3 < 3
```

Result:

```text
false
```

The loop ends.

# 6.4 Standard `for` Loop Versus `while` Loop

## Standard `for` Loop

```js
for (
  let index = 0;
  index < products.length;
  index++
) {
  console.log(products[index]);
}
```

## `while` Loop

```js
let index = 0;

while (index < products.length) {
  console.log(products[index]);

  index++;
}
```

Both produce the same output.

The main difference is organization.

The `for` loop groups:

* Initialization.
* Condition.
* Update.

The `while` loop separates them.

---

# 6.5 Why Use a `while` Loop?

A `while` loop is useful when:

* You do not know exactly how many repetitions will occur.
* Repetition depends on a changing condition.
* The counter changes in a non-standard way.
* You are waiting for a condition to become false.
* The loop should continue until some special event occurs.

Example:

```js
while (currentPage < totalPages) {
  currentPage++;
}
```

For simple array rendering, a standard `for` loop is often more compact.

Still, learning `while` is important because it teaches explicit loop control.

# 6.7 Building a Reusable Render Function

```js
function renderProducts() {
  productList.innerHTML = "";

  let index = 0;

  while (index < products.length) {
    const product = products[index];

    const listItem =
      document.createElement("li");

    listItem.textContent =
      `${index + 1}. ${product.name}`;

    productList.appendChild(listItem);

    index++;
  }
}
```

The `index` variable is created fresh each time the function runs.

This is important.

Incorrect:

```js
let index = 0;

function renderProducts() {
  while (index < products.length) {
    // Render
    index++;
  }
}
```

After the first render, `index` equals `products.length`.

The second render displays nothing because the condition is already false.

Correct:

```js
function renderProducts() {
  let index = 0;

  while (index < products.length) {
    // Render
    index++;
  }
}
```

---

# 6.8 Clearing Before Rendering

```js
productList.innerHTML = "";
```

This removes previously rendered elements before rebuilding the list.

Without clearing, repeated renders create duplicates.

---

# 6.9 Empty-State Rendering

```js
function renderProducts() {
  productList.innerHTML = "";

  if (products.length === 0) {
    const emptyMessage =
      document.createElement("li");

    emptyMessage.textContent =
      "No products available.";

    productList.appendChild(emptyMessage);

    return;
  }

  let index = 0;

  while (index < products.length) {
    const product = products[index];

    const listItem =
      document.createElement("li");

    listItem.textContent = product.name;

    productList.appendChild(listItem);

    index++;
  }
}
```

Expected output for an empty array:

```text
No products available.
```

# 6.10 Avoiding Infinite Loops

The most common `while` loop error is forgetting to update the condition.

Incorrect:

```js
let index = 0;

while (index < products.length) {
  console.log(products[index]);
}
```

`index` remains `0`.

If the array is not empty, the condition remains true forever.

Correct:

```js
let index = 0;

while (index < products.length) {
  console.log(products[index]);

  index++;
}
```

> Always ask: What changes during each repetition so the condition eventually becomes false?

---

# 6.11 Counter Placement Matters

Correct:

```js
let index = 0;

while (index < products.length) {
  console.log(products[index]);

  index++;
}
```

Incorrect:

```js
let index = 0;

while (index < products.length) {
  index++;

  console.log(products[index]);
}
```

This increments before reading the current item.

The first item is skipped.

The final access may produce `undefined`.

---

# 6.12 Rendering in Reverse

A `while` loop can move backwards.

```js
let index = products.length - 1;

while (index >= 0) {
  console.log(products[index].name);

  index--;
}
```

For:

```js
[
  { name: "Laptop" },
  { name: "Phone" },
  { name: "Keyboard" },
]
```

Expected output:

```text
Keyboard
Phone
Laptop
```

---

# 6.13 Skipping Items

Suppose inactive items should not be displayed.

```js
const products = [
  {
    id: 1,
    name: "Laptop",
    isActive: true,
  },
  {
    id: 2,
    name: "Phone",
    isActive: false,
  },
  {
    id: 3,
    name: "Keyboard",
    isActive: true,
  },
];
```

A safe `while` loop:

```js
let index = 0;

while (index < products.length) {
  const product = products[index];

  if (product.isActive) {
    console.log(product.name);
  }

  index++;
}
```

Expected output:

```text
Laptop
Keyboard
```

Notice that `index++` still runs for every item.

If the increment were placed only inside the `if`, inactive items could create an infinite loop.

---

# 6.14 Stopping Early

You can use `break`.

```js
let index = 0;

while (index < products.length) {
  if (products[index].id === selectedId) {
    console.log(products[index]);

    break;
  }

  index++;
}
```

This stops as soon as the matching item is found.

This ability will be useful during Update and Delete operations.

---

# 6.15 Rendering Product Cards

A `while` loop can create complex DOM structures.

```js
let index = 0;

while (index < products.length) {
  const product = products[index];

  const card =
    document.createElement("article");

  const title =
    document.createElement("h2");

  const price =
    document.createElement("p");

  title.textContent = product.name;

  price.textContent =
    `KSh ${product.price.toLocaleString()}`;

  card.append(title, price);

  productList.appendChild(card);

  index++;
}
```

The loop pattern is identical to a standard `for` loop.

Only the control structure changes.

## Example 2: Rendering Product Objects into the DOM

### Problem

Render product names and prices using a `while` loop.

### Input Data

```js
const products = [
  {
    id: 1,
    name: "Laptop",
    price: 50000,
  },
  {
    id: 2,
    name: "Phone",
    price: 25000,
  },
  {
    id: 3,
    name: "Keyboard",

  },
];
```

### Complete Code

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8" />

  <meta
    name="viewport"
    content="width=device-width, initial-scale=1.0"
  />

  <title>Products with while Loop</title>

  <style>
    body {
      max-width: 700px;
      margin: 0 auto;
      padding: 24px;
      font-family: Arial, sans-serif;
    }

    .product-item {
      margin-bottom: 10px;
      padding: 12px;
      border: 1px solid #cccccc;
      border-radius: 6px;
    }
  </style>
</head>

<body>
  <h1>Products</h1>

  <ul id="product-list"></ul>

  <script>
    const products = [
      {
        id: 1,
        name: "Laptop",
        price: 50000,
      },
      {
        id: 2,
        name: "Phone",
        price: 25000,
      },
      {
        id: 3,
        name: "Keyboard",
        price: 3000,
      },
    ];

    const productList =
      document.getElementById("product-list");

    function renderProducts() {
      productList.innerHTML = "";

      if (products.length === 0) {
        const emptyMessage =
          document.createElement("li");

        emptyMessage.textContent =
          "No products available.";

        productList.appendChild(emptyMessage);

        return;
      }

      let index = 0;

      while (index < products.length) {
        const product = products[index];

        const listItem =
          document.createElement("li");

        listItem.classList.add(
          "product-item"
        );

        listItem.dataset.id = product.id;

        listItem.textContent =
          `${index + 1}. ${product.name} — ` +
          `KSh ${product.price.toLocaleString()}`;

        productList.appendChild(listItem);

        index++;
      }
    }

    renderProducts();
  </script>
</body>
</html>
```

### Step-by-Step Explanation

#### Step 1: Clear the old DOM

```js
productList.innerHTML = "";
```

#### Step 2: Handle empty state

```js
if (products.length === 0) {
  // Display a message.
}
```

#### Step 3: Initialize the loop counter

```js
let index = 0;
```

This happens inside the render function.

#### Step 4: Check each valid index

```js
while (index < products.length)
```

#### Step 5: Access the product

```js
const product = products[index];
```

#### Step 6: Build the DOM element

```js
const listItem =
  document.createElement("li");
```

#### Step 7: Append the element

```js
productList.appendChild(listItem);
```

#### Step 8: Increment the counter

```js
index++;
```

### Expected DOM Output

```text
Products

1. Laptop — KSh 50,000
2. Phone — KSh 25,000
3. Keyboard — KSh 3,000
```

### Final State

The products array remains unchanged.

### Common Mistakes

#### Declaring the counter outside `renderProducts()`

A second render may not work.

#### Forgetting to clear the list

Repeated renders create duplicates.

#### Forgetting the final increment

The browser may freeze because of an infinite loop.

```html
      document.getElementById("product-list");

        const emptyMessage =

        emptyMessage.textContent =
          "No products available.";

            "Out of stock";

          status.textContent =

          quantity,

      }
    }
```

# 6.17 `while` Loop Versus Standard `for` Loop

| Feature                           | Standard `for`                 | `while`                       |
| --------------------------------- | ------------------------------ | ----------------------------- |
| Initialization                    | In loop header                 | Before loop                   |
| Condition                         | In loop header                 | In loop header                |
| Update                            | In loop header                 | Inside body                   |
| Compactness                       | More compact                   | More spread out               |
| Infinite-loop risk                | Lower for standard array loops | Higher if update is forgotten |
| Custom control                    | High                           | High                          |
| Common for array rendering        | Very common                    | Less common                   |
| Best for unknown repetition count | Sometimes                      | Often                         |

---

# 6.18 When a `while` Loop Is Better

A `while` loop may be clearer when the loop continues until a changing condition occurs.

Example:

```js
while (remainingAttempts > 0) {
  // Ask again.
}
```

Example:

```js
while (!isValid) {
  // Continue validation process.
}
```

Example:

```js
while (currentItem !== null) {
  // Process linked data.
}
```

For normal array rendering, a standard `for` loop or `for...of` is often easier to read.

# 6.19 Common Beginner Mistakes

## Mistake 1: Forgetting Initialization

Incorrect:

```js
while (index < products.length) {
  console.log(products[index]);

  index++;
}
```

If `index` has not been declared, the code fails.

---

## Mistake 2: Forgetting Increment

```js
let index = 0;

while (index < products.length) {
  console.log(products[index]);
}
```

This may create an infinite loop.

---

## Mistake 3: Incrementing Before Reading

```js
let index = 0;

while (index < products.length) {
  index++;

  console.log(products[index]);
}
```

The first item is skipped.

---

## Mistake 4: Declaring Counter Outside Render Function

```js
let index = 0;

function renderProducts() {
  while (index < products.length) {
    index++;
  }
}
```

Later renders may display nothing.

---

## Mistake 5: Updating the Counter Only in an `if`

```js
let index = 0;

while (index < products.length) {
  if (products[index].isActive) {
    console.log(products[index].name);

    index++;
  }
}
```

If an item is inactive, the loop never moves forward.

Correct:

```js
let index = 0;

while (index < products.length) {
  if (products[index].isActive) {
    console.log(products[index].name);
  }

  index++;
}
```

---

## Mistake 6: Using `<=`

```js
while (index <= products.length)
```

The loop performs one invalid access.

---

## Mistake 7: Forgetting to Clear the DOM

Repeated renders create duplicates.

---

## Mistake 8: Mutating State During Rendering

Rendering should not change product values.

## Key Takeaways

* A `while` loop repeats while a condition remains true.
* The counter is declared before the loop.
* The condition controls whether another repetition occurs.
* The counter must be updated manually.
* Forgetting the update may create an infinite loop.
* For arrays, the common condition is `index < array.length`.
* `array[index]` accesses the current item.
* A reusable render function should reset the counter each time.
* The container should be cleared before rendering.
* Empty arrays should display a helpful message.
* `while` loops offer flexible condition control.
* Standard `for` loops are often more compact for array rendering.
* Rendering should read state without mutating it.
* Every possible loop path must eventually move toward a false condition.
