# 16. Re-rendering After an Update

## 16.1 What You Will Learn

JavaScript state and the DOM are separate.

Suppose the update succeeds:

```js
products[1].name =
  "Smartphone";
```

The state now contains:

```js
{
  id: 205,
  name: "Smartphone",
}
```

But the DOM may still display:

```text
Phone
```

until the application calls:

```js
renderProducts();
```

The correct flow is:

```text
Update state
    ↓
Reset temporary UI state
    ↓
Render current state
    ↓
User sees the updated DOM
```

---

# 16.2 Why State Changes Are Not Automatically Visible

Plain JavaScript does not automatically watch arrays and objects for changes.

This code changes only JavaScript memory:

```js
selectedProduct.name =
  "Smartphone";
```

The browser does not automatically know:

* Which DOM element represents the product.
* Which text should change.
* Whether count or totals should change.
* Whether CSS classes should change.

The application must explicitly update the DOM.

---

# 16.3 What Re-rendering Means

Re-rendering means running the render function again after state changes.

```js
renderProducts();
```

The function reads the current state and rebuilds the displayed list.

Initial render:

```text
Laptop
Phone
```

State update:

```js
products[1].name =
  "Smartphone";
```

Re-render:

```text
Laptop
Smartphone
```

---

# 16.4 The Correct Update Order

A successful Update operation should normally follow this sequence:

```text
1. Read submitted form values
2. Normalize values
3. Validate values
4. Find the product by ID
5. Update the matching object
6. Confirm that an update occurred
7. Reset the form to create mode
8. Render the updated state
9. Display a success message
```

Example:

```js
const updatedProductName =
  data.name;

let wasUpdated = false;

for (
  let index = 0;
  index < products.length;
  index++
) {
  if (
    products[index].id ===
    editingProductId
  ) {
    products[index].name =
      updatedProductName;

    wasUpdated = true;
    break;
  }
}

if (!wasUpdated) {
  showError(
    "The selected product was not found."
  );

  return;
}

resetFormToCreateMode();
renderProducts();
showSuccess(
  "Product updated successfully."
);
```

---

# 16.5 Why Rendering Should Happen After State Mutation

Incorrect order:

```js
renderProducts();

products[index].name =
  data.name;
```

The render function sees the old state.

Correct order:

```js
products[index].name =
  data.name;

renderProducts();
```

The render function sees the new state.

# 16.6 Reusable Render Function

```js
function renderProducts() {
  productList.innerHTML = "";

  productCount.textContent =
    `Total products: ${products.length}`;

  if (products.length === 0) {
    renderEmptyState();
    return;
  }

  for (
    let index = 0;
    index < products.length;
    index++
  ) {
    renderProductCard(
      products[index],
      index
    );
  }
}
```

This function is used after:

* Initial page load.
* Create.
* Update.
* Delete.
* Reset.
* Filtering.
* Sorting.

---

# 16.7 Why the Container Must Be Cleared

The first line should usually be:

```js
productList.innerHTML = "";
```

Without it, every re-render appends another copy.

State:

```js
[
  { name: "Laptop" },
  { name: "Phone" },
]
```

First render:

```text
Laptop
Phone
```

Second render without clearing:

```text
Laptop
Phone
Laptop
Phone
```

The state contains two items, but the DOM displays four.

---

# 16.8 Clearing Does Not Delete State

This line:

```js
productList.innerHTML = "";
```

clears only the displayed HTML children.

It does not change:

```js
products
```

State remains available and is used to rebuild the list immediately.

---

# 16.9 Rendering Should Not Change State

Incorrect:

```js
function renderProducts() {
  products[0].quantity++;

  // Display products.
}
```

Every render changes the quantity.

Rendering should only read state.

Correct:

```js
function renderProducts() {
  productList.innerHTML = "";

  for (
    let index = 0;
    index < products.length;
    index++
  ) {
    const product =
      products[index];

    // Display product.
  }
}
```

---

# 16.10 Reset Edit Mode Before or After Rendering?

Both sequences can work:

```js
resetFormToCreateMode();
renderProducts();
```

or:

```js
renderProducts();
resetFormToCreateMode();
```

However, if rendering uses:

```js
editingProductId
```

to highlight the edited card, reset first when you want the highlight removed immediately.

Recommended:

```js
resetFormToCreateMode();
renderProducts();
```

Now the render function sees:

```js
editingProductId === null;
```

# 16.12 Clearing Old Messages

Before submission:

```js
clearError();
clearSuccess();
```

Functions:

```js
function clearError() {
  errorMessage.textContent = "";
}

function clearSuccess() {
  successMessage.textContent = "";
}
```

This prevents old feedback from remaining visible during a new operation.

# 16.13 Preserve the Updated Name Before Reset

Suppose the success message should include the updated product name.

The form will be reset after the update.

Store the name first:

```js
const updatedName = data.name;
```

Then:

```js
resetFormToCreateMode();
renderProducts();

showSuccess(
  `${updatedName} was updated successfully.`
);
```

---

# 16.14 Handling a Failed Update

If the selected ID is stale:

```js
let wasUpdated = false;
```

After searching:

```js
if (!wasUpdated) {
  showError(
    "The product could not be updated because it no longer exists."
  );

  resetFormToCreateMode();
  renderProducts();

  return;
}
```

Whether to reset the form depends on the application's design.

For a missing item, resetting is usually safest because edit mode is no longer valid.

---

# 16.15 Render Only After a Real State Change

Invalid form:

```text
Name: empty
```

Validation fails.

Do not update state.

A render is usually unnecessary because the product list did not change.

Instead:

```js
showError(
  "Product name is required."
);

return;
```

Render only when:

* State changed.
* UI-derived output must be refreshed.
* A stale display needs correction.

---

# 16.16 Re-rendering Derived Values

A render function may update more than the list.

Examples:

```js
productCount.textContent =
  `Total products: ${products.length}`;
```

Inventory value:

```js
totalValue.textContent =
  `Total inventory value: KSh ${calculatedTotal}`;
```

Out-of-stock count:

```js
outOfStockCount.textContent =
  `Out of stock: ${count}`;
```

All these values should reflect the current state after an update.

---

# 16.17 Re-rendering Conditional Classes

Suppose quantity changes from:

```text
0
```

to:

```text
5
```

Before update, the card may have:

```html
class="product-card out-of-stock"
```

After the update and re-render:

```html
class="product-card"
```

A complete re-render recalculates the class from state.

---

# 16.18 Re-rendering Button Data

Each action button should continue using the stable ID.

```js
editButton.dataset.id =
  product.id;

deleteButton.dataset.id =
  product.id;
```

The item's editable fields may change, but its ID remains the same.

---

# 16.19 Event Delegation Survives Re-rendering

If the parent has one listener:

```js
productList.addEventListener(
  "click",
  handleProductListClick
);
```

then newly rendered buttons still work.

Why?

The listener is attached to:

```js
productList
```

which remains in the DOM.

The child cards and buttons may be replaced, but click events bubble to the parent.

This is one major benefit of event delegation.

---

# 16.20 Direct Child Listeners and Re-rendering

Suppose listeners are attached directly inside the render function:

```js
editButton.addEventListener(
  "click",
  handleEdit
);
```

When the list is cleared, those buttons and their listeners are removed.

The new buttons need new listeners.

This can work, but event delegation is easier for dynamic lists.

# 16.21 Full Update Function

```js
function updateProduct(
  data
) {
  const productId =
    editingProductId;

  let wasUpdated = false;

  for (
    let index = 0;
    index < products.length;
    index++
  ) {
    if (
      products[index].id ===
      productId
    ) {
      products[index].name =
        data.name;

      products[index].price =
        data.price;

      products[index].quantity =
        data.quantity;

      products[index].category =
        data.category;

      products[index].isActive =
        data.isActive;

      wasUpdated = true;
      break;
    }
  }

  if (!wasUpdated) {
    showError(
      "The selected product no longer exists."
    );

    resetFormToCreateMode();
    renderProducts();

    return false;
  }

  resetFormToCreateMode();
  renderProducts();

  showSuccess(
    `${data.name} was updated successfully.`
  );

  return true;
}
```

---

# 16.22 Why Return a Boolean?

The function can return:

```js
true
```

for success and:

```js
false
```

for failure.

Usage:

```js
const updateSucceeded =
  updateProduct(data);

if (updateSucceeded) {
  console.log(
    "Update complete."
  );
}
```

This makes the function easier to coordinate and test.

---

# 16.23 Submit Handler with Create and Update

```js
function handleProductSubmit(
  event
) {
  event.preventDefault();

  clearError();
  clearSuccess();

  const rawData =
    readProductForm();

  const result =
    validateProductForm(rawData);

  if (!result.isValid) {
    showError(result.message);

    focusInvalidField(
      result.field
    );

    return;
  }

  if (
    editingProductId === null
  ) {
    createProduct(
      result.normalizedData
    );

    return;
  }

  updateProduct(
    result.normalizedData
  );
}
```

This completes mode-based form behavior.

---

# 16.24 Update and Re-render Architecture

```text
handleProductSubmit()
├── readProductForm()
├── validateProductForm()
├── updateProduct()
│   ├── locate matching object
│   ├── mutate editable properties
│   ├── detect failure
│   ├── resetFormToCreateMode()
│   ├── renderProducts()
│   └── showSuccess()
└── return
```

# 16.25 Three Complete Examples

## Example 1: State Update Followed by Console Rendering

### Problem

Update an item and print the current state after the update.

### Input Data

```js
let products = [
  {
    id: 1,
    name: "Laptop",
  },
  {
    id: 2,
    name: "Phone",
  },
];
```

### Complete Code

```js
let products = [
  {
    id: 1,
    name: "Laptop",
  },
  {
    id: 2,
    name: "Phone",
  },
];

function renderProducts() {
  console.log(
    "Current products:"
  );

  for (
    let index = 0;
    index < products.length;
    index++
  ) {
    console.log(
      products[index].name
    );
  }
}

function updateProductName(
  productId,
  newName
) {
  let wasUpdated = false;

  for (
    let index = 0;
    index < products.length;
    index++
  ) {
    if (
      products[index].id ===
      productId
    ) {
      products[index].name =
        newName;

      wasUpdated = true;
      break;
    }
  }

  if (!wasUpdated) {
    console.log(
      "Product not found."
    );

    return;

  2,
  "Smartphone"
);
```

    id: 2,
    name: "Smartphone",
  },
]
```

### Common Mistakes

* Calling the render function before the update.
* Updating the wrong ID.
* Rendering despite a failed update without explaining the failure.

Return `true` or `false` from `updateProductName()`.

      name,

        return {
          isValid: false,
          message:
            "Price is required.",
          field: "price",
        };
      }

      const price =
        Number(data.priceText);

      if (
        !Number.isFinite(price) ||
        price < 0
      ) {
        return {
          isValid: false,
          message:
            "Price must be a non-negative number.",
          field: "price",
        };
      }

      if (
        data.quantityText === ""
      ) {
        return {
          isValid: false,
          message:
            "Quantity is required.",
          field: "quantity",
        };
      }

      const quantity =
        Number(data.quantityText);

      if (
        !Number.isInteger(quantity) ||
        quantity < 0
      ) {
        return {
          isValid: false,
          message:
            "Quantity must be a non-negative whole number.",
          field: "quantity",
        };
      }

      if (data.category === "") {
        return {
          isValid: false,
          message:
            "Please select a category.",
          field: "category",
        };
      }

      return {
        isValid: true,
        message: "",
        field: null,
        normalizedData: {
          name: data.name,
          price,
          quantity,
          category: data.category,
          isActive: data.isActive,
        },
      };
    }

    function focusInvalidField(
      field
    ) {
      if (field === "name") {
        productNameInput.focus();
      } else if (field === "price") {
        productPriceInput.focus();
      } else if (
        field === "quantity"
      ) {
        productQuantityInput.focus();
      } else if (
        field === "category"
      ) {
        productCategorySelect.focus();
      }
    }

    function generateProductId() {
      const id = nextProductId;

      nextProductId++;

      return id;
    }

    function loadProductIntoForm(
      product
    ) {
      productNameInput.value =
        product.name;

      productPriceInput.value =
        product.price;

      productQuantityInput.value =
        product.quantity;

      productCategorySelect.value =
        product.category;

      productActiveInput.checked =
        product.isActive;
    }

    function setCreateModeUI() {
      formTitle.textContent =
        "Add Product";

      submitButton.textContent =
        "Add Product";

      cancelEditButton.hidden =
        true;
    }

    function setEditModeUI() {
      formTitle.textContent =
        "Edit Product";

      submitButton.textContent =
        "Update Product";

      cancelEditButton.hidden =
        false;
    }

    function resetFormToCreateMode() {
      editingProductId = null;

      productForm.reset();

      productActiveInput.checked =
        true;

      setCreateModeUI();

      productNameInput.focus();
    }

    function createProduct(data) {
      const product = {
        id: generateProductId(),
        name: data.name,
        price: data.price,
        quantity: data.quantity,
        category: data.category,
        isActive: data.isActive,
      };

      products.push(product);

      resetFormToCreateMode();
      renderProducts();

      showSuccess(
        `${product.name} was added successfully.`
      );
    }

    function updateProduct(data) {
      let wasUpdated = false;

      for (
        let index = 0;
        index < products.length;
        index++
      ) {
        if (
          products[index].id ===
          editingProductId
        ) {
          products[index].name =
            data.name;

          products[index].price =
            data.price;

          products[index].quantity =
            data.quantity;

          products[index].category =
            data.category;

          products[index].isActive =
            data.isActive;

          wasUpdated = true;
          break;
        }
      }

      if (!wasUpdated) {
        showError(
          "The selected product no longer exists."
        );

        resetFormToCreateMode();
        renderProducts();

        return;
      }

      const updatedName =
        data.name;

      resetFormToCreateMode();
      renderProducts();

      showSuccess(
        `${updatedName} was updated successfully.`
      );
    }

    function startEditingProduct(
      productId
    ) {
      clearError();
      clearSuccess();

      const product =
        findProductById(productId);

      if (product === null) {
        showError(
          "The selected product was not found."
        );

        return;
      }

      editingProductId =
        product.id;

      loadProductIntoForm(product);
      setEditModeUI();

      productNameInput.focus();
      productNameInput.select();
    }

    function renderEmptyState() {
      const emptyState =
        document.createElement("li");

      emptyState.textContent =
        "No products available.";

      productList.appendChild(
        emptyState
      );
    }

    function renderProductCard(
      product,
      index
    ) {
      const card =
        document.createElement("li");

      const name =
        document.createElement("h3");

      const details =
        document.createElement("p");

      const status =
        document.createElement("p");

      const editButton =
        document.createElement("button");

      card.classList.add(
        "product-card"
      );

      card.dataset.id =
        product.id;

      name.textContent =
        `${index + 1}. ${product.name}`;

      details.textContent =
        `KSh ${product.price.toLocaleString()} — ` +
        `Quantity: ${product.quantity} — ` +
        `Category: ${product.category}`;

      status.textContent =
        product.isActive
          ? "Status: Active"
          : "Status: Inactive";

      editButton.type =
        "button";

      editButton.textContent =
        "Edit";

      editButton.dataset.action =
        "edit";

      editButton.dataset.id =
        product.id;

      card.append(
        name,
        details,
        status,
        editButton
      );

      productList.appendChild(card);
    }

    function renderProducts() {
      productList.innerHTML = "";

      productCount.textContent =
        `Total products: ${products.length}`;

      if (products.length === 0) {
        renderEmptyState();
        return;
      }

      for (
        let index = 0;
        index < products.length;
        index++
      ) {
        renderProductCard(
          products[index],
          index
        );
      }
    }

    function handleProductSubmit(
      event
    ) {
      event.preventDefault();

      clearError();
      clearSuccess();

      const rawData =
        readProductForm();

      const result =
        validateProductForm(
          rawData
        );

      if (!result.isValid) {
        showError(result.message);

        focusInvalidField(
          result.field
        );

        return;
      }

      if (
        editingProductId === null
      ) {
        createProduct(
          result.normalizedData
        );

        return;
      }

      updateProduct(
        result.normalizedData
      );
    }

    productList.addEventListener(
      "click",
      function (event) {
        const editButton =
          event.target.closest(
            '[data-action="edit"]'
          );

        if (editButton === null) {
          return;
        }

        const productId =
          Number(
            editButton.dataset.id
          );

        if (
          !Number.isInteger(
            productId
          ) ||
          productId <= 0
        ) {
          showError(
            "Invalid product ID."
          );

          return;
        }

        startEditingProduct(
          productId
        );
      }
    );

    cancelEditButton.addEventListener(
      "click",
      function () {
        clearError();
        clearSuccess();

        resetFormToCreateMode();
        renderProducts();
      }
    );

    productForm.addEventListener(
      "submit",
      handleProductSubmit
    );

    setCreateModeUI();
    renderProducts();
  </script>
</body>
</html>
```

## Key Takeaways

* Updating JavaScript state does not automatically update the DOM.
* Re-rendering means displaying the current state again.
* The state must change before rendering.
* A reusable render function should clear and rebuild the list.
* Clearing the DOM does not clear state.
* Rendering should read state without mutating it.
* Edit mode should reset after a successful Update.
* Derived counts, totals, statuses, and classes should be refreshed.
* Missing items should be handled safely.
* Event delegation keeps dynamic buttons functional after re-rendering.
* A complete Update flow ends only when the user sees the new state in the DOM.
