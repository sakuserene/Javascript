# 12. Finding an Item by ID

## 12.1 What You Will Learn

Before updating an item, the application must answer:

> Which exact object should be changed?

Suppose the state contains:

```js
let products = [
  {
    id: 101,
    name: "Laptop",
  },
  {
    id: 205,
    name: "Phone",
  },
  {
    id: 309,
    name: "Keyboard",
  },
];
```

The user clicks the Edit button for ID `205`.

The application must:

1. Read the selected ID.
2. Loop through the array.
3. Compare every product ID.
4. Stop when the correct product is found.
5. Store the matching object.
6. Handle the case where no match exists.

---

# 12.2 The Update Operation Begins with Read

Update is not only about assigning new values.

A complete Update flow begins by reading the selected item.

```text
User clicks Edit
        ↓
Read selected ID
        ↓
Find matching item
        ↓
Load existing values
        ↓
User changes values
        ↓
Validate changes
        ↓
Update matching object
        ↓
Render updated state
```

Therefore, locating the item is part of the Update process.

# 12.3 Basic Plain-Loop Search

```js
const selectedId = 205;

let selectedProduct = null;

for (
  let index = 0;
  index < products.length;
  index++
) {
  if (
    products[index].id === selectedId
  ) {
    selectedProduct =
      products[index];

    break;
  }
}
```

After the loop:

```js
selectedProduct
```

contains:

```js
{
  id: 205,
  name: "Phone",
}
```

If no match exists, it remains:

```js
null
```

---

# 12.4 Why Initialize with `null`?

```js
let selectedProduct = null;
```

`null` clearly means:

```text
No product has been found yet.
```

If the loop finds a match:

```js
selectedProduct = products[index];
```

If not, the value remains `null`.

This makes the result easy to test.

```js
if (selectedProduct === null) {
  console.log("Product not found.");
}
```

---

# 12.5 Syntax Breakdown

Consider:

```js
if (
  products[index].id === selectedId
)
```

## `products`

The array containing all product objects.

## `products[index]`

The current product during this loop repetition.

## `.id`

The stable identity of the current product.

## `selectedId`

The ID requested by the user or event handler.

## `===`

Strict comparison.

The condition is true only when both:

* Values match.
* Types match.

---

# 12.6 Step-by-Step Search

State:

```js
const products = [
  {
    id: 101,
    name: "Laptop",
  },
  {
    id: 205,
    name: "Phone",
  },
  {
    id: 309,
    name: "Keyboard",
  },
];
```

Search:

```js
const selectedId = 205;
```

### First repetition

```js
index = 0;
```

Comparison:

```js
101 === 205;
```

Result:

```text
false
```

Continue.

### Second repetition

```js
index = 1;
```

Comparison:

```js
205 === 205;
```

Result:

```text
true
```

Store:

```js
selectedProduct =
  products[1];
```

Then:

```js
break;
```

The search ends.

---

# 12.7 Why Use `break`?

Once a unique ID match is found, there is no reason to continue.

```js
break;
```

Benefits:

* Stops unnecessary work.
* Makes the intention clear.
* Prevents accidental later matches if data contains duplicate IDs.
* Improves efficiency for large arrays.

Without `break`, the loop checks every remaining item.

---

# 12.8 Handling a Missing ID

Search:

```js
const selectedId = 999;
```

No product has ID `999`.

After the loop:

```js
selectedProduct === null;
```

Handle it:

```js
if (selectedProduct === null) {
  console.log(
    "Product with that ID was not found."
  );
}
```

A missing ID should not cause the application to crash.

# 12.9 Reusable `findProductById()` Function

```js
function findProductById(productId) {
  for (
    let index = 0;
    index < products.length;
    index++
  ) {
    if (
      products[index].id ===
      productId
    ) {
      return products[index];
    }
  }

  return null;
}
```

Usage:

```js
const selectedProduct =
  findProductById(205);
```

Result:

```js
{
  id: 205,
  name: "Phone",
}
```

---

# 12.10 Returning Instead of Using `break`

Inside a function, `return` can stop both:

* The loop.
* The function.

```js
function findProductById(productId) {
  for (
    let index = 0;
    index < products.length;
    index++
  ) {
    if (
      products[index].id ===
      productId
    ) {
      return products[index];
    }
  }

  return null;
}
```

This is often cleaner than:

```js
let selectedProduct = null;

for (...) {
  if (...) {
    selectedProduct =
      products[index];

    break;
  }
}

return selectedProduct;
```

Both approaches are valid.

# 12.11 Finding an Item Versus Finding Its Index

These are different operations.

## Finding the item

```js
let selectedProduct = null;

for (
  let index = 0;
  index < products.length;
  index++
) {
  if (
    products[index].id === selectedId
  ) {
    selectedProduct =
      products[index];

    break;
  }
}
```

Result:

```js
{
  id: 205,
  name: "Phone"
}
```

## Finding the index

```js
let selectedIndex = -1;

for (
  let index = 0;
  index < products.length;
  index++
) {
  if (
    products[index].id === selectedId
  ) {
    selectedIndex = index;

    break;
  }
}
```

Result:

```text
1
```

# 12.12 When You Need the Object

Use the object when you need to:

* Read existing values.
* Display item details.
* Load a form.
* Update properties directly.
* Check item status.

Example:

```js
selectedProduct.name;
selectedProduct.price;
selectedProduct.quantity;
```

# 12.13 When You Need the Index

Use the index when you need to:

* Replace an object at a position.
* Delete using `splice()`.
* Compare neighboring items.
* Reorder items.
* Update using index-based syntax.

Example:

```js
products[selectedIndex].name =
  "Updated Phone";
```

Later, Delete will rely heavily on finding the index.

# 12.14 Sentinel Values

A sentinel value represents “not found.”

For an object:

```js
let selectedProduct = null;
```

For an index:

```js
let selectedIndex = -1;
```

Why `-1`?

Valid array indexes begin at:

```text
0
```

Therefore:

```text
-1
```

clearly means:

```text
No valid position was found.
```

# 12.15 String IDs from the DOM

Rendered button:

```html
<button
  data-id="205"
  data-action="edit"
>
  Edit
</button>
```

JavaScript:

```js
const selectedId =
  button.dataset.id;
```

Result:

```text
"205"
```

Type:

```text
string
```

State ID:

```js
205
```

Type:

```text
number
```

Strict comparison fails:

```js
205 === "205";
```

Result:

```text
false
```

---

# 12.16 Converting DOM IDs

Convert with `Number()`:

```js
const selectedId =
  Number(button.dataset.id);
```

Now:

```js
205 === selectedId;
```

returns:

```text
true
```

Always remember:

> Values read from `dataset` are strings.

---

# 12.17 Validating the Converted ID

Conversion can fail.

```js
const selectedId =
  Number(button.dataset.id);
```

Check:

```js
if (
  !Number.isInteger(selectedId)
) {
  showError(
    "The selected product ID is invalid."
  );

  return;
}
```

For this application, IDs are positive integers.

A stronger check:

```js
if (
  !Number.isInteger(selectedId) ||
  selectedId <= 0
) {
  showError(
    "The selected product ID is invalid."
  );

  return;
}
```

# 12.18 Object References Matter

When you find an object:

```js
const selectedProduct =
  findProductById(205);
```

`selectedProduct` refers to the same object stored in the array.

If you write:

```js
selectedProduct.name =
  "Smartphone";
```

then the array also reflects the change.

```js
console.log(
  products[1].name
);
```

Output:

```text
Smartphone
```

This happens because both references point to the same object.

---

# 12.19 Direct Reference Update

```js
const selectedProduct =
  findProductById(205);

if (selectedProduct !== null) {
  selectedProduct.price = 27000;
}
```

Final state:

```js
[
  {
    id: 101,
    name: "Laptop",
    price: 50000,
  },
  {
    id: 205,
    name: "Phone",
    price: 27000,
  },
]
```

This is a mutable update.

Immutable updates will be taught later with `map()`.

---

# 12.20 Do Not Change the ID During Update

Incorrect:

```js
selectedProduct.id = 500;
```

An item's identity should remain stable.

Update editable properties such as:

```js
selectedProduct.name;
selectedProduct.price;
selectedProduct.quantity;
```

Do not normally update:

```js
selectedProduct.id;
```

---

# 12.21 Searching by Name Versus ID

Searching by name:

```js
if (
  products[index].name ===
  selectedName
)
```

can be unreliable because names may:

* Repeat.
* Change.
* Differ in capitalization.
* Contain extra whitespace.

IDs are more reliable.

```js
if (
  products[index].id ===
  selectedId
)
```

# 12.22 Finding from an Edit Button

Rendered button:

```js
editButton.dataset.id =
  product.id;

editButton.dataset.action =
  "edit";
```

Click handler:

```js
editButton.addEventListener(
  "click",
  function (event) {
```

# 12.23 Event Delegation Preview

A parent list can listen for all child clicks.

```js
productList.addEventListener(
```

### Problem

Find a product with ID `205` using a standard `for` loop.

### Input Data

```js
const products = [
  {
    id: 101,

  },
```

### Complete Code

```js
const products = [
  {
    id: 101,
    name: "Laptop",
  },
  {
    id: 205,
    name: "Phone",
  },
  {
    id: 309,
    name: "Keyboard",
  },
];

const selectedId = 205;

let selectedProduct = null;

for (
  let index = 0;
  index < products.length;
  index++
) {
  const product =
    products[index];

  if (
    product.id === selectedId
  ) {
    selectedProduct = product;

    break;
  }

  console.log(

  );
} else {
  console.log(

  );

```

2. Initialize the result with `null`.
3. Loop through each product.
4. Compare the current ID.

6. Stop with `break`.
7. Check the result after the loop.

```text
{
  id: 205,
  name: "Phone"
}
```

The array is unchanged.

This is a Read operation.

### Common Mistakes

* Comparing the name instead of the ID.
* Forgetting `break`.
* Using assignment instead of comparison.
* Starting the loop at index `1`.

## Example 2: Reusable Search Function

### Problem

Create a reusable function that returns a product or `null`.

### Complete Code

```js
const products = [
  {
    id: 101,
    name: "Laptop",
    price: 50000,
  },
  {
    id: 205,
    name: "Phone",
    price: 25000,
  },
  {
    id: 309,
    name: "Keyboard",
    price: 3000,
  },
];

function findProductById(productId) {
  for (
    let index = 0;
    index < products.length;
    index++
  ) {
    if (
      products[index].id ===
      productId
    ) {
      return products[index];
    }
  }

  return null;
}

const firstResult =
  findProductById(309);

const secondResult =
  findProductById(999);

console.log(firstResult);
console.log(secondResult);
```

### Expected Console Output

```text
{
  id: 309,
  name: "Keyboard",

```

The products array remains unchanged.

#### Forgetting `return null`

The function returns `undefined` automatically.

#### Returning too early

Incorrect:

```js

  if (...) {
    return products[index];
  }

}
```

This stops after checking only the first item.

Correct:

```js
return null;
```

must appear after the loop.

## Example 3: Find a Product from a DOM Button

### Problem

Render product cards with Edit buttons and identify the clicked product.

### Complete Code

```html
<!DOCTYPE html>
<html lang="en">
<head>

  <meta
    name="viewport"
    content="width=device-width, initial-scale=1.0"
```

# 12.25 Finding with a Loop Versus Modern `find()`

## Plain Loop

```js
let selectedProduct = null;

for (
  let index = 0;
  index < products.length;
  index++
) {
  if (
    products[index].id ===
    selectedId
  ) {
    selectedProduct =
      products[index];

    break;
  }
}
```

## Array Method

```js
const selectedProduct =
  products.find(
    (product) =>
      product.id === selectedId
  );
```

Both versions search sequentially.

The array method performs the loop internally.

For now, continue using the plain-loop version so the search process is fully visible.

`find()` will be taught in detail later.

---

# 12.26 Which Version Gives More Control?

A plain loop makes it easy to:

* Track the current index.
* Stop with `break`.
* Count comparisons.
* Search conditionally.
* Add custom logging.
* Search from a custom starting position.
* Search backwards.

`find()` is usually shorter and more common in modern JavaScript.

Understanding the loop first makes `find()` easier to understand.

## Key Takeaways

* Update begins by locating the correct item.
* Stable IDs should be used instead of array indexes.
* A standard `for` loop can search one object at a time.
* Initialize an object result with `null`.
* Initialize an index result with `-1`.
* Stop searching when the item is found.
* A reusable search function should return the object or `null`.
* `dataset` values are strings and may require numeric conversion.
* Strict comparison prevents unintended type coercion.
* Missing IDs must be handled safely.
* A found object is a reference to the object in state.
* Mutating that reference changes state.
* Item IDs should remain unchanged.
* The next lesson will load the found item's existing values into the edit form.
