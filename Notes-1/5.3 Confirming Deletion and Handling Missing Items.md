# 19. Confirming Deletion and Handling Missing Items

## 19.1 What You Will Learn

Delete is destructive.

After an item is removed with:

```js
products.splice(index, 1);
```

the object is no longer in the main array.

A confirmation step helps prevent accidental deletion.

The safe flow is:

```text
User clicks Delete
        ↓
Read and validate ID
        ↓
Find selected product
        ↓
Ask for confirmation
        ↓
If canceled, stop
        ↓
Re-find current index
        ↓
If missing, stop safely
        ↓
Delete with splice()
        ↓
Reset related UI state
        ↓
Re-render
```

---

# 19.2 Why Confirmation Matters

Users may click Delete accidentally.

Possible causes include:

* Clicking the wrong button.
* Tapping twice on a mobile device.
* Confusing Edit and Delete buttons.
* Misreading the selected product.
* Acting before reviewing the product name.

A confirmation prompt gives the user a final chance to cancel.

---

# 19.3 Basic `confirm()` Syntax

```js
const shouldDelete =
  window.confirm(
    "Are you sure?"
  );
```

The function returns a boolean.

If the user clicks **OK**:

```js
shouldDelete === true;
```

If the user clicks **Cancel**:

```js
shouldDelete === false;
```

---

# 19.4 Shorter Syntax

Because `window` is global in the browser, you may write:

```js
const shouldDelete =
  confirm(
    "Are you sure?"
  );
```

Both versions work.

For clarity, this lesson will often use:

```js
window.confirm()
```

---

# 19.5 Stopping When the User Cancels

```js
const shouldDelete =
  window.confirm(
    "Delete this product?"
  );

if (!shouldDelete) {
  return;
}
```

If the user selects Cancel:

* The function stops.
* State remains unchanged.
* `splice()` does not run.
* The DOM remains unchanged.

---

# 19.6 Include the Product Name

Less helpful:

```text
Are you sure?
```

More helpful:

```text
Are you sure you want to delete Phone?
```

Code:

```js
const shouldDelete =
  window.confirm(
    `Are you sure you want to delete ${product.name}?`
  );
```

The message should clearly identify the target.

---

# 19.7 Find the Product Before Confirmation

Before asking for confirmation, find the selected product.

```js
const product =
  findProductById(productId);

if (product === null) {
  showError(
    "The selected product no longer exists."
  );

  return;
}
```

This allows the confirmation prompt to include the current product name.

---

# 19.8 Why Re-find the Index After Confirmation?

A confirmation prompt pauses the current JavaScript flow until the user responds.

In a simple synchronous browser application, state usually does not change during this pause.

However, a safer Delete design is:

1. Find the product for the confirmation message.
2. Ask for confirmation.
3. Re-find the current index immediately before deletion.

```js
const productIndex =
  findProductIndexById(
    productId
  );
```

This avoids relying on an old index.

---

# 19.9 Do Not Store a Long-Lived Index

Less safe:

```js
const indexToDelete =
  findProductIndexById(
    productId
  );

const shouldDelete =
  window.confirm(
    "Delete product?"
  );

products.splice(
  indexToDelete,
  1
);
```

Safer:

```js
const shouldDelete =
  window.confirm(
    "Delete product?"
  );

if (!shouldDelete) {
  return;
}

const currentIndex =
  findProductIndexById(
    productId
  );

if (currentIndex === -1) {
  return;
}

products.splice(
  currentIndex,
  1
);
```

Stable IDs should be preserved longer than indexes.

# 19.10 Complete Confirmation Function

```js
function confirmProductDeletion(
  product
) {
  return window.confirm(
    `Are you sure you want to delete ${product.name}?`
  );
}
```

Usage:

```js
const shouldDelete =
  confirmProductDeletion(
    product
  );

if (!shouldDelete) {
  return;
}
```

This function has one responsibility:

```text
Ask the user whether deletion should continue.
```

# 19.11 Cancellation Is Not an Error

If the user clicks Cancel, the application should not normally display:

```text
Deletion failed.
```

The system behaved correctly.

You may:

* Display nothing.
* Display a neutral message.
* Log the cancellation.

Optional:

```js
showInfo(
  "Deletion canceled."
);
```

Do not treat a deliberate cancellation like an application error.

# 19.12 Handling an Invalid ID

Read from the button:

```js
const productId =
  Number(
    deleteButton.dataset.id
  );
```

Validate:

```js
if (
  !Number.isInteger(productId) ||
  productId <= 0
) {
  showError(
    "The selected product ID is invalid."
  );

  return;
}
```

Do not continue to lookup or deletion.

---

# 19.13 Handling a Missing Product Before Confirmation

```js
const product =
  findProductById(productId);

if (product === null) {
  showError(
    "The selected product could not be found."
  );

  renderProducts();

  return;
}
```

Re-rendering may remove stale buttons from the page if the DOM was outdated.

---

# 19.14 Handling a Missing Index After Confirmation

```js
const productIndex =
  findProductIndexById(
    productId
  );

if (productIndex === -1) {
  showError(
    "The product no longer exists."
  );

  renderProducts();

  return;
}
```

This prevents:

```js
products.splice(-1, 1);
```

from deleting the final product accidentally.

---

# 19.15 Repeated Delete Clicks

A user may click the Delete button multiple times rapidly.

In synchronous deletion:

1. The first click deletes the product.
2. The list re-renders.
3. The old button is removed.

If another stale event is processed, the item may no longer exist.

Missing-item checks make this safe.

```js
if (product === null) {
  showError(
    "The product has already been deleted."
  );

  return;
}
```

---

# 19.16 Disable the Delete Button During the Operation

For synchronous deletion, confirmation blocks interaction temporarily.

For future asynchronous deletion, disabling is useful:

```js
deleteButton.disabled = true;
```

If deletion is canceled:

```js
deleteButton.disabled = false;
```

For this basic lesson, missing-item handling is the main protection.

---

# 19.17 Delete Result Objects

Instead of returning only the deleted object or `null`, a function can return a structured result.

```js
function deleteProductById(
  productId
) {
  const productIndex =
    findProductIndexById(
      productId
    );

  if (productIndex === -1) {
    return {
      success: false,
      deletedProduct: null,
      message:
        "Product not found.",
    };
  }

  const removedProducts =
    products.splice(
      productIndex,
      1
    );

  return {
    success: true,
    deletedProduct:
      removedProducts[0],
    message:
      "Product deleted.",
  };
}
```

This gives the caller clear information.

---

# 19.18 Confirmation Should Not Be Inside Low-Level State Logic

A reusable state function should ideally focus on state.

State logic:

```js
function deleteProductById(
  productId
) {
  // Find and remove.
}
```

UI logic:

```js
function handleDeleteRequest(
  productId
) {
  // Ask user.
  // Call state logic.
  // Show messages.
  // Render.
}
```

This separation makes the delete function easier to test.

---

# 19.19 Recommended Separation

```text
handleDeleteRequest(productId)
├── validate ID
├── find product
├── ask for confirmation
├── call deleteProductById(productId)
├── reset related UI state
├── renderProducts()
└── show feedback
```

State function:

```text
deleteProductById(productId)
├── find current index
├── handle missing index
├── splice one object
└── return result
```

---

# 19.20 Reusable Delete Request Function

```js
function handleDeleteRequest(
  productId
) {
  clearError();
  clearSuccess();

  const product =
    findProductById(
      productId
    );

  if (product === null) {
    showError(
      "The selected product no longer exists."
    );

    renderProducts();
    return false;
  }

  const shouldDelete =
    confirmProductDeletion(
      product
    );

  if (!shouldDelete) {
    return false;
  }

  const result =
    deleteProductById(
      productId
    );

  if (!result.success) {
    showError(
      result.message
    );

    renderProducts();
    return false;
  }

  if (
    editingProductId ===
    productId
  ) {
    resetFormToCreateMode();
  }

  renderProducts();

  showSuccess(
    `${result.deletedProduct.name} was deleted successfully.`
  );

  return true;
}
```

---

# 19.21 Canceling an Active Edit After Deletion

If:

```js
editingProductId ===
  productId
```

then the deleted product is loaded in the form.

Reset:

```js
resetFormToCreateMode();
```

Otherwise, the form points to an object that no longer exists.

---

# 19.22 Deleting Another Product While Editing

Suppose the user is editing Laptop:

```js
editingProductId = 101;
```

and deletes Phone:

```js
productId = 205;
```

This condition is false:

```js
editingProductId ===
  productId
```

The Laptop edit remains active.

This is correct because the active product still exists.

---

# 19.23 Confirmation Message Security

Use product names as plain text in application DOM.

For `window.confirm()`, the value is displayed as text by the browser.

When displaying custom confirmation content in the DOM, prefer:

```js
messageElement.textContent =
  product.name;
```

Do not insert untrusted product names using:

```js
messageElement.innerHTML =
  product.name;
```

---

# 19.24 Native Confirmation Limitations

`window.confirm()` is useful for learning because it is:

* Built into the browser.
* Synchronous.
* Simple.
* Easy to understand.

Limitations:

* Appearance cannot be customized easily.
* It blocks JavaScript execution.
* Browser styling differs.
* It offers limited accessibility control.
* It is less flexible than a custom modal.

A custom confirmation modal can be introduced later.

---

# 19.25 Custom Confirmation State Preview

A custom modal might use:

```js
let pendingDeleteProductId =
  null;
```

Flow:

```text
Click Delete
    ↓
Store pending ID
    ↓
Show modal
    ↓
User confirms
    ↓
Re-find by ID
    ↓
Delete
```

Stable IDs remain important.

This lesson will continue using `window.confirm()`.

---

# 19.26 Confirmation Does Not Guarantee the Item Still Exists

The user may confirm a product that has already been deleted by another process in a more complex application.

Therefore, confirmation is not a replacement for current-state validation.

Always re-check:

```js
const index =
  findProductIndexById(
    productId
  );
```

before mutation.

# 19.27 Three Complete Examples

## Example 1: Basic Confirmation Without the DOM

### Problem

Delete a product only when a simulated confirmation value is true.

### Input Data

```js
let products = [
  {
    id: 1,
    name: "Laptop",
  },
  {
    id: 2,
    name: "Phone",
  },
];
```

### Complete Code

```js
let products = [
  {
    id: 1,
    name: "Laptop",
  },
  {
    id: 2,
    name: "Phone",
  },
];

function findProductIndexById(
  productId
) {
  for (
    let index = 0;
    index < products.length;
    index++
  ) {
    if (
      products[index].id ===
      productId
    ) {
      return index;
    }
  }

  return -1;
}

function deleteProductById(
  productId,
  shouldDelete
) {
  if (!shouldDelete) {
    return {
      success: false,
      reason: "canceled",
    };
  }

  const productIndex =
    findProductIndexById(
      productId
    );

  if (productIndex === -1) {
    return {
      success: false,
      reason: "not-found",
    };
  }

  const removed =
    products.splice(
      productIndex,
      1
    );

  return {
    success: true,
    deletedProduct:
      removed[0],
  };
}

const result =
  deleteProductById(
    2,
    true
  );

console.log(result);
console.log(products);
```

### Expected Console Output

```text
{
  success: true,
  deletedProduct: {
    id: 2,
    name: "Phone"
  }
}

[
  {
    id: 1,
    name: "Laptop"
  }
]
```

### Final State

Phone is removed.

### Common Mistakes

* Deleting before checking the confirmation.
* Treating cancellation as a missing-item error.
* Failing to return after cancellation.

### Small Modification Challenge

Change:

```js
true
```

to:

```js
false
```

Confirm that state remains unchanged.

### Problem

Use `window.confirm()` before removing a selected product.

        index++
      ) {
        if (
          products[index].id ===

        }
      }

      return null;

    function findProductIndexById(

## Example 3: Confirmed Deletion with Dynamic Product Cards

### Problem

Build dynamic Delete buttons with:

* Stable IDs.
* Event delegation.
* Confirmation.
* Missing-item handling.
* Re-rendering.
* Edit-mode safety.

### Complete Code

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8" />

  <meta
    name="viewport"
    content="width=device-width, initial-scale=1.0"
  />

  <title>Confirmed Product Deletion</title>

  <style>
    * {
      box-sizing: border-box;
    }

    body {
      max-width: 760px;
      margin: 0 auto;
      padding: 24px;
      font-family: Arial, sans-serif;
      line-height: 1.5;
    }

      padding: 0;
      list-style: none;
    }

    .product-card {
      margin-bottom: 12px;
      padding: 14px;

      border-radius: 6px;

    .actions {
      display: flex;

    }

<body>
  <h1>Product Inventory</h1>

  <p
    id="error-message"
    role="alert"
  ></p>

  <p
    id="success-message"
    role="status"
  ></p>

  <p id="product-count"></p>

  <ul id="product-list"></ul>

  <script>
    let products = [
      {
        id: 101,
        name: "Laptop",
        price: 50000,
      },
      {
        id: 205,
        name: "Phone",
        price: 25000,
      },
      {
        id: 309,
        name: "Keyboard",
        price: 3000,
      },
    ];

    let editingProductId = null;

    const errorMessage =
      document.getElementById(
        "error-message"
      );

    const successMessage =
      document.getElementById(
        "success-message"
      );

    const productCount =
      document.getElementById(
        "product-count"
      );

    const productList =
      document.getElementById(
        "product-list"
      );

    function showError(message) {
      errorMessage.textContent =
        message;
    }

    function clearError() {
      errorMessage.textContent = "";
    }

    function showSuccess(message) {
      successMessage.textContent =
        message;
    }

    function clearSuccess() {
      successMessage.textContent = "";
    }

    function findProductById(
      productId
    ) {
      for (
        let index = 0;
        index < products.length;
        index++
      ) {
        if (
          products[index].id ===
          productId
        ) {
          return products[index];
        }
      }

      return null;
    }

    function findProductIndexById(
      productId
    ) {
      for (
        let index = 0;
        index < products.length;
        index++
      ) {
        if (
          products[index].id ===
          productId
        ) {
          return index;
        }
      }

      return -1;
    }

    function confirmProductDeletion(
      product
    ) {
      return window.confirm(
        `Are you sure you want to delete ${product.name}?`
      );
    }

    function deleteProductById(
      productId
    ) {
      const productIndex =
        findProductIndexById(
          productId
        );

      if (
        productIndex === -1
      ) {
        return {
          success: false,
          deletedProduct: null,
          message:
            "The product was not found.",
        };
      }

      const removedProducts =
        products.splice(
          productIndex,
          1
        );

      return {
        success: true,
        deletedProduct:
          removedProducts[0],
        message:
          "Product deleted.",
      };
    }

    function resetFormToCreateMode() {
      editingProductId = null;
    }

    function renderProducts() {
      productList.innerHTML = "";

      productCount.textContent =
        `Total products: ${products.length}`;

      if (products.length === 0) {
        const emptyState =
          document.createElement("li");

        emptyState.textContent =
          "No products available.";

        productList.appendChild(
          emptyState
        );

        return;
      }

      for (
        let index = 0;
        index < products.length;
        index++
      ) {
        const product =
          products[index];

        const card =
          document.createElement("li");

        const name =
          document.createElement("h2");

        const price =
          document.createElement("p");

        const actions =
          document.createElement("div");

        const editButton =
          document.createElement("button");

        const deleteButton =
          document.createElement("button");

        card.classList.add(
          "product-card"
        );

        actions.classList.add(
          "actions"
        );

        name.textContent =
          product.name;

        price.textContent =
          `KSh ${product.price.toLocaleString()}`;

        editButton.type =
          "button";

        editButton.textContent =
          "Edit";

        editButton.dataset.action =
          "edit";

        editButton.dataset.id =
          product.id;

        deleteButton.type =
          "button";

        deleteButton.textContent =
          "Delete";

        deleteButton.dataset.action =
          "delete";

        deleteButton.dataset.id =
          product.id;

        actions.append(
          editButton,
          deleteButton
        );

        card.append(
          name,
          price,
          actions
        );

        productList.appendChild(
          card
        );
      }
    }

    function handleDeleteRequest(
      productId
    ) {
      clearError();
      clearSuccess();

      const product =
        findProductById(
          productId
        );

      if (product === null) {
        showError(
          "The selected product no longer exists."
        );

        renderProducts();
        return false;
      }

      const shouldDelete =
        confirmProductDeletion(
          product
        );

      if (!shouldDelete) {
        return false;
      }

      const result =
        deleteProductById(
          productId
        );

      if (!result.success) {
        showError(
          result.message
        );

        renderProducts();
        return false;
      }

      if (
        editingProductId ===
        productId
      ) {
        resetFormToCreateMode();
      }

      renderProducts();

      showSuccess(
        `${result.deletedProduct.name} was deleted successfully.`
      );

      return true;
    }

    productList.addEventListener(
      "click",
      function (event) {
        const clickedButton =
          event.target.closest(
            "button"
          );

        if (
          clickedButton === null
        ) {
          return;
        }

        const action =
          clickedButton.dataset.action;

        const productId =
          Number(
            clickedButton.dataset.id
          );

        if (
          !Number.isInteger(
            productId
          ) ||
          productId <= 0
        ) {
          showError(
            "Invalid product ID."
          );

          return;
        }

        if (action === "edit") {
          editingProductId =
            productId;

          showSuccess(
            `Editing product ID ${productId}.`
          );

          return;
        }

        if (action === "delete") {
          handleDeleteRequest(
            productId
          );
        }
      }
    );

    renderProducts();
  </script>
</body>
</html>
```

## Key Takeaways

* Delete confirmation helps prevent accidental destructive actions.
* `window.confirm()` returns `true` or `false`.
* Cancellation must stop the Delete flow before state mutation.
* Cancellation is a valid user choice, not an application error.
* Find the product before confirmation so the target is clearly identified.
* Store stable IDs rather than long-lived indexes.
* Re-find the current index immediately before `splice()`.
* Always check for `-1`.
* Handle repeated and stale Delete actions safely.
* Keep confirmation UI separate from low-level state mutation.
* Reset edit mode only when its product is deleted.
* The next lesson will combine confirmation, deletion, feedback, UI reset, and re-rendering into the complete Delete operation.
