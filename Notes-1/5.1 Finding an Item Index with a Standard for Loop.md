# 17. Finding an Item Index

## 17.1 What You Will Learn

The Delete operation usually follows this flow:

```text
User clicks Delete
        ↓
Read the selected item's ID
        ↓
Convert and validate the ID
        ↓
Find the item's array index
        ↓
Handle a missing item
        ↓
Ask for confirmation
        ↓
Remove the item
        ↓
Re-render the DOM
```

This lesson focuses on:

```text
Find the item's array index
```

---

# 17.2 Why Delete Often Needs an Index

JavaScript's `splice()` method removes items using their positions.

Example:

```js
products.splice(1, 1);
```

This means:

```text
Begin at index 1
Remove 1 item
```

Therefore, before using `splice()`, the application must know the selected item's current index.

---

# 17.3 ID Versus Index

Consider:

```js
const products = [
  {
    id: 101,
    name: "Laptop",
  },
  {
    id: 205,
    name: "Phone",
  },
  {
    id: 309,
    name: "Keyboard",
  },
];
```

The values are:

| Product  |    ID | Index |
| -------- | ----: | ----: |
| Laptop   | `101` |   `0` |
| Phone    | `205` |   `1` |
| Keyboard | `309` |   `2` |

The ID identifies the item.

The index identifies its current array position.

Do not confuse them.

---

# 17.4 Basic Index Search

```js
const selectedId = 205;

let selectedIndex = -1;

for (
  let index = 0;
  index < products.length;
  index++
) {
  if (
    products[index].id === selectedId
  ) {
    selectedIndex = index;

    break;
  }
}
```

After the loop:

```js
selectedIndex === 1;
```

---

# 17.5 Why Initialize with `-1`?

Valid array indexes begin at:

```text
0
```

Therefore:

```text
-1
```

cannot represent a real array position.

It clearly means:

```text
The item was not found.
```

Example:

```js
if (selectedIndex === -1) {
  console.log(
    "Product not found."
  );
}
```

---

# 17.6 Step-by-Step Search

State:

```js
const products = [
  {
    id: 101,
    name: "Laptop",
  },
  {
    id: 205,
    name: "Phone",
  },
  {
    id: 309,
    name: "Keyboard",
  },
];
```

Selected ID:

```js
const selectedId = 205;
```

### First repetition

```js
index = 0;
```

Comparison:

```js
101 === 205;
```

Result:

```text
false
```

### Second repetition

```js
index = 1;
```

Comparison:

```js
205 === 205;
```

Result:

```text
true
```

Store:

```js
selectedIndex = 1;
```

Then:

```js
break;
```

---

# 17.7 Why Use `break`?

IDs should be unique.

After finding the matching item, no additional search is needed.

```js
break;
```

prevents unnecessary work.

It also prevents later duplicate IDs from replacing the first matching index, although duplicate IDs should never exist in valid state.

# 17.8 Reusable Index Search Function

```js
function findProductIndexById(
  productId
) {
  for (
    let index = 0;
    index < products.length;
    index++
  ) {
    if (
      products[index].id ===
      productId
    ) {
      return index;
    }
  }

  return -1;
}
```

Usage:

```js
const selectedIndex =
  findProductIndexById(205);
```

Result:

```text
1
```

Missing ID:

```js
const selectedIndex =
  findProductIndexById(999);
```

Result:

```text
-1
```

---

# 17.9 Why `return -1` Must Be After the Loop

Incorrect:

```js
function findProductIndexById(
  productId
) {
  for (
    let index = 0;
    index < products.length;
    index++
  ) {
    if (
      products[index].id ===
      productId
    ) {
      return index;
    }

    return -1;
  }
}
```

This checks only the first product.

If the first product does not match, the function immediately returns `-1`.

Correct:

```js
function findProductIndexById(
  productId
) {
  for (
    let index = 0;
    index < products.length;
    index++
  ) {
    if (
      products[index].id ===
      productId
    ) {
      return index;
    }
  }

  return -1;
}
```

# 17.10 Finding an Object Versus Finding an Index

## Find the object

```js
function findProductById(
  productId
) {
  for (
    let index = 0;
    index < products.length;
    index++
  ) {
    if (
      products[index].id ===
      productId
    ) {
      return products[index];
    }
  }

  return null;
}
```

Result:

```js
{
  id: 205,
  name: "Phone",
}
```

## Find the index

```js
function findProductIndexById(
  productId
) {
  for (
    let index = 0;
    index < products.length;
    index++
  ) {
    if (
      products[index].id ===
      productId
    ) {
      return index;
    }
  }

  return -1;
}
```

Result:

```text
1
```

---

# 17.11 When to Find the Object

Find the object when you need to:

* Load values into a form.
* Read properties.
* Display details.
* Update properties directly.
* Check status.

Example:

```js
const product =
  findProductById(205);

console.log(product.name);
```

---

# 17.12 When to Find the Index

Find the index when you need to:

* Delete with `splice()`.
* Replace an array entry.
* Move an item.
* Compare neighboring items.
* Insert before or after an item.

Example:

```js
const index =
  findProductIndexById(205);
```

Later:

```js
products.splice(index, 1);
```

---

# 17.13 Why Not Use the ID as the Index?

Incorrect:

```js
products.splice(
  selectedId,
  1
);
```

If:

```js
selectedId === 205;
```

JavaScript tries to remove an item at index `205`.

The array may contain only three items.

Nothing useful is removed.

Correct:

```js
const selectedIndex =
  findProductIndexById(
    selectedId
  );

products.splice(
  selectedIndex,
  1
);
```

---

# 17.14 Why Not Use `products[selectedId]`?

Incorrect:

```js
const product =
  products[selectedId];
```

This treats the ID as an array position.

IDs and indexes are unrelated.

For ID `205`, this attempts to access:

```js
products[205];
```

Result:

```text
undefined
```

---

# 17.15 Indexes Can Change

Initial state:

```js
[
  {
    id: 101,
    name: "Laptop",
  },
  {
    id: 205,
    name: "Phone",
  },
  {
    id: 309,
    name: "Keyboard",
  },
]
```

Phone index:

```text
1
```

After Laptop is removed:

```js
[
  {
    id: 205,
    name: "Phone",
  },
  {
    id: 309,
    name: "Keyboard",
  },
]
```

Phone index:

```text
0
```

Phone ID remains:

```text
205
```

This is why you should find the current index immediately before deletion.

---

# 17.16 Do Not Store a Long-Lived Delete Index

Risky:

```js
let productIndexToDelete = 1;
```

If sorting or deletion happens before confirmation, index `1` may point to a different product.

Safer:

```js
let productIdToDelete = 205;
```

Then, immediately before deletion:

```js
const index =
  findProductIndexById(
    productIdToDelete
  );
```

Stable IDs should be stored longer than indexes.

Rendered button:

```html

>
  Delete
</button>
```

Read:

```js
const selectedId =
  Number(

  );
```

Remember:

```js
deleteButton.dataset.id
```

returns a string.

# 17.18 Validating a Selected ID

```js
if (
  !Number.isInteger(selectedId) ||
  selectedId <= 0
) {
  showError(
    "Invalid product ID."
  );

  return;
}
```

This rejects:

```text
NaN
0
-5
2.5
```

---

# 17.19 Preparing for a Missing Item

```js
const selectedIndex =
  findProductIndexById(
    selectedId
  );

if (selectedIndex === -1) {
  showError(
    "The selected product no longer exists."
  );

  return;
}
```

Do not call:

```js
splice(-1, 1);
```

without checking.

That would remove the final array item.

---

# 17.20 Why `splice(-1, 1)` Is Dangerous

In JavaScript, a negative index counts from the end.

```js
products.splice(-1, 1);
```

removes the last product.

Therefore, if a missing search returns `-1` and you forget to check it, the wrong item is deleted.

> **Important warning:** Always verify that the index is not `-1` before passing it to `splice()`.

# 17.21 Event Delegation for Delete Buttons

```js
productList.addEventListener(
  "click",
  function (event) {
    const deleteButton =
      event.target.closest(
        '[data-action="delete"]'
      );

    if (deleteButton === null) {
      return;
    }

    const selectedId =
      Number(
        deleteButton.dataset.id
      );

    if (
      !Number.isInteger(
        selectedId
      ) ||
      selectedId <= 0
    ) {
      showError(
        "Invalid product ID."
      );

      return;
    }

    const selectedIndex =
      findProductIndexById(
        selectedId
      );

    if (
      selectedIndex === -1
    ) {
      showError(
        "Product not found."
      );

      return;
    }

    console.log(
      "Product index:",
      selectedIndex
    );
  }
);
```

The item is not deleted yet.

# 17.22 Finding Both Object and Index

Sometimes you need both.

```js
function findProductEntryById(
  productId
) {
  for (
    let index = 0;
    index < products.length;
    index++
  ) {
    if (
      products[index].id ===
      productId
    ) {
      return {
        product:
          products[index],

        index,
      };
    }
  }

  return null;
}
```

Usage:

```js
const result =
  findProductEntryById(205);
```

Result:

```js
{
  product: {
    id: 205,
    name: "Phone",
  },
  index: 1,
}
```

---

# 17.23 Why Both Values May Be Useful

Before deletion, the application may need the product name for confirmation.

Example:

```text
Delete Phone?
```

The object gives:

```js
result.product.name;
```

The index gives:

```js
result.index;
```

Later:

```js
products.splice(
  result.index,
  1
);
```

---

# 17.24 Safer Confirmation Preparation

```js
function prepareProductDeletion(
  productId
) {
  const result =
    findProductEntryById(
      productId
    );

  if (result === null) {
    showError(
      "The selected product no longer exists."
    );

    return null;
  }

  return result;
}
```

The next lesson will use this information with confirmation and `splice()`.

# 17.25 Three Complete Examples

## Example 1: Find an Index Without the DOM

### Problem

Find the current index of product ID `205`.

### Input Data

```js
const products = [
  {
    id: 101,
    name: "Laptop",
  },
  {
    id: 205,
    name: "Phone",
  },
  {
    id: 309,
    name: "Keyboard",
  },
];
```

### Complete Code

```js
const products = [
  {
    id: 101,
    name: "Laptop",
  },
  {
    id: 205,
    name: "Phone",
  },
  {
    id: 309,
    name: "Keyboard",
  },
];

const selectedId = 205;

let selectedIndex = -1;

for (
  let index = 0;
  index < products.length;
  index++
) {
  if (
    products[index].id ===
    selectedId
  ) {

  }
}

console.log(selectedIndex);
```

### Step-by-Step Explanation

1. Store the requested ID.
2. Set the result to `-1`.
3. Loop from index `0`.
4. Compare the current product ID.
5. Store the matching index.
6. Stop the loop.

### Expected Console Output

```text
1
```

### Final State

The array remains unchanged.

Finding an index is a Read operation.

### Common Mistakes

* Initializing with `0`.
* Using the ID directly as an index.
* Forgetting `break`.
* Comparing strings and numbers.

Confirm the output is:

```text
-1
```

### Problem

Create a function that returns an index or `-1`.

### Complete Code

```js
const products = [
  {

  findProductIndexById(101)

console.log(

```

```text

```

### Problem

# 17.26 Plain Loop Versus `findIndex()` Preview

## Plain Loop

```js
let selectedIndex = -1;

for (
  let index = 0;
  index < products.length;
  index++
) {
  if (
    products[index].id ===
    selectedId
  ) {
    selectedIndex = index;
    break;
  }
}
```

## Array Method

```js
const selectedIndex =
  products.findIndex(
    (product) =>
      product.id === selectedId
  );
```

Both versions return:

* A valid index when found.
* `-1` when missing.

The plain loop makes the internal search visible.

`findIndex()` will be taught later.

---

# 17.27 Why a Plain Loop Is Useful

A standard loop gives complete control.

You can:

* Count comparisons.
* Search backwards.
* Stop with `break`.
* Add custom conditions.
* Log every checked item.
* Search only part of an array.
* Return both the object and index.

# 17.28 Common Beginner Mistakes

## Mistake 1: Initializing with `0`

```js
let selectedIndex = 0;
```

If no match exists, the first product may be mistaken for the target.

Use:

```js
let selectedIndex = -1;
```

---

## Mistake 2: Checking with `if (!selectedIndex)`

Index `0` is valid but falsy.

Incorrect:

```js
if (!selectedIndex) {
  console.log(
    "Not found"
  );
}
```

Correct:

```js
if (selectedIndex === -1) {
  console.log(
    "Not found"
  );
}
```

---

## Mistake 3: Passing ID to `splice()`

```js
products.splice(
  productId,
  1
);
```

The ID is not the array position.

---

## Mistake 4: Using `splice(-1, 1)`

This removes the last item.

Always check the result first.

---

## Mistake 5: Returning `-1` Inside the Loop

The function checks only the first item.

---

## Mistake 6: Storing an Index for Too Long

Indexes may change after sorting or deletion.

Store a stable ID and calculate the index near the deletion operation.

---

## Mistake 7: String and Number Mismatch

```js
205 === "205";
```

returns:

```text
false
```

Convert dataset values.

---

## Mistake 8: Starting the Loop at `1`

The first product is skipped.

---

## Mistake 9: Using `<=`

The loop accesses one invalid position.

---

## Mistake 10: Finding the DOM Position Instead of the State Position

Filtered DOM order may not match the original state array.

Search the state by stable ID.

## Key Takeaways

* Delete commonly requires the target item's current array index.
* IDs identify objects; indexes identify current positions.
* IDs should remain stable.
* Indexes may change after sorting, filtering, insertion, or deletion.
* A plain loop can locate the current index by comparing stable IDs.
* Use `-1` to represent a missing index.
* Never treat index `0` as missing.
* Never pass a missing `-1` result directly to `splice()`.
* `splice(-1, 1)` removes the last item.
* Store stable IDs for pending actions and calculate indexes near the operation.
* The next lesson will remove the item safely using `splice()`.
