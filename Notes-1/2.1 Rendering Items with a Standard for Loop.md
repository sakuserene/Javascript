# 5. Reading Items with a Standard `for` Loop

## 5.1 What You Will Learn

This lesson focuses on the **Read** part of CRUD.

Read means:

* Accessing existing items.
* Displaying items.
* Inspecting item properties.
* Turning JavaScript data into visible DOM elements.

You will use a standard `for` loop before learning modern methods such as `forEach()`.

### Why This Matters

A CRUD application may contain many items.

```js
const products = [
  {
    id: 1,
    name: "Laptop",
  },
  {
    id: 2,
    name: "Phone",
  },
  {
    id: 3,
    name: "Keyboard",
  },
];
```

Instead of writing separate code for every product, a loop repeats the same rendering steps.

Without a loop:

```js
console.log(products[0].name);
console.log(products[1].name);
console.log(products[2].name);
```

With a loop:

```js
for (
  let index = 0;
  index < products.length;
  index++
) {
  console.log(products[index].name);
}
```

The loop works even when the number of products changes.

---

# 5.2 Standard `for` Loop Syntax

A standard `for` loop has three main parts.

```js
for (
  let index = 0;
  index < products.length;
  index++
) {
  console.log(products[index]);
}
```

The general structure is:

```js
for (initialization; condition; update) {
  // Repeated code
}
```

---

## Initialization

```js
let index = 0;
```

This creates the loop counter.

Arrays begin at index `0`, so the loop usually begins at `0`.

For this array:

```js
const products = [
  "Laptop",
  "Phone",
  "Keyboard",
];
```

The indexes are:

```text
Laptop   → index 0
Phone    → index 1
Keyboard → index 2
```

---

## Condition

```js
index < products.length;
```

The loop continues while this condition is true.

If:

```js
products.length === 3
```

then valid indexes are:

```text
0
1
2
```

The loop stops when:

```js
index === 3
```

because:

```js
3 < 3
```

is false.

---

## Update

```js
index++;
```

This increases the index by `1` after every repetition.

The values become:

```text
0
1
2
```

---

## Loop Body

```js
{
  console.log(products[index]);
}
```

This code runs once for every valid index.

---

# 5.3 Step-by-Step Loop Execution

Consider:

```js
const products = [
  "Laptop",
  "Phone",
  "Keyboard",
];

for (
  let index = 0;
  index < products.length;
  index++
) {
  console.log(products[index]);
}
```

### First Repetition

```js
index = 0;
```

Condition:

```js
0 < 3;
```

Result:

```text
true
```

Accessed value:

```js
products[0];
```

Output:

```text
Laptop
```

---

### Second Repetition

```js
index = 1;
```

Condition:

```js
1 < 3;
```

Result:

```text
true
```

Output:

```text
Phone
```

---

### Third Repetition

```js
index = 2;
```

Condition:

```js
2 < 3;
```

Result:

```text
true
```

Output:

```text
Keyboard
```

---

### Fourth Check

```js
index = 3;
```

Condition:

```js
3 < 3;
```

Result:

```text
false
```

The loop ends.

# 5.4 Accessing Array Items

Inside the loop:

```js
products[index]
```

represents the current item.

Example:

```js
const product = products[index];
```

This creates a helpful local variable.

Instead of repeatedly writing:

```js
products[index].name;
products[index].price;
products[index].quantity;
```

you can write:

```js
const product = products[index];

product.name;
product.price;
product.quantity;
```

This improves readability.

# 5.6 Rendering into the DOM

HTML:

```html
<ul id="product-list"></ul>
```

JavaScript:

```js
const productList =
  document.getElementById("product-list");

for (
  let index = 0;
  index < products.length;
  index++
) {
  const product = products[index];

  const listItem =
    document.createElement("li");

  listItem.textContent = product.name;

  productList.appendChild(listItem);
}
```

Expected DOM:

```text
Laptop
Phone
Keyboard
```

---

# 5.7 Why Use `document.createElement()`?

This code creates a real DOM element:

```js
const listItem =
  document.createElement("li");
```

The browser now has an `<li>` object in memory.

It is not visible yet.

Set its text:

```js
listItem.textContent = product.name;
```

Append it:

```js
productList.appendChild(listItem);
```

Now it becomes visible.

---

# 5.8 Why Use `textContent`?

Use:

```js
listItem.textContent = product.name;
```

This inserts the product name as plain text.

Suppose a user entered:

```html
<strong>Laptop</strong>
```

With `textContent`, the page displays:

```text
<strong>Laptop</strong>
```

The browser does not treat the value as HTML.

> **Security warning:** Do not insert untrusted product names using `innerHTML`.

Unsafe:

```js
listItem.innerHTML = product.name;
```

Safer:

```js
listItem.textContent = product.name;
```

---

# 5.9 Adding CSS Classes

Create an element:

```js
const listItem =
  document.createElement("li");
```

Add a class:

```js
listItem.classList.add("product-item");
```

Generated HTML:

```html
<li class="product-item"></li>
```

Add multiple classes:

```js
listItem.classList.add(
  "product-item",
  "in-stock"
);
```

Generated HTML:

```html
<li class="product-item in-stock"></li>
```

---

# 5.10 Storing IDs with `dataset`

Inside the render loop:

```js
listItem.dataset.id = product.id;
```

Generated HTML:

```html
<li data-id="1">Laptop</li>
```

This will later help determine which item the user clicked.

Remember:

```js
listItem.dataset.id
```

returns a string.

If the value is:

```text
"1"
```

convert it before comparing with numeric state IDs:

```js
const selectedId =
  Number(listItem.dataset.id);
```

---

# 5.11 Clearing the Container Before Rendering

A reusable render function should clear old elements before creating new ones.

```js
function renderProducts() {
  productList.innerHTML = "";

  for (
    let index = 0;
    index < products.length;
    index++
  ) {
    // Create and append elements.
  }
}
```

Without clearing:

```js
renderProducts();
renderProducts();
```

may display duplicates.

With clearing:

```js
renderProducts();
renderProducts();
```

still displays only one copy of every product.

---

# 5.12 Empty-State Rendering

State:

```js
let products = [];
```

Render function:

```js
function renderProducts() {
  productList.innerHTML = "";

  if (products.length === 0) {
    const emptyMessage =
      document.createElement("li");

    emptyMessage.textContent =
      "No products available.";

    productList.appendChild(emptyMessage);

    return;
  }

  for (
    let index = 0;
    index < products.length;
    index++
  ) {
    const product = products[index];

    const listItem =
      document.createElement("li");

    listItem.textContent = product.name;

    productList.appendChild(listItem);
  }
}
```

Expected DOM:

```text
No products available.
```

# 5.13 Rendering Item Numbers

Suppose the first product has ID `15`.

```js
const products = [
  {
    id: 15,
    name: "Laptop",
  },
  {
    id: 28,
    name: "Phone",
  },
];
```

You may want visible numbering to begin at `1`.

Use:

```js
index + 1
```

Example:

```js
listItem.textContent =
  `${index + 1}. ${product.name}`;
```

Expected DOM:

```text
1. Laptop
2. Phone
```

The visible number is not the same as the product ID.

You can still store the real ID:

```js
listItem.dataset.id = product.id;
```

Generated HTML:

```html
<li data-id="15">1. Laptop</li>
<li data-id="28">2. Phone</li>
```

---

# 5.14 Display Number Versus Item ID

These values have different responsibilities.

| Value        | Purpose                   |
| ------------ | ------------------------- |
| `index`      | Current array position    |
| `index + 1`  | User-friendly list number |
| `product.id` | Stable item identifier    |

Example:

```js
const products = [
  {
    id: 25,
    name: "Laptop",
  },
];
```

Values:

```text
Index: 0
Display number: 1
Product ID: 25
```

Do not confuse them.

---

# 5.15 Rendering Multiple Child Elements

A product card may contain:

* A product name.
* A price.
* A quantity.
* An Edit button.
* A Delete button.

Instead of placing everything directly into one text string, create separate elements.

```js
const productCard =
  document.createElement("article");

const nameElement =
  document.createElement("h2");

const priceElement =
  document.createElement("p");

const quantityElement =
  document.createElement("p");

nameElement.textContent = product.name;

priceElement.textContent =
  `Price: KSh ${product.price}`;

quantityElement.textContent =
  `Quantity: ${product.quantity}`;

productCard.appendChild(nameElement);
productCard.appendChild(priceElement);
productCard.appendChild(quantityElement);
```

This gives more control over layout and styling.

---

# 5.16 Using `append()`

Instead of:

```js
productCard.appendChild(nameElement);
productCard.appendChild(priceElement);
productCard.appendChild(quantityElement);
```

you can write:

```js
productCard.append(
  nameElement,
  priceElement,
  quantityElement
);
```

`append()` can add several nodes at once.

Both are valid.

This course will use whichever form makes the example clearest.

---

# 5.17 Creating Action Buttons

Inside a render loop:

```js
const editButton =
  document.createElement("button");

editButton.type = "button";
editButton.textContent = "Edit";
editButton.dataset.id = product.id;

const deleteButton =
  document.createElement("button");

deleteButton.type = "button";
deleteButton.textContent = "Delete";
deleteButton.dataset.id = product.id;
```

Generated HTML:

```html
<button type="button" data-id="1">
  Edit
</button>

<button type="button" data-id="1">
  Delete
</button>
```

The buttons are not functional yet.

Later lessons will connect click events to Update and Delete operations.

---

# 5.18 Avoiding Event Listeners Inside Early Render Examples

You may be tempted to write:

```js
deleteButton.addEventListener(
  "click",
  function () {
    // Delete product.
  }
);
```

inside the render loop.

This can work, but repeated rendering creates new buttons and new listeners.

For large dynamic lists, event delegation is often cleaner.

For now, you may render the buttons without implementing behavior.

A later subtopic will explain event delegation in detail.

---

# 5.19 Formatting Prices

A simple display:

```js
priceElement.textContent =
  `KSh ${product.price}`;
```

Output:

```text
KSh 50000
```

Formatted using `toLocaleString()`:

```js
priceElement.textContent =
  `KSh ${product.price.toLocaleString()}`;
```

Output:

```text
KSh 50,000
```

This does not change the numeric state value.

State remains:

```js
price: 50000
```

The formatted version is only used for display.

---

# 5.20 Rendering Conditional Text

State:

```js
{
  id: 1,
  name: "Laptop",
  quantity: 0
}
```

You can display different text depending on quantity.

```js
if (product.quantity === 0) {
  quantityElement.textContent =
    "Out of stock";
} else {
  quantityElement.textContent =
    `Quantity: ${product.quantity}`;
}
```

Expected output:

```text
Out of stock
```

The state is not changed.

The DOM output is derived from state.

# 5.21 Three Complete Examples

## Example 1: Basic Array Rendering Without the DOM

### Problem

Use a standard `for` loop to read and display every item name in the console.

### Input Data

```js
const itemNames = [
  "Laptop",
  "Phone",
  "Keyboard",
];
```

### Complete Code

```js
const itemNames = [
  "Laptop",
  "Phone",
  "Keyboard",
];

for (
  let index = 0;
  index < itemNames.length;
  index++
) {
  console.log(itemNames[index]);
}
```

### Step-by-Step Explanation

#### Initialization

```js
let index = 0;
```

The loop begins at the first array index.

#### Condition

```js
index < itemNames.length;
```

The loop runs while the index is valid.

#### Current item

```js
itemNames[index]
```

This accesses the item at the current position.

#### Update

```js
index++;
```

The loop moves to the next index.

### Expected Console Output

```text
Laptop
Phone
Keyboard
```

### Final State

The array is unchanged:

```js
[
  "Laptop",
  "Phone",
  "Keyboard",
]
```

The loop only reads the items.

### Common Mistakes

#### Starting at `1`

Incorrect:

```js
let index = 1;
```

This skips `"Laptop"`.

#### Using `<=`

Incorrect:

```js
index <= itemNames.length;
```

The final repetition accesses an invalid index.

Expected extra value:

```text
undefined
```

#### Forgetting `index++`

The loop may continue forever because the condition never changes.

### Small Modification Challenge

Change the output to:

```text
1. Laptop
2. Phone
3. Keyboard
```

Use `index + 1`.

## Example 2: Rendering Product Objects into a List

### Problem

Display products containing names, prices, and quantities in the browser.

### Input Data

```js
const products = [
  {
    id: 1,
    name: "Laptop",
    price: 50000,
    quantity: 2,
  },
  {
    id: 2,
    name: "Phone",
    price: 25000,
    quantity: 4,
  },
  {
    id: 3,
    name: "Keyboard",
    price: 3000,
    quantity: 5,
  },
];
```

### Complete Code

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8" />

  <meta
    name="viewport"
    content="width=device-width, initial-scale=1.0"
  />

  <title>Product List</title>

  <style>
    body {
      max-width: 700px;
      margin: 0 auto;
      padding: 24px;
      font-family: Arial, sans-serif;
    }

    .product-item {
      margin-bottom: 10px;
      padding: 12px;
      border: 1px solid #cccccc;
      border-radius: 6px;
    }
  </style>
</head>

<body>
  <h1>Products</h1>

  <ul id="product-list"></ul>

  <script>
    const products = [
      {
        id: 1,
        name: "Laptop",
        price: 50000,
        quantity: 2,
      },
      {
        id: 2,
        name: "Phone",
        price: 25000,
        quantity: 4,
      },
      {
        id: 3,
        name: "Keyboard",
        price: 3000,
        quantity: 5,
      },
    ];

    const productList =
      document.getElementById("product-list");

    function renderProducts() {
      productList.innerHTML = "";

      if (products.length === 0) {
        const emptyMessage =
          document.createElement("li");

        emptyMessage.textContent =
          "No products available.";

        productList.appendChild(emptyMessage);

        return;
      }

      for (
        let index = 0;
        index < products.length;
        index++
      ) {
        const product = products[index];

        const listItem =
          document.createElement("li");

        listItem.classList.add(
          "product-item"
        );

        listItem.dataset.id = product.id;

        listItem.textContent =
          `${index + 1}. ${product.name} — ` +
          `KSh ${product.price.toLocaleString()} — ` +
          `Quantity: ${product.quantity}`;

        productList.appendChild(listItem);
      }
    }

    renderProducts();
  </script>
</body>
</html>
```

### Step-by-Step Explanation

#### Step 1: Select the list

```js
const productList =
  document.getElementById("product-list");
```

#### Step 2: Clear old content

```js
productList.innerHTML = "";
```

#### Step 3: Check for an empty array

```js
if (products.length === 0) {
  // Display an empty message.
}
```

#### Step 4: Loop through products

```js
for (
  let index = 0;
  index < products.length;
  index++
) {
  const product = products[index];
}
```

#### Step 5: Create the list item

```js
const listItem =
  document.createElement("li");
```

#### Step 6: Add styling and ID data

```js
listItem.classList.add("product-item");
listItem.dataset.id = product.id;
```

#### Step 7: Set safe display text

```js
listItem.textContent =
  `${index + 1}. ${product.name} — ` +
  `KSh ${product.price.toLocaleString()} — ` +
  `Quantity: ${product.quantity}`;
```

#### Step 8: Append the item

```js
productList.appendChild(listItem);
```

### Expected DOM Output

```text
Products

1. Laptop — KSh 50,000 — Quantity: 2
2. Phone — KSh 25,000 — Quantity: 4
3. Keyboard — KSh 3,000 — Quantity: 5
```

### Final State

The state remains unchanged:

```js
[
  {
    id: 1,
    name: "Laptop",
    price: 50000,
    quantity: 2,
  },
  {
    id: 2,
    name: "Phone",

    quantity: 4,

    id: 3,

```

### Common Mistakes

#### Clearing the list inside the loop

This causes earlier items to disappear.

#### Using the index as the ID

Incorrect:

```js

```

Use:

```js
listItem.dataset.id = product.id;
```

#### Forgetting to call `renderProducts()`

```text

```

instead of:

```text

```

for products whose quantity is zero.

# 5.22 Standard Loop Versus Manual Rendering

## Manual Rendering

```js
const firstItem =
  document.createElement("li");

firstItem.textContent =
  products[0].name;

productList.appendChild(firstItem);

const secondItem =
  document.createElement("li");

secondItem.textContent =
  products[1].name;

productList.appendChild(secondItem);
```

Problems:

* Repeated code.
* Only supports a fixed number of products.
* Difficult to maintain.
* Easy to forget an item.
* Does not scale.

---

## Standard `for` Loop

```js
for (
  let index = 0;
  index < products.length;
  index++
) {
  const product = products[index];

  const listItem =
    document.createElement("li");

  listItem.textContent = product.name;

  productList.appendChild(listItem);
}
```

Benefits:

* Works with any array length.
* Avoids repeated code.
* Gives direct control over the index.
* Allows `break` and `continue`.
* Makes the looping process visible to beginners.

---

# 5.23 When a Standard `for` Loop Is Useful

A standard `for` loop is especially useful when:

* You need the current index.
* You need to stop early using `break`.
* You need to skip an item using `continue`.
* You need to move forwards or backwards.
* You need a custom step size.
* You need full control over loop execution.
* You are learning how array iteration works internally.

Example with `break`:

```js
for (
  let index = 0;
  index < products.length;
  index++
) {
  if (products[index].id === selectedId) {
    console.log(products[index]);
    break;
  }
}
```

# 5.24 Common Beginner Mistakes

## Mistake 1: Starting at Index `1`

Incorrect:

```js
for (
  let index = 1;
  index < products.length;
  index++
)
```

The first item at index `0` is skipped.

---

## Mistake 2: Using `<=`

Incorrect:

```js
index <= products.length;
```

The last repetition accesses:

```js
products[products.length]
```

This is `undefined`.

Correct:

```js
index < products.length;
```

---

## Mistake 3: Forgetting the Increment

Incorrect:

```js
for (
  let index = 0;
  index < products.length;
)
```

The index remains `0`.

This can create an infinite loop.

---

## Mistake 4: Incrementing in the Wrong Direction

Incorrect:

```js
index--;
```

when the loop begins at `0` and expects to move forward.

---

## Mistake 5: Clearing Inside the Loop

Incorrect:

```js
for (...) {
  productList.innerHTML = "";
}
```

Only the last item may remain.

---

## Mistake 6: Appending Outside the Loop

Incorrect:

```js
for (...) {
  const listItem =
    document.createElement("li");

  listItem.textContent =
    products[index].name;
}

productList.appendChild(listItem);
```

`listItem` may be out of scope.

Even if scope were changed, only one item would be appended.

---

## Mistake 7: Using `innerHTML` for User Data

Unsafe:

```js
listItem.innerHTML = product.name;
```

Safer:

```js
listItem.textContent = product.name;
```

---

## Mistake 8: Confusing ID and Index

Incorrect:

```js
listItem.dataset.id = index;
```

Correct:

```js
listItem.dataset.id = product.id;
```

---

## Mistake 9: Forgetting the Initial Render

A correct render function does nothing until it is called.

```js
renderProducts();
```

---

## Mistake 10: Mutating State During Rendering

Incorrect:

```js
products[index].quantity += 1;
```

inside the render function.

Rendering should read state, not change it.

# 5.31 Refactoring Preparation

Later, this standard loop:

```js
for (
  let index = 0;
  index < products.length;
  index++
) {
  const product = products[index];

  console.log(product.name);
}
```

will be rewritten using:

```js
products.forEach((product) => {
  console.log(product.name);
});
```

Both versions loop internally.

The standard loop makes each step explicit:

* Starting index.
* Condition.
* Increment.
* Current item access.

`forEach()` hides those mechanics behind a method.

You should understand the standard loop first so array methods do not feel like magic.

## Key Takeaways

* Reading is the CRUD operation used to access and display existing data.
* A standard `for` loop provides direct control over array iteration.
* Array indexes begin at `0`.
* The common condition is `index < array.length`.
* `array[index]` accesses the current item.
* A local variable such as `const product = products[index]` improves readability.
* `document.createElement()` creates DOM elements.
* `textContent` safely displays user-controlled text.
* `appendChild()` attaches elements to the DOM.
* `classList.add()` applies CSS classes.
* `dataset` stores item IDs and action information.
* A render function should clear the container before rebuilding it.
* Empty arrays should display a helpful message.
* `index + 1` creates user-friendly numbering.
* Indexes, display numbers, and stable IDs are different.
* Rendering should read state without modifying it.
* Standard loops should be understood before modern array methods.
