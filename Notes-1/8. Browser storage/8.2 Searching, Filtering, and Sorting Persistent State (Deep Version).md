# 1. The Core Principle: Derived State

## What You Will Learn

Why we NEVER modify the main state when searching or filtering.

---

## Concept Explanation

You have:

```js
let products = [...]; // main state
```

If you do:

```js
products = products.filter(p => p.price > 1000);
```

You have **destroyed data permanently** ❌

---

## Correct Approach

```js
const visibleProducts = products.filter(...);
```

---

## Mental Model

```text
Main State (products) → NEVER mutate for UI
Derived State (visibleProducts) → safe for display
```

---

## Real Flow

```text
User types search
→ compute derived list
→ render derived list
→ original state remains intact
```

# 2. Searching Items

## What You Will Learn

* Case-insensitive search
* Partial matching
* Safe string comparison

---

## Concept Explanation

User input is unpredictable:

```text
Laptop
laptop
LAPTOP
lap
```

We normalize:

```js
text.toLowerCase()
```

---

## Syntax Breakdown

```js
product.name.toLowerCase().includes(searchTerm)
```

| Part          | Meaning       |
| ------------- | ------------- |
| toLowerCase() | normalize     |
| includes()    | partial match |
| searchTerm    | user input    |

---

## Example 1 (Basic)

```js
const items = ["Laptop", "Phone", "Keyboard"];

const result = items.filter(item =>
  item.toLowerCase().includes("lap")
);

console.log(result);
```

### Output

```text
["Laptop"]
```

---

## Example 2 (Objects)

```js
const products = [
  { name: "Laptop" },
  { name: "Phone" },
];

const result = products.filter(product =>
  product.name.toLowerCase().includes("pho")
);

console.log(result);
```

---

## Example 3 (DOM + State)

```js
function getVisibleProducts() {
  const term = searchInput.value.toLowerCase();

  return products.filter(product =>
    product.name.toLowerCase().includes(term)
  );
}
```

---

## Final State

```js
products // unchanged
visibleProducts // filtered
```

---

## Common Mistakes

❌ Using `===` instead of `includes()`
❌ Not normalizing case
❌ Mutating original array

# 3. Filtering Items

## Concept Explanation

Filtering applies conditions:

```text
Show only:
- In-stock items
- Out-of-stock items
```

---

## Syntax

```js
products.filter(product => product.quantity === 0)
```

---

## Example 1 (Basic)

```js
const numbers = [1,2,3,4];

const result = numbers.filter(n => n > 2);

console.log(result);
```

---

## Example 2 (Objects)

```js
const products = [
  { name: "Laptop", quantity: 2 },
  { name: "Mouse", quantity: 0 }
];

const result = products.filter(p => p.quantity === 0);
```

---

## Example 3 (DOM + Logic)

```js
function applyFilter(product) {
  if (filter === "in-stock") return product.quantity > 0;
  if (filter === "out-of-stock") return product.quantity === 0;
  return true;
}
```

---

## Flow

```text
products → filter → derived list → render
```

# 4. Sorting Items

## What You Will Learn

* How sorting works internally
* Why sorting is dangerous

---

## Critical Warning

```js
products.sort(...)
```

⚠️ MUTATES original array ❌

---

## Safe Approach

```js
[...products].sort(...)
```

---

## Syntax Breakdown

```js
(a, b) => a.price - b.price
```

| Case     | Result     |
| -------- | ---------- |
| negative | a before b |
| positive | b before a |
| 0        | unchanged  |

---

## Example 1 (Numbers)

```js
[3,1,2].sort((a,b) => a - b);
```

---

## Example 2 (Objects)

```js
products.sort((a,b) => a.price - b.price);
```

---

## Example 3 (Safe DOM)

```js
function sortProducts(list) {
  return [...list].sort((a,b) => a.price - b.price);
}
```

---

## Common Mistakes

❌ Sorting original array
❌ Wrong comparator
❌ Forgetting spread operator

---

# 5. Combining Search + Filter + Sort

## Concept Explanation

We build a pipeline:

```text
products
→ search
→ filter
→ sort
→ render
```

---

## Implementation

```js
function getVisibleProducts() {
  const term = searchInput.value.toLowerCase();

  let result = products.filter(product =>
    product.name.toLowerCase().includes(term)
  );

  result = result.filter(product => {
    if (filter === "in-stock") return product.quantity > 0;
    if (filter === "out-of-stock") return product.quantity === 0;
    return true;
  });

  result = [...result].sort((a,b) => a.price - b.price);

  return result;
}
```

---

## Full Flow

```text
User input
→ derive list
→ render
→ UI updates
```

# 6. Persisting UI State

## What You Will Learn

Keep search/filter/sort after refresh.

---

## Save UI State

```js
localStorage.setItem("uiState", JSON.stringify({
  search: searchInput.value,
  filter,
  sort
}));
```

---

## Load UI State

```js
function loadUIState() {
  const stored = localStorage.getItem("uiState");

  if (!stored) return;

  const state = JSON.parse(stored);

  searchInput.value = state.search;
  filterSelect.value = state.filter;
  sortSelect.value = state.sort;
}
```

---

## Why This Matters

```text
User refreshes → keeps search/filter → better UX
```

# 7. Rendering with Derived State

```js
function renderProducts() {
  const visibleProducts = getVisibleProducts();

  list.innerHTML = "";

  visibleProducts.forEach(product => {
    const li = document.createElement("li");
    li.textContent = product.name;

    list.appendChild(li);
  });
}
```
