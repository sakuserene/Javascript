# 1. Core Principle: Derived State

## Concept Explanation

We must never modify the main state when searching or filtering.

Incorrect:

```js
products = products.filter(...)
```

This permanently removes items.

---

## Correct Approach

Use a **derived array**:

```js
const visibleProducts = products.filter(...)
```

---

## Flow

```text
Main State (products)
        ↓
Derived State (visibleProducts)
        ↓
Render UI
```

# 2. Searching Items

## What You Will Learn

* How to search text input
* How to match strings safely
* Why normalization is important

---

## Concept Explanation

Users may type:

```text
phone
Phone
PHONE
```

We normalize:

```js
text.toLowerCase()
```

---

## Syntax Breakdown

```js
product.name.toLowerCase().includes(searchTerm)
```

* `toLowerCase()` → normalize text
* `includes()` → partial match

---

## Example 1: Basic

```js
const items = ["Laptop", "Phone"];

const result = items.filter(item =>
  item.toLowerCase().includes("phone")
);

console.log(result);
```

### Output

```text
["Phone"]
```

---

## Example 2: Objects

```js
const products = [
  { name: "Laptop" },
  { name: "Phone" }
];

const result = products.filter(product =>
  product.name.toLowerCase().includes("lap")
);

console.log(result);
```

---

## Example 3: DOM Integration

```js
function getVisibleProducts() {
  const term = searchInput.value.toLowerCase();

  return products.filter(product =>
    product.name.toLowerCase().includes(term)
  );
}
```

---

## Common Mistakes

* Not normalizing strings
* Using `===` instead of `includes`
* Modifying main state

# 3. Filtering Items

## Concept Explanation

Filtering selects items based on conditions.

Example:

```text
Show only out-of-stock items
```

---

## Syntax Breakdown

```js
products.filter(product => product.quantity === 0)
```

---

## Example 1: Basic

```js
const numbers = [1,2,3,4];

const result = numbers.filter(n => n > 2);

console.log(result);
```

---

## Example 2: Objects

```js
const products = [
  { name: "Laptop", quantity: 2 },
  { name: "Keyboard", quantity: 0 }
];

const result = products.filter(p => p.quantity === 0);
```

---

## Example 3: DOM Integration

```js
function applyFilter(product) {
  if (filter === "in-stock") return product.quantity > 0;
  if (filter === "out-of-stock") return product.quantity === 0;
  return true;
}
```

---

# 4. Sorting Items

## Concept Explanation

Sorting changes order.

⚠️ Important:

```js
products.sort(...)
```

mutates the original array ❌

---

## Safe Approach

```js
[...products].sort(...)
```

---

## Syntax Breakdown

```js
array.sort((a, b) => a.price - b.price)
```

---

## Example 1: Numbers

```js
[3,1,2].sort((a,b) => a - b);
```

---

## Example 2: Objects

```js
products.sort((a,b) => a.price - b.price);
```

---

## Example 3: Safe DOM Sorting

```js
function sortProducts(list) {
  return [...list].sort((a,b) => a.price - b.price);
}
```

---

## Common Mistakes

* Sorting original array
* Forgetting spread operator
* Incorrect comparator

# 5. Combining Search + Filter + Sort

## Concept Explanation

We combine steps:

```text
products
 → filter (search)
 → filter (status)
 → sort
 → render
```

---

## Implementation

```js
function getVisibleProducts() {
  const term = searchInput.value.toLowerCase();

  let result = products.filter(product =>
    product.name.toLowerCase().includes(term)
  );

  result = result.filter(product => {
    if (filter === "in-stock") return product.quantity > 0;
    if (filter === "out-of-stock") return product.quantity === 0;
    return true;
  });

  result = [...result].sort((a,b) => a.price - b.price);

  return result;
}
```

# 6. Persisting UI Controls

## Concept Explanation

We store:

* search term
* filter value
* sort option

---

## Save UI State

```js
localStorage.setItem("uiState", JSON.stringify({
  search: searchInput.value,
  filter: filterValue,
  sort: sortValue
}));
```

---

## Load UI State

```js
function loadUIState() {
  const stored = localStorage.getItem("uiState");

  if (!stored) return;

  const state = JSON.parse(stored);

  searchInput.value = state.search;
  filterSelect.value = state.filter;
  sortSelect.value = state.sort;
}
```

---

# 7. Rendering with Derived State

```js
function renderProducts() {
  const visibleProducts = getVisibleProducts();

  productList.innerHTML = "";

  visibleProducts.forEach(product => {
    renderProductCard(product);
  });
}
```

# 9. Common Mistakes

* Mutating state when filtering
* Sorting without copying array
* Forgetting to re-render
* Not saving UI state
* Not restoring UI state

---

# 10. Best Practices

* Always derive visible state
* Keep main state unchanged
* Normalize text comparisons
* Use pure functions
* Persist UI preferences
* Re-render after changes
