By the end of this lesson, you will:

* Build automatic save + load systems
* Handle corrupted or missing data
* Keep state, storage, and DOM in sync

---

# 1. The Core Problem

## What You Will Learn

Why your CRUD app loses data after refresh.

## Concept Explanation

Your app currently works like this:

```text
User adds item → stored in JS array → shown in DOM
```

But:

```text
Refresh page → JS restarts → array resets → DOM cleared ❌
```

### Why?

Because:

* JavaScript state lives in **memory (RAM)**
* Memory is wiped when page reloads

---

# 2. The Solution: localStorage

## Concept Explanation

`localStorage` allows you to:

```text
Store data inside the browser → survives refresh
```

### Key Rule

> localStorage ONLY stores strings

---

# 3. The Data Conversion Problem

## Problem

You want to store:

```js
let products = [
  { id: 1, name: "Laptop" }
];
```

But localStorage only accepts:

```text
"string"
```

---

## Solution

### Step 1: Convert Object → String

```js
JSON.stringify(products)
```

### Step 2: Convert String → Object

```js
JSON.parse(string)
```

---

# 4. Example 1 (Basic – No DOM)

## Problem

Store and retrieve simple data.

```js
const numbers = [1, 2, 3];

localStorage.setItem("numbers", JSON.stringify(numbers));

const stored = localStorage.getItem("numbers");
const parsed = JSON.parse(stored);

console.log(parsed);
```

### Output

```text
[1, 2, 3]
```

---

## Final State

```js
numbers = [1, 2, 3];
```

---

## Common Mistake

```js
localStorage.setItem("numbers", numbers); ❌
```

Stores:

```text
"1,2,3"
```

# 5. Example 2 (Intermediate – Objects)

## Problem

Store products.

```js
const products = [
  { id: 1, name: "Laptop", price: 50000 }
];

localStorage.setItem("products", JSON.stringify(products));

const loaded = JSON.parse(localStorage.getItem("products"));

console.log(loaded);
```

---

## Output

```text
[{ id: 1, name: "Laptop", price: 50000 }]
```

# 6. Example 3 (DOM + CRUD Integration)

## Problem

Persist products in a CRUD app.

---

## Step 1: State

```js
let products = [];
```

---

## Step 2: Save Function

```js
function saveProducts() {
  localStorage.setItem("products", JSON.stringify(products));
}
```

---

## Step 3: Load Function

```js
function loadProducts() {
  const stored = localStorage.getItem("products");

  if (!stored) return [];

  try {
    return JSON.parse(stored);
  } catch {
    return [];
  }
}
```

---

## Step 4: Initialize State

```js
let products = loadProducts();
```

---

## Step 5: Create Flow

```js
function createProduct(data) {
  const newProduct = {
    id: Date.now(),
    ...data
  };

  products = [...products, newProduct];

  saveProducts();
  renderProducts();
}
```

---

## Full Flow

```text
User submits form
→ Event handler
→ Update state (products array)
→ Save to localStorage
→ Re-render DOM
```

---

# 7. Important Concepts

## 1. State is STILL the source of truth

```text
localStorage ≠ state
```

Correct flow:

```text
localStorage → load → state → render DOM
```

---

## 2. Always Save AFTER State Change

Wrong:

```js
saveProducts();
products.push(item);
```

Correct:

```js
products.push(item);
saveProducts();
```

---

## 3. Handle Empty Storage

```js
if (!stored) return [];
```

---

## 4. Handle Invalid JSON

```js
try {
  JSON.parse(...)
} catch {}
```

# 8. ID Problem After Refresh

## Problem

IDs reset after reload.

---

## Solution

```js
function getNextId() {
  return products.length === 0
    ? 1
    : Math.max(...products.map(p => p.id)) + 1;
}
```

❌ Not using JSON.parse
❌ Saving before updating state
❌ Not re-rendering
