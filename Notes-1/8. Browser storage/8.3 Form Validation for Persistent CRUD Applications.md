# 1. Why Validation is Important

## Concept Explanation

Without validation:

```text
User enters invalid data → stored in state → saved to localStorage → broken app ❌
```

With validation:

```text
User enters data → validate → accept OR reject → clean state ✅
```

# 3. Validating Empty Input

## Concept

Empty values cause broken UI and logic.

---

## Example

```js
if (name.trim() === "") {
  showError("Name is required");
  return;
}
```

---

## Explanation

* `trim()` removes spaces
* Prevents `"   "` input

---

# 4. Validating Numbers

## Concept

User input is always a string.

---

## Example

```js
const priceValue = Number(priceInput.value);

if (isNaN(priceValue) || priceValue <= 0) {
  showError("Invalid price");
  return;
}
```

---

## Common Mistake

Not converting string → number.

---

# 5. Preventing Duplicates

## Using Plain Loop

```js
let exists = false;

for (const product of products) {
  if (product.name === name) {
    exists = true;
    break;
  }
}
```

---

## Using `some()`

```js
const exists = products.some(
  product => product.name === name
);
```

---

## Recommendation

Use `some()` for readability.

---

# 6. Validating All Items

## Using `every()`

```js
const valid = products.every(
  product => product.price > 0
);
```

---

## Explanation

Returns:

* `true` → all valid
* `false` → at least one invalid

---

# 7. Central Validation Function

## Concept

Keep validation logic separate.

---

## Example

```js
function validateProduct(data) {

  if (data.name.trim() === "") {
    return "Name is required";
  }

  if (products.some(p => p.name === data.name)) {
    return "Duplicate product name";
  }

  if (data.price <= 0) {
    return "Invalid price";
  }

  if (data.quantity < 0) {
    return "Invalid quantity";
  }

  return null;
}
```

---

# 8. Using Validation in CRUD

## Create

```js
function createProduct(data) {

  const error = validateProduct(data);

  if (error) {
    showError(error);
    return;
  }

  products = [...products, data];

  saveProducts();
  renderProducts();
}
```

---

# 9. Displaying Errors in the DOM

## Example

```js
function showError(message) {
  errorBox.textContent = message;
  errorBox.classList.add("visible");
}
```

---

## Clearing Errors

```js
function clearError() {
  errorBox.textContent = "";
}
```

# 10. Preventing Form Submission

```js
form.addEventListener("submit", function(event) {
  event.preventDefault();
});
```

# 11. Three Complete Examples

## Example 1: Basic Validation

```js
const name = "";

if (name.trim() === "") {
  console.log("Invalid");
}
```

---

## Example 2: Duplicate Check

```js
const products = [
  { name: "Laptop" }
];

const exists = products.some(
  p => p.name === "Laptop"
);

console.log(exists);
```

---

## Example 3: Full DOM Validation

```js
form.addEventListener("submit", function(event) {
  event.preventDefault();

  const data = {
    name: nameInput.value,
    price: Number(priceInput.value),
    quantity: Number(quantityInput.value)
  };

  const error = validateProduct(data);

  if (error) {
    showError(error);
    return;
  }

  createProduct(data);
});
```
