By the end of this lesson, you will:

* Understand why validation is critical in CRUD apps
* Prevent invalid data from entering state
* Prevent corrupted data from being saved in localStorage
* Validate user input step-by-step before state updates

* Handle real-world edge cases (duplicates, empty input, invalid numbers)
* Integrate validation into the full CRUD flow

# 1. The Core Problem

## What You Will Learn

Why validation must happen BEFORE updating state.

---

## Concept Explanation

Without validation:

```text
User enters bad data
→ state updated
→ localStorage saved
→ UI breaks
→ bugs everywhere ❌
```

---

## Example of Broken State

```js
products = [
  { id: 1, name: "", price: -2000, quantity: "abc" }
];
```

This creates:

* Empty names ❌
* Negative prices ❌
* Invalid numbers ❌

---

## Correct Flow

```text
User input
→ VALIDATE
→ if valid → update state
→ if invalid → show error
```

---

# 2. Types of Validation

You must validate:

| Type      | Example                 |
| --------- | ----------------------- |
| Empty     | ""                      |
| Duplicate | "Laptop" already exists |
| Numeric   | price must be number    |
| Range     | price > 0               |
| Logical   | quantity ≥ 0            |

# 3. Validating Empty Input

## Concept Explanation

Users can enter:

```text
""
"   "
```

We must block both.

---

## Syntax

```js
if (name.trim() === "") {
  return "Name is required";
}
```

---

## Example 1 (Basic)

```js
const name = "   ";

if (name.trim() === "") {
  console.log("Invalid");
}
```

### Output

```text
Invalid
```

---

## Example 2 (Objects)

```js
const product = { name: "" };

if (product.name.trim() === "") {
  console.log("Invalid product");
}
```

---

## Example 3 (DOM)

```js
form.addEventListener("submit", function(e) {
  e.preventDefault();

  const name = nameInput.value;

  if (name.trim() === "") {
    showError("Name is required");
    return;
  }
});
```

---

## Common Mistakes

❌ Not using `trim()`
❌ Allowing whitespace

# 4. Validating Numbers

## Concept Explanation

All input values are strings:

```js
priceInput.value // "50000"
```

---

## Convert to Number

```js
const price = Number(priceInput.value);
```

---

## Validation

```js
if (isNaN(price) || price <= 0) {
  return "Invalid price";
}
```

---

## Example 1 (Basic)

```js
const value = "abc";

console.log(Number(value)); // NaN
```

---

## Example 2 (Objects)

```js
const product = { price: -100 };

if (product.price <= 0) {
  console.log("Invalid");
}
```

---

## Example 3 (DOM)

```js
const price = Number(priceInput.value);

if (isNaN(price) || price <= 0) {
  showError("Enter valid price");
  return;
}
```

---

## Common Mistakes

❌ Not converting string → number
❌ Accepting NaN
❌ Allowing negative values

# 5. Preventing Duplicates

## Concept Explanation

Duplicate names break logic:

```text
Two "Laptop" items → confusion ❌
```

---

## Plain Loop

```js
let exists = false;

for (const p of products) {
  if (p.name === name) {
    exists = true;
    break;
  }
}
```

---

## Array Method

```js
const exists = products.some(
  p => p.name.toLowerCase() === name.toLowerCase()
);
```

---

## Example 1

```js
const items = ["Laptop"];

const exists = items.some(i => i === "Laptop");
```

---

## Example 2

```js
const exists = products.some(p =>
  p.name === "Laptop"
);
```

---

## Example 3 (DOM)

```js
if (products.some(p => p.name === name)) {
  showError("Duplicate product");
  return;
}
```

---

## Common Mistakes

❌ Case-sensitive comparison
❌ Not checking duplicates

---

# 6. Validating All Items

## Concept Explanation

Sometimes validate entire state:

```text
Ensure ALL products are valid
```

---

## Using `every()`

```js
const valid = products.every(
  p => p.price > 0 && p.quantity >= 0
);
```

---

## Example

```js
const valid = [1,2,3].every(n => n > 0);
```

---

## Meaning

* `true` → all valid
* `false` → at least one invalid

---

# 7. Central Validation Function

## Concept Explanation

Instead of scattering validation:

```text
Create ONE validation function
```

---

## Implementation

```js
function validateProduct(data) {

  if (data.name.trim() === "") {
    return "Name is required";
  }

  if (products.some(p =>
    p.name.toLowerCase() === data.name.toLowerCase()
  )) {
    return "Duplicate product name";
  }

  if (isNaN(data.price) || data.price <= 0) {
    return "Invalid price";
  }

  if (data.quantity < 0) {
    return "Invalid quantity";
  }

  return null;
}
```

---

## Why This Matters

```text
Clean → reusable → testable
```

# 8. Full CRUD Flow with Validation

```js
form.addEventListener("submit", function(e) {
  e.preventDefault();

  const data = {
    name: nameInput.value,
    price: Number(priceInput.value),
    quantity: Number(quantityInput.value)
  };

  const error = validateProduct(data);

  if (error) {
    showError(error);
    return;
  }

  createProduct(data);
  saveProducts();
  renderProducts();
});
```

---

## Full Flow

```text
User submits form
→ collect input
→ validate
→ if invalid → show error
→ if valid → update state
→ save
→ render
```

# 9. Displaying Errors in the DOM

```js
function showError(message) {
  errorBox.textContent = message;
  errorBox.style.display = "block";
}
```

---

## Clearing Errors

```js
function clearError() {
  errorBox.textContent = "";
}
```
