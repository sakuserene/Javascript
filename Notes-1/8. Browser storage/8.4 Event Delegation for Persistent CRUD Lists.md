# 1. The Problem Without Event Delegation

## Concept Explanation

Suppose you render items like this:

```js
items.forEach(item => {
  const button = document.createElement("button");

  button.addEventListener("click", () => {
    deleteItem(item.id);
  });
});
```

### Problem

Every render:

```text
New buttons → New listeners → Memory waste ❌
```

Also:

```text
Re-render → old listeners lost ❌
```

---

# 2. What is Event Delegation?

## Concept Explanation

Instead of adding listeners to each child:

```text
Add ONE listener to parent
```

Then detect which child was clicked.

---

## Flow

```text
User clicks button
→ Event bubbles to parent
→ Parent handles event
→ Identify clicked element
→ Perform action
```

---

# 3. Event Bubbling

Events move upward in the DOM:

```text
button → li → ul → body → document
```

We take advantage of this.

---

# 4. Basic Example

```js
list.addEventListener("click", function(event) {
  console.log(event.target);
});
```

* `event.target` → actual clicked element

# 5. Using dataset

## Concept Explanation

We store IDs in HTML:

```html
<button data-id="1">Delete</button>
```

---

## Access in JS

```js
event.target.dataset.id
```

---

# 6. Handling Multiple Actions

## HTML Example

```html
<button data-action="delete" data-id="1">Delete</button>
<button data-action="edit" data-id="1">Edit</button>
```

---

## JavaScript

```js
list.addEventListener("click", function(event) {

  const action = event.target.dataset.action;
  const id = Number(event.target.dataset.id);

  if (!action) return;

  if (action === "delete") {
    deleteItem(id);
  }

  if (action === "edit") {
    startEdit(id);
  }

});
```

---

# 7. Three Complete Examples

## Example 1: Basic Click Detection

```js
document.body.addEventListener("click", function(event) {
  console.log(event.target.tagName);
});
```

---

## Example 2: Action Handling

```js
list.addEventListener("click", function(event) {

  const action = event.target.dataset.action;

  if (action === "delete") {
    console.log("Delete clicked");
  }

});
```

---

## Example 3: Full CRUD Integration

```js
list.addEventListener("click", function(event) {

  const action = event.target.dataset.action;
  const id = Number(event.target.dataset.id);

  if (!action) return;

  if (action === "delete") {
    products = products.filter(p => p.id !== id);
    saveProducts();
    renderProducts();
  }

  if (action === "edit") {
    loadEditForm(id);
  }

  if (action === "toggle") {
    products = products.map(p =>
      p.id === id
        ? { ...p, completed: !p.completed }
        : p
    );

    saveProducts();
    renderProducts();
  }

});
```

# 8. Why Event Delegation is Important

## Advantages

* Only ONE listener
* Works for dynamic elements
* No duplicate listeners
* Better performance
* Cleaner code

---

# 9. Comparing Approaches

## Without Delegation

```js
button.addEventListener("click", ...)
```

Problems:

* Many listeners
* Hard to manage
* Breaks on re-render

---

## With Delegation

```js
parent.addEventListener("click", ...)
```

Advantages:

* Scalable
* Clean
* Reliable

---

# 10. Common Mistakes

* Forgetting to check `dataset.action`
* Using string ID instead of number
* Not checking if target is a button
* Triggering events from wrong element
* Missing re-render after action

---

# 11. Best Practices

* Use one listener per container
* Use `dataset` for IDs and actions
* Validate action before executing
* Keep logic simple
* Always re-render after state change
