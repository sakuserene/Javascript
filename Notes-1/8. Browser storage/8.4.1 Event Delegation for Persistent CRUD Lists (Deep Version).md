# 1. The Core Problem

## What You Will Learn

Why attaching listeners directly to elements breaks in CRUD apps.

---

## Concept Explanation

In CRUD apps:

```text
You constantly re-render the DOM
```

Example:

```js
function renderProducts() {
  list.innerHTML = "";

  products.forEach(product => {
    const btn = document.createElement("button");

    btn.addEventListener("click", () => {
      deleteProduct(product.id);
    });

    list.appendChild(btn);
  });
}
```

---

## Problems

```text
Every render:
→ New elements created
→ New listeners attached
→ Old listeners destroyed

Results:
❌ Memory waste
❌ Duplicate listeners
❌ Hard debugging
```

---

# 2. The Solution: Event Delegation

## Concept Explanation

Instead of attaching listeners to each button:

```text
Attach ONE listener to parent container
```

---

## Flow

```text
User clicks child element
→ Event bubbles up
→ Parent catches event
→ Identify clicked element
→ Perform action
```

---

# 3. Event Bubbling (Deep Understanding)

## Concept

Events move upward:

```text
button → li → ul → body → document
```

---

## Example

```js
document.body.addEventListener("click", (e) => {
  console.log(e.target);
});
```

---

## Why This Matters

We intercept events at a higher level.

---

# 4. Identifying the Clicked Element

## Using event.target

```js
list.addEventListener("click", function(event) {
  console.log(event.target);
});
```

---

## Problem

User might click:

* Button
* Icon inside button
* Text inside button

---

## Solution: Closest Element

```js
const button = event.target.closest("button");
```

---

# 5. Using dataset for Actions

## HTML

```html
<button data-action="delete" data-id="1">Delete</button>
<button data-action="edit" data-id="1">Edit</button>
```

---

## JavaScript

```js
const action = button.dataset.action;
const id = Number(button.dataset.id);
```

---

# 6. Handling Multiple Actions

## Full Handler

```js
list.addEventListener("click", function(event) {

  const button = event.target.closest("button");

  if (!button) return;

  const action = button.dataset.action;
  const id = Number(button.dataset.id);

  if (!action) return;

  if (action === "delete") {
    deleteProduct(id);
  }

  if (action === "edit") {
    startEdit(id);
  }

  if (action === "toggle") {
    toggleProduct(id);
  }

});
```

## Example 2 (Action Detection)

```js
});
```

---

## Example 3 (Full CRUD Integration)

```js
list.addEventListener("click", function(e) {

  const btn = e.target.closest("button");

  if (!btn) return;

  const action = btn.dataset.action;
  const id = Number(btn.dataset.id);

  if (action === "delete") {
    products = products.filter(p => p.id !== id);
  }

  if (action === "edit") {
    loadEditForm(id);
  }

  if (action === "toggle") {
    products = products.map(p =>
      p.id === id
        ? { ...p, active: !p.active }
        : p
    );
  }

  saveProducts();
  renderProducts();

});
```

---

## Full Flow

```text
User clicks button
→ event bubbles
→ parent listener triggered
→ identify button
→ extract action + id
→ update state
→ save
→ re-render
```

# 8. Why Event Delegation is Powerful

## Advantages

* One listener only
* Works after re-render
* No duplication
* Better performance
* Cleaner architecture

---

# 9. Comparing Approaches

## Without Delegation

```js
button.addEventListener(...)
```

Problems:

❌ Many listeners
❌ Breaks on re-render

---

## With Delegation

```js
parent.addEventListener(...)
```

Advantages:

✅ Scalable
✅ Efficient

# 10. Edge Cases

# 11. Common Mistakes (Deep)

❌ Missing null checks

❌ Multiple listeners attached
