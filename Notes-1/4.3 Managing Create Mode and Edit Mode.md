# 14. Managing Create Mode and Edit Mode

## 14.1 What You Will Learn

A CRUD form can be designed in two ways:

1. Separate Create and Update forms.
2. One reusable form that changes mode.

This course will use one reusable form.

The form behaves differently depending on:

```js
editingProductId
```

If:

```js
editingProductId === null
```

the form is in create mode.

If:

```js
editingProductId !== null
```

the form is in edit mode.

---

# 14.2 Why Use One Form?

One form avoids duplicated HTML.

Without reuse, you might create:

```html
<form id="create-product-form">
  <!-- Product fields -->
</form>

<form id="edit-product-form">
  <!-- Same product fields again -->
</form>
```

Problems:

* Repeated markup.
* Repeated validation.
* Repeated event logic.
* Greater risk of inconsistent fields.
* More code to maintain.

A single form is simpler:

```html
<form id="product-form">
  <!-- Shared product fields -->
</form>
```

JavaScript changes its behavior based on mode.

---

# 14.3 The Mode State Variable

```js
let editingProductId = null;
```

This variable is temporary UI state.

It does not represent a product property.

It represents the form's current purpose.

### Create mode

```js
editingProductId = null;
```

### Edit mode

```js
editingProductId = 205;
```

---

# 14.4 Checking the Current Mode

Create a helper function:

```js
function isEditingProduct() {
  return editingProductId !== null;
}
```

Usage:

```js
if (isEditingProduct()) {
  console.log("Edit mode");
} else {
  console.log("Create mode");
}
```

This improves readability.

---

# 14.5 Create-Mode UI

HTML elements:

```js
const formTitle =
  document.getElementById(
    "form-title"
  );

const submitButton =
  document.getElementById(
    "submit-button"
  );

const cancelEditButton =
  document.getElementById(
    "cancel-edit-button"
  );

const editingMessage =
  document.getElementById(
    "editing-message"
  );
```

Create-mode function:

```js
function setCreateModeUI() {
  formTitle.textContent =
    "Add Product";

  submitButton.textContent =
    "Add Product";

  cancelEditButton.hidden =
    true;

  editingMessage.textContent =
    "Create a new product.";

```

# 14.6 Edit-Mode UI

```js
function setEditModeUI(product) {
  formTitle.textContent =
    "Edit Product";

  submitButton.textContent =
    "Update Product";

  cancelEditButton.hidden =
    false;

  editingMessage.textContent =
    `Editing: ${product.name} ` +
    `(ID: ${product.id})`;
}
```

The UI now reflects the current action.

# 14.7 Entering Edit Mode

```js
function startEditingProduct(
  productId
) {
  clearError();

  const product =
    findProductById(productId);

  if (product === null) {
    showError(
      "The selected product was not found."
    );

    return;
  }

  editingProductId =
    product.id;

  loadProductIntoForm(product);
  setEditModeUI(product);

  productNameInput.focus();
  productNameInput.select();
}
```

The order matters:

1. Find the item.
2. Store the stable ID.
3. Load the values.
4. Update the UI.

---

# 14.8 Leaving Edit Mode

Create a reusable reset function:

```js
function exitEditMode() {
  editingProductId = null;

  productForm.reset();

  productActiveInput.checked =
    true;

  clearError();

  setCreateModeUI();

  productNameInput.focus();
}
```

This returns the form to create mode.

---

# 14.9 Cancel Edit Behavior

```js
cancelEditButton.addEventListener(
  "click",
  function () {
    exitEditMode();
  }
);
```

The button must use:

```html
type="button"
```

Otherwise, it may submit the form.

---

# 14.10 Why Cancel Must Not Change State

Suppose state contains:

```js
{
  id: 205,
  name: "Phone",
}
```

The user changes the form input to:

```text
Smartphone
```

Then clicks Cancel.

State should remain:

```js
{
  id: 205,
  name: "Phone",
}
```

Only the temporary form values should be cleared.

This works because state is not changed until valid submission.

---

# 14.11 Form Submission Must Check the Mode

A shared submit handler needs a decision.

```js
function handleProductSubmit(event) {
  event.preventDefault();

  if (editingProductId === null) {
    console.log(
      "Create a new product."
    );
  } else {
    console.log(
      "Update an existing product."
    );
  }
}
```

Later, the branches will call:

```js
createProductFromForm();
```

or:

```js
updateProductFromForm();
```

---

# 14.12 Recommended Submit Structure

```js
function handleProductSubmit(event) {
  event.preventDefault();

  clearError();

  const rawData =
    readProductForm();

  const validationResult =
    validateProductForm(rawData);

  if (!validationResult.isValid) {
    showError(
      validationResult.message
    );

    focusInvalidField(
      validationResult.field
    );

    return;
  }

  if (editingProductId === null) {
    handleCreateProduct(
      validationResult.normalizedData
    );
  } else {
    handleUpdateProduct(
      validationResult.normalizedData
    );
  }
}
```

The form data and validation can be reused in both modes.

# 14.13 Create Branch

```js
function handleCreateProduct(data) {
  const product = {
    id: generateProductId(),
    name: data.name,
    price: data.price,
    quantity: data.quantity,
    category: data.category,
    isActive: data.isActive,
  };

  products.push(product);

  exitEditMode();
  renderProducts();
}
```

Although `exitEditMode()` sounds edit-specific, it resets the shared form to create mode.

You may choose a more general name:

```js
resetProductFormToCreateMode();
```

# 14.14 Update Branch Preview

```js
function handleUpdateProduct(data) {
  console.log(
    "Update product ID:",
    editingProductId
  );

  console.log(data);
}
```

The actual object mutation will be taught in the next subtopic.

# 14.15 Why the Editing ID Must Persist

The ID must survive between:

1. Clicking Edit.
2. Changing input values.
3. Submitting the form.

Therefore, it must be declared outside those event handlers.

Correct:

```js
let editingProductId = null;
```

Incorrect:

```js
function startEditingProduct(id) {
  let editingProductId = id;
}
```

The local variable disappears when the function ends.

The submit handler cannot access it.

---

# 14.16 Avoid Hidden Input as the Main State

You could store the ID in HTML:

```html
<input
  id="editing-product-id"
  type="hidden"
/>
```

But for this lesson, JavaScript should remain the main source of truth:

```js
let editingProductId = null;
```

A hidden input can be altered in browser tools.

Important actions should always validate the ID against the current state.

# 14.17 Switching Directly Between Items

Suppose the user is editing Laptop and then clicks Edit on Phone.

The application should:

1. Replace `editingProductId`.
2. Load Phone values.
3. Update the edit message.
4. Keep the form in edit mode.

```js
startEditingProduct(205);
```

No special reset is required before loading the next item.

The new selection replaces the previous temporary form values.

---

# 14.18 Unsaved Changes Warning

In a more advanced application, switching items could discard unsaved input.

You might track:

```js
let hasUnsavedChanges = false;
```

Then ask for confirmation.

For now, clicking another Edit button will replace the form values.

This behavior should be documented clearly.

# 14.19 Form Input Events and Dirty State

Optional UI state:

```js
let isFormDirty = false;
```

Listen for input:

```js
productForm.addEventListener(
  "input",
  function () {
    isFormDirty = true;
  }
);
```

After reset:

```js
isFormDirty = false;
```

This can help detect unsaved changes.

It is not required for basic CRUD.

# 14.20 Stale Edit Mode

A stale edit state occurs when:

```js
editingProductId = 205;
```

but the product no longer exists.

This might happen if:

* It was deleted elsewhere.
* State was reset.
* Data changed asynchronously.
* The UI is outdated.

Before updating, search again by ID.

```js
const product =
  findProductById(
    editingProductId
  );

if (product === null) {
  showError(
    "The product no longer exists."
  );

  exitEditMode();
  return;
}
```

---

# 14.21 Do Not Store the Selected Object Permanently

You might consider:

```js
let editingProduct = product;
```

This stores a direct reference.

It can work, but an ID is safer because:

* The object might be removed.
* State might be replaced immutably.
* Sorting does not matter.
* The application can revalidate existence before saving.
* IDs serialize easily.

Prefer:

```js
let editingProductId = product.id;
```

---

# 14.22 Highlighting the Edited Card

You can derive a CSS class during rendering.

```js
if (
  product.id ===
  editingProductId
) {
  card.classList.add(
    "currently-editing"
  );
}
```

This visually links the form to the selected product.

The highlight is derived from UI state.

---

# 14.23 Why Re-render After Entering Edit Mode?

If the render function applies an editing class, call:

```js
renderProducts();
```

after setting:

```js
editingProductId =
  product.id;
```

Then the selected card can be highlighted.

However, if rendering only displays product data and no edit indicator, re-rendering is not required merely to load the form.

---

# 14.24 Preserve Edit State During Rendering

A render function should not reset:

```js
editingProductId
```

Incorrect:

```js
function renderProducts() {
  editingProductId = null;
}
```

Rendering should display state, not destroy UI mode.

---

# 14.25 Disable Editing the Same Item

Optional behavior:

```js
editButton.disabled =
  product.id === editingProductId;
```

The currently edited product's Edit button becomes disabled.

This is a derived DOM state.

# 14.26 Duplicate Name Validation During Edit

Suppose Phone is being renamed to Laptop.

The duplicate check should ignore the item being edited.

Plain-loop version:

```js
function productNameExists(
  name,
  ignoredProductId = null
) {
  const normalizedName =
    name.trim().toLowerCase();

  for (
    let index = 0;
    index < products.length;
    index++
  ) {
    const product =
      products[index];

    if (
      product.id ===
      ignoredProductId
    ) {
      continue;
    }

    const existingName =
      product.name
        .trim()
        .toLowerCase();

    if (
      existingName ===
      normalizedName
    ) {
      return true;
    }
  }

  return false;
}
```

Usage in create mode:

```js
productNameExists(name);
```

Usage in edit mode:

```js
productNameExists(
  name,
  editingProductId
);
```

---

# 14.27 Why Ignore the Current Product?

State:

```js
[
  {
    id: 101,
    name: "Laptop",
  },
  {
    id: 205,
    name: "Phone",
  },
]
```

When editing Phone without changing its name, the duplicate checker sees `"Phone"` in state.

Without ignoring ID `205`, it would incorrectly reject its own existing name.

---

# 14.28 Reusing Validation in Both Modes

```js
const duplicateExists =
  productNameExists(
    data.name,
    editingProductId
  );
```

In create mode:

```js
editingProductId === null
```

No product is ignored.

In edit mode:

```js
editingProductId === 205
```

The current product is ignored.

---

# 14.29 Mode-Based Validation Message

You may use different messages:

```js
const actionWord =
  editingProductId === null
    ? "create"
    : "update";
```

Example:

```js
showError(
  `Unable to ${actionWord} the product.`

```

This is optional.

```html
      productPriceInput.value =
        product.price;

      productQuantityInput.value =
        product.quantity;

      productCategorySelect.value =
        product.category;

      productActiveInput.checked =
        product.isActive;
    }

    function setCreateModeUI() {
      formTitle.textContent =
        "Add Product";

      submitButton.textContent =
        "Add Product";

      cancelEditButton.hidden =
        true;

      editingMessage.textContent =
        "Create a new product.";
    }

    function setEditModeUI(
      product
    ) {
      formTitle.textContent =
        "Edit Product";

      submitButton.textContent =
        "Update Product";

      cancelEditButton.hidden =
        false;

      editingMessage.textContent =
        `Editing: ${product.name} ` +
        `(ID: ${product.id})`;
    }

    function resetFormToCreateMode() {
      editingProductId = null;

      productForm.reset();

      productActiveInput.checked =
        true;

      clearError();
      setCreateModeUI();

      productNameInput.focus();
    }

    function startEditingProduct(
      productId
    ) {
      clearError();

      const product =
        findProductById(productId);

      if (product === null) {
        showError(
          "The selected product no longer exists."
        );

        resetFormToCreateMode();
        return;
      }

      editingProductId =
        product.id;

      loadProductIntoForm(product);
      setEditModeUI(product);
      renderProducts();

      productNameInput.focus();
      productNameInput.select();
    }

    function renderProducts() {
      productList.innerHTML = "";

      for (
        let index = 0;
        index < products.length;
        index++
      ) {
        const product =
          products[index];

        const card =
          document.createElement("li");

        const name =
          document.createElement("h3");

        const details =
          document.createElement("p");

        const editButton =
          document.createElement("button");

        card.classList.add(
          "product-card"
        );

        if (
          product.id ===
          editingProductId
        ) {
          card.classList.add(
            "currently-editing"
          );
        }

        name.textContent =
          product.name;

        details.textContent =
          `KSh ${product.price.toLocaleString()} — ` +
          `Quantity: ${product.quantity}`;

        editButton.type =
          "button";

        editButton.textContent =
          "Edit";

        editButton.dataset.action =
          "edit";

        editButton.dataset.id =
          product.id;

        editButton.disabled =
          product.id ===
          editingProductId;

        card.append(
          name,
          details,
          editButton
        );

        productList.appendChild(
          card
        );
      }
    }

    productList.addEventListener(
      "click",
      function (event) {
        const editButton =
          event.target.closest(
            '[data-action="edit"]'
          );

        if (editButton === null) {
          return;
        }

        const productId =
          Number(
            editButton.dataset.id
          );

        if (
          !Number.isInteger(
            productId
          ) ||
          productId <= 0
        ) {
          showError(
            "Invalid product ID."
          );

          return;
        }

        startEditingProduct(
          productId
        );
      }
    );

    cancelEditButton.addEventListener(
      "click",
      function () {
        resetFormToCreateMode();
        renderProducts();
      }
    );

    productForm.addEventListener(
      "submit",
      function (event) {
        event.preventDefault();

        clearError();

        const data =
          readProductForm();

        if (data.name === "") {
          showError(
            "Product name is required."
          );

          productNameInput.focus();
          return;
        }

        if (
          editingProductId ===
          null
        ) {
          const newProduct = {
            id: generateProductId(),
            name: data.name,
            price: data.price,
            quantity: data.quantity,
            category: data.category,
            isActive: data.isActive,
          };

          products.push(
            newProduct
          );

          resetFormToCreateMode();
          renderProducts();

          return;
        }

        console.log(
          "Update will be implemented in the next lesson.",
          {
            editingProductId,
            data,
          }
        );
      }
    );

    setCreateModeUI();
    renderProducts();
  </script>
</body>
</html>
```

# 14.31 Create Mode Versus Edit Mode

| Feature               | Create Mode    | Edit Mode               |
| --------------------- | -------------- | ----------------------- |
| `editingProductId`    | `null`         | Stable item ID          |
| Form title            | Add Product    | Edit Product            |
| Submit text           | Add Product    | Update Product          |
| Cancel button         | Hidden         | Visible                 |
| Initial form values   | Empty/default  | Existing item values    |
| Submit result         | New object     | Existing object changed |
| ID generation         | Yes            | No                      |
| Existing ID preserved | Not applicable | Yes                     |

# 14.32 Common Beginner Mistakes

## Mistake 1: Generating a New ID During Update

Update must preserve the current ID.

---

## Mistake 2: Forgetting to Reset Mode After Success

The next submission may update the old item accidentally.

---

## Mistake 3: Form UI and Mode State Disagree

Example:

```js
editingProductId = null;
```

but button still displays:

```text
Update Product
```

Keep internal and visual state synchronized.

---

## Mistake 4: Cancel Button Submits the Form

Use:

```html
type="button"
```

---

## Mistake 5: Duplicate Validation Rejects the Current Item

Ignore `editingProductId` during duplicate checking.

---

## Mistake 6: Storing an Index

Sorting or filtering may update the wrong product.

---

## Mistake 7: Resetting State Instead of Form State

Incorrect:

```js
products = [];
```

Correct:

```js
productForm.reset();
```

---

## Mistake 8: Render Function Exits Edit Mode

Rendering should preserve UI state.

---

## Mistake 9: Changing Inputs Mutates State Immediately

Temporary form values should remain separate until submission.

---

## Mistake 10: Missing Item Leaves Form in Broken Edit Mode

Reset safely when a stale ID is discovered.
