# 1. The Core Problem

## What You Will Learn

Why files cannot access each other by default.

---

## Concept Explanation

If you have:

```js
// state.js
let items = [];
```

And in another file:

```js
// logic.js
console.log(items); ❌
```

This will fail.

---

## Why?

```text
Each file is isolated (its own scope)
```

---

# 2. The Solution: Export and Import

## Concept

We must explicitly share data:

```text
export → make available
import → use in another file
```

---

# 3. Named Exports

## Syntax

```js
export const items = [];

export function addItem() {}
```

---

## Import

```js
import { items, addItem } from "./state.js";
```

---

## Key Rule

```text
Names MUST match exactly
```

---

# 4. Default Exports

## Syntax

```js
export default function addItem() {}
```

---

## Import

```js
import addItem from "./logic.js";
```

---

## Key Difference

| Type    | Import Style |
| ------- | ------------ |
| Named   | { name }     |
| Default | anyName      |

---

# 5. Syntax Breakdown

```js
import { items } from "./state.js";
```

| Part         | Meaning        |
| ------------ | -------------- |
| import       | keyword        |
| { items }    | what to import |
| "./state.js" | file path      |

# 6. Three Complete Examples

---

## Example 1: Basic Export/Import

### state.js

```js
export const numbers = [1,2,3];
```

### main.js

```js
import { numbers } from "./state.js";

console.log(numbers);
```

---

### Output

```text
[1,2,3]
```

---

## Example 2: Functions

### logic.js

```js
export function add(a, b) {
  return a + b;
}
```

### main.js

```js
import { add } from "./logic.js";

console.log(add(2,3));
```

---

### Output

```text
5
```

---

## Example 3: Full CRUD Connection

### state.js

```js
export let items = [];

export function setItems(newItems) {
  items = newItems;
}
```

---

### logic.js

```js
import { items, setItems } from "./state.js";

export function createItem(data) {
  const newItem = {
    id: Date.now(),
    ...data
  };

  setItems([...items, newItem]);
}
```

---

### ui.js

```js
import { items } from "./state.js";

export function render(list) {
  list.innerHTML = "";

  items.forEach(item => {
    const li = document.createElement("li");
    li.textContent = item.name;
    list.appendChild(li);
  });
}
```

---

### main.js

```js
import { createItem } from "./logic.js";
import { render } from "./ui.js";

const list = document.getElementById("list");

createItem({ name: "Laptop" });
render(list);
```

---

## Flow

```text
main.js → logic.js → state.js → ui.js → DOM
```

# 7. Important Rules

## 1. Use `.js` extension

```js
import { x } from "./file.js"; ✅
```

---

## 2. Use relative paths

```js
"./file.js"
```

---

## 3. Use type="module"

```html
<script type="module" src="main.js"></script>
```

---

# 8. Common Errors

## ❌ Error 1: Missing .js

```js
import { x } from "./file"; ❌
```

---

## ❌ Error 2: Wrong path

```js
import { x } from "file.js"; ❌
```

---

## ❌ Error 3: Not using module script

```html
<script src="main.js"></script> ❌
```

---

## ❌ Error 4: Name mismatch

```js
export const items = [];
import { item } from "./state.js"; ❌
```

# 9. Named vs Default — When to Use

## Named Export

Use when:

* Multiple exports
* Clear structure

---

## Default Export

Use when:

* One main function per file
