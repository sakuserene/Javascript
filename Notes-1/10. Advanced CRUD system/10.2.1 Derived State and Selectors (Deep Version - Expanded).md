# 1. Understanding Derived State (Deep Concept)

## What You Will Learn

What derived state really means in real applications.

---

## Concept Explanation

You have two types of data:

### 1. Stored State

```js
let products = [
  { id: 1, price: 1000, quantity: 2 }
];
```

### 2. Derived State

```js
const total = products.reduce(
  (sum, p) => sum + p.price * p.quantity,
  0
);
```

---

## Key Idea

```text
Derived State = A FUNCTION of your existing state
```

---

## Important Rule

> ❗ Derived state must NEVER be stored permanently

---

# 2. Why Storing Derived State is Dangerous

## Bad Example

```js
let state = {
  products: [...],
  total: 10000
};
```

---

## What Goes Wrong

```text
User updates product price
→ products updated
→ total NOT updated
→ UI shows incorrect value ❌
```

---

## Real Bug Scenario

```text
Total says: 10,000
Actual total: 12,000
```

---

## Correct Approach

```js
function getTotal(products) {
  return products.reduce(
    (sum, p) => sum + p.price * p.quantity,
    0
  );
}
```

---

# 3. What is a Selector?

## Definition

```text
A selector is a function that computes derived data from state
```

---

## Example

```js
function getTotal(products) {
  return products.reduce(
    (sum, p) => sum + p.price * p.quantity,
    0
  );
}
```

---

## Why Use Selectors?

* Centralize logic
* Avoid duplication
* Improve readability
* Make testing easier

---

# 4. Syntax Breakdown (Important)

```js
products.reduce((sum, product) => {
  return sum + product.price * product.quantity;
}, 0);
```

### Breakdown

| Part     | Meaning             |
| -------- | ------------------- |
| products | state array         |
| reduce() | loops through array |
| sum      | accumulator         |
| product  | current item        |
| 0        | initial value       |

---

# 5. Three Complete Examples

---

## Example 1: Basic (No DOM)

### Problem

Calculate sum of numbers.

```js
const numbers = [1, 2, 3, 4];

function getSum(arr) {
  return arr.reduce((sum, n) => sum + n, 0);
}

console.log(getSum(numbers));
```

---

### Output

```text
10
```

---

### Final State

```js
numbers // unchanged
```

---

### Common Mistake

```js
numbers = numbers.reduce(...) ❌
```

---

## Example 2: Intermediate (Objects)

### Problem

Calculate inventory value.

```js
const products = [
  { name: "Laptop", price: 1000, quantity: 2 },
  { name: "Mouse", price: 500, quantity: 1 }
];

function getInventoryValue(products) {
  return products.reduce(
    (total, product) => total + product.price * product.quantity,
    0
  );
}

console.log(getInventoryValue(products));
```

---

### Output

```text
2500
```

---

### Step-by-Step

```text
Laptop → 1000 * 2 = 2000
Mouse → 500 * 1 = 500
Total = 2500
```

---

## Example 3: DOM + CRUD Application

### Problem

Display total in UI.

```js
function renderTotal() {
  const total = getInventoryValue(products);

  totalElement.textContent = `Total: KSh ${total}`;
}
```

---

### Flow

```text
User adds item
→ state updated
→ selector runs
→ total recalculated
→ DOM updated
```

---

### Expected DOM Output

```text
Total: KSh 2500
```

# 6. Types of Selectors in CRUD Apps

---

## 1. Total Selector

```js
function getTotal(products) {
  return products.reduce(
    (sum, p) => sum + p.price * p.quantity,
    0
  );
}
```

---

## 2. Count Selector

```js
function getCount(items) {
  return items.length;
}
```

---

## 3. Filter Selector

```js
function getCompleted(tasks) {
  return tasks.filter(t => t.completed);
}
```

---

## 4. Find Selector

```js
function getById(list, id) {
  return list.find(item => item.id === id);
}
```

---

## 5. Search Selector

```js
function searchProducts(products, term) {
  return products.filter(p =>
    p.name.toLowerCase().includes(term.toLowerCase())
  );
}
```

---

# 7. Combining Selectors (Advanced)

## Concept

Selectors can be chained.

---

## Example

```js
function getInStock(products) {
  return products.filter(p => p.quantity > 0);
}

function getInStockTotal(products) {
  return getInStock(products).reduce(
    (sum, p) => sum + p.price * p.quantity,
    0
  );
}
```

---

## Flow

```text
products → filter → reduce → result
```

---

# 8. Derived State in Rendering

## Important Pattern

```js
function renderProducts() {
  const visibleProducts = getVisibleProducts(products);

  list.innerHTML = "";

  visibleProducts.forEach(product => {
    const li = document.createElement("li");
    li.textContent = product.name;
    list.appendChild(li);
  });
}
```

---

## Key Idea

```text
Render ALWAYS uses derived state
```

---

# 9. Performance Considerations

## Problem

Selectors run frequently:

```text
Every render → recompute
```

---

## Optimization: Memoization

```js
let lastProducts = null;
let lastResult = null;

function getTotalMemo(products) {

  if (products === lastProducts) {
    return lastResult;
  }

  lastProducts = products;
  lastResult = getTotal(products);

  return lastResult;
}
```

---

## When to Optimize

* Large datasets
* Frequent renders
* Complex calculations

---

# 10. Edge Cases

* Empty array → total = 0
* Missing values → handle safely
* Invalid numbers → NaN issues
* Large lists → performance concerns

---

# 11. Common Mistakes (Deep)

❌ Storing derived state
❌ Mutating state inside selector
❌ Mixing DOM logic in selectors
❌ Not reusing selectors
❌ Incorrect reduce usage

---

# 12. Best Practices

* Keep selectors pure
* Never mutate state
* Always compute values
* Reuse selectors
* Combine selectors carefully
* Optimize only when needed
