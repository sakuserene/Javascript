# 1. The Concept of State History

## Explanation

Instead of storing only current state:

```js
let products = [];
```

We store:

```js
const stateHistory = {
  past: [],
  present: [],
  future: []
};
```

---

## Meaning

| Property | Description     |
| -------- | --------------- |
| past     | Previous states |
| present  | Current state   |
| future   | Undone states   |

---

# 2. Why This Matters

Without history:

```text
User deletes item → cannot recover ❌
```

With history:

```text
Undo → restore deleted item ✅
```

---

# 3. Saving State Snapshots

## Concept

Each time state changes:

```text
Move current → past
Set new state → present
Clear future
```

---

## Implementation

```js
function saveState(newState) {
  stateHistory.past.push(stateHistory.present);

  stateHistory.present = newState;

  stateHistory.future = [];
}
```

---

# 4. Undo Operation

## Concept

```text
Move present → future
Restore last past → present
```

---

## Implementation

```js
function undo() {
  if (stateHistory.past.length === 0) return;

  const previous = stateHistory.past.pop();

  stateHistory.future.push(stateHistory.present);

  stateHistory.present = previous;

  renderProducts();
}
```

---

# 5. Redo Operation

## Concept

```text
Move present → past
Restore future → present
```

---

## Implementation

```js
function redo() {
  if (stateHistory.future.length === 0) return;

  const next = stateHistory.future.pop();

  stateHistory.past.push(stateHistory.present);

  stateHistory.present = next;

  renderProducts();
}
```

# 6. Three Complete Examples

## Example 1: Basic Numbers

## Example 2: CRUD State


# 7. Preventing Duplicate States

## Problem

```text
Saving same state multiple times ❌
```

---

## Solution

```js
function isSameState(a, b) {
  return JSON.stringify(a) === JSON.stringify(b);
}
```

---

# 8. Limiting History Size

## Problem

Too much memory usage.

---

## Solution

```js
const MAX_HISTORY = 10;

if (stateHistory.past.length > MAX_HISTORY) {
  stateHistory.past.shift();
}
```

---

# 9. Immutable Snapshots

## Important Rule

Never store references.

---

## Correct

```js
stateHistory.past.push([...stateHistory.present]);
```

---

## Wrong

```js
stateHistory.past.push(stateHistory.present);
```

---

# 10. Common Mistakes

* Mutating past states
* Not clearing future
* Not limiting history
* Saving duplicate states
* Forgetting to re-render

---

# 11. Best Practices

* Use immutable updates
* Limit history size
* Avoid duplicates
* Keep logic clean
* Separate history logic
