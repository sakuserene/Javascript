# 1. What is Derived State?

## Concept Explanation

Derived state is:

```text
Data that is CALCULATED from existing state
```

---

## Example

```js
const products = [
  { price: 1000, quantity: 2 }
];

const total = products.reduce(
  (sum, p) => sum + p.price * p.quantity,
  0
);
```

---

## Key Rule

> ❗ NEVER store derived state inside your main state

---

## Why?

```text
Stored value → can become outdated ❌
Derived value → always correct ✅
```

---

# 2. The Problem with Storing Derived State

## Bad Example

```js
let state = {
  products: [...],
  totalValue: 10000
};
```

---

## What Goes Wrong

```text
User updates product
→ totalValue NOT updated ❌
→ UI shows wrong data ❌
```

---

## Correct Approach

```js
const totalValue = calculateTotal(products);
```

# 3. Selectors (Core Concept)

## What You Will Learn

Selectors = functions that compute derived state.

---

## Example

```js
function getTotalValue(products) {
  return products.reduce(
    (sum, p) => sum + p.price * p.quantity,
    0
  );
}
```

---

## Why Use Selectors?

```text
✔ Reusable
✔ Clean
✔ Testable
✔ Centralized logic
```

---

# 4. Three Complete Examples

---

## Example 1 (Basic Numbers)

```js
const numbers = [1,2,3];

function getSum(arr) {
  return arr.reduce((sum, n) => sum + n, 0);
}

console.log(getSum(numbers));
```

### Output

```text
6
```

---

## Example 2 (Objects)

```js
const products = [
  { price: 1000, quantity: 2 },
  { price: 500, quantity: 1 }
];

function getTotal(products) {
  return products.reduce(
    (sum, p) => sum + p.price * p.quantity,
    0
  );
}

console.log(getTotal(products));
```

---

## Example 3 (DOM + CRUD)

```js
function renderTotal() {
  const total = getTotal(products);

  totalElement.textContent = "Total: " + total;
}
```

---

## Flow

```text
State → selector → computed value → render → UI
```

# 5. Common Selectors in CRUD Apps

## 1. Total Value

```js
function getTotal(products) {
  return products.reduce(
    (sum, p) => sum + p.price * p.quantity,
    0
  );
}
```

---

## 2. Completed Items

```js
function getCompleted(tasks) {
  return tasks.filter(t => t.completed);
}
```

---

## 3. Count

```js
function getCount(items) {
  return items.length;
}
```

---

## 4. Find by ID

```js
function getById(list, id) {
  return list.find(item => item.id === id);
}
```

---

## 5. Filtered List

```js
function getVisible(products, term) {
  return products.filter(p =>
    p.name.toLowerCase().includes(term)
  );
}
```

---

# 6. Combining Selectors

## Concept

Selectors can call other selectors.

---

## Example

```js
function getInStock(products) {
  return products.filter(p => p.quantity > 0);
}

function getInStockTotal(products) {
  return getInStock(products).reduce(
    (sum, p) => sum + p.price * p.quantity,
    0
  );
}
```

# 7. Performance Considerations

## Problem

Selectors run often:

```text
Every render → recompute
```

---

## Optimization Idea

```text
Cache results (memoization)
```

---

## Basic Example

```js
let lastProducts = null;
let lastResult = null;

function getTotalMemo(products) {

  if (products === lastProducts) {
    return lastResult;
  }

  lastProducts = products;
  lastResult = getTotal(products);

  return lastResult;
}
```
