# 1. The Core Problem

## What You Will Learn

Why basic CRUD apps cannot reverse actions.

---

## Concept Explanation

Normal CRUD:

```text
User deletes item → state updated → item gone forever ❌
```

No recovery.

---

## Real-World Expectation

```text
User deletes item → clicks Undo → item restored ✅
```

---

# 2. The Solution: State History

## Concept Explanation

We store history of state changes.

---

## Structure

```js
const stateHistory = {
  past: [],
  present: [],
  future: []
};
```

---

## Meaning

| Part    | Role            |
| ------- | --------------- |
| past    | previous states |
| present | current state   |
| future  | undone states   |

---

# 3. State Transition Flow

## On Every Change

```text
1. Move present → past
2. Set new present
3. Clear future
```

---

## Visual Flow

```text
Past:   [A, B]
Present: C
Future: []

New change → D

Past:   [A, B, C]
Present: D
Future: []
```

# 4. Saving State (Deep Understanding)

## Problem

If you mutate state:

```js
stateHistory.past.push(stateHistory.present);
```

You store reference ❌

---

## Correct Approach

```js
function cloneState(state) {
  return JSON.parse(JSON.stringify(state));
}
```

---

## Save Function

```js
function saveState(newState) {

  if (JSON.stringify(stateHistory.present) === JSON.stringify(newState)) {
    return; // avoid duplicates
  }

  stateHistory.past.push(cloneState(stateHistory.present));

  stateHistory.present = cloneState(newState);

  stateHistory.future = [];
}
```

---

## Why This Matters

```text
Without cloning → all history changes together ❌
With cloning → true snapshots ✅
```

---

# 5. Undo Operation (Deep)

## Concept

```text
Move present → future
Restore last past → present
```

---

## Implementation

```js
function undo() {

  if (stateHistory.past.length === 0) return;

  const previous = stateHistory.past.pop();

  stateHistory.future.push(cloneState(stateHistory.present));

  stateHistory.present = previous;

  renderProducts();
}
```

---

## Flow

```text
Past: [A, B]
Present: C

Undo →

Past: [A]
Present: B
Future: [C]
```

---

# 6. Redo Operation (Deep)

## Concept

```text
Move present → past
Restore future → present
```

---

## Implementation

```js
function redo() {

  if (stateHistory.future.length === 0) return;

  const next = stateHistory.future.pop();

  stateHistory.past.push(cloneState(stateHistory.present));

  stateHistory.present = next;

  renderProducts();
}
```

---

## Flow

```text
Past: [A]
Present: B
Future: [C]

Redo →

Past: [A, B]
Present: C
Future: []
```

# 8. Preventing Duplicate States

## Problem

```text
Same state saved multiple times ❌
```

---

## Solution

```js
if (JSON.stringify(oldState) === JSON.stringify(newState)) {
  return;
}
```

---

# 9. Limiting History Size

## Problem

Unlimited history = memory issues.

---

## Solution

```js
const MAX_HISTORY = 20;

if (stateHistory.past.length > MAX_HISTORY) {
  stateHistory.past.shift();
}
```

# 10. Edge Cases

* Undo with no history
* Redo with empty future
* Repeated undo clicks
* Large state objects
* Rapid updates

---

# 11. Common Mistakes (Deep)

❌ Not cloning state
❌ Mutating snapshots
❌ Not clearing future
❌ Saving duplicate states
❌ Forgetting render

---

# 12. Best Practices

* Always clone state
* Avoid storing references
* Limit history size
* Avoid duplicates
* Separate history logic
