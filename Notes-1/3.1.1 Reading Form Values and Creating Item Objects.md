# 8. Creating Items with Plain JavaScript

## 8.1 What You Will Learn

Create is the CRUD operation that adds new data.

In a browser application, Create usually begins with a form.

The flow is:

```text
User enters values
        ↓
User submits the form
        ↓
Submit event handler runs
        ↓
JavaScript reads input values
        ↓
Values are cleaned and converted
        ↓
A new object is created
        ↓
The object is added to state
        ↓
The form is cleared
        ↓
The updated state is rendered
```

This lesson focuses on reading form values and creating an object.

Detailed validation will be covered in the next subtopic.

---

# 8.2 Basic HTML Form Structure

A product form may contain:

```html
<form id="product-form">
  <label for="product-name">
    Product Name
  </label>

  <input
    id="product-name"
    name="productName"
    type="text"
  />

  <label for="product-price">
    Price
  </label>

  <input
    id="product-price"
    name="productPrice"
    type="number"
  />

  <label for="product-quantity">
    Quantity
  </label>

  <input
    id="product-quantity"
    name="productQuantity"
    type="number"
  />

  <button type="submit">
    Add Product
  </button>
</form>
```

The form groups related inputs.

The submit button triggers the form's `submit` event.

---

# 8.3 Selecting the Form and Inputs

```js
const productForm =
  document.getElementById("product-form");

const productNameInput =
  document.getElementById("product-name");

const productPriceInput =
  document.getElementById("product-price");

const productQuantityInput =
  document.getElementById(
    "product-quantity"
  );
```

Each variable stores a reference to a DOM element.

At this point, you have selected the inputs, but you have not yet read their current values.

---

# 8.4 Listening for Form Submission

```js
productForm.addEventListener(
  "submit",
  function (event) {
    console.log("Form submitted");
  }
);
```

The callback runs when:

* The user clicks the submit button.
* The user presses Enter inside an eligible form input.

Listening to the form's `submit` event is more complete than listening only to the button's `click` event.

---

# 8.5 Preventing the Default Submission

HTML forms normally reload or navigate the page when submitted.

Use:

```js
event.preventDefault();
```

Complete example:

```js
productForm.addEventListener(
  "submit",
  function (event) {
    event.preventDefault();

    console.log("JavaScript is handling the form.");
  }
);
```

Without `preventDefault()`, your temporary JavaScript state may appear to disappear when the page reloads.

---

# 8.6 Reading Text Input Values

HTML:

```html
<input
  id="product-name"
  type="text"
/>
```

JavaScript:

```js
const productName =
  productNameInput.value;
```

If the user entered:

```text
Laptop
```

then:

```js
productName
```

contains:

```text
Laptop
```

Use `.trim()` to remove unnecessary spaces:

```js
const productName =
  productNameInput.value.trim();
```

Input:

```text
"   Laptop   "
```

Result:

```text
"Laptop"
```

---

# 8.7 Input Values Are Usually Strings

Even number inputs commonly return string values through `.value`.

HTML:

```html
<input
  id="product-price"
  type="number"
  value="50000"
/>
```

JavaScript:

```js
console.log(productPriceInput.value);
console.log(typeof productPriceInput.value);
```

Expected output:

```text
50000
string
```

This is important because your application probably needs a numeric price.

---

# 8.8 Converting Input Values into Numbers

Use `Number()`:

```js
const productPrice =
  Number(productPriceInput.value);

const productQuantity =
  Number(productQuantityInput.value);
```

Example:

```js
console.log(typeof productPrice);
```

Expected output:

```text
number
```

You can also use:

```js
productPriceInput.valueAsNumber
```

Example:

```js
const productPrice =
  productPriceInput.valueAsNumber;
```

However, an empty number input may produce `NaN`.

For beginner clarity, this course will frequently use:

```js
Number(input.value)
```

and validate the result explicitly.

---

# 8.9 Reading Checkbox Values

HTML:

```html
<label>
  <input
    id="product-active"
    type="checkbox"
  />

  Active product
</label>
```

JavaScript:

```js
const productActiveInput =
  document.getElementById("product-active");

const isActive =
  productActiveInput.checked;
```

Expected values:

```text
true
```

or:

```text
false
```

Use `.checked`, not `.value`, when you need the checkbox's selected state.

---

# 8.10 Reading Select Values

HTML:

```html
<select id="product-category">
  <option value="computers">
    Computers
  </option>

  <option value="phones">
    Phones
  </option>

  <option value="accessories">
    Accessories
  </option>
</select>
```

JavaScript:

```js
const categoryInput =
  document.getElementById(
    "product-category"
  );

const category =
  categoryInput.value;
```

If the user selected Phones:

```text
phones
```

is returned.

---

# 8.11 Creating an Object from Form Data

After reading the values:

```js
const productName =
  productNameInput.value.trim();

const productPrice =
  Number(productPriceInput.value);

const productQuantity =
  Number(productQuantityInput.value);
```

Create the object:

```js
const newProduct = {
  id: 1,
  name: productName,
  price: productPrice,
  quantity: productQuantity,
};
```

This object now groups all values for one product.

---

# 8.12 Object Property Shorthand

When the property name and variable name match, JavaScript allows shorthand.

Long form:

```js
const newProduct = {
  name: productName,
  price: productPrice,
  quantity: productQuantity,
};
```

Shorthand only works when names match exactly:

```js
const name = productName;
const price = productPrice;
const quantity = productQuantity;

const newProduct = {
  name,
  price,
  quantity,
};
```

In many form examples, descriptive input variables remain clearer:

```js
const newProduct = {
  id: nextProductId,
  name: productName,
  price: productPrice,
  quantity: productQuantity,
};
```

---

# 8.13 Reading Form Data Is Not Yet Create

This code only reads values:

```js
const productName =
  productNameInput.value.trim();
```

This code only creates an object in memory:

```js
const newProduct = {
  id: 1,
  name: productName,
};
```

The application state changes only when the object is added to the state array:

```js
products.push(newProduct);
```

The complete Create operation is:

```js
const newProduct = {
  id: 1,
  name: productName,
};

products.push(newProduct);
```

---

# 8.14 Adding an Object with `push()`

Starting state:

```js
let products = [];
```

New object:

```js
const newProduct = {
  id: 1,
  name: "Laptop",
  price: 50000,
  quantity: 2,
};
```

Add it:

```js
products.push(newProduct);
```

Final state:

```js
[
  {
    id: 1,
    name: "Laptop",
    price: 50000,
    quantity: 2,
  },
]
```

`push()` changes the original array.

This is a mutable Create operation.

---

# 8.15 Adding an Object with the Spread Operator

You can also create a new array:

```js
products = [
  ...products,
  newProduct,
];
```

This is an immutable-style Create operation.

Starting state:

```js
[
  {
    id: 1,
    name: "Laptop",
  },
]
```

New item:

```js
{
  id: 2,
  name: "Phone"
}
```

Updated state:

```js
[
  {
    id: 1,
    name: "Laptop",
  },
  {
    id: 2,
    name: "Phone",
  },
]
```

Both approaches add an item.

## Mutable

```js
products.push(newProduct);
```

## Immutable

```js
products = [
  ...products,
  newProduct,
];
```

The plain-loop application will initially use `push()` because it makes the state change easy to observe.

---

# 8.16 Keeping State Outside the Event Handler

Correct:

```js
let products = [];

productForm.addEventListener(
  "submit",
  function (event) {
    event.preventDefault();

    products.push(newProduct);
  }
);
```

Incorrect:

```js
productForm.addEventListener(
  "submit",
  function (event) {
    event.preventDefault();

    let products = [];

    products.push(newProduct);
  }
);
```

The incorrect version creates a new empty array every time the form is submitted.

Previously created products are lost.

---

# 8.17 Clearing Form Values

After successful creation:

```js
productNameInput.value = "";
productPriceInput.value = "";
productQuantityInput.value = "";
```

For a checkbox:

```js
productActiveInput.checked = false;
```

For a select:

```js
categoryInput.value = "";
```

You can also reset the whole form:

```js
productForm.reset();
```

`reset()` restores the inputs to their initial HTML values.

---

# 8.18 Focusing the First Input

After clearing the form:

```js
productNameInput.focus();
```

This allows the user to add another product without clicking the input again.

Typical order:

```js
productForm.reset();
productNameInput.focus();
```

---

# 8.19 Re-Rendering After Creation

After state changes:

```js
products.push(newProduct);
```

call:

```js
renderProducts();
```

Correct flow:

```js
products.push(newProduct);

productForm.reset();

renderProducts();
```

The array changes first.

The DOM is then rebuilt from the updated state.

---

# 8.20 Generating IDs Temporarily

A simple beginner approach:

```js
let nextProductId = 1;
```

During creation:

```js
const newProduct = {
  id: nextProductId,
  name: productName,
  price: productPrice,
  quantity: productQuantity,
};
```

Then increment:

```js
nextProductId++;
```

After creating three products, IDs become:

```text
1
2
3
```

ID generation will be covered in more detail in a later subtopic.

# 8.21 The Complete Create Flow

```js
let products = [];
let nextProductId = 1;

productForm.addEventListener(
  "submit",
  function (event) {
    event.preventDefault();

    const productName =
      productNameInput.value.trim();

    const productPrice =
      Number(productPriceInput.value);

    const productQuantity =
      Number(productQuantityInput.value);

    const newProduct = {
      id: nextProductId,
      name: productName,
      price: productPrice,
      quantity: productQuantity,
    };

    products.push(newProduct);

    nextProductId++;

    productForm.reset();

    renderProducts();

    productNameInput.focus();
  }
);
```

This code does not yet contain complete validation.

That comes next.

# 8.22 Syntax Breakdown

Consider:

```js
const productName =
  productNameInput.value.trim();
```

## `productNameInput`

A reference to the text input.

## `.value`

Reads the current entered value.

## `.trim()`

Removes whitespace from both ends.

## `productName`

Stores the cleaned string.

---

Consider:

```js
const productPrice =
  Number(productPriceInput.value);
```

## `productPriceInput.value`

Returns the input value as a string.

## `Number()`

Attempts to convert the string into a number.

## Returned result

A number or `NaN` when conversion fails.

---

Consider:

```js
const newProduct = {
  id: nextProductId,
  name: productName,
  price: productPrice,
  quantity: productQuantity,
};
```

## `newProduct`

A variable containing one complete product object.

## `id`

A unique identifier.

## `name`

The cleaned product name.

## `price`

The converted numeric price.

## `quantity`

The converted numeric quantity.

---

Consider:

```js
products.push(newProduct);
```

## `products`

The main state array.

## `push()`

Adds the object to the end of the array.

## Return value

`push()` returns the new array length.

Example:

```js
const newLength =
  products.push(newProduct);

console.log(newLength);
```

If the array had two products, the output is:

```text
3
```

# 8.23 Three Complete Examples

## Example 1: Read One Form Value

### Problem

Read a shopping item name from a form and create an object.

Do not add it to a persistent list yet.

### Input HTML

```html
<form id="shopping-form">
  <input
    id="shopping-name"
    type="text"
  />

  <button type="submit">
    Create Object
  </button>
</form>
```

### Complete Code

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8" />

  <meta
    name="viewport"
    content="width=device-width, initial-scale=1.0"
  />

  <title>Create Shopping Item</title>
</head>

<body>
  <h1>Shopping Item</h1>

  <form id="shopping-form">
    <label for="shopping-name">
      Item Name
    </label>

    <input
      id="shopping-name"
      type="text"
      autocomplete="off"
    />

    <button type="submit">
      Create Object
    </button>
  </form>

  <p id="result"></p>

  <script>
    const shoppingForm =
      document.getElementById(
        "shopping-form"
      );

    const shoppingNameInput =
      document.getElementById(
        "shopping-name"
      );

    const result =
      document.getElementById("result");

    shoppingForm.addEventListener(
      "submit",
      function (event) {
        event.preventDefault();

        const itemName =
          shoppingNameInput.value.trim();

        const newItem = {
          id: 1,
          name: itemName,
          purchased: false,
        };

        console.log(newItem);

        result.textContent =
          `Created item: ${newItem.name}`;
      }
    );
  </script>
</body>
</html>
```

### Step-by-Step Explanation

#### Step 1: Select the form

```js
const shoppingForm =
  document.getElementById(
    "shopping-form"
  );
```

#### Step 2: Listen for submission

```js
shoppingForm.addEventListener(
  "submit",
  function (event) {
    // Handle submission.
  }
);
```

#### Step 3: Stop page reload

```js
event.preventDefault();
```

#### Step 4: Read and clean the input

```js
const itemName =
  shoppingNameInput.value.trim();
```

#### Step 5: Create the object

```js
const newItem = {
  id: 1,
  name: itemName,
  purchased: false,
};
```

#### Step 6: Display the result

```js
result.textContent =
  `Created item: ${newItem.name}`;
```

### Expected Console Output for `Milk`

```text
{
  id: 1,
  name: "Milk",
  purchased: false
}
```

### Expected DOM Output

```text
Created item: Milk
```

### Final State

No persistent items array was changed.

The object exists only inside the submit callback during that execution.

### Common Mistakes

#### Forgetting `.value`

```js
const itemName = shoppingNameInput;
```

This stores the element, not the entered text.

#### Forgetting `preventDefault()`

The form may reload the page.

#### Assuming object creation automatically updates an array

No array is changed until `push()` is used.

### Small Modification Challenge

Add a checkbox for `purchased` and store its `.checked` value in the object.

## Example 3: Create and Render Products

### Problem

Create products from form values, store them in state, and render the current state.

### Complete Code

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8" />

  <meta
    name="viewport"
    content="width=device-width, initial-scale=1.0"
  />

  <title>Create and Render Products</title>

  <style>
    * {
      box-sizing: border-box;
    }

    body {
      max-width: 760px;
      margin: 0 auto;
      padding: 24px;
      font-family: Arial, sans-serif;
      line-height: 1.5;
    }

    form {
      display: grid;
      gap: 10px;
      margin-bottom: 24px;
    }

    input,
    button {
      padding: 10px;
    }

    #product-list {
      padding: 0;
      list-style: none;
    }

    .product-item {
      margin-bottom: 10px;
      padding: 12px;
      border: 1px solid #cccccc;
      border-radius: 6px;
    }
  </style>
</head>

<body>
  <h1>Product Inventory</h1>

  <form id="product-form">
    <label for="product-name">
      Product Name
    </label>

    <input
      id="product-name"
      type="text"
      autocomplete="off"
    />

    <label for="product-price">
      Price
    </label>

    <input
      id="product-price"
      type="number"
    />

    <label for="product-quantity">
      Quantity
    </label>

    <input
      id="product-quantity"
      type="number"
    />

    <button type="submit">
      Add Product
    </button>
  </form>

  <p id="product-count"></p>

  <ul id="product-list"></ul>

  <script>
    let products = [];
    let nextProductId = 1;

    const productForm =
      document.getElementById(
        "product-form"
      );

    const productNameInput =
      document.getElementById(
        "product-name"
      );

    const productPriceInput =
      document.getElementById(
        "product-price"
      );

    const productQuantityInput =
      document.getElementById(
        "product-quantity"
      );

    const productCount =
      document.getElementById(
        "product-count"
      );

    const productList =
      document.getElementById(
        "product-list"
      );

    function renderProducts() {
      productList.innerHTML = "";

      productCount.textContent =
        `Total products: ${products.length}`;

      if (products.length === 0) {
        const emptyMessage =
          document.createElement("li");

        emptyMessage.textContent =
          "No products available.";

        productList.appendChild(
          emptyMessage
        );

        return;
      }

      for (
        let index = 0;
        index < products.length;
        index++
      ) {
        const product = products[index];

        const listItem =
          document.createElement("li");

        listItem.classList.add(
          "product-item"
        );

        listItem.dataset.id = product.id;

        listItem.textContent =
          `${index + 1}. ${product.name} — ` +
          `KSh ${product.price.toLocaleString()} — ` +
          `Quantity: ${product.quantity}`;

        productList.appendChild(
          listItem
        );
      }
    }

    productForm.addEventListener(
      "submit",
      function (event) {
        event.preventDefault();

        const productName =
          productNameInput.value.trim();

        const productPrice =
          Number(productPriceInput.value);

        const productQuantity =
          Number(
            productQuantityInput.value
          );

        const newProduct = {
          id: nextProductId,
          name: productName,
          price: productPrice,
          quantity: productQuantity,
        };

        products.push(newProduct);

        nextProductId++;

        productForm.reset();

        renderProducts();

        productNameInput.focus();
      }
    );

    renderProducts();
  </script>
</body>
</html>
```

# 8.24 Mutable Versus Immutable Create

## Mutable Create

```js
products.push(newProduct);
```

### What Happens

* The existing array is changed.
* Its length increases.
* The array reference stays the same.

---

## Immutable Create

```js
products = [
  ...products,
  newProduct,
];
```

### What Happens

* A new array is created.
* Existing items are copied into it.
* The new item is added at the end.
* The array variable receives a new reference.

---

## Comparison

| Feature                              | `push()`          | Spread operator |
| ------------------------------------ | ----------------- | --------------- |
| Changes original array               | Yes               | No              |
| Creates a new array                  | No                | Yes             |
| Simple for beginners                 | Yes               | Fairly simple   |
| Common in plain JavaScript           | Yes               | Yes             |
| Common in state libraries/frameworks | Sometimes avoided | Very common     |

Both approaches are valid in plain JavaScript.

# 8.25 Common Beginner Mistakes

## Mistake 1: Reading the Element Instead of Its Value

Incorrect:

```js
const productName =
  productNameInput;
```

Correct:

```js
const productName =
  productNameInput.value;
```

---

## Mistake 2: Forgetting `.trim()`

Whitespace-only names may be accepted.

---

## Mistake 3: Keeping Numeric Input as a String

Incorrect:

```js
const price =
  productPriceInput.value;
```

Correct:

```js
const price =
  Number(productPriceInput.value);
```

---

## Mistake 4: Using `.value` for a Checkbox

Incorrect:

```js
const isActive =
  productActiveInput.value;
```

Correct:

```js
const isActive =
  productActiveInput.checked;
```

---

## Mistake 5: Forgetting `preventDefault()`

The page reloads.

---

## Mistake 6: Recreating State in the Handler

```js
let products = [];
```

inside the submission callback destroys earlier records.

---

## Mistake 7: Creating an Object but Not Adding It

```js
const newProduct = {
  id: 1,
  name: "Laptop",
};
```

The object is not in state until:

```js
products.push(newProduct);
```

---

## Mistake 8: Pushing Before the Object Is Complete

Ensure all properties are prepared before changing state.

---

## Mistake 9: Rendering Before State Changes

Correct order:

```js
products.push(newProduct);
renderProducts();
```

---

## Mistake 10: Clearing the Form Before Reading Values

Incorrect:

```js
productForm.reset();

const productName =
  productNameInput.value;
```

The value has already been cleared.

## Key Takeaways

* Create begins when a user submits new information.
* Form values are read from DOM inputs.
* Text input is accessed with `.value`.
* Text should usually be cleaned with `.trim()`.
* Input values are normally strings.
* Numeric values should be explicitly converted.
* Checkbox state is read with `.checked`.
* Select values are read with `.value`.
* A new object groups related form values.
* Creating an object alone does not update application state.
* `push()` adds the object to the state array.
* The spread operator can add items immutably.
* State must remain outside event handlers.
* The form should reset only after successful creation.
* The render function should run after state changes.
* The complete Create flow is:

```text
Submit
→ prevent default
→ read values
→ clean and convert
→ create object
→ add to state
→ reset form
→ render
```
