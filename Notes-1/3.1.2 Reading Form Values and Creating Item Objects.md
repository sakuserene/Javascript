# 8. Reading Form Values and Creating Item Objects

## 8.1 What You Will Learn

Create is the first operation in CRUD.

In a browser application, Create usually begins with a form.

Example form:

```html
<form id="product-form">
  <input id="product-name" type="text" />
  <input id="product-price" type="number" />
  <input id="product-quantity" type="number" />
  <button type="submit">Add Product</button>
</form>
```

The user enters information.

JavaScript reads the entered values and creates an object.

Example result:

```js
{
  id: 1,
  name: "Laptop",
  price: 50000,
  quantity: 2
}
```

That object is added to the main state array.

---

# 8.2 The Create Operation

Create means adding a new record to state.

Initial state:

```js
let products = [];
```

New object:

```js
const newProduct = {
  id: 1,
  name: "Laptop",
  price: 50000,
  quantity: 2,
};
```

Add to state:

```js
products.push(newProduct);
```

Final state:

```js
[
  {
    id: 1,
    name: "Laptop",
    price: 50000,
    quantity: 2,
  },
]
```

In a DOM application, this usually happens after a form is submitted.

---

# 8.3 Basic Product Form

```html
<form id="product-form">
  <div>
    <label for="product-name">
      Product Name
    </label>

    <input
      id="product-name"
      name="productName"
      type="text"
    />
  </div>

  <div>
    <label for="product-price">
      Price
    </label>

    <input
      id="product-price"
      name="productPrice"
      type="number"
    />
  </div>

  <div>
    <label for="product-quantity">
      Quantity
    </label>

    <input
      id="product-quantity"
      name="productQuantity"
      type="number"
    />
  </div>

  <button type="submit">
    Add Product
  </button>
</form>
```

Each input collects one property.

| Form Input   | Object Property |
| ------------ | --------------- |
| Product name | `name`          |
| Price        | `price`         |
| Quantity     | `quantity`      |

The ID will be generated by JavaScript.

# 8.4 Selecting the Form and Inputs

```js
const productForm =
  document.getElementById("product-form");

const productNameInput =
  document.getElementById("product-name");

const productPriceInput =
  document.getElementById("product-price");

const productQuantityInput =
  document.getElementById("product-quantity");
```

Each variable stores a reference to one DOM element.

These references allow JavaScript to:

* Read user input.
* Clear input values.
* Set focus.
* Add or remove classes.
* Disable or enable controls.

---

# 8.5 Listening for Form Submission

```js
productForm.addEventListener(
  "submit",
  function (event) {
    console.log("Form submitted");
  }
);
```

The callback runs when the form is submitted.

Submission can happen when the user:

* Clicks the submit button.
* Presses Enter in a form field.

Listening for the form's `submit` event is more reliable than listening only for the button's `click` event.

---

# 8.6 The Event Object

The callback receives an event object.

```js
productForm.addEventListener(
  "submit",
  function (event) {
    console.log(event.type);
  }
);
```

Expected output:

```text
submit
```

The event object describes what happened.

Useful properties and methods include:

```js
event.type;
event.target;
event.currentTarget;
event.preventDefault();
```

---

# 8.7 Why `preventDefault()` Is Required

A normal HTML form tries to submit to a server or reload the page.

In a JavaScript-managed CRUD application, that default behavior is usually prevented.

```js
productForm.addEventListener(
  "submit",
  function (event) {
    event.preventDefault();

    console.log("JavaScript is handling the form.");
  }
);
```

Without `preventDefault()`:

* The page may reload.
* Unsaved JavaScript state may disappear.
* Console output may disappear quickly.
* The user may be taken to another page.
* Your render logic may not complete.

---

# 8.8 Reading Text Input

HTML:

```html
<input
  id="product-name"
  type="text"
/>
```

JavaScript:

```js
const productName =
  productNameInput.value;
```

If the user enters:

```text
Laptop
```

then:

```js
productName === "Laptop";
```

Form input values are read using `.value`.

---

# 8.9 Trimming Text Input

A user may enter:

```text
"   Laptop   "
```

Use `.trim()`:

```js
const productName =
  productNameInput.value.trim();
```

Result:

```text
Laptop
```

This removes whitespace from the beginning and end.

It does not remove spaces inside the name.

```text
Gaming Laptop
```

remains:

```text
Gaming Laptop
```

---

# 8.10 Reading Number Inputs

HTML:

```html
<input
  id="product-price"
  type="number"
/>
```

JavaScript:

```js
const productPrice =
  productPriceInput.value;
```

Even though the input type is `number`, `.value` returns a string.

If the user enters:

```text
50000
```

then:

```js
typeof productPrice;
```

returns:

```text
string
```

This is important.

---

# 8.11 Converting Input Strings to Numbers

Use `Number()`:

```js
const productPrice =
  Number(productPriceInput.value);

const productQuantity =
  Number(productQuantityInput.value);
```

Now:

```js
typeof productPrice;
```

returns:

```text
number
```

The created object should store numeric values as numbers.

Correct:

```js
{
  price: 50000,
  quantity: 2
}
```

Less useful:

```js
{
  price: "50000",
  quantity: "2"
}
```

Numbers are easier to:

* Add.
* Compare.
* Multiply.
* Validate.
* Use in calculations.

---

# 8.12 Using `valueAsNumber`

Number inputs also support:

```js
const productPrice =
  productPriceInput.valueAsNumber;
```

This returns a number directly.

However, an empty number input may produce:

```text
NaN
```

For beginners, this course will often use:

```js
Number(input.value)
```

and validate the result carefully.

---

# 8.13 Reading Checkbox Values

HTML:

```html
<input
  id="product-active"
  type="checkbox"
/>
```

Use `.checked`:

```js
const isActive =
  productActiveInput.checked;
```

Result:

```js
true
```

or:

```js
false
```

Do not use `.value` to determine whether a checkbox is selected.

---

# 8.14 Reading Select Values

HTML:

```html
<select id="product-category">
  <option value="computers">
    Computers
  </option>

  <option value="phones">
    Phones
  </option>
</select>
```

JavaScript:

```js
const category =
  productCategorySelect.value;
```

Expected value:

```text
computers
```

or:

```text
phones
```

---

# 8.15 Creating an Object from Form Values

After reading the values:

```js
const productName =
  productNameInput.value.trim();

const productPrice =
  Number(productPriceInput.value);

const productQuantity =
  Number(productQuantityInput.value);
```

Create an object:

```js
const newProduct = {
  id: 1,
  name: productName,
  price: productPrice,
  quantity: productQuantity,
};
```

This object combines the form data into one structured record.

---

# 8.16 Property Shorthand

When the variable name matches the property name, JavaScript allows shorthand.

Long form:

```js
const newProduct = {
  name: productName,
  price: productPrice,
  quantity: productQuantity,
};
```

In this example, the names do not match exactly.

If the variables are named:

```js
const name = productNameInput.value.trim();
const price = Number(productPriceInput.value);
const quantity = Number(productQuantityInput.value);
```

then shorthand can be used:

```js
const newProduct = {
  id: 1,
  name,
  price,
  quantity,
};
```

Both versions are valid.

---

# 8.17 Generating an ID

A new object needs a unique ID.

For simple beginner applications, use a counter:

```js
let nextProductId = 1;
```

When creating:

```js
const newProduct = {
  id: nextProductId,
  name,
  price,
  quantity,
};
```

After creation:

```js
nextProductId++;
```

First product:

```text
ID 1
```

Second product:

```text
ID 2
```

Third product:

```text
ID 3
```

---

# 8.18 Why the ID Counter Must Be Outside the Handler

Correct:

```js
let nextProductId = 1;

productForm.addEventListener(
  "submit",
  function (event) {
    event.preventDefault();

    const newProduct = {
      id: nextProductId,
      name: "Laptop",
    };

    nextProductId++;
  }
);
```

Incorrect:

```js
productForm.addEventListener(
  "submit",
  function (event) {
    let nextProductId = 1;

    const newProduct = {
      id: nextProductId,
      name: "Laptop",
    };

    nextProductId++;
  }
);
```

In the incorrect version, every submission begins with:

```js
nextProductId = 1;
```

Every item receives the same ID.

---

# 8.19 Adding the Object to State

State:

```js
let products = [];
```

Create an object:

```js
const newProduct = {
  id: nextProductId,
  name,
  price,
  quantity,
};
```

Add it:

```js
products.push(newProduct);
```

`push()` mutates the existing array by adding one value to the end.

---

# 8.20 Complete Create Handler

```js
let products = [];
let nextProductId = 1;

productForm.addEventListener(
  "submit",
  function (event) {
    event.preventDefault();

    const name =
      productNameInput.value.trim();

    const price =
      Number(productPriceInput.value);

    const quantity =
      Number(productQuantityInput.value);

    const newProduct = {
      id: nextProductId,
      name,
      price,
      quantity,
    };

    products.push(newProduct);

    nextProductId++;

    renderProducts();
  }
);
```

This performs the Create operation.

Detailed validation will be added next.

---

# 8.21 Clearing the Form

After successful creation:

```js
productNameInput.value = "";
productPriceInput.value = "";
productQuantityInput.value = "";
```

This prepares the form for the next product.

You can also use:

```js
productForm.reset();
```

`reset()` restores each field to its original HTML value.

If fields have no initial value, they become empty.

---

# 8.22 Returning Focus to the First Input

After creation:

```js
productNameInput.focus();
```

This allows the user to begin typing the next item immediately.

A good successful Create flow is:

```js
products.push(newProduct);
nextProductId++;
productForm.reset();
productNameInput.focus();
renderProducts();
```

---

# 8.23 Updating State Before Rendering

Correct:

```js
products.push(newProduct);

renderProducts();
```

Incorrect:

```js
renderProducts();

products.push(newProduct);
```

The incorrect version renders the old state.

The newly created product may not appear until a later render.

---

# 8.24 Do Not Recreate State Inside the Handler

Incorrect:

```js
productForm.addEventListener(
  "submit",
  function (event) {
    event.preventDefault();

    let products = [];

    products.push(newProduct);
  }
);
```

Every submission creates a new empty array.

Only the most recently added product remains available in that handler.

Correct:

```js
let products = [];

productForm.addEventListener(
  "submit",
  function (event) {
    event.preventDefault();

    products.push(newProduct);
  }
);
```

The state persists across submissions.

---

# 8.25 Mutable Addition with `push()`

```js
products.push(newProduct);
```

This modifies the existing array.

Before:

```js
[
  {
    id: 1,
    name: "Laptop",
  },
]
```

After:

```js
[
  {
    id: 1,
    name: "Laptop",
  },
  {
    id: 2,
    name: "Phone",
  },
]
```

This approach is straightforward and appropriate for plain JavaScript applications.

---

# 8.26 Immutable Addition with the Spread Operator

An alternative is:

```js
products = [
  ...products,
  newProduct,
];
```

This creates a new array.

Before:

```js
[
  {
    id: 1,
    name: "Laptop",
  },
]
```

After:

```js
[
  {
    id: 1,
    name: "Laptop",
  },
  {
    id: 2,
    name: "Phone",
  },
]
```

The result looks the same, but the array reference is different.

---

# 8.27 Mutable Versus Immutable Create

## Mutable

```js
products.push(newProduct);
```

### Characteristics

* Changes the existing array.
* Simple.
* Efficient.
* Common in small plain JavaScript applications.

## Immutable

```js
products = [
  ...products,
  newProduct,
];
```

### Characteristics

* Creates a new array.
* Keeps the old array reference unchanged.
* Common in state-management patterns and frameworks.
* Requires `let` because `products` is reassigned.

Both approaches add the item.

---

# 8.28 Why Object Data Types Matter

Suppose form values are not converted:

```js
const newProduct = {
  price: productPriceInput.value,
  quantity: productQuantityInput.value,
};
```

State:

```js
{
  price: "50000",
  quantity: "2"
}
```

Now:

```js
newProduct.quantity + 1;
```

may produce:

```text
"21"
```

because string concatenation occurs.

With numeric conversion:

```js
const quantity =
  Number(productQuantityInput.value);
```

then:

```js
quantity + 1;
```

produces:

```text
3
```

---

# 8.29 The Full Create Flow

```text
1. User enters form values
2. User submits form
3. Submit event occurs
4. preventDefault() stops page reload
5. JavaScript reads input values
6. Text is trimmed
7. Numeric strings are converted
8. A unique ID is assigned
9. A new object is created
10. The object is added to state
11. The ID counter is incremented
12. The form is reset
13. Input focus is restored
14. The render function runs
15. The DOM displays the updated state
```

# 8.30 Three Complete Examples

## Example 1: Create an Object Without the DOM

### Problem

Create a product object from separate values and add it to an array.

### Input Data

```js
const name = "Laptop";
const price = 50000;
const quantity = 2;
```

### Complete Code

```js
let products = [];

let nextProductId = 1;

const name = "Laptop";
const price = 50000;
const quantity = 2;

const newProduct = {
  id: nextProductId,
  name,
  price,
  quantity,
};

products.push(newProduct);

nextProductId++;

console.log(products);
console.log(nextProductId);
```

### Step-by-Step Explanation

#### Step 1: Create the state

```js
let products = [];
```

#### Step 2: Create the ID counter

```js
let nextProductId = 1;
```

#### Step 3: Create the product object

```js
const newProduct = {
  id: nextProductId,
  name,
  price,
  quantity,
};
```

#### Step 4: Add it to state

```js
products.push(newProduct);
```

#### Step 5: Prepare the next ID

```js
nextProductId++;
```

### Expected Console Output

```text
[
  {
    id: 1,
    name: "Laptop",
    price: 50000,
    quantity: 2
  }
]

2
```

### Final State

```js
[
  {
    id: 1,
    name: "Laptop",
    price: 50000,
    quantity: 2,
  },
]
```

### Common Mistakes

#### Incrementing before object creation

```js
nextProductId++;
```

before creating the object causes the first ID to be `2`.

#### Forgetting to add the object

Creating `newProduct` alone does not update state.

#### Declaring `products` with `const` and then reassigning it

`push()` works with a `const` array, but immutable reassignment does not.

### Small Modification Challenge

Add a second product named `"Phone"` with:

```js
price: 25000
quantity: 4
```

Ensure it receives ID `2`.

```html
const name =

const price =
  Number(productPriceInput.value);
```

```js
const newProduct = {
  id: nextProductId,
  name,
  price,
  quantity,
};
```

```js
products.push(newProduct);
```



```js
productNameInput.focus();
```

### Expected Input

```text
Name: Laptop

Quantity: 2
```

### Expected Console Output

```text
[
  {
    id: 1,
    name: "Laptop",
    price: 50000,
    quantity: 2
  }
]
```

### Final State

```js
[
  {
    id: 1,
    name: "Laptop",

  },
]
```

#### Forgetting `Number()`

Price and quantity remain strings.

The page reloads.

```html
    let nextProductId = 1;

    const productForm =
      document.getElementById("product-form");

    const productNameInput =
      document.getElementById("product-name");

    const productPriceInput =
      document.getElementById("product-price");

    const productQuantityInput =
      document.getElementById(
        "product-quantity"
      );

    const productCount =
      document.getElementById("product-count");

    const productList =
      document.getElementById("product-list");

    function renderProducts() {
      productList.innerHTML = "";

      productCount.textContent =
        `Total products: ${products.length}`;

      if (products.length === 0) {
        const emptyState =
          document.createElement("li");

        emptyState.classList.add(
          "empty-state"
        );

        emptyState.textContent =
          "No products available.";

        productList.appendChild(emptyState);

        return;
      }

      for (
        let index = 0;
        index < products.length;
        index++
      ) {
        const product = products[index];

        const productCard =
          document.createElement("li");

        const productName =
          document.createElement("h2");

        const productPrice =
          document.createElement("p");

        const productQuantity =
          document.createElement("p");

        productCard.classList.add(
          "product-card"
        );

        productCard.dataset.id =
          product.id;

        productName.textContent =
          `${index + 1}. ${product.name}`;

        productPrice.textContent =
          `Price: KSh ${product.price.toLocaleString()}`;

        productQuantity.textContent =
          `Quantity: ${product.quantity}`;

        productCard.append(
          productName,
          productPrice,
          productQuantity
        );

        productList.appendChild(
          productCard
        );
      }
    }

    productForm.addEventListener(
      "submit",
      function (event) {
        event.preventDefault();

        const name =
          productNameInput.value.trim();

        const price =
          Number(productPriceInput.value);

        const quantity =
          Number(
            productQuantityInput.value
          );

        const newProduct = {
          id: nextProductId,
          name,
          price,
          quantity,
        };

        products.push(newProduct);

        nextProductId++;

        productForm.reset();

        productNameInput.focus();

        renderProducts();
      }
    );

    renderProducts();
  </script>
</body>
</html>
```

# 8.31 Create Using `push()` Versus Spread

## Using `push()`

```js
products.push(newProduct);
```

State array reference:

```text
Same array
```

## Using Spread

```js
products = [
  ...products,
  newProduct,

```

State array reference:

```text
New array
```

Both result in the new product being part of the state.

For this plain JavaScript stage, `push()` will be used most often.

# 8.32 State and Form Data Are Different

Before submission, the input may contain:

```text
Laptop
```

But the state may still be:

```js
[]
```

Typing into the form does not automatically create a product.

Only after the submit handler reads the values and updates the array does the item become part of application state.

```text
Input DOM state
      ↓
Submit event
      ↓
JavaScript object
      ↓
Application state
```

The form and the application array are separate.

# 8.34 Common Beginner Mistakes

## Mistake 1: Reading an Element Instead of Its Value

Incorrect:

```js
const name = productNameInput;
```

Correct:

```js
const name = productNameInput.value;
```

---

## Mistake 2: Not Trimming Text

```js
const name = productNameInput.value;
```

A whitespace-only value may be accepted.

Better:

```js
const name =
  productNameInput.value.trim();
```

---

## Mistake 3: Treating Number Input as a Number Automatically

Incorrect assumption:

```js
const price =
  productPriceInput.value;
```

`price` is a string.

Convert it:

```js
const price =
  Number(productPriceInput.value);
```

---

## Mistake 4: Recreating State

Incorrect:

```js
productForm.addEventListener(
  "submit",
  function () {
    let products = [];
  }
);
```

---

## Mistake 5: Recreating the ID Counter

Incorrect:

```js
productForm.addEventListener(
  "submit",
  function () {
    let nextProductId = 1;
  }
);
```

---

## Mistake 6: Forgetting `preventDefault()`

The page reloads.

---

## Mistake 7: Resetting Too Early

Incorrect:

```js
productForm.reset();

const name =
  productNameInput.value;
```

The name is already empty.

---

## Mistake 8: Rendering Before State Changes

Incorrect:

```js
renderProducts();

products.push(newProduct);
```

---

## Mistake 9: Incrementing the ID Only Sometimes

Every successful creation must update the counter exactly once.

---

## Mistake 10: Using the Array Length as a Permanent ID

```js
id: products.length + 1
```

This may produce duplicate IDs after deletion.

A separate increasing counter is safer for this stage.

## Key Takeaways

* Create begins when the user submits form data.
* Forms should normally use the `submit` event.
* `preventDefault()` stops the normal page reload.
* Text inputs are read using `.value`.
* Text should usually be trimmed before storage.
* Form values are strings by default.
* Numeric values should be converted before being stored.
* Checkboxes are read using `.checked`.
* Form values are combined into an object.
* A unique ID identifies the new item.
* ID counters must persist outside event handlers.
* `push()` adds an object to the existing state array.
* The spread operator can add an item immutably.
* State should be updated before rendering.
* Form reset clears the UI but does not delete state.
* The render function should run after successful creation.
* The next lesson will add full validation before items are allowed into state.
