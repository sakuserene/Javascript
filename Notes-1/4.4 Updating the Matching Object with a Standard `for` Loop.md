# 15. Understanding Object Updates

Updating means changing the properties of an existing object.

Example:

Before:

```js
{
  id: 205,
  name: "Phone",
  quantity: 4,
}
```

After:

```js
{
  id: 205,
  name: "Smartphone",
  quantity: 6,
}
```

The object still represents the same product.

Only its properties changed.

---

# 15.1 What Actually Changes?

Only editable fields.

Usually:

```text
✔ name
✔ price
✔ quantity
✔ category
✔ active status
```

Normally **not**:

```text
✘ id
```

The ID is permanent.

---

# 15.2 The Update Flow

```text
User clicks Edit
        ↓
Form loads values
        ↓
User changes values
        ↓
User clicks Update
        ↓
Read form values
        ↓
Validate
        ↓
Find matching object
        ↓
Update object
        ↓
Exit edit mode
        ↓
Render updated state
```

# 15.3 Why Search Again?

You already know:

```js
editingProductId
```

But you still search again.

Why?

Because state may have changed.

Someone could have:

* deleted the item
* filtered state
* reloaded data

Always verify.

---

# 15.4 Mutable Update

Suppose:

```js
const product = {
  id: 205,
  name: "Phone",
};
```

Updating:

```js
product.name =
  "Smartphone";
```

The object itself changes.

This is called a **mutable update**.

---

# 15.5 Plain Loop Update

The standard solution:

```js
for (
  let index = 0;
  index < products.length;
  index++
) {
  if (
    products[index].id ===
    editingProductId
  ) {
    products[index].name =
      data.name;

    break;
  }
}
```

The loop stops after updating.

---

# 15.6 Why Break?

Suppose IDs are unique.

Once found:

```text
205
```

there is no need to continue searching.

Without:

```js
break;
```

the loop checks every remaining object.

That wastes time.

---

# 15.7 Never Change the ID

Incorrect:

```js
products[index].id =
  generateProductId();
```

Correct:

```js
products[index].id
```

remains unchanged.

IDs identify objects.

Changing IDs creates identity problems.

---

# 15.8 Updating Multiple Properties

```js
products[index].name =
  data.name;

products[index].price =
  data.price;

products[index].quantity =
  data.quantity;

products[index].category =
  data.category;

products[index].isActive =
  data.isActive;
```

Update only editable fields.

---

# 15.9 Detect Whether Update Happened

Create a flag.

```js
let wasUpdated =
  false;
```

Inside loop:

```js
wasUpdated = true;
```

After loop:

```js
if (!wasUpdated) {
  showError(
    "Product not found."
  );

  return;
}
```

This safely handles stale IDs.

---

# 15.10 Complete Update Function

```js
function updateProduct(
  data
) {
  let wasUpdated =
    false;

  for (
    let index = 0;
    index < products.length;
    index++
  ) {
    if (
      products[index].id ===
      editingProductId
    ) {
      products[index].name =
        data.name;

      products[index].price =
        data.price;

      products[index].quantity =
        data.quantity;

      products[index].category =
        data.category;

      products[index].isActive =
        data.isActive;

      wasUpdated =
        true;

      break;
    }
  }

  if (!wasUpdated) {
    showError(
      "Product not found."
    );

    return;
  }

  resetFormToCreateMode();

  renderProducts();
}
```

---

# 15.11 Why Reset After Updating?

Suppose:

```text
Phone
```

becomes

```text
Smartphone
```

If edit mode stays active:

```text
Update Product
```

the next submission may accidentally update it again.

Returning to create mode avoids confusion.

---

# 15.12 Update Only After Validation

Never do:

```js
updateProduct(data);
```

before validation.

Correct order:

```text
Read inputs

↓

Validate

↓

Update state
```

---

# 15.13 Prevent Empty Names

Before updating:

```js
if (
  data.name === ""
) {
  showError(
    "Name required."
  );

  return;
}
```

Never store invalid state.

---

# 15.14 Prevent Negative Prices

```js
if (
  data.price < 0
) {
  showError(
    "Invalid price."
  );

  return;
}
```

---

# 15.15 Prevent Negative Quantities

```js
if (
  data.quantity < 0
) {
  showError(
    "Invalid quantity."
  );

  return;
}
```

---

# 15.16 Prevent Duplicate Names

Reuse:

```js
productNameExists(
  data.name,
  editingProductId
);
```

If:

```js
true
```

stop the update.

---

# 15.17 Why Ignore the Current Product?

Suppose:

```text
Phone
```

remains

```text
Phone
```

That is not a duplicate.

Ignoring:

```js
editingProductId
```

allows this.

---

# 15.18 Update Only One Object

State:

```js
[
  { id: 1 },
  { id: 2 },
  { id: 3 },
]
```

Only:

```js
id === editingProductId
```

changes.

Everything else remains untouched.

---

# 15.19 Mutable Updates and References

Suppose:

```js
const selected =
  products[1];
```

Later:

```js
products[1].name =
  "Smartphone";
```

Now:

```js
selected.name
```

also becomes:

```text
Smartphone
```

Both variables reference the same object.

---

# 15.20 Object Identity

Even after updating:

```js
products[1]
```

is still the same object.

Only properties changed.

Identity is preserved.

# 15.21 Three Complete Examples

## Example 1 — Updating Without the DOM

### Problem

Update one product.

### Code

```js
const products = [
  {
    id: 1,
    name: "Laptop",
  },
  {
    id: 2,
    name: "Phone",
  },
];

const editingProductId =
  2;

for (
  let index = 0;
  index < products.length;
  index++
) {
  if (
    products[index].id ===
    editingProductId
  ) {
    products[index].name =
      "Smartphone";

    break;
  }
}

console.log(products);
```

### Expected Console Output

```text
[
  { id: 1, name: "Laptop" },
  { id: 2, name: "Smartphone" }
]
```

### Final State

```js
[
  {
    id: 1,
    name: "Laptop",
  },
  {
    id: 2,
    name: "Smartphone",
  },
]
```

### Common Mistakes

* Forgetting `break`.
* Comparing string ID to number.
* Updating every object.

### Challenge

Also update quantity.

## Example 2 — Updating Inventory

### Problem

Update product quantity.

### Code

```js
const products = [
  {
    id: 10,
    quantity: 2,
  },
  {
    id: 20,
    quantity: 5,
  },
];

const editingProductId =
  20;

const newQuantity =
  9;

let updated =
  false;

for (
  let index = 0;
  index < products.length;
  index++
) {
  if (
    products[index].id ===
    editingProductId
  ) {
    products[index].quantity =
      newQuantity;

    updated = true;

    break;
  }
}

console.log(updated);
console.log(products);
```

### Expected Output

```text
true

[
  { id:10, quantity:2 },
  { id:20, quantity:9 }
]
```

### Final State

Only one quantity changed.

# 15.22 Plain Loop vs Array Method Preview

Using Plain Loop

```js
for (
  let index = 0;
  index < products.length;
  index++
) {
  if (
    products[index].id ===
    editingProductId
  ) {
    products[index].name =
      data.name;

    break;
  }
}
```

Using Array Method

```js
products =
  products.map(
    (product) =>
      product.id ===
      editingProductId
        ? {
            ...product,
            name: data.name,
          }
        : product
  );
```

You will learn the second approach later.
