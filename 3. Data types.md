


# Lesson 3: Data Types

## Phase 1: Programming Fundamentals

## Learning Objectives

By the end of this lesson you will be able to:

- Explain what a data type is.
- Identify JavaScript's common primitive data types.
- Create and use numbers, strings, and booleans.
- Understand `null` and `undefined`.
- Use `typeof` to inspect a value's type.

---

## What Is a Data Type?

A **data type** tells JavaScript what kind of value it is working with.

Think of a warehouse with different labeled boxes:

- 📦 Numbers
- 📦 Text
- 📦 True/False values

The label helps JavaScript know how to work with the value.

---

## Numbers

```javascript
const age = 25;
const price = 19.99;

console.log(age);
console.log(price);
```

Numbers can be whole numbers or decimals.

---

## Strings

Strings are text wrapped in quotes.

```javascript
const name = "Alice";
const language = 'JavaScript';

console.log(name);
console.log(language);
```

---

## Booleans

A boolean has only two possible values.

```javascript
const isLoggedIn = true;
const hasPermission = false;
```

Use booleans to represent yes/no or on/off states.

---

## Undefined

A variable declared without a value is `undefined`.

```javascript
let score;

console.log(score);
```

---

## Null

`null` represents an intentional absence of a value.

```javascript
const middleName = null;
```

Unlike `undefined`, `null` is assigned deliberately.

---

## Using typeof

```javascript
console.log(typeof 42);          // "number"
console.log(typeof "Hello");     // "string"
console.log(typeof true);        // "boolean"
console.log(typeof undefined);   // "undefined"
console.log(typeof null);        // "object" (historical JavaScript quirk)
```

Remember: `typeof null` returning `"object"` is a long-standing language quirk.

---

## Common Beginner Mistakes

- Forgetting quotes around text.
- Assuming `"5"` and `5` are the same.
- Confusing `null` with `undefined`.

---

## Best Practices

- Use meaningful variable names.
- Store numbers as numbers, not text.
- Use `null` intentionally.
- Check types with `typeof` while learning.

---

## Mini Quiz

1. What is a data type?
2. Which type stores text?
3. Which type stores true/false?
4. What is the difference between `null` and `undefined`?
5. What does `typeof` do?

---

2. Create a string variable called `city`.
3. Create a boolean called `isStudent`.
4. Declare a variable without assigning a value and print it.
5. Print the type of all four variables using `typeof`.

---

## Real-world Challenge

Create variables for a product in an online store:

- Product name
- Price
- In stock (true/false)
- Description (or `null` if unavailable)

Print each value and its type.

---

## Summary

You learned:

- Data types classify values.
- Numbers store numeric values.
- Strings store text.
- Booleans store true/false.
- `undefined` means no value has been assigned.
- `null` intentionally represents no value.
- `typeof` helps inspect types.
