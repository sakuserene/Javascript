# Lesson 8: Expressions

## Phase 1: Programming Fundamentals

## Learning Objectives

By the end of this lesson you will be able to:

- Explain what an expression is.
- Distinguish expressions from statements.
- Predict how JavaScript evaluates expressions.
- Build expressions using values, variables, and operators.
- Improve expression readability.

---

# What Is an Expression?

An **expression** is any piece of JavaScript code that produces a value.

Examples:

```javascript
10 + 5
```

Produces:

```text
15
```

```javascript
"Hello"
```

Produces the string `"Hello"`.

```javascript
true
```

Produces the boolean `true`.

---

# Mental Model

Think of an expression as a **machine**:

```text
Input values
      │
      ▼
 Expression
      │
      ▼
 One resulting value
```

---

# Expressions vs Statements

An **expression** creates a value.

```javascript
5 + 5
```

A **statement** performs an action.

```javascript
const total = 5 + 5;
```

The expression is `5 + 5`.

The statement is the entire line.

---

# Examples

```javascript
const age = 20;

console.log(age + 5);
```

`age + 5` is an expression.

---

```javascript
const first = "Hello";
const second = "World";

console.log(first + " " + second);
```

Output:

```text
Hello World
```

---

# Operator Precedence

JavaScript follows mathematical precedence.

```javascript
2 + 3 * 4
```

Result:

```text
14
```

Use parentheses to make intent clear.

```javascript
(2 + 3) * 4
```

Result:

```text
20
```

---

# Common Beginner Mistakes

- Forgetting parentheses.
- Confusing expressions with statements.
- Creating expressions without using the result.
- Writing overly complex expressions.

---

# Best Practices

- Keep expressions simple.
- Use parentheses for readability.
- Break large expressions into variables.
- Predict the result before running code.

---

# Mini Quiz

1. What is an expression?
2. What is the difference between an expression and a statement?
3. What value does `6 * 2` produce?
4. Why use parentheses?
5. Is `true && false` an expression?

---

# Practice Mode

## 10 Practice Questions

1. Is `100` an expression?
2. Is `"JavaScript"` an expression?
3. What is `8 + 2 * 5`?
4. What is `(8 + 2) * 5`?
5. Predict `"Hi" + "!"`.
6. Predict `10 / 2 + 3`.
7. Predict `5 > 3`.
8. Is `const x = 5;` an expression or statement?
9. Why keep expressions readable?
10. Give one example of an expression.

---

## 5 Coding Exercises

1. Print the result of `20 + 30`.
2. Create a variable and use it in an expression.
3. Use parentheses to change an arithmetic result.
4. Concatenate two strings with an expression.
5. Create a boolean expression that evaluates to `true`.

---

## Real-world Challenge

A bookstore sells books for 15 units each.

- Store the price in a variable.
- Store a quantity in another variable.
- Create an expression to calculate the total.
- Print the total.

---

# Summary

You learned:

- Expressions always produce a value.
- Statements perform actions.
- Expressions can use values, variables, and operators.
- Parentheses improve clarity and control evaluation order.
- Simple expressions are easier to read and debug.
