

# Lesson 4: Operators

## Phase 1: Programming Fundamentals

## Learning Objectives

By the end of this lesson, you will be able to:

- Explain what an operator is.
- Use arithmetic operators.
- Use assignment operators.
- Understand comparison operators.
- Understand logical operators.
- Predict the result of simple expressions.

---

# What Is an Operator?

An **operator** tells JavaScript to perform an action on one or more values.

Example:

```javascript
5 + 3
```

Here, `+` is the operator.

Think of operators as the **verbs** in a sentence—they tell JavaScript what to do.

---

# Arithmetic Operators

```javascript
10 + 5   // 15
10 - 5   // 5
10 * 5   // 50
10 / 5   // 2
10 % 3   // 1 (remainder)
2 ** 3   // 8 (power)
```

---

# Assignment Operators

```javascript
let score = 10;

score += 5; // 15
score -= 2; // 13
score *= 2; // 26
score /= 2; // 13
```

These are shortcuts for updating variables.

---

# Comparison Operators

Comparison operators return a boolean.

```javascript
5 === 5   // true
5 !== 4   // true
5 > 3     // true
5 < 3     // false
5 >= 5    // true
5 <= 4    // false
```

Prefer **strict equality (`===`)** and **strict inequality (`!==`)** because they avoid automatic type conversion.

---

# Logical Operators

```javascript
true && true   // true
true && false  // false

true || false  // true
false || false // false

!true          // false
!false         // true
```

Mental model:

- `&&` = AND (both must be true)
- `||` = OR (at least one must be true)
- `!` = NOT (reverse the value)

---

# Expressions

An **expression** is code that produces a value.

```javascript
5 + 5
```

produces:

```text
10
```

---

# Common Beginner Mistakes

- Confusing `=` with `===`.
- Forgetting `%` returns the remainder.
- Using `==` instead of `===`.
- Mixing strings and numbers unexpectedly.

---

# Best Practices

- Prefer `===` and `!==`.
- Use parentheses to make expressions clear.
- Keep expressions simple and readable.

---

# Mini Quiz

1. What is an operator?
2. What does `%` return?
3. What is the difference between `=` and `===`?
4. What does `&&` mean?
5. Why is `===` preferred over `==`?

---

# Practice Mode

## 10 Practice Questions

1. What is `8 + 4`?
2. What is `8 - 4`?
3. What is `8 * 4`?
4. What is `8 / 4`?
5. What is `8 % 3`?
6. Predict `5 > 8`.
7. Predict `5 === "5"`.
8. Predict `5 !== 6`.
9. Predict `true || false`.
10. Predict `!true`.

---

## 5 Coding Exercises

1. Print the result of `12 + 8`.
2. Create `let score = 20;` then add 10 using `+=`.
3. Compare `10` and `20` using `>`.
4. Compare `"10"` and `10` using `===`.
5. Print the result of `true && false`.

---

## Real-world Challenge

A game awards 100 points.

- Store the score.
- Add a 50-point bonus.
- Check whether the score is at least 120.
- Print the final score and the comparison result.

---

# Summary

You learned:

- Operators perform actions.
- Arithmetic operators perform calculations.
- Assignment operators update variables.
- Comparison operators return booleans.
- Logical operators combine boolean values.
- Expressions produce values.
