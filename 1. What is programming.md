## 1. What Is Programming?

Programming is the process of giving a computer a precise set of instructions to solve a problem or complete a task.

A computer is powerful, but it does not independently understand what you want. It follows instructions.

Examples of tasks completed by programs include:

- Calculating the total price in a shopping cart
- Displaying a message on a website
- Checking whether a password is correct
- Playing a video
- Sending a chat message
- Finding the fastest route to a destination
- Saving a user's notes
- Processing a payment

A **program** is a collection of instructions written for a computer.

A **programmer** is a person who designs, writes, tests, and improves those instructions.

**Programming** is the complete problem-solving process used to create the program.

Programming is not mainly about memorizing syntax. It is mainly about:

1. Understanding a problem
2. Breaking it into smaller steps
3. Expressing those steps precisely
4. Testing the result
5. Fixing mistakes
6. Improving the solution

## 2. A Real-World Analogy: A Cooking Recipe

Imagine that you are giving someone instructions for making tea.

A vague instruction would be:

> Make some tea.

A more useful sequence would be:

1. Fill a kettle with water.
2. Heat the water until it boils.
3. Put a tea bag in a cup.
4. Pour the hot water into the cup.
5. Wait three minutes.
6. Remove the tea bag.
7. Serve the tea.

This sequence is similar to a program.

The person following the recipe is like the computer. The recipe writer is like the programmer. The recipe is like the code.

A computer needs instructions to be:

- Ordered
- Clear
- Specific
- Complete
- Unambiguous

For example, the instruction â€œwait a littleâ€ is unclear. The instruction â€œwait three minutesâ€ is more precise.

This precision is one of the most important habits in programming.

## 4. What Is an Algorithm?

An **algorithm** is a clear, ordered set of steps for solving a problem.

An algorithm does not have to be written in a programming language.

Example problem:

> Determine whether a person is old enough to enter an event.

Possible algorithm:

1. Ask for the person's age.
2. Compare the age with the minimum required age.
3. If the age is high enough, allow entry.
4. Otherwise, deny entry.

Later, you will convert algorithms like this into JavaScript.

A strong programmer usually thinks about the steps before writing code.

### Algorithm Example: Find the Larger Number

Problem:

> Given two numbers, determine which one is larger.

Algorithm:

1. Receive the first number.
2. Receive the second number.
3. Compare the numbers.
4. Report the larger number.
5. If they are equal, report that they are equal.

You do not yet need to know the JavaScript syntax for this algorithm. The goal is to practice thinking in steps.

## 5. What Is Code?

**Code** is a set of instructions written in a programming language.

Here is a JavaScript instruction:

```javascript
console.log("Hello, world!");
```

This tells JavaScript to display the text:

```text
Hello, world!
```

### Syntax Explanation

Programming languages have grammar rules called **syntax**.

In this code:

```javascript
console.log("Hello, world!");
```

- `console` gives us access to the JavaScript console.
- `.` accesses something belonging to `console`.
- `log` is an operation that displays a value.
- `(` and `)` contain the value given to `log`.
- `"Hello, world!"` is text.
- `;` marks the end of the instruction.

JavaScript often works without the semicolon in simple cases, but using semicolons consistently can make instruction boundaries clearer while you are learning.

You are not expected to memorize every part yet. For now, read the line as:

> Display the text â€œHello, world!â€ in the console.

## 6. Source Code and Machine Instructions

Humans usually write **source code** using a programming language such as JavaScript.

Computers ultimately operate using low-level machine instructions.

A JavaScript engine translates and executes JavaScript code so that the computer can perform the requested actions.

Popular JavaScript engines include engines used inside web browsers and server environments. You do not need to memorize their names in this lesson.

Use this model:

```text
You write JavaScript
        â†“

        â†“

```

JavaScript is a **high-level programming language**. This means its syntax is designed to be much easier for humans to read than raw machine instructions.

### Example 5: Order Matters

```javascript
console.log("Put on your shoes");

console.log("Walk outside");
```

## 10. Programming Is Problem Solving

Consider this task:

> Calculate the total cost of three items.

A beginner may immediately try to write code.

A programmer should first clarify the problem:

- What are the prices?
- Are taxes included?
- Are discounts possible?
- Should the answer be rounded?
- In what currency should the result appear?

Then the programmer can break the task into steps:

1. Get the first price.
2. Get the second price.
3. Get the third price.
4. Add them together.
5. Display the total.

A future JavaScript version could look like this:

```javascript
console.log(12 + 8 + 5);
```

Output:

```text
25
```

The code is short because the problem has already been reduced to a clear sequence.

## 11. Bugs and Debugging

A **bug** is a mistake or defect in a program.

**Debugging** is the process of finding, understanding, and fixing bugs.

There are several common categories of mistakes.

### Syntax Error

The code breaks a JavaScript grammar rule.

```javascript
console.log("Hello";
```

The closing parenthesis is missing.

Correct version:

```javascript
console.log("Hello");
```

### Logic Error

The code runs, but it produces the wrong result.

```javascript
console.log(10 - 5);
```

If the goal was to add the numbers, the program is logically wrong even though the syntax is valid.

Correct version:

```javascript
console.log(10 + 5);
```

### Runtime Error

The program starts running but encounters a problem while executing.

You will study runtime errors in more detail later.

Errors are a normal part of programming. Professional developers encounter and fix errors every day.

A useful debugging attitude is:

```text
Expected result
      versus
Actual result
      â†“
Find the first point where they differ
```

## 12. Common Beginner Mistakes

### Mistake 1: Trying to Memorize Everything

You do not need to memorize the entire language.

Focus on:

- Understanding concepts
- Reading code
- Practicing frequently
- Knowing how to investigate problems

### Mistake 2: Writing Code Before Understanding the Problem

First describe the problem in ordinary language. Then list the steps. Then write code.

### Mistake 3: Assuming the Computer Knows What You Intended

The computer follows what you wrote, not what you meant.

### Mistake 4: Ignoring Error Messages

Error messages contain useful clues. Read them slowly.

### Mistake 5: Copying Code Without Predicting Its Behavior

Before running code, ask:

- What will happen?
- What will be displayed?
- In what order will the instructions run?

### Mistake 6: Being Afraid of Mistakes

Mistakes are feedback. A broken program gives you information about what you need to investigate.

### Mistake 7: Changing Many Things at Once

When debugging, make one small change and test again. This helps you identify what fixed or caused the problem.

## 13. Best Practices

At this stage, practice these habits:

1. Read code from top to bottom.
2. Predict the output before running a program.
3. Break large problems into small steps.
4. Use clear and exact language.
5. Test one small piece at a time.
6. Read error messages.
7. Type examples yourself instead of only copying them.
8. Explain your reasoning in words.
9. Keep your code neatly formatted.
10. Treat mistakes as part of the learning process.
