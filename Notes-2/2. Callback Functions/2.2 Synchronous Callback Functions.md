# Synchronous Callback Functions

In the previous lesson, we learned what a callback is.

We defined it as:

> **A callback is a function passed into another function so that the receiving function can execute it later.**

Notice something important.

The definition never said **when** "later" is.

Sometimes "later" means:

- immediately
- after one second
- after ten minutes
- after a network request
- after a user clicks a button

This lesson studies the first category.

## Synchronous Callbacks

These callbacks execute **immediately** while the current function is still running.

# What Does "Synchronous" Mean?

The word **synchronous** means:

> **One task starts only after the previous task has completely finished.**

Think of standing in a supermarket queue.

```
Customer 1

↓

Customer 2

↓

Customer 3

↓

Customer 4
```

Customer 4 cannot pay before Customer 1 finishes.

Everything happens **in order**.

JavaScript executes synchronous code the same way.

---

# JavaScript Is Single-Threaded

Before discussing callbacks, we need to understand an important property of JavaScript.

JavaScript executes code on a **single thread**.

This means:

- One instruction executes at a time.
- One function executes at a time.
- One statement executes at a time.

Imagine a worker with only one pair of hands.

```
Task A

↓

Finish

↓

Task B

↓

Finish

↓

Task C
```

There is no simultaneous execution on the main JavaScript thread.

# A Normal Function Call

Example

```javascript
function first() {

    console.log("First");

}

first();

console.log("Done");
```

Output

```text
First

Done
```

Execution

```
Call first()

↓

Print

↓

Return

↓

Print Done
```

Everything waits for the previous task.

# Now Add a Callback

```javascript
function greet() {

    console.log("Hello");

}

function execute(callback) {

    callback();

    console.log("Finished");

}

execute(greet);
```

Output

```text
Hello

Finished
```

Notice something.

The callback executes **before**

```javascript
console.log("Finished");
```

Why?

Because synchronous callbacks execute immediately.

# Why Use Synchronous Callbacks?

Suppose we have a reusable calculator.

```javascript
function calculate(a, b, operation) {

    return operation(a, b);

}
```

Different operations

```javascript
function add(a, b) {

    return a + b;

}

function subtract(a, b) {

    return a - b;

}

function multiply(a, b) {

    return a * b;

}
```

Calls

```javascript
calculate(10, 5, add);

calculate(10, 5, subtract);

calculate(10, 5, multiply);
```

The calculator never changes.

Only the callback changes.

This avoids duplicate code.

# Example 2 — Intermediate

Suppose we validate input.

```javascript
function processUser(user, validator) {

    if (validator(user)) {

        console.log("Accepted");

    } else {

        console.log("Rejected");

    }

}
```

Validator

```javascript
function isAdult(user) {

    return user.age >= 18;

}
```

Call

```javascript
processUser(

    {

        name: "Alice",

        age: 20

    },

    isAdult

);
```

Output

```text
Accepted
```

Notice

The reusable algorithm never changes.

Different validators can be supplied.

# Example 3 — Real-World Application

Imagine a Point of Sale system.

Every receipt must be formatted differently.

```javascript
function printReceipt(data, formatter) {

    const receipt = formatter(data);

    console.log(receipt);

}
```

Formatter

```javascript
function simpleFormatter(data) {

    return

`${data.receipt} - KES ${data.total}`;

}
```

Usage

```javascript
printReceipt(

    {

        receipt: "RCT-001",

        total: 500

    },

    simpleFormatter

);
```

Output

```text
RCT-001 - KES 500
```

Professional software constantly uses synchronous callbacks for configurable processing.

# Built-in JavaScript Examples

Many JavaScript methods use synchronous callbacks.

Example

```javascript
const numbers = [

    1,

    2,

    3

];

numbers.forEach(function(number) {

    console.log(number);

});
```

Output

```text
1

2

3
```

Notice

Each callback executes immediately.

The next iteration waits until the current callback finishes.

---

Another example

```javascript
const doubled = numbers.map(function(number) {

    return number * 2;

});
```

Output

```text
[2, 4, 6]
```

Again,

callbacks execute synchronously.

# Common Mistakes

## Mistake 1 — Assuming Every Callback Is Asynchronous

Many beginners think

```
Callback

↓

Means

↓

Asynchronous
```

This is incorrect.

Callbacks can be:

- synchronous
- asynchronous

The callback itself does not determine this.

The receiving function does.

---

## Mistake 2 — Executing Instead of Passing

Incorrect

```javascript
run(sayHello());
```

Correct

```javascript
run(sayHello);
```

Always pass the function itself.

---

## Mistake 3 — Forgetting the Return Value

Suppose

```javascript
function calculate(a, b, operation) {

    operation(a, b);

}
```

Nothing is returned.

Correct

```javascript
function calculate(a, b, operation) {

    return operation(a, b);

}
```

---

# Best Practices

- Keep synchronous callbacks short and focused.
- Give callback parameters descriptive names:

```javascript
callback

validator

formatter

transform

handler
```

- Avoid deeply nested callback chains when a simple loop or helper function improves readability.
- Return callback results when appropriate.

---

# Performance Considerations

Because synchronous callbacks execute immediately, they block the current execution until they finish.

Example

```javascript
numbers.forEach(number => {

    expensiveCalculation(number);

});
```

If `expensiveCalculation()` takes a long time,

the entire application waits.

Keep synchronous callbacks efficient.

# Interview Questions

## Beginner

**Q:** What is a synchronous callback?

**Answer:**

A callback that executes immediately while the current function is still running.

---

## Intermediate

**Q:** Are all callbacks asynchronous?

**Answer:**

No.

Callbacks can be synchronous or asynchronous depending on how the receiving function executes them.

---

## Advanced

**Q:** Why do methods like `map()` and `forEach()` use synchronous callbacks?

**Answer:**

Because they process data immediately during the current execution rather than scheduling work for later.
