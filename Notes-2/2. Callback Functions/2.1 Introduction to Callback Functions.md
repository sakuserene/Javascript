# What is a Callback Function?

Let's begin with a precise definition.

> **A callback function is a function that is passed into another function so that it can be executed later.**

Notice something important.

The callback itself **does not decide when it will execute.**

Instead,

the receiving function decides.

This is the biggest difference from ordinary function calls.

---

# Breaking Down the Definition

Suppose we write

```javascript
function greet() {

    console.log("Hello");

}
```

This is an ordinary function.

Nothing special yet.

Now consider

```javascript
function execute(task) {

    task();

}
```

Then

```javascript
execute(greet);
```

Question:

Which function actually executes first?

Many beginners answer:

```
greet()
```

Incorrect.

The execution order is

```
execute()

↓

task()

↓

greet()
```

The callback waits until the receiving function decides to execute it.

---

# Why Were Callback Functions Invented?

Imagine JavaScript without callbacks.

Suppose you want to write a reusable function.

Without callbacks

```javascript
function printGreeting() {

    console.log("Hello");

}
```

What if tomorrow you want it to print

```
Welcome
```

or

```
Goodbye
```

You would need three different functions.

```javascript
printHello();

printWelcome();

printGoodbye();
```

As software grows,

this approach quickly becomes impossible to maintain.

Instead,

JavaScript allows us to separate

```
Algorithm

↓

Behavior
```

The algorithm remains reusable.

The behavior changes through callbacks.

---

# Real-Life Analogy

Imagine ordering food at a restaurant.

You place an order.

The chef cooks.

When the meal is ready,

the waiter calls your name.

Notice

You don't constantly interrupt the chef asking:

```
Is it ready?

Is it ready?

Is it ready?
```

Instead,

you provide a way to notify you later.

A callback works similarly.

```
Main Function

↓

Receives Callback

↓

Performs Work

↓

Work Finished

↓

Execute Callback
```

The callback is simply the notification mechanism.

# Callback vs Ordinary Function Call

Ordinary call

```javascript
greet();
```

Execution

```text
Call greet()

↓

Immediately Execute

↓

Return
```

---

Callback

```javascript
execute(greet);
```

Execution

```text
Call execute()

↓

Store Callback

↓

Continue

↓

Later

↓

Execute Callback

↓

Return
```

Notice the delay.

The callback doesn't execute immediately.

It executes **when instructed**.

Consider

```javascript

    console.log("Hello");

}

    callback();

}

execute(greet);
```

Let's examine every step.

Creation Phase

↓

Execution Phase

JavaScript reaches

```

↓

↓

```

Notice

The parameter now points to exactly the same function object.

---

## Step 3

Execution

```javascript

```

becomes

```

Output

```

# Why Is It Called a Callback?

The name becomes easier to understand when we examine the flow.

Suppose

```javascript
execute(greet);
```

Think of it like this.

```
Main Function

↓

Do Your Work

↓

When Finished

↓

Call Back

↓

greet()
```

The receiving function literally

**calls back**

the supplied function.

Hence the name:

```
Callback
```

# Example 1 — Beginner

```javascript
function greet() {

    console.log("Hello!");

}

function run(callback) {

    callback();

}

run(greet);
```

Output

```text
Hello!
```

### Explanation

1. `greet` is created.
2. `run` is created.
3. `run(greet)` passes the function.
4. `callback()` executes `greet()`.
5. `"Hello!"` is printed.

---

# Example 2 — Intermediate

Suppose we want reusable mathematical operations.

```javascript
function calculate(a, b, operation) {

    return operation(a, b);

}

function add(a, b) {

    return a + b;

}

function multiply(a, b) {

    return a * b;

}
```

Calls

```javascript
console.log(
    calculate(10, 20, add)
);

console.log(
    calculate(10, 20, multiply)
);
```

Output

```text
30

200
```

Notice

The calculation algorithm never changes.

Only the callback changes.

# Example 3 — Real-World Application

Imagine a POS system.

Different stores require different receipt formats.

Instead of writing

```javascript
printCashReceipt();

printCardReceipt();

printMobileReceipt();
```

Create one reusable printer.

```javascript
function printReceipt(formatter) {

    formatter();

}
```

Different formatting functions

```javascript
function cashFormat() {

    console.log("Cash Receipt");

}

function cardFormat() {

    console.log("Card Receipt");

}
```

Usage

```javascript
printReceipt(cashFormat);

printReceipt(cardFormat);
```

Output

```text
Cash Receipt

Card Receipt
```

Professional software uses this pattern constantly.

# Common Mistakes

## Mistake 1 — Calling Instead of Passing

Incorrect

```javascript
run(greet());
```

Execution

```
greet()

↓

Returns undefined

↓

run(undefined)
```

This is **not** passing a callback.

Correct

```javascript
run(greet);
```

Pass the function itself.

---

## Mistake 2 — Thinking the Callback Executes Immediately

Callbacks execute **when the receiving function decides.**

Not necessarily when they are passed.

---

## Mistake 3 — Confusing the Callback with the Higher-Order Function

Remember

```
Higher-Order Function

↓

Receives Callback
```

They are different concepts.

---

# Best Practices

- Name callback parameters descriptively:

```javascript
callback

handler

listener

formatter

validator

transform
```

instead of

```javascript
x

fn

value
```

- Keep callback functions focused on one task.
- Pass functions, not function calls.
- Design Higher-Order Functions so they clearly control when callbacks execute.

---

# Performance Considerations

Passing a function as a callback does **not** copy its code.

Only a reference to the function object is passed.

This makes callbacks lightweight and efficient.

Modern JavaScript engines optimize callback invocation heavily.

---

# Key Takeaways

- A callback is a function passed to another function.
- The receiving function controls **when** the callback executes.
- Callbacks separate reusable algorithms from changing behavior.
- Passing a callback means passing a **function reference**, not executing the function.
- Callback functions are the foundation of JavaScript's event-driven and asynchronous programming model.
