# DOM Events --- Part 3 (Section 2)

## Understanding `target` vs `currentTarget`

### `event.target`

The **target** is the element where the event originally occurred.

### `event.currentTarget`

The **currentTarget** is the element whose listener is currently
executing.

When listeners are attached to parent elements, these two values may
differ.

## Example

``` html
<div id="parent">
  <button id="save">Save</button>
</div>
```

``` javascript
const parent = document.getElementById("parent");

parent.addEventListener("click", function (event) {
  console.log("target:", event.target.id);
  console.log("currentTarget:", event.currentTarget.id);
});
```

Clicking the button prints:

``` text
target: save
currentTarget: parent
```

## Other Important Properties

### `event.bubbles`

Returns `true` if the event bubbles through ancestor elements.

### `event.cancelable`

Indicates whether the event's default behaviour can be prevented.

### `event.defaultPrevented`

Becomes `true` after `event.preventDefault()` has been called.

### `event.isTrusted`

-   `true` → generated by a real user interaction.
-   `false` → generated programmatically (for example with
    `dispatchEvent()`).

## Debugging

``` javascript
button.addEventListener("click", (event) => {
  console.log(event);
});
```

Inspecting the Event Object in DevTools is one of the fastest ways to
understand browser behaviour.

## Best Practices

-   Use `target` to identify the originating element.
-   Use `currentTarget` to identify the element handling the listener.
-   Avoid assuming both properties are always identical.
