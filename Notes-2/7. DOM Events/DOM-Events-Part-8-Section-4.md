# DOM Events --- Part 8 (Section 4)

## Accessibility

-   Use custom events to communicate state changes, not to replace
    semantic HTML.
-   Ensure UI updates triggered by custom events remain accessible to
    keyboard and assistive technology users.
-   Announce important changes when appropriate.

## Security

-   Treat data in `event.detail` as untrusted if it originates from user
    input.
-   Validate and sanitize data before using it.
-   Avoid exposing sensitive information in event payloads.

## Interview Questions

**Q:** What is the difference between built-in events and custom events?

**A:** Built-in events are generated by the browser, while custom events
are created and dispatched by your application.

**Q:** Why is the `detail` property useful?

**A:** It allows developers to attach structured data to a custom event
so listeners receive the information they need.
