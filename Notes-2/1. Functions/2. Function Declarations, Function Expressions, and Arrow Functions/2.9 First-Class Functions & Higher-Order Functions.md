# Functions as First-Class Citizens

Up until now, we've learned **how to create functions**.

We have studied:

- Function Declarations
- Function Expressions
- Arrow Functions

However, we haven't yet answered one of JavaScript's biggest questions.

> **Why are functions so special?**

The answer is:

> **Because JavaScript treats functions as first-class citizens.**

This single feature is responsible for nearly every modern JavaScript pattern.

Without it, JavaScript would not have:

- Callback functions
- Event listeners
- Promises
- Async/Await
- React Components
- Express Middleware
- Functional Programming
- Array methods like `map()`, `filter()`, and `reduce()`

Understanding this lesson changes how you think about JavaScript.

---

# What Does "First-Class Citizen" Mean?

The term **first-class citizen** comes from programming language theory.

It describes any value that can be treated like every other value in the language.

In JavaScript, numbers are first-class values.

Example

```javascript
const age = 25;
```

The number can be:

- Stored
- Copied
- Assigned
- Passed
- Returned

Likewise,

Strings

```javascript
const name = "Alice";
```

can also be:

- Stored
- Assigned
- Passed
- Returned

JavaScript gives **functions** exactly the same freedom.

---

# Definition

A language has **first-class functions** if functions can be:

- Stored inside variables
- Stored inside arrays
- Stored inside objects
- Passed into functions
- Returned from functions
- Created dynamically
- Assigned to other variables

JavaScript satisfies all of these.

---

# Why Was This Introduced?

Imagine JavaScript without first-class functions.

Suppose you wanted a function to perform different operations.

Without first-class functions, you might write:

```javascript
if (operation === "add") {

}

if (operation === "subtract") {

}

if (operation === "multiply") {

}
```

As the application grows, this becomes difficult to maintain.

Instead,

JavaScript allows you to pass the desired behavior itself.

Instead of passing data,

you pass a function.

This is much more flexible.

---

# Functions Are Objects

This surprises many beginners.

Functions are not just executable code.

Internally,

they are special kinds of objects.

Example

```javascript
function greet() {

}
```

Conceptually

```
Memory

↓

Function Object

↓

Executable Code
```

Because functions are objects,

they can be stored just like any other object.

---

# Internal Engine Behaviour

Suppose

```javascript
function greet() {

    console.log("Hello");

}
```

During the Creation Phase,

JavaScript creates

```text
Function Object
```

Then

```text
greet

↓

Function Object
```

Notice

The variable

```
greet
```

does not contain the function itself.

It contains a **reference** to the function object.

This is exactly how objects work.

---

# Storing Functions in Variables

Example

```javascript
function greet() {

    console.log("Hello");

}

const sayHello = greet;
```

Memory

```text
greet

↓

Function Object

↑

sayHello
```

Both variables point to the same function.

Calling either

```javascript
greet();

sayHello();
```

produces

```text
Hello

Hello
```

---

# Functions Inside Arrays

Because functions are values,

they can be stored in arrays.

Example

```javascript
const operations = [

    function () {

        console.log("Add");

    },

    function () {

        console.log("Subtract");

    }

];
```

Calling

```javascript
operations[0]();
```

Output

```text
Add
```

Calling

```javascript
operations[1]();
```

Output

```text
Subtract
```

Notice

Arrays can contain:

- Numbers
- Strings
- Objects
- Functions

---

# Functions Inside Objects

Objects can also store functions.

Example

```javascript
const calculator = {

    add(a, b) {

        return a + b;

    },

    subtract(a, b) {

        return a - b;

    }

};
```

Call

```javascript
calculator.add(5, 3);
```

Output

```text
8
```

Functions stored inside objects are commonly called **methods**.

---

# Passing Functions as Arguments

This is where JavaScript becomes powerful.

Suppose

```javascript
function greet() {

    console.log("Hello");

}
```

Now

```javascript
function execute(fn) {

    fn();

}
```

Notice

```
fn
```

is a parameter.

But instead of receiving a number,

it receives a function.

Call

```javascript
execute(greet);
```

Output

```text
Hello
```

---

# Internal Behaviour

Memory

```text
greet

↓

Function Object
```

Call

```javascript
execute(greet);
```

Parameter

```
fn

↓

Function Object
```

Execution

```javascript
fn();
```

becomes

```javascript
greet();
```

---

# Mental Model

Imagine giving someone a remote control.

Instead of telling them

> "Turn on the television."

you simply hand them the remote.

The remote represents the behavior.

Passing functions works similarly.

Instead of passing instructions as text,

you pass executable behavior.

# Returning Functions

Functions can also return functions.

Example

```javascript
function createGreeting() {

    return function () {

        console.log("Hello");

    };

}
```

Call

```javascript
const greet = createGreeting();

greet();
```

Output

```text
Hello
```

Notice

The outer function created another function.

That function became the return value.

This concept will later lead us to:

- Closures
- Factory Functions
- Functional Programming

# Example 2 — Intermediate

```javascript

```

# Example 3 — Real-World Application

Suppose a POS system allows different payment methods.

```javascript
function processPayment(strategy) {

    strategy();

}

function cashPayment() {

    console.log("Cash Paid");

}

function cardPayment() {

    console.log("Card Paid");

}
```

Usage

```javascript
processPayment(cashPayment);

processPayment(cardPayment);
```

Output

```text
Cash Paid

Card Paid
```

Instead of writing dozens of `if` statements,

the behavior itself is supplied.

This is called the **Strategy Pattern**, a common software design pattern.

# What Is a Higher-Order Function?

Now we can define an important term.

A **Higher-Order Function** is a function that does **at least one** of the following:

1. Accepts one or more functions as arguments.
2. Returns a function.

If either condition is true,

it is a higher-order function.

---

# Examples

Accepts a function

```javascript
run(callback);
```

Returns a function

```javascript
createGreeting();
```

Both are higher-order functions.

---

# Why Are Higher-Order Functions Important?

Without them,

none of these would exist:

```javascript
setTimeout()

map()

filter()

reduce()

forEach()

sort()

Promise.then()

Promise.catch()

addEventListener()
```

Almost every modern JavaScript API depends on higher-order functions.

---

# Common Mistakes

## Mistake 1 — Calling Instead of Passing

Incorrect

```javascript
run(greet());
```

This calls the function immediately.

Correct

```javascript
run(greet);
```

You pass the function itself.

---

## Mistake 2 — Thinking Parentheses Are Optional Everywhere

Remember

```
greet
```

means

```
The function
```

while

```
greet()
```

means

```
Execute the function
```

These are completely different.

---

# Best Practices

- Treat functions as reusable pieces of behavior.
- Pass functions instead of writing repetitive conditional logic.
- Give higher-order functions descriptive parameter names like:
  - `callback`
  - `handler`
  - `listener`
  - `transform`
  - `predicate`
- Avoid unnecessary anonymous functions when a named function improves readability.

---

# Performance Considerations

Creating functions has a small memory cost, but modern JavaScript engines optimize function objects efficiently.

The flexibility and maintainability gained from first-class functions far outweigh the minimal overhead in most applications.

---

# Key Takeaways

- Functions are **first-class citizens** in JavaScript.
- Functions are special objects that can be treated like other values.
- Functions can be:
  - Stored
  - Passed
  - Returned
  - Assigned
- A **Higher-Order Function** either:
  - accepts a function, or
  - returns a function.
- First-class functions are the foundation for callbacks, promises, event listeners, and modern JavaScript programming.
