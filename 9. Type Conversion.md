# Lesson 9: Type Conversion

## Phase 1: Programming Fundamentals

## Learning Objectives

By the end of this lesson, you will be able to:

- Explain type conversion.
- Distinguish implicit and explicit conversion.
- Convert between strings, numbers, and booleans.
- Predict common conversion results.
- Avoid common type-conversion bugs.

---

# What Is Type Conversion?

**Type conversion** is changing a value from one data type to another.

Example:

```javascript
const age = Number("25");
console.log(age);
```

Output:

```text
25
```

The string `"25"` becomes the number `25`.

---

# Why Is It Important?

User input from `prompt()` is returned as a **string**.

```javascript
const a = prompt("First:");
const b = prompt("Second:");

console.log(a + b);
```

If the user enters `5` and `6`, the output is:

```text
56
```

To perform math:

```javascript
const a = Number(prompt("First:"));
const b = Number(prompt("Second:"));

console.log(a + b);
```

Output:

```text
11
```

---

# Explicit Conversion

You intentionally convert a value.

## Number()

```javascript
Number("42");      // 42
Number("3.14");    // 3.14
Number("hello");   // NaN
```

`NaN` means **Not a Number**.

## String()

```javascript
String(42);      // "42"
String(true);    // "true"
```

## Boolean()

```javascript
Boolean(1);      // true
Boolean(0);      // false
Boolean("");     // false
Boolean("Hi");   // true
```

---

# Implicit Conversion

Sometimes JavaScript converts values automatically.

```javascript
"5" + 1
```

Result:

```text
"51"
```

The number becomes text.

Another example:

```javascript
"5" * 2
```

Result:

```text
10
```

JavaScript converts `"5"` into a number for multiplication.

Automatic conversion can be confusing, so prefer explicit conversion.

---

# Truthy and Falsy

Values treated as `false` include:

- `false`
- `0`
- `""`
- `null`
- `undefined`
- `NaN`

Many other values are **truthy**, including:

- Non-empty strings
- Non-zero numbers
- Objects
- Arrays

---

# Common Beginner Mistakes

- Assuming `"5"` equals `5`.
- Forgetting `Number()` with user input.
- Ignoring `NaN`.
- Relying on automatic conversion.

---

# Best Practices

- Use `Number()`, `String()`, and `Boolean()` explicitly.
- Prefer `===` over `==`.
- Validate user input before using it.
- Check values with `typeof` when learning.

---

# Mini Quiz

1. What is type conversion?
2. What does `Number("8")` return?
3. What does `String(25)` return?
4. What does `Boolean(0)` return?
5. What does `NaN` mean?

---

# Practice Mode

## 10 Practice Questions

1. Convert `"15"` to a number.
2. Convert `100` to a string.
3. Convert `true` to a string.
4. Convert `""` to a boolean.
5. Convert `"Hello"` to a boolean.
6. Predict `"10" + 5`.
7. Predict `"10" - 5`.
8. Predict `Boolean(null)`.
9. Why is explicit conversion safer?
10. What does `typeof Number("9")` return?

---

## 5 Coding Exercises

1. Convert `"50"` into a number and print it.
2. Convert `99` into a string and print it.
3. Convert `0` into a boolean and print it.
4. Ask for a user's age with `prompt()`, convert it, and print it.
5. Print the type before and after converting `"42"` to a number.

---

## Real-world Challenge

Create a bill calculator.

Requirements:

- Ask for the price of two products.
- Convert both inputs to numbers.
- Calculate the total.
- Print the total.
- Print the type of the total using `typeof`.

---

# Summary

You learned:

- Type conversion changes one data type into another.
- Explicit conversion uses `Number()`, `String()`, and `Boolean()`.
- Implicit conversion happens automatically and can be surprising.
- `NaN` represents an invalid numeric conversion.
- Explicit conversion makes code clearer and more reliable.
