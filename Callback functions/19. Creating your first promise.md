# Module 19 — Creating Your First Promise
Congratulations!
You now know:
What a Promise is.
Why Promises were introduced.
The three Promise states.
Now it's time to create your own Promise.
This is where many beginners get confused, so we'll go slowly and explain every single line.
Lesson Objectives
By the end of this lesson, you will understand:
How to create a Promise.
What new Promise() does.
What resolve() is.
What reject() is.
Why Promises start in the Pending state.
How a Promise changes state.
The Syntax
A Promise is created using the Promise constructor.
const promise = new Promise(function (resolve, reject) {

});
Let's break this down.
Understanding new Promise()
Look at this code.
const promise = new Promise(function (resolve, reject) {

});
There are three important parts.
Part 1
new Promise()
This creates a new Promise object.
Think of it as creating an empty package that will eventually contain either:
A success result.
An error.
Immediately after creation, its state is:
Pending
Part 2
function (resolve, reject) {

}
Notice something familiar?
We're passing a function.
Just like callbacks!
That function is called the executor function.
JavaScript automatically executes it immediately after the Promise is created.
Part 3
Inside the executor function, JavaScript gives us two special functions:
resolve
and
reject
We did not create them.
JavaScript provides them automatically.
Think of them as two buttons.
Promise

     Pending

       │

 ┌─────┴─────┐

 ▼           ▼

resolve()   reject()

Success      Failure
Only one of these buttons should be pressed.
Your First Promise
const promise = new Promise(function (resolve, reject) {

    resolve("Promise completed successfully.");

});
Let's explain every line.
Line 1
const promise =
Create a variable named promise.
Line 2
new Promise(...)
Create a Promise.
State:
Pending
Line 3
function(resolve, reject)
JavaScript immediately runs this function.
Line 4
resolve("Promise completed successfully.");
We tell JavaScript:
"This Promise finished successfully."
Now the state changes.
Pending
      ↓
Fulfilled
The Promise now contains the value:
"Promise completed successfully."

      ↓

      ↓

      ↓

Using reject()
Now let's fail the Promise.
const promise = new Promise(function (resolve, reject) {

    reject("Something went wrong.");

});
State changes:
Pending
      ↓
Rejected
The Promise now stores an error instead of a success value.
Visual Animation
Create Promise
      ↓
Pending
      ↓
reject()
      ↓
Rejected
      ↓
Store Error
A Promise Can Only Settle Once
This is very important.
Consider this code.
const promise = new Promise(function (resolve, reject) {

    resolve("Success");

    reject("Failure");

});
What happens?
The Promise becomes:
Pending
      ↓
Fulfilled
The second call to reject() is ignored.
Likewise:
const promise = new Promise(function (resolve, reject) {

    reject("Failure");

    resolve("Success");

});
Result:
Pending
      ↓
Rejected
The second call is ignored.
A Promise settles only once.
Real-World Analogy
Imagine you're taking an exam.
Possible outcomes:
Exam

      ↓

Teacher Marks Paper

      ↓

Pass

or

Fail
Once the result is published:
You cannot immediately change from Pass to Fail or vice versa.
The result is final.
Promises behave the same way.

Suppose you're checking whether a student passed.

Important Note
Creating a Promise does not automatically display its result.
For example:
const promise = new Promise(function(resolve, reject){

    resolve("Hello");

});
Nothing is printed.
Why?
Because the Promise simply stores the result.
To use the result, we need .then().
You'll learn that in the next lesson.

Common Beginner Mistakes
Mistake 1
Calling both.
resolve("Success");

reject("Error");
Only the first call works.
Mistake 2
Thinking resolve() prints something.
Wrong.
resolve("Hello");
This stores the value.
It does not display it.
Mistake 3
Thinking reject() crashes the program.
It doesn't.
It simply changes the Promise state to Rejected.
Summary
Today you learned:
new Promise() creates a Promise.
Every Promise starts as Pending.
resolve() changes it to Fulfilled.
reject() changes it to Rejected.
A Promise can settle only once.
The result is stored until someone uses it.
