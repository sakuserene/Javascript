# Module 8 — Event Listeners and Callback Functions
This is one of the most important modules in your callback journey.
After this lesson, you'll understand why callbacks are the foundation of interactive web applications.
Every time you click a button, type into an input, submit a form, or move your mouse, a callback function is involved.
What Is an Event?
An event is simply something that happens in the browser.
Examples:
🖱️ User clicks a button.
⌨️ User presses a key.
📝 User types in an input box.
📄 A page finishes loading.
🖼️ An image finishes loading.
🖱️ User moves the mouse.
Think of an event as an action that happens.
Real-World Analogy
Imagine you're at a restaurant.
You tell the waiter:
"When my food is ready, please call me."
What happens?
You place your order

↓

↓

↓

The waiter doesn't call you immediately.
He waits until the event ("food is ready") happens.
The call is like a callback.
Why Do We Need Event Listeners?
Imagine this button in HTML:
HTML
<button id="saveBtn">Save</button>
Should JavaScript run the save code immediately when the page loads?
No!
It should wait until the user clicks the button.
So JavaScript needs a way to say:
"When this button is clicked, run this function."
That's exactly what addEventListener() does.
Basic Syntax
element.addEventListener("eventName", callback);
Example:
button.addEventListener("click", callback);
Let's understand each part.
element
The HTML element you want to watch.
Example:
const button = document.getElementById("saveBtn");
"click"
The event you're waiting for.
Examples include:
"click"
"submit"
"input"
"change"
"keydown"
"mouseover"
callback
The function that should run later, when the event happens.
First Example
HTML
HTML
<button id="helloBtn">Say Hello</button>
JavaScript
const button = document.getElementById("helloBtn");

button.addEventListener("click", function () {
    console.log("Hello!");
});

Why Don't We Write Parentheses?
Correct:
button.addEventListener("click", sayHello);
Wrong:
button.addEventListener("click", sayHello());
Why?
Because:
sayHello()
means:
Execute now.
But:
sayHello
means:
Pass the function so the browser can execute it later.
This is exactly the same concept you learned earlier with run(sayHello).
Using a Named Function
function sayHello() {
    console.log("Hello!");
}

button.addEventListener("click", sayHello);

Using an Anonymous Function
Instead of creating a separate function:
function sayHello() {
    console.log("Hello");
}
You can write:
button.addEventListener("click", function () {
    console.log("Hello");
});
This is called an anonymous callback function because it has no name.
Real-World Example 1 — Login Button
HTML
HTML
<button id="loginBtn">Login</button>
JavaScript
const loginButton = document.getElementById("loginBtn");

loginButton.addEventListener("click", function () {
    console.log("Checking username...");
    console.log("Checking password...");
    console.log("Logging in...");
});
Nothing happens until the user clicks.
Real-World Example 2 — Delete Button
deleteButton.addEventListener("click", function () {
    console.log("Deleting item...");
});
The deletion only happens after the click event.
Real-World Example 3 — Typing in an Input
HTML
HTML
<input id="name">
JavaScript
const input = document.getElementById("name");

input.addEventListener("input", function () {
    console.log("User is typing...");
});
Every time the user types a character, the callback runs.
The Browser Is the One Calling Your Function
This is a key idea.
When you write:
button.addEventListener("click", function () {
    console.log("Clicked!");
});
You never call the callback yourself.
The browser does.

Multiple Event Listeners
You can attach more than one callback to the same button.
button.addEventListener("click", function () {
    console.log("First callback");
});

button.addEventListener("click", function () {
    console.log("Second callback");
});
Clicking the button prints:
First callback
Second callback
Each registered callback is executed.
Common Beginner Mistakes
Mistake 1: Calling the Function Immediately
Wrong:
button.addEventListener("click", sayHello());
This runs immediately when the page loads.
Correct:
button.addEventListener("click", sayHello);
Mistake 2: Thinking the Callback Runs Automatically
The callback only runs after the event occurs.
If nobody clicks the button, the callback never runs.
Summary
An event listener has three parts:
element.addEventListener(
    "eventName",
    callbackFunction
);
The browser:
Stores your callback.
Waits for the event.
Executes the callback when the event happens.
This is another example of passing a function to another function, just like:
run(sayHello);
The difference is that now the browser decides when to call your function.
