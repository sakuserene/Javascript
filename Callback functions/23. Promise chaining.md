# Module 23 – Promise Chaining: Passing Results from One .then() to Another
Welcome back!

Now it's time to learn one of the most powerful features of Promises:
Promise Chaining.
Promise chaining allows you to perform multiple asynchronous tasks one after another in a clean and readable way.
Lesson Objectives

Why Do We Need Promise Chaining?
Imagine you're registering for an online course.
You cannot skip steps.
The process looks like this:
Create Account
      ↓
Verify Email
      ↓
Login
      ↓
Open Dashboard
Each step depends on the previous one.
You cannot log in before creating an account.
You cannot open the dashboard before logging in.
Promises let us write this sequence in a clean way.
What is Promise Chaining?
Promise chaining means connecting multiple .then() methods together.
Example:
promise
    .then(function () {

    })
    .then(function () {

    })
    .then(function () {

    });
Each .then() waits for the previous one to finish.
Your First Promise Chain
const promise = new Promise(function (resolve) {

    resolve(10);

});

promise
    .then(function (number) {

        console.log(number);

        return number + 5;

    })
    .then(function (result) {

        console.log(result);

    });
Output
10
15
Understanding Every Step
Step 1
The Promise resolves.
resolve(10);
The Promise stores:
10
Step 2
The first .then() executes.
.then(function (number) {

    console.log(number);

    return number + 5;

})
The parameter becomes:
number = 10
It prints:
10
Then it returns:
return 15;
Step 3
The returned value automatically becomes the input for the next .then().
.then(function (result) {

    console.log(result);

});
Now:
result = 15
Output:
15
Visual Flow
resolve(10)
      │
      ▼
First .then()
number = 10
      │
return 15
      │
      ▼
Second .then()
result = 15
      │
console.log(result)
Another Example
Promise.resolve(5)

    .then(function (number) {

        return number * 2;

    })

    .then(function (number) {

        return number + 10;

    })

    .then(function (number) {

        console.log(number);

    });
Output
20
Let's see how it happens.
Start

5

↓

5 × 2 = 10

↓

10 + 10 = 20

↓

Print 20

Returning Values
You don't have to return another Promise.
You can return a normal value.
Example:
Promise.resolve(50)

    .then(function (price) {

        return price * 0.9;

    })

    .then(function (discountedPrice) {

        console.log(discountedPrice);

    });
Output
45
Flow:
50

↓

Apply 10% Discount

↓

45

↓

Print 45
Returning Another Promise
Sometimes a .then() returns another Promise.
Example:
Promise.resolve(5)

    .then(function (number) {

        return Promise.resolve(number * 3);

    })

    .then(function (result) {

        console.log(result);

    });
Output
15
JavaScript automatically waits for the returned Promise to finish.
This is one of the biggest advantages of Promise chaining.
Promise Chaining vs Callback Hell
Earlier, you learned Callback Hell.
Example:
login(function () {

    getUser(function () {

        getOrders(function () {

            processPayment(function () {

                console.log("Done");

            });

        });

    });

});
Notice how the code keeps moving to the right.
This becomes difficult to read.
Now compare it with Promise chaining.
login()

    .then(function () {

        return getUser();

    })

    .then(function () {

        return getOrders();

    })

    .then(function () {

        return processPayment();

    })

    .then(function () {

        console.log("Done");

    });
Much cleaner.
Much easier to read.
What Happens If You Forget return?
Look at this example.
Promise.resolve(10)

    .then(function (number) {

        number + 5;

    })

    .then(function (result) {

        console.log(result);

    });
Output
undefined
Why?
Because nothing was returned.
Correct version:
Promise.resolve(10)

    .then(function (number) {

        return number + 5;

    })

    .then(function (result) {

        console.log(result);

    });
Output
15
Always remember:
If the next .then() needs a value, you must return it.
Common Beginner Mistakes
Mistake 1
Forgetting return.
Wrong:
.then(function (value) {

    value + 10;

})
Correct:
.then(function (value) {

    return value + 10;

})
Mistake 2
Thinking each .then() starts from the original Promise.
Wrong.
Each .then() receives the value returned by the previous .then().
Mistake 3
Returning nothing.
If you don't return anything,
the next .then() receives:
undefined
