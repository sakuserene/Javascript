# Module 25 – Error Handling with async and await Using try...catch
Welcome back!
In the previous lesson, you learned:
async functions always return a Promise.
await pauses an async function until a Promise settles.
But an important question remains:
What happens if the Promise fails?
Suppose you write:
async function getData() {

    const data = await Promise.reject("Network Error");

    console.log(data);

}

getData();
Will console.log(data) run?
No.
The Promise is rejected.
An error occurs.
The rest of the function stops executing unless you handle the error.
That's why JavaScript provides try...catch.

By the end of this lesson, you will understand:
What try...catch is.
Why try...catch is needed with async and await.
How try...catch works.
What happens when an error occurs.
Real-world examples of error handling.

What is try?
The try block contains code that might fail.
Example:
try {

    console.log("Start");

} catch (error) {

}
Think of try as saying:
"Attempt to execute this code."
What is catch?
The catch block runs only if an error occurs inside the try block.
Example:
try {

    console.log(name);

} catch (error) {

    console.log(error);

}
Since name doesn't exist...
JavaScript throws an error.
Instead of crashing...
The error is caught.
Basic Syntax
try {

    // Code that may fail

} catch (error) {

    // Handle the error

}
Your First Example
try {

    console.log(user);

} catch (error) {

    console.log("Something went wrong.");

}
Output
Something went wrong.
Notice that the program doesn't crash.
Using try...catch with async and await
Suppose we have this Promise.
function login() {

    return Promise.reject("Wrong Password");

}
Now use await.
async function start() {

    try {

        const message = await login();

        console.log(message);

    } catch (error) {

        console.log(error);

    }

}

start();
Output
Wrong Password
Step-by-Step Execution
Step 1
start() begins.
start();
Step 2
JavaScript enters the try block.
try {

}
Step 3
await login() runs.
await login();
The Promise is rejected.
Rejected
Step 4
Execution immediately jumps to catch.
catch (error) {

    console.log(error);

}
The error becomes:
Wrong Password
Output
Wrong Password

Example 1 – Login System
function login(password) {

    return new Promise(function (resolve, reject) {

        if (password === "admin123") {

            resolve("Login Successful");

        } else {

            reject("Invalid Password");

        }

    });

}
Using try...catch
async function start() {

    try {

        const message = await login("12345");

        console.log(message);

    } catch (error) {

        console.log(error);

    }

}

start();
Output
Invalid Password
Example 2 – Downloading a File

    }

Example 3 – Successful Promise
function getName() {

    return Promise.resolve("Adi");

}
async function displayName() {

    try {

        const name = await getName();

        console.log(name);

    } catch (error) {

        console.log(error);

    }

}

displayName();

Adi
Notice that catch never runs because no error occurred.
What Happens After catch?
Execution continues after the catch block.
Example
async function example() {

    try {

        await Promise.reject("Error");

    } catch (error) {

        console.log(error);

    }

    console.log("Program Continues");

}

example();
Output
Error
Program Continues
This is important.
Handling an error allows the rest of your program to continue running.
Using finally with try...catch
You can also use finally.
try {

    console.log("Trying...");

} catch (error) {

    console.log(error);

} finally {

    console.log("Finished.");

}
Output
Trying...
Finished.
If an error occurs:
try {

    console.log(user);

} catch (error) {

    console.log("User not found.");

} finally {

    console.log("Finished.");

}
Output
User not found.
Finished.
The finally block always executes.

Common Beginner Mistakes
Mistake 1
Using await without try...catch.
Wrong
const data = await fetchData();
If the Promise rejects...
The error isn't handled.
Mistake 2
Thinking catch always runs.
Wrong.
catch only runs if an error occurs.
Mistake 3
Thinking finally replaces catch.
Wrong.
finally is for cleanup.
catch is for handling errors.
Comparing Promise .catch() and try...catch
Promise Style
login()

    .then(function (message) {

        console.log(message);

    })

    .catch(function (error) {

        console.log(error);

    });
Async/Await Style
try {

    const message = await login();

    console.log(message);

} catch (error) {

    console.log(error);

}
Both do the same thing.
The second style is usually easier to read.
Summary
Today you learned:
try contains code that may fail.
catch handles errors.
finally always runs.
await should usually be placed inside a try block.
try...catch is the standard way to handle errors with async and await.
