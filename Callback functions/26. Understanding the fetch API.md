Module 1 – Why Do We Need the Fetch API?
Imagine you open an online shopping website.
You want to see all available laptops.
Does your browser already know the products?
No.
The products are stored on a server somewhere on the internet.
Your browser has to ask the server for them.

Module 4 – What is fetch()?
fetch() is a built-in JavaScript function used to send HTTP requests.
Basic syntax:
fetch("https://jsonplaceholder.typicode.com/users");
Question:
What does fetch() return?
Answer:
A Promise.
Why?
Because communication over the internet takes time.
JavaScript doesn't know when the server will respond.
So instead of returning the data immediately, it returns a Promise.

Module 6 – Receiving the Response
Since fetch() returns a Promise, we use .then().
fetch("https://jsonplaceholder.typicode.com/users")
    .then(function (response) {
        console.log(response);
    });
Question:
What is response?
It is a Response object returned by the server.
It contains information such as:
Status code
Headers
Body
Response type
Example:
fetch("https://jsonplaceholder.typicode.com/users")
    .then(function (response) {
        console.log(response.status);
    });
Possible output:
200
Status code 200 means the request was successful.
Module 7 – Where is the Actual Data?
If you print the response:
fetch("https://jsonplaceholder.typicode.com/users")
    .then(function (response) {
        console.log(response);
    });
You won't immediately see the users.
Instead, you'll see something like:
Response {
    body: ...
    headers: ...
    status: 200
}
The actual data is stored inside the response body.
We need to extract it.
Module 8 – Understanding JSON
Most servers send data as JSON.
JSON stands for:
JavaScript Object Notation
Example of JSON:
{
    "name": "John",
    "age": 20
}
Or an array:
[
    {
        "name": "John"
    },
    {
        "name": "Mary"
    }
]
JSON looks like JavaScript objects.
However, JSON is simply text.
We must convert it into JavaScript objects before we can use it.
Module 9 – Converting JSON into JavaScript Objects
The Response object has a method called .json().
Example:
fetch("https://jsonplaceholder.typicode.com/users")
    .then(function (response) {
        return response.json();
    })
    .then(function (users) {
        console.log(users);
    });
Execution flow:
fetch()

↓

Server Responds

↓

Response Object

↓

response.json()

↓

Convert JSON

↓

JavaScript Objects

↓

Print Users
Notice that there are two .then() methods.
Why?
The first waits for the HTTP response.
The second waits for the JSON conversion.
Module 10 – Why Does response.json() Return a Promise?
Many beginners ask:
"If the response has already arrived, why does .json() return another Promise?"
Because converting JSON into JavaScript objects also takes time.
Especially when the response contains thousands of records.
So JavaScript returns another Promise while the conversion happens.
Module 11 – Using async and await
Instead of writing:
fetch("https://jsonplaceholder.typicode.com/users")
    .then(function (response) {
        return response.json();
    })
    .then(function (users) {
        console.log(users);
    });
We can write:
async function getUsers() {
    const response = await fetch("https://jsonplaceholder.typicode.com/users");

    const users = await response.json();

    console.log(users);
}

getUsers();
Many developers prefer this because it reads from top to bottom.

Module 14 – Error Handling
Internet connections sometimes fail.
Servers sometimes go offline.
Always use try...catch.
async function getUsers() {
    try {
        const response = await fetch("https://jsonplaceholder.typicode.com/users");

        const users = await response.json();

        console.log(users);
    } catch (error) {
        console.log(error);
    }
}

getUsers();
This prevents your application from crashing if an error occurs.
Module 15 – Common Beginner Mistakes
Mistake 1
Forgetting await
Wrong:
const response = fetch(url);
Correct:
const response = await fetch(url);
Mistake 2
Printing the Response object instead of the data
Wrong:
console.log(response);
Correct:
const data = await response.json();

console.log(data);
Mistake 3
Forgetting to await .json()
Wrong:
const users = response.json();
Correct:
const users = await response.json();
