# Module 22 – Understanding .finally(): Running Code No Matter What Happens
Welcome back!
So far, you've learned two important Promise methods:
.then() → Runs when the Promise is fulfilled.
.catch() → Runs when the Promise is rejected.
But what if you want to execute some code whether the Promise succeeds or fails?
For example:
Hide a loading spinner.
Close a database connection.
Stop a loading animation.
Enable a disabled button.
Log that an operation has finished.
Should you write the same code inside both .then() and .catch()?
No.
JavaScript provides another method called .finally().
Lesson Objectives
By the end of this lesson, you will understand:
What .finally() is.
Why .finally() exists.
When .finally() executes.
The difference between .then(), .catch(), and .finally().
Real-world situations where .finally() is useful.
Why Do We Need .finally()?
Imagine you're downloading a file.
Before the download starts, you display a loading spinner.
User Clicks Download
        ↓
Show Loading Spinner
        ↓
Download Starts
Two things can happen.
Case 1: Download Succeeds
Download Starts
        ↓
Download Complete
        ↓
Hide Loading Spinner
Case 2: Download Fails
Download Starts
        ↓
Network Error
        ↓
Hide Loading Spinner
Notice something?
Whether the download succeeds or fails...
You still want to hide the loading spinner.
Instead of writing:
.then(function () {

    hideSpinner();

})

.catch(function () {

    hideSpinner();

});
You simply write:
.finally(function () {

    hideSpinner();

});
Much cleaner.
Basic Syntax
promise.finally(function () {

});
Read it like this:
"When everything is finished, run this function."
Your First Example
const promise = new Promise(function (resolve, reject) {

    resolve("Download Complete");

});

promise
    .then(function (message) {

        console.log(message);

    })
    .finally(function () {

        console.log("Finished.");

    });
Output
Download Complete
Finished.
Example with a Rejected Promise
const promise = new Promise(function (resolve, reject) {

    reject("Network Error");

});

promise
    .catch(function (error) {

        console.log(error);

    })
    .finally(function () {

        console.log("Finished.");

    });
Output
Network Error
Finished.
Notice something important.
.finally() executes in both situations.
Execution Flow
If the Promise succeeds:
Create Promise
       ↓
Pending
       ↓
resolve()
       ↓
.then()
       ↓
.finally()
If the Promise fails:
Create Promise
       ↓
Pending
       ↓
reject()
       ↓
.catch()
       ↓
.finally()
.finally() always comes last.
Does .finally() Receive Data?
Let's try.
const promise = Promise.resolve("Hello");

promise.finally(function (result) {

    console.log(result);

});
Output
undefined
Why?
Because .finally() does not receive:
the resolved value
the rejected error
Its purpose is cleanup, not processing results.
Compare .then(), .catch(), and .finally()
const promise = new Promise(function (resolve, reject) {

    resolve("Success");

});

promise
    .then(function (message) {

        console.log("then:", message);

    })
    .catch(function (error) {

        console.log("catch:", error);

    })
    .finally(function () {

        console.log("finally: Finished");

    });
Output
then: Success
finally: Finished
Now change:
resolve("Success");
to:
reject("Something went wrong");
Output
catch: Something went wrong
finally: Finished
Real-World Example 1 – Login Screen
Suppose a user clicks the Login button.
Before sending the request:
showLoadingSpinner();
When the request finishes:
hideLoadingSpinner();
Instead of writing:
login()
    .then(function () {

        hideLoadingSpinner();

    })
    .catch(function () {

        hideLoadingSpinner();

    });
Use:
login()
    .finally(function () {

        hideLoadingSpinner();

    });
Cleaner and easier to maintain.

Transfer Money
        ↓
Processing
        ↓
Success or Failure
        ↓
Print Receipt
        ↓
Close Transaction Window
No matter what happens...
The transaction window should close.
That is a perfect job for .finally().
Real-World Example 3 – File Upload
Choose File
      ↓
Upload Begins
      ↓

     /      \
   Yes       No
    |         |
Show File   Show Error
      \     /

 Hide Progress Bar
The progress bar should disappear in both cases.
Use .finally().
Another Example
const internet = true;

const download = new Promise(function (resolve, reject) {

    if (internet) {

        resolve("Download Complete");

    } else {

        reject("No Internet");

    }

});

download
    .then(function (message) {

        console.log(message);

    })
    .catch(function (error) {

        console.log(error);

    })
    .finally(function () {

        console.log("Closing connection...");

    });
Output
Download Complete
Closing connection...
If internet = false
Output
No Internet
Closing connection...
When Should You Use .finally()?
Use .finally() when you need to:
Hide a loading spinner.
Stop a loading animation.
Close a database connection.
Close a file.
Re-enable a button.
Remove a progress bar.
Log that an operation has finished.
Perform cleanup tasks.
Common Beginner Mistakes
Mistake 1
Thinking .finally() only runs after success.
Wrong.
It runs after both success and failure.
Mistake 2
Trying to get the resolved value.
.finally(function(result){

    console.log(result);

});
Wrong.
result will be undefined.
Use .then() if you need the successful value.
Mistake 3
Trying to get the error.
.finally(function(error){

    console.log(error);

});
Wrong.
Use .catch() if you need the error.

Summary
Today you learned:
.finally() runs after a Promise settles.
It runs after both success and failure.
It does not receive the resolved value.
It does not receive the rejected error.
It is mainly used for cleanup tasks.
