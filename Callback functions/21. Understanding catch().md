# Module 21 – Understanding .catch(): Handling Errors in Promises
Congratulations!
In the previous lesson, you learned that:
resolve() is used when a Promise succeeds.
.then() receives the successful value from a Promise.
But what happens if something goes wrong?
For example:
The internet connection is lost.
A user enters the wrong password.
A payment fails.
A file cannot be found.
A server crashes.
Should the program stop completely?
No.
JavaScript provides another Promise method called .catch().
Its job is to receive and handle errors.
Instead of allowing your program to fail silently or crash, .catch() gives you a place to respond to errors.
Lesson Objectives
By the end of this lesson, you will understand:
What .catch() is.
Why .catch() exists.
How reject() and .catch() work together.
How JavaScript passes errors from reject() to .catch().
Why .catch() makes asynchronous code easier to manage.
Why Do We Need .catch()?
Imagine you order a laptop online.
Normally the process looks like this.
Place Order
     ↓
Payment Successful
     ↓
Shipping
     ↓
Delivered
Everything goes well.
But what if your payment is declined?
Place Order
     ↓
Payment Failed
     ↓
Notify Customer
The online store doesn't pretend everything worked.
Instead, it informs you that something went wrong.
That notification is exactly what .catch() does.
Basic Syntax
promise.catch(function (error) {

});
Read it like this:
"If something goes wrong, run this function."
Your First Example
const promise = new Promise(function (resolve, reject) {

    reject("Something went wrong.");

});

promise.catch(function (error) {

    console.log(error);

});
Output
Something went wrong.
Understanding Every Line
Let's break the code down.
Step 1
Create the Promise.
const promise = new Promise(function (resolve, reject) {

});
Current state:
Pending
Step 2
Reject the Promise.
reject("Something went wrong.");
The Promise changes state.
Pending
   ↓
Rejected
The Promise now stores the error message.
"Something went wrong."
Step 3
JavaScript reaches:
promise.catch(function (error) {

    console.log(error);

});
Since the Promise is rejected...
JavaScript automatically executes the callback function.
It passes the stored error into the parameter named error.
Execution Flow
reject("Something went wrong.")
            │
            ▼
Promise Stores Error
            │
            ▼
.catch()
            │
            ▼
Callback Executes
            │
            ▼
error = "Something went wrong."
            │
            ▼
console.log(error)
Output
Something went wrong.
Where Does error Come From?
Look carefully.
reject("Invalid Password");
Later...
promise.catch(function (error) {

    console.log(error);

});
Question:
Who assigned the value to error?
Answer:
JavaScript.
You never write:
error = "Invalid Password";
JavaScript does that automatically.
Whatever you pass into reject() becomes the parameter inside .catch().
Think of it like this.
reject("Invalid Password")
          │
          ▼
Promise Stores Error
          │
          ▼
.catch(function(error){
          │
          ▼
error = "Invalid Password"
The Parameter Name Can Be Anything
The parameter does not have to be called error.
These examples all work.
promise.catch(function (err) {

    console.log(err);

});
promise.catch(function (message) {

    console.log(message);

});
promise.catch(function (reason) {

    console.log(reason);

});
Just like callback parameters, the name is your choice.
Example 1 – Login System
const login = new Promise(function (resolve, reject) {

    const password = "12345";

    if (password === "admin123") {

        resolve("Login Successful");

    } else {

        reject("Wrong Password");

    }

});
Now handle the result.
login
    .then(function (message) {

        console.log(message);

    })
    .catch(function (error) {

        console.log(error);

    });
Output
Wrong Password

Using .then() and .catch() Together
This is the most common pattern.
promise
    .then(function (result) {

        console.log(result);

    })
    .catch(function (error) {

        console.log(error);

    });
Think of it like this.
Was the Promise successful?

        │
   ┌────┴────┐
   │         │
  Yes        No
   │         │
.then()   .catch()
Only one path is taken.
Visual Flow
Create Promise
      │
      ▼
   Pending
      │
      ▼
Was it successful?
      │
 ┌────┴────┐
 │         │
 ▼         ▼
resolve() reject()
 │         │
 ▼         ▼
.then()  .catch()
Real-World Analogy
Imagine applying for a passport.
Submit Application
        │
        ▼
Government Reviews
        │
        ▼
Application Approved?
     │           │
    Yes         No
     │           │
Passport     Rejection
 Issued        Letter
Receiving the passport is like .then().
Receiving the rejection letter is like .catch().
Common Beginner Mistakes
Mistake 1
Thinking .catch() always runs.
Wrong.
It only runs when the Promise is rejected.
Mistake 2
Thinking .then() and .catch() both execute.
Wrong.
If the Promise is fulfilled:
.then() runs
If the Promise is rejected:
.catch() runs
Only one executes.
Mistake 3
Thinking reject() stops JavaScript.
Wrong.
reject() simply changes the Promise state to Rejected.
.catch() is responsible for handling the error.
Summary
Today you learned:
reject() changes a Promise from Pending to Rejected.
The value passed into reject() becomes the parameter inside .catch().
.catch() receives and handles Promise errors.
The function inside .catch() is also a callback.
.then() handles success.
.catch() handles failure.

promise
