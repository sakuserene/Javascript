# Module 7 (Part 8) — sort()
Congratulations! You've reached the last major callback-based array method in this module.
Today you'll learn:
sort() → Arrange items in a specific order.
This is one of the most useful methods in real-world applications because almost every app allows users to sort data.
Examples:
📱 Sort contacts alphabetically.
🛒 Sort products by price.
🎓 Sort students by score.
📅 Sort tasks by due date.
💰 Sort bank transactions by amount.
Why Does sort() Exist?
Suppose you have this array.
const numbers = [40, 10, 30, 20];
You want:
[10, 20, 30, 40]
Instead of writing your own sorting algorithm, JavaScript provides:
numbers.sort();
Easy, right?
Well...
There's something important you need to know.
First Surprise
Look at this.
const numbers = [100, 25, 5];

numbers.sort();

console.log(numbers);
What do you expect?
Most beginners expect:
[5, 25, 100]
But the actual output is:
[100, 25, 5]
or in some cases:
[100, 5, 25]
Why?
Because by default, JavaScript sorts as strings, not as numbers.
It compares:
"100"

"25"

"5"
Alphabetically.
That isn't what we want for numbers.
The Solution: A Callback
Instead of letting JavaScript guess, we tell it exactly how to compare two values.
numbers.sort(function(a, b) {
    return a - b;
});
Output
[5, 25, 100]
What Are a and b?
This callback is different from the others.
Instead of receiving one item, it receives two items.
function(a, b)
JavaScript picks two values and asks:
"Which one should come first?"
Understanding the Return Value
This is the most important part.
return a - b;
The callback returns a number.
Not true or false.
If the result is negative
a comes before b
If the result is positive
b comes before a
If the result is zero
Leave them as they are.
Step-by-Step Example
const numbers = [30, 10];
JavaScript compares:
a = 30

b = 10
Callback:
return a - b;
Becomes:
30 - 10 = 20
Positive number.
So JavaScript swaps them.
Result:
[10, 30]
Another Example
a = 10

b = 30
Callback:
10 - 30 = -20
Negative.
Keep them as they are.
Visual Flow
[40,10,30,20]

↓

sort()

↓

Compare 40 and 10

↓

40 - 10 = 30

↓

Swap

↓

Compare 30 and 20

↓

...

↓

Finished

↓

[10,20,30,40]
Ascending Order
Smallest → Largest
const numbers = [40, 10, 30, 20];

numbers.sort(function(a, b) {
    return a - b;
});

console.log(numbers);
Output
[10,20,30,40]
Descending Order
Largest → Smallest
const numbers = [40, 10, 30, 20];

numbers.sort(function(a, b) {
    return b - a;
});

console.log(numbers);
Output
[40,30,20,10]
Notice:
We simply reversed the subtraction.
Real-World Example 1 — Student Scores
const scores = [65, 90, 45, 80];

scores.sort(function(a, b) {
    return b - a;
});

console.log(scores);
Output
[90,80,65,45]
Highest score first.

Sorting Objects
This is where callbacks become powerful.
Suppose we have:
const students = [
    { name: "John", score: 80 },
    { name: "Mary", score: 95 },
    { name: "Adi", score: 70 }
];
Sort by score.
students.sort(function(a, b) {
    return a.score - b.score;
});

console.log(students);
Output
[
  { name: "Adi", score: 70 },
  { name: "John", score: 80 },
  { name: "Mary", score: 95 }
]
Sort by Name
const students = [
    { name: "John" },
    { name: "Mary" },
    { name: "Adi" }
];

students.sort(function(a, b) {
    return a.name.localeCompare(b.name);
});

console.log(students);
Output
[
  { name: "Adi" },
  { name: "John" },
  { name: "Mary" }
]
localeCompare() is a built-in string method that compares two strings alphabetically.
Does sort() Create a New Array?
No.
Unlike map() and filter(), sort() changes the original array.
Example:
const numbers = [3, 1, 2];

numbers.sort(function(a, b) {
    return a - b;
});

console.log(numbers);
Output
[1,2,3]
The original array has been modified.
Real-World Example 3 — Todo App
const tasks = [
    { title: "Study", priority: 2 },
    { title: "Exercise", priority: 3 },
    { title: "Sleep", priority: 1 }
];

tasks.sort(function(a, b) {
    return a.priority - b.priority;
});

console.log(tasks);
Output
[
  { title: "Sleep", priority: 1 },
  { title: "Study", priority: 2 },
  { title: "Exercise", priority: 3 }
]
Common Beginner Mistakes
Mistake 1
Sorting numbers without a callback.
numbers.sort();
This may not sort numbers correctly because it compares them as strings.
Mistake 2
Returning true or false.
Wrong:
numbers.sort(function(a, b) {
    return a > b;
});
sort() expects a number, not a boolean.
Correct:
numbers.sort(function(a, b) {
    return a - b;
});
