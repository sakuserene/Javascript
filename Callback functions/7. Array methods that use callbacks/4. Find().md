# Module 7 (Part 4) — find()
So far you've learned:
✅ forEach() → Do something with every item.
✅ map() → Transform every item into a new array.
✅ filter() → Keep all items that match a condition.
Now we'll learn:
find() → Return the first item that matches a condition.
This is one of the most useful array methods in real-world applications.
Why Does find() Exist?
Imagine you have a list of users.
const users = [
    { id: 1, name: "Adi" },
    { id: 2, name: "John" },
    { id: 3, name: "Mary" }
];
You only want the user whose id is 2.
Do you want all matching users?
No.
There should only be one.
Using filter() would give:
[
    { id: 2, name: "John" }
]
Notice that's an array.
But you don't want an array.
You want the actual user object.
That's why find() exists.
What Does find() Do?
find() asks every item:
"Are you the one I'm looking for?"
If the callback returns:
true → Stop searching and return that item.
false → Continue searching.
Unlike filter(), find() stops as soon as it finds the first match.
Basic Syntax
array.find(callback);
First Example
const numbers = [5, 10, 15, 20];

const result = numbers.find(function(number) {
    return number > 12;
});

console.log(result);
Output
15
Why?
Let's see the execution.

Compare filter() and find()
Suppose we have:
const numbers = [2, 4, 6, 8];
Using filter()
const result = numbers.filter(function(number) {
    return number > 3;
});

console.log(result);
Output:
[4, 6, 8]
All matching numbers are returned.
Using find()
const result = numbers.find(function(number) {
    return number > 3;
});

console.log(result);
Output:
4
Only the first matching number is returned.
Real-World Example 1 — Find a Student
const students = [
    { id: 1, name: "Alice" },
    { id: 2, name: "Brian" },
    { id: 3, name: "Carol" }
];

const student = students.find(function(student) {
    return student.id === 2;
});

console.log(student);
Output
{ id: 2, name: "Brian" }
Real-World Example 2 — Shopping Cart
const products = [
    { id: 101, name: "Laptop" },
    { id: 102, name: "Phone" },
    { id: 103, name: "Keyboard" }
];

const product = products.find(function(product) {
    return product.id === 103;
});

console.log(product);
Output
{ id: 103, name: "Keyboard" }
This is exactly how many shopping websites locate a product by its ID.
Real-World Example 3 — Login System
Imagine you have registered users.
const users = [
    { username: "adi", password: "1234" },
    { username: "john", password: "5678" }
];
Find the user named "john".
const user = users.find(function(user) {
    return user.username === "john";
});

console.log(user);
Output:
{ username: "john", password: "5678" }
What If Nothing Is Found?
const numbers = [2, 4, 6];

const result = numbers.find(function(number) {
    return number > 10;
});

console.log(result);
Output:
undefined
Why?
Because no number satisfied the condition.
Common Beginner Mistake
Some beginners think find() returns an array.
It doesn't.
const result = numbers.find(function(number) {
    return number > 5;
});
If the first matching number is 6, then:
result
is:
6
Not:
[6]
Callback Receives More Than One Parameter
Like forEach(), map(), and filter(), the callback can also receive:
const fruits = ["Apple", "Banana", "Orange"];

const result = fruits.find(function(fruit, index, array) {
    console.log(index, fruit);
    return fruit === "Banana";
});
The callback receives:
Current item
Current index
Entire array

Summary
Use find() when:
You need only one item.
You want the first matching item.
You're searching by ID, username, email, product code, etc.
Remember:
filter() → Returns an array of all matches.
find() → Returns the first matching item.
If nothing matches, find() returns undefined.

Next Lesson
Next you'll learn some() and every().
These two methods don't return an array or an item. Instead, they answer yes/no questions by returning either true or false.
For example:
some() → "Does at least one student pass the exam?"
every() → "Did all students pass the exam?"
These are incredibly useful for validation, permissions, form checking, and business rules.
