# Module 7 (Part 5) — some()
So far you've learned:
✅ forEach() → Do something with every item.
✅ map() → Transform every item.
✅ filter() → Keep matching items.
✅ find() → Return the first matching item.
Now you'll learn another very useful method:
some() → Checks if at least one item matches a condition.
Unlike the previous methods, some() does not return an array or an object.
It returns only:
true
or
false
Why Does some() Exist?
Imagine you are a teacher.
These are your students' marks.
const scores = [30, 45, 80, 25];
Question:
Did at least one student pass?
Notice you're not asking:
Who passed?
How many passed?
Which marks passed?
You're only asking:
Is there at least one pass?
That's exactly what some() answers.
Basic Syntax
array.some(callback);
First Example
const scores = [30, 45, 80, 25];

const hasPassed = scores.some(function(score) {
    return score >= 50;
});

console.log(hasPassed);
Output
true
Why?
Because 80 is greater than or equal to 50.

What If Nothing Matches?
const scores = [10, 20, 30];

const hasPassed = scores.some(function(score) {
    return score >= 50;
});

console.log(hasPassed);
Output
false
Because no student passed.
Real-World Example 1 — Login Permissions
const roles = ["viewer", "editor", "admin"];

const isAdmin = roles.some(function(role) {
    return role === "admin";
});

console.log(isAdmin);
Output
true
This is useful for checking whether a user has a required role.
Real-World Example 2 — Shopping Cart
const cart = [
    { name: "Laptop", inStock: true },
    { name: "Phone", inStock: false },
    { name: "Mouse", inStock: true }
];
Check if any product is out of stock.
const hasOutOfStock = cart.some(function(product) {
    return product.inStock === false;
});

console.log(hasOutOfStock);
Output
true
Real-World Example 3 — Form Validation
Suppose a user fills in three fields.
const fields = ["John", "", "Kenya"];
Check if any field is empty.
const hasEmptyField = fields.some(function(field) {
    return field === "";
});

console.log(hasEmptyField);
Output
true
A form can use this to prevent submission until all required fields are filled.
How some() Thinks
Imagine a security guard asking:
Does anyone have a VIP ticket?
Person 1
No
↓
Continue
Person 2
No
↓
Continue
Person 3
Yes
↓
Stop immediately
↓
Answer:
Yes
That's exactly how some() works.
Common Beginner Mistake
Some beginners think some() returns the matching item.
Wrong:
const result = numbers.some(function(number) {
    return number > 5;
});
If the array is:
[2, 4, 6]
The output is:
true
Not:
6
If you want the actual item, use:
find()
find() vs some()
Suppose:
const numbers = [2, 4, 6, 8];
Using find()
const result = numbers.find(function(number) {
    return number > 5;
});

console.log(result);
Output
6
Using some()
const result = numbers.some(function(number) {
    return number > 5;
});

console.log(result);
Output
true
When Should You Use some()?
Use it when you need a yes/no answer, such as:
Does any student fail?
Does any product cost more than KSh 10,000?
Does any user have admin rights?
Does any task remain incomplete?
Does any input field contain an error?
Summary
some():
Loops through the array.
Calls your callback for each item.
Stops at the first true.
Returns:
true if at least one item matches.
false if no items match.
