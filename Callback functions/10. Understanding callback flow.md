# Module 10 — Understanding Callback Flow
This is one of the most important lessons in the entire course.
Until now, you've learned how to write callbacks.
Now you'll learn how JavaScript thinks about callbacks.
Once you understand this flow, callbacks in event listeners, timers, array methods, and later Promises and async/await will make much more sense.
The Big Idea
A callback follows this journey:
Create Function

↓

Pass Function

↓

Another Function Receives It

↓

Stores It (or uses it immediately)

↓

Some Condition Happens

↓

JavaScript Calls the Callback

↓

Program Continues
Notice something important:
The function that receives the callback is usually NOT the function that executes it immediately.
Sometimes it executes it immediately (like forEach()).
Sometimes later (like setTimeout()).
Sometimes when the user clicks (like addEventListener()).

Common Beginner Misunderstanding
Many beginners think:
setTimeout(callback, 3000);
means:
"Wait 3 seconds before continuing."
It doesn't.
It means:
"Start a timer, continue running the rest of the code, and execute the callback later."
That's a huge difference.
