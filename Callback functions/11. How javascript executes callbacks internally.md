# Module 11 — How JavaScript Executes Callbacks Internally
This is one of the most important JavaScript concepts you'll ever learn.
Many developers can write callback functions.
Far fewer understand how JavaScript actually runs them.
By the end of this lesson, you'll understand:
✅ Call Stack
✅ Web APIs (Browser APIs)
✅ Callback Queue (Task Queue)
✅ Event Loop
✅ Microtask Queue
✅ Macrotask Queue
Once you understand these, Promises and async/await will become much easier.
The Problem JavaScript Had to Solve
JavaScript has one Call Stack.
That means:
JavaScript can only execute one thing at a time.
This is called single-threaded.

What Is the Call Stack?
Think of the Call Stack as a stack of plates.

The function on top runs first.
When it finishes, it's removed.
Then the next one runs.
This is called Last In, First Out (LIFO).
The last function added is the first one to finish.

What Is the Event Loop?
Think of the Event Loop as a traffic police officer.
Its only job is to ask:
Is the Call Stack empty?
If:
No
Wait.
If:
Yes
Take the next callback from the queue and put it on the stack.

Microtasks vs. Macrotasks
There are actually two main queues.
Macrotask Queue
Contains callbacks from:
setTimeout()
setInterval()
DOM events
Microtask Queue
Contains callbacks from:
Promises (.then())
queueMicrotask()
Microtasks have higher priority.
Example
console.log("Start");

setTimeout(() => {
    console.log("Timeout");
}, 0);

Promise.resolve().then(() => {
    console.log("Promise");
});

console.log("End");
Output:
Start
End
Promise
Timeout
Why?
Because:
Normal code runs first.
Microtasks (Promise) run next.
Macrotasks (setTimeout) run last.

JavaScript always empties the Microtask Queue before taking a callback from the Macrotask Queue.

Summary
Think of JavaScript like this:
Call Stack → Where functions execute.
Web APIs → Browser features that handle timers and events.
Callback Queue → Where finished callbacks wait.
Event Loop → Moves callbacks to the Call Stack when it's empty.
Microtasks → High-priority callbacks (Promises).
Macrotasks → Lower-priority callbacks (setTimeout, events).

What's Next?
You've now mastered callback functions at a deep level.
The next step is to learn Promises, which solve many of the problems that arise when callbacks become deeply nested (often called "callback hell"). After Promises, you'll move on to async/await, the modern and most readable way to write asynchronous JavaScript. Understanding callbacks first makes those topics much easier.
