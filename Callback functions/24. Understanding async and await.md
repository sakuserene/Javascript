# Module 24 – Understanding async and await
Welcome back!
So far, you've learned:
How to create Promises.
How to use .then().
How to use .catch().
How to use .finally().
How to chain multiple .then() methods.
Although Promise chaining is much cleaner than Callback Hell, long chains can still become difficult to read.
For example:
getUser()

    .then(function (user) {

        return getOrders(user);

    })

    .then(function (orders) {

        return calculateTotal(orders);

    })

    .then(function (total) {

        console.log(total);

    })

    .catch(function (error) {

        console.log(error);

    });
This is perfectly valid JavaScript.
But imagine a program with 20 or 30 asynchronous operations.
The code becomes difficult to follow.
To solve this problem, JavaScript introduced async and await.

Why Do We Need async and await?
Imagine you're making tea.
The steps are:
Boil Water
      ↓
Add Tea Leaves
      ↓
Pour Into Cup
      ↓
Drink Tea
Each step depends on the previous one.
If someone explains the process like this:
Boil Water

↓

Wait

↓

Add Tea Leaves

↓

Wait

↓

Pour Tea

↓

Wait

↓

Drink
It's easy to understand.
Now imagine writing it using many .then() calls.
It works...
But it becomes harder to read.
async and await allow us to write asynchronous code in the same top-to-bottom style.
What is async?
async is a keyword placed before a function.
Example:
async function greet() {

}
Adding async tells JavaScript:
"This function works with Promises."
Your First Async Function
async function greet() {

    return "Hello";

}
Question:
What does this function return?
Many beginners think it returns:
Hello
Actually...
It returns a Promise.
Equivalent code:
function greet() {

    return Promise.resolve("Hello");

}
Both functions return exactly the same thing.
Proof
async function greet() {

    return "Hello";

}

console.log(greet());
Output
Promise { "Hello" }
Notice that the value is automatically wrapped inside a Promise.
What is await?
await tells JavaScript:
"Wait until this Promise finishes before continuing."
Example:
const result = await promise;
Think of it like this.
Promise Starts

↓

Wait...

↓

Promise Finishes

↓

Continue Program
Where Can You Use await?
You cannot write:
const result = await promise;
by itself.
This is invalid.
await can only be used inside an async function.
Correct example:
async function example() {

    const result = await promise;

}
Remember this rule:
await must be inside an async function.
First Example
function getMessage() {

    return Promise.resolve("Hello JavaScript");

}

async function displayMessage() {

    const message = await getMessage();

    console.log(message);

}

displayMessage();
Output
Hello JavaScript
Step-by-Step Execution
Step 1
JavaScript creates the Promise.
Promise.resolve("Hello JavaScript")
State
Fulfilled
Step 2
await pauses execution.
const message = await getMessage();
JavaScript waits.
Waiting...
Step 3
The Promise finishes.
The resolved value becomes:
message = "Hello JavaScript"
Step 4
Execution continues.
console.log(message);
Output
Hello JavaScript
Visual Flow
Call async function

↓

Call Promise

↓

await

↓

Pause

↓

Promise Resolves

↓

Continue Execution

↓

Print Result
Comparing .then() and await
Using .then():
getMessage()

    .then(function (message) {

        console.log(message);

    });
Using await:
async function showMessage() {

    const message = await getMessage();

    console.log(message);

}

showMessage();
Both produce the same output.
The second version is usually easier to read.
Real-World Example – Online Shopping
Suppose you buy a laptop.
Steps:
Login
      ↓
Add Item
      ↓
Pay
      ↓
Receive Receipt
With Promises:
login()

    .then(function () {

        return addItem();

    })

    .then(function () {

        return pay();

    })

    .then(function () {

        return getReceipt();

    });
Using async and await:
async function shop() {

    await login();

    await addItem();

    await pay();

    await getReceipt();

}
Notice how the code now reads like normal English.
Real-World Example – Cooking
Boil Water

↓

Cook Rice

↓

Serve Food

↓

Eat
JavaScript:
async function cookDinner() {

    await boilWater();

    await cookRice();

    await serveFood();

    console.log("Dinner Ready");

Each step waits for the previous one.
Returning Values
async function getNumber() {

    return 50;

}

async function showNumber() {

    const number = await getNumber();

    console.log(number);

}

showNumber();
Output
50
Even though getNumber() returns 50, JavaScript automatically wraps it inside a Promise.
Waiting for Multiple Promises
Suppose you have two asynchronous tasks.
async function example() {

    const user = await getUser();

    const orders = await getOrders();

    console.log(user);

    console.log(orders);

}
Each await waits for its Promise before moving on.
Common Beginner Mistakes
Mistake 1
Using await outside an async function.
Wrong
const data = await fetchData();
Correct
async function loadData() {

    const data = await fetchData();

}
Mistake 2
Thinking await stops the whole program.
Wrong.
It pauses only the current async function.
Other JavaScript code can continue running.
Mistake 3
Thinking async removes Promises.
Wrong.
Promises still exist.
async and await are simply a cleaner way to use them.
Promise Chain vs Async/Await
Promise Chaining
login()

    .then(function () {

        return loadProfile();

    })

    .then(function () {

        return loadSettings();

    })

    .then(function () {

        console.log("Done");

    });
Async/Await
async function start() {

    await login();

    await loadProfile();

    await loadSettings();

    console.log("Done");

}

start();
Both do the same thing.
The second version is often easier to read and maintain.
Summary
Today you learned:
async marks a function as asynchronous.
Every async function returns a Promise.
await waits for a Promise to settle.
await can only be used inside an async function.
async and await make asynchronous code easier to read.
