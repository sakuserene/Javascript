# Module 20 – Understanding .then(): Receiving the Result from a Promise
Congratulations!
In the previous lesson, you learned how to create a Promise.
You also noticed something interesting.
When you write:
const promise = new Promise(function (resolve) {
    resolve("Hello");
});
nothing is printed on the screen.
Why?
Because resolve() does not print anything.
It simply stores the value inside the Promise.
Today's lesson is about learning how to retrieve that stored value.
We do this using .then().
Lesson Objectives
By the end of this lesson, you will understand:
What .then() is.
Why .then() exists.
How .then() receives data from resolve().
Why the function inside .then() is a callback.
How JavaScript knows when to execute .then().
Why Do We Need .then()?
Imagine you order a pizza.
You place the order.
The restaurant tells you:
"We'll call you when your pizza is ready."
You don't immediately receive the pizza.
Instead, you continue with your day.
Later...
The pizza is ready.
The restaurant calls you.
You go and collect it.
Flow:
Order Pizza
      ↓
Wait
      ↓
Pizza Ready
      ↓
Restaurant Calls
      ↓
Collect Pizza
That phone call is exactly like .then().
.then() waits until the Promise finishes successfully.
Basic Syntax
promise.then(function (result) {

});
Let's understand every part.
Part 1
promise
This is the Promise object.
Inside it is a value that will become available later.
Part 2
.then()
Read it like this:
"When this Promise succeeds, do something."
Think of .then() as saying:
"I'll wait."
Part 3
function (result) {

}
Does this look familiar?
It should.
This is a callback function.
Just like:
numbers.forEach(function (number) {

});
or
button.addEventListener("click", function () {

});
The function passed into .then() is also a callback.
The only difference is when JavaScript executes it.
Your First Example
const promise = new Promise(function (resolve) {
    resolve("Hello JavaScript!");
});

promise.then(function (result) {
    console.log(result);
});
Output
Hello JavaScript!
Let's Follow the Execution Step by Step
Step 1
JavaScript creates the Promise.
const promise = new Promise(...);
State:
Pending
Step 2
JavaScript executes the executor function.
resolve("Hello JavaScript!");
Now the Promise changes.
Pending
   ↓
Fulfilled
The Promise stores:
"Hello JavaScript!"
Step 3
JavaScript reaches:
promise.then(function (result) {
    console.log(result);
});
Since the Promise has already been fulfilled...
JavaScript automatically executes the callback.
It takes the stored value:
"Hello JavaScript!"
and places it inside the parameter named result.
Flow:
resolve("Hello JavaScript!")
           ↓
Promise Stores Value
           ↓
.then()
           ↓
Callback Executes
           ↓
result = "Hello JavaScript!"
           ↓
console.log(result)
Output
Hello JavaScript!
Where Does result Come From?
Look carefully.
resolve("Welcome");
Later...
promise.then(function (result) {
    console.log(result);
});
Question:
Who created result?
Answer:
JavaScript.
You never assign anything yourself.
JavaScript automatically passes whatever you give to resolve() into the callback parameter.
Think of it like this:
resolve("Welcome")
        │
        ▼
Promise stores "Welcome"
        │
        ▼
.then(function(result){
        │
        ▼
result = "Welcome"
Changing the Parameter Name
The parameter does not have to be named result.
Example
promise.then(function (data) {
    console.log(data);
});
Or
promise.then(function (message) {
    console.log(message);
});
Or
promise.then(function (user) {
    console.log(user);
});
These all work perfectly.
Why?
Because parameter names are just variable names.
Example Using a Number
const promise = new Promise(function (resolve) {
    resolve(500);
});

promise.then(function (price) {
    console.log(price);
});
Output
500
Example Using an Object
const promise = new Promise(function (resolve) {
    resolve({
        name: "Adi",
        age: 25
    });
});

promise.then(function (user) {
    console.log(user.name);
    console.log(user.age);
});
Output
Adi
25
Example Using an Array
const promise = new Promise(function (resolve) {
    resolve(["Apple", "Banana", "Orange"]);
});

promise.then(function (fruits) {

    fruits.forEach(function (fruit) {
        console.log(fruit);
    });

});
Output
Apple
Banana
Orange
Notice something interesting.
Inside .then(), we're using another callback.
The callback inside forEach().
Callbacks can be used together.
Real-World Example
Suppose you're buying a laptop online.
Flow
Place Order
      ↓

      ↓
Payment Successful
      ↓
Generate Receipt
      ↓
Receive Receipt
JavaScript version
paymentPromise.then(function (receipt) {
    console.log(receipt);
});
The receipt only becomes available after payment succeeds.
Another Example
Imagine exam results.
Flow

     ↓
Teacher Marks Papers
     ↓
Results Ready
     ↓
Student Receives Grade
JavaScript version

.then() Still Uses Callbacks
Earlier you learned:
function run(callback) {
    callback();
}
Now compare that with:
promise.then(function (result) {
    console.log(result);
});
Both receive a callback.
The difference is:
run() executes its callback immediately.
.then() waits until the Promise succeeds.
Execution Flow
Create Promise
      ↓
Pending
      ↓
resolve("Hello")
      ↓
Fulfilled
      ↓
.then()
      ↓
Callback Executes
      ↓
Print Result
Common Beginner Mistakes
Mistake 1
Thinking .then() executes immediately.
Wrong.
It waits for the Promise to become Fulfilled.
Mistake 2
Thinking resolve() prints something.
Wrong.
resolve("Hello");
This stores the value.
It does not display it.
Mistake 3
Thinking the parameter must be called result.
Wrong.
These are all correct.
.then(function (result) {

});
.then(function (data) {

});
.then(function (user) {

});
.then(function (anything) {

});
The parameter name is entirely your choice.
Summary
Today you learned:
.then() receives the successful value from a Promise.
Whatever you pass into resolve() becomes the parameter inside .then().
The function inside .then() is a callback.
.then() executes only after the Promise becomes Fulfilled.
