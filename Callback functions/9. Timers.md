# Module 9 — Timers: setTimeout() and setInterval()
This module introduces one of the most important ideas in JavaScript:
A callback doesn't have to run immediately. It can run later.
This is what makes JavaScript feel responsive and allows it to perform tasks without freezing the page.
Imagine This Situation
You order a pizza.
The cashier says:
"Your pizza will be ready in 20 minutes."
Do you:
❌ Stand there staring at the oven?
Or do you:
✅ Go shopping, chat with friends, or do something else until it's ready?
Most people do other things.
When the pizza is ready, the restaurant calls your number.
That call is the callback.
Why Do Timers Exist?
Suppose you want to display a welcome message 3 seconds after a page loads.
Without timers, JavaScript would only know how to run it immediately.
Timers allow you to say:
"Run this function later."
setTimeout()
setTimeout() runs a callback once, after a specified delay.
Syntax
setTimeout(callback, delayInMilliseconds);
Example:
setTimeout(function () {
    console.log("Hello after 3 seconds!");
}, 3000);
Output (after 3 seconds)
Hello after 3 seconds!
Understanding 3000
JavaScript measures time in milliseconds.

JavaScript doesn't stop while waiting.

setInterval()
Unlike setTimeout(), which runs once,
setInterval() runs over and over again until you stop it.
Syntax
setInterval(callback, delayInMilliseconds);
Example:
setInterval(function () {
    console.log("Tick");
}, 1000);
Output:
Tick
Tick
Tick
Tick
Tick
...
It prints every second.
Real-World Example 3 — Digital Clock
A digital clock updates every second.
setInterval(function () {
    console.log(new Date().toLocaleTimeString());
}, 1000);
Every second, the callback runs and displays the current time.

Stopping setInterval()
setInterval() keeps running until you stop it.
const timerId = setInterval(function () {
    console.log("Running...");
}, 1000);

setTimeout(function () {
    clearInterval(timerId);
    console.log("Stopped!");
}, 5000);
Output
Running...
Running...
Running...
Running...
Running...
Stopped!
Explanation:
setInterval() runs every second.
After 5 seconds, another timer (setTimeout) calls clearInterval().
The interval stops.
Real-World Example 4 — Countdown
let count = 5;

const timer = setInterval(function () {
    console.log(count);

    count--;

    if (count === 0) {
        console.log("Go!");
        clearInterval(timer);
    }
}, 1000);
Output:
5
4
3
2
1
Go!
Why Are These Callbacks?
Look carefully.
setTimeout(function () {
    console.log("Hello");
}, 2000);
You're not calling the function.
You're passing it to setTimeout().
setTimeout() stores it and calls it later.
Exactly like:
run(sayHello);
The difference is:
run() decides when to call it.
setTimeout() decides when to call it.
addEventListener() lets the browser decide when to call it.
Common Beginner Mistakes
Mistake 1: Calling the Function Immediately
Wrong:
setTimeout(sayHello(), 2000);
This executes sayHello() immediately.
Correct:
setTimeout(sayHello, 2000);
Pass the function, don't call it.
Mistake 2: Confusing Seconds and Milliseconds
Wrong:
setTimeout(callback, 5);
This waits 5 milliseconds, not 5 seconds.
Correct:
setTimeout(callback, 5000);
