# Introduction

Every loop asks one question repeatedly:

```text
Should I continue?
```

If the answer never becomes **false**, the loop never ends.

This is called an **infinite loop**.

Professional developers rarely debug loops by guessing.

Instead, they ask:

1. What changes every iteration?
2. What never changes?
3. Will the condition eventually become false?

# Loop Invariants

This is a computer science concept that experienced programmers use constantly.

## Definition

A **loop invariant** is something that remains true before and after every iteration.

Example

```js
let total = 0;

for (let i = 0; i < 5; i++) {
    total += i;
}
```

Notice:

```text
i always increases.

‚Üì

total always stores the sum of all processed values.

‚Üì

The condition depends only on i.
```

Those facts remain true throughout the loop.

Understanding invariants makes debugging much easier.

# Debugging Loops Like a Professional

Professional developers don't start by changing code.

They trace the loop.

Imagine this loop.

```js
let i = 1;

while (i <= 8) {
    console.log(i);

    i += 2;
}
```

Create a table.

| Iteration | i Before | Condition | Printed | i After |
|-----------|---------:|-----------|---------|---------:|
| 1 | 1 | true | 1 | 3 |
| 2 | 3 | true | 3 | 5 |
| 3 | 5 | true | 5 | 7 |
| 4 | 7 | true | 7 | 9 |
| 5 | 9 | false | ‚Äî | Stop |

Notice how easy bugs become to spot.

# Common Mistakes

| Mistake | Why It Happens | Fix |
|----------|----------------|-----|
| Forgetting the update | Condition never changes | Update the controlling variable |
| Updating in the wrong direction | Value moves away from the stopping condition | Move toward the exit condition |
| Wrong comparison operator | Condition stays true longer than expected | Verify the comparison carefully |
| Updating a different variable | The condition variable never changes | Update the same variable used in the condition |

---

# Performance

| Operation | Time | Space |
|-----------|------|-------|
| One iteration | O(1) + body | O(1) |
| n iterations | O(n) | O(1) |

> **Warning**
>
> An infinite loop has **no finite time complexity** because it never finishes.

---

# Best Practices

- Before running a loop, ask:
  - What changes?
  - What stays the same?
  - When will the condition become false?
- Draw a trace table for complex loops.
- Keep update logic simple.
- Avoid modifying the loop variable in multiple places.

> **Tip**
>
> If you cannot explain why a loop will stop, don't run it yet. Trace it first.

# Summary

A loop ends only when its condition becomes false. To determine whether this will happen, you must analyze how the loop variable changes over time. By tracing memory, understanding progress toward the stopping condition, and identifying loop invariants, you can reason about loop behavior without relying on trial and error.

---

# Key Takeaways

- Infinite loops occur when the stopping condition is never reached.
- Every loop should make measurable progress toward termination.
- The update expression must move the controlling variable toward making the condition false.
- Loop invariants describe what remains true throughout the loop.
- Trace tables are one of the most effective debugging tools.

---

# Glossary

| Term | Definition |
|------|------------|
| Infinite Loop | A loop that never reaches its stopping condition. |
| Termination | The point where a loop exits because its condition is false. |
| Loop Invariant | A property that remains true throughout every iteration. |
| Trace Table | A table showing variable values at each iteration. |
| Progress | A change that moves the loop closer to termination. |
