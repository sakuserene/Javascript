# Introduction

Every loop has one job:

```text
Repeat until the condition becomes false.
```

But here's the question:

> **What makes the condition eventually become false?**

The answer is:

> **The update expression.**

Without an update, most loops never stop.

---

# Mental Model

Imagine climbing stairs.

```text
Step 0

↓

Step 1

↓

Step 2

↓

Step 3

↓

Destination
```

If you never move to the next step...

```text
Step 0

↓

Step 0

↓

Step 0

↓

Step 0

↓

Forever...
```

That's an infinite loop.

---

# Theory

Consider:

```js
let i = 0;

while (i < 5) {
    console.log(i);
    i++;
}
```

The loop stops because `i` changes.

Without:

```js
i++;
```

the condition remains:

```text
0 < 5

↓

true

↓

Forever
```

---

# Why Loops Need Updates

Every iteration should move the loop closer to its stopping point.

Think of it as a countdown.

```text
Start

↓

Condition

↓

Body

↓

Update

↓

Closer to Finish
```

If the update doesn't affect the condition, the loop may never end.

# Postfix vs Prefix

This is one of the most misunderstood topics in JavaScript.

There are two increment operators:

```js
i++
```

and

```js
++i
```

Both increase `i` by **1**.

The difference is **when** the new value is returned.

---

## Postfix (`i++`)

1. Return the current value.
2. Then increment.

```js
let i = 5;

console.log(i++);
```

Execution

```text
Current value

5

↓

Print

5

↓

Increment

6
```

Final memory

```text
i → 6
```

Output

```text
5
```

---

## Prefix (`++i`)

1. Increment first.
2. Return the new value.

```js
let i = 5;

console.log(++i);
```

Execution

```text
Increment

6

↓

Print

6
```

Memory

```text
i → 6
```

Output

```text
6
```

---

## Comparison

| Expression | Printed | Final Value |
|------------|---------|-------------|
| `i++` | Old value | Incremented |
| `++i` | New value | Incremented |

# Internal Mechanics

Consider:

```js
i++;
```

The engine performs something like this:

```text
Read i

↓

Current Value

↓

Store Current Value (if needed)

↓

Add 1

↓

Write Back

↓

Return Result
```

For:

```js
++i
```

the order changes:

```text
Read i

↓

Add 1

↓

Write Back

↓

Return Updated Value
```

This distinction matters when the expression is used inside another expression, such as `console.log(i++)`.

# Common Mistakes

| Mistake | Why It Happens | Fix |
|----------|----------------|-----|
| Confusing `i++` with `++i` | Both increment, but return different values | Understand the order of operations |
| Forgetting to update the loop variable | The condition never changes | Always verify the update affects the condition |
| Updating by the wrong amount | The loop skips or repeats values | Choose an update that matches your algorithm |
| Using `i *= 2` unintentionally | Multiplication grows values quickly | Trace updates manually before running the code |

---

# Performance

| Operation | Time | Space |
|-----------|------|-------|
| `++` | O(1) | O(1) |
| `--` | O(1) | O(1) |
| `+=` | O(1) | O(1) |
| `-=` | O(1) | O(1) |
| `*=` | O(1) | O(1) |
| `/=` | O(1) | O(1) |

# Best Practices

- Use `i++` or `i--` for simple counting.
- Use `+=` or `-=` when skipping values.
- Prefer `i++` in loops unless you specifically need the updated value immediately.
- Ensure the update moves the loop toward its stopping condition.
- Trace updates on paper when debugging.

> **Tip**
>
> In a standard `for` loop, `i++` and `++i` usually behave the same because the returned value is ignored. The difference becomes important when the expression's result is used, such as in assignments or function calls.

# Summary

Update expressions are the engine that drives a loop forward. They modify variables so that the loop's condition eventually becomes `false`. JavaScript offers multiple update operators, each suited to different scenarios. Understanding how they work internally—and the distinction between prefix and postfix operators—will help you write predictable and efficient loops.

---

# Key Takeaways

- Loops rely on updates to make progress.
- `i++` is equivalent to `i = i + 1`.
- `i--` is equivalent to `i = i - 1`.
- `+=`, `-=`, `*=`, `/=`, and `%=` are shorthand assignment operators.
- `i++` returns the old value before incrementing.
- `++i` increments first, then returns the new value.
- In most `for` loops, `i++` and `++i` produce the same behavior because the returned value is ignored.

---

# Glossary

| Term | Definition |
|------|------------|
| Increment | Increase a value, usually by one. |
| Decrement | Decrease a value, usually by one. |
| Prefix Operator | Updates the value before returning it. |
| Postfix Operator | Returns the current value before updating it. |
| Compound Assignment | A shorthand that combines an operation with assignment (e.g., `+=`). |
