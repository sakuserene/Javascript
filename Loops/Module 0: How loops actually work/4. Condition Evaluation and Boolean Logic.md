# Introduction

Every loop asks one question repeatedly:

```text
Should I continue?
```

The answer must always become one of two values.

```text
true

or

false
```

There is no third option.

If the answer is:

```text
true
```

the loop continues.

If the answer is:

```text
false
```

the loop stops.

Everything else in a loop depends on this decision.

---

# Mental Model

Imagine a security checkpoint.

```text
             Enter?
                │
        ┌───────┴────────┐
        │                │
      YES              NO
        │                │
 Continue Loop       Exit Loop
```

The loop body never decides whether to continue.

Only the **condition** does.

---

# Theory

Consider:

```js
let i = 0;

while (i < 5) {
    console.log(i);
    i++;
}
```

Every iteration starts with the same question.

```text
Is i less than 5?
```

Iteration 1

```text
0 < 5

↓

true
```

Iteration 2

```text
1 < 5

↓

true
```

Iteration 3

```text
2 < 5

↓

true
```

Eventually

```text
5 < 5

↓

false
```

Loop stops.

# How JavaScript Evaluates Conditions

Suppose JavaScript sees:

```js
i < 5
```

Internally it performs something similar to:

```text
Read value of i

↓

Read value of 5

↓

Compare

↓

Produce Boolean
```

Notice something important.

The condition does **not** change memory.

It only **reads** memory.

---

## Memory Example

```js
let i = 2;
```

Memory

```text
+---------+
| i | 2   |
+---------+
```

Condition

```js
i < 5
```

JavaScript reads memory.

```text
Read i

↓

2

↓

Compare

↓

2 < 5

↓

true
```

Memory is unchanged.

# Truthy and Falsy Values

A loop condition doesn't have to return `true` or `false` explicitly.

JavaScript converts values to booleans when necessary.

---

# Logical Operators

Sometimes one comparison isn't enough.

---

## AND (`&&`)

Both conditions must be true.

```js

```

Diagram

```text
Condition A

TRUE

AND

Condition B

TRUE

↓

Overall

TRUE
```

If either side is false:

```text
FALSE
```

---

## OR (`||`)

Only one condition needs to be true.

```js
score >= 90 || bonus === true
```

Diagram

```text
TRUE

OR

FALSE

↓

TRUE
```

---

## NOT (`!`)

Reverses a boolean.

```js
!true
```

↓

```text
false
```

---

```js
!false
```

↓

```text
true
```

# Short-Circuit Evaluation

JavaScript avoids unnecessary work.

Example:

```js
false && something()
```

The engine already knows:

```text
FALSE

AND

Anything

↓

FALSE
```

There is no need to evaluate `something()`.

It stops immediately.

Likewise:

```js
true || something()
```

becomes:

```text
TRUE

OR

Anything

↓

TRUE
```

Again, JavaScript skips the second expression.

> **Tip**
>
> Short-circuit evaluation improves performance by avoiding unnecessary calculations.

# Internal Mechanics

When evaluating a loop condition, the engine roughly performs:

```text
Read Variables

↓

Evaluate Expression

↓

Convert Result to Boolean (if needed)

↓

true ?

↓

YES → Execute Body

NO → Exit Loop
```

Notice:

The condition only **reads** values.

The update expression changes them.

# Common Mistakes

| Mistake | Why It Happens | Fix |
|----------|----------------|-----|
| Using `=` instead of `===` | Assignment changes the variable instead of comparing it | Use comparison operators in conditions |
| Using `==` unintentionally | Type coercion can produce unexpected results | Prefer `===` and `!==` |
| Forgetting truthy/falsy conversion | JavaScript automatically converts values | Learn the falsy values |
| Assuming conditions modify variables | Conditions only read values | Updates happen separately |

---

# Performance

| Operation | Time | Space |
|-----------|------|-------|
| Read Variable | O(1) | O(1) |
| Comparison | O(1) | O(1) |
| Boolean Conversion | O(1) | O(1) |
| Logical Operator | O(1) | O(1) |

---

# Best Practices

- Prefer `===` over `==`.
- Keep conditions simple.
- Separate condition logic from update logic.
- Read conditions as English sentences.
- Remember that conditions never modify memory.

> **Warning**
>
> Avoid writing overly complex conditions inside loops. If the condition is difficult to read, consider breaking it into smaller variables with descriptive names.

# Summary

A loop continues only while its condition evaluates to a truthy value. The JavaScript engine repeatedly reads values from memory, evaluates the condition, converts the result to a boolean if necessary, and decides whether to continue or exit. Understanding boolean logic and truthy/falsy values is essential for writing correct loops.

---

# Key Takeaways

- Every loop depends on a boolean condition.
- Conditions read memory; they do not modify it.
- JavaScript converts non-boolean values to `true` or `false` when needed.
- Only eight values are falsy.
- `&&`, `||`, and `!` allow more expressive conditions.
- Short-circuit evaluation can avoid unnecessary work.

---

# Glossary

| Term | Definition |
|------|------------|
| Boolean | A value that is either `true` or `false`. |
| Truthy | A value that converts to `true`. |
| Falsy | A value that converts to `false`. |
| Comparison Operator | An operator that compares two values. |
| Logical Operator | An operator that combines or negates boolean expressions. |
| Short-Circuit Evaluation | Stopping evaluation as soon as the final result is known. |
