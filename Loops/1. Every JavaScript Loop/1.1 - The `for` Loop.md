# Introduction

Suppose someone asks you:

> Print the numbers from **1 to 10**.

Without loops, you might write:

```js
console.log(1);
console.log(2);
console.log(3);
console.log(4);
console.log(5);
console.log(6);
console.log(7);
console.log(8);
console.log(9);
console.log(10);
```

This works.

But imagine printing:

- 1 to 100
- 1 to 10,000
- Every student in a school
- Every product in an inventory
- Every task in a task manager

Writing one statement for each item would be impossible.

The solution is **repetition**.

The `for` loop lets us describe repetition instead of writing repeated code.

---

# Why Does the `for` Loop Exist?

The `for` loop exists because programmers often know **before the loop starts**:

- where to begin,
- when to stop,
- and how to move to the next step.

Imagine climbing stairs.

```text
Step 1

↓

Step 2

↓

Step 3

↓

Step 4

↓

Destination
```

You know:

- where you're starting,
- your destination,
- and how you'll move.

A `for` loop describes exactly that.

# Anatomy of the `for` Loop

General syntax:

```js
for (initialization; condition; update) {
    // body
}
```

Let's separate the parts.

```js
for (
    let i = 0;     // Initialization
    i < 5;         // Condition
    i++            // Update
) {
    console.log(i); // Body
}
```

The four parts are:

1. Initialization
2. Condition
3. Body
4. Update

You already learned these in Module 0.

Now we'll see how the `for` loop combines them.

---

# Internal Mechanics

The JavaScript engine **does not** treat a `for` loop as magic.

Conceptually, this:

```js
for (let i = 0; i < 3; i++) {
    console.log(i);
}
```

behaves similarly to:

```js
let i = 0;

while (i < 3) {
    console.log(i);
    i++;
}
```

The `for` loop is simply a compact way of writing:

- initialize,
- check,
- execute,
- update,
- repeat.

---

# Syntax

Basic form:

```js
for (initialization; condition; update) {
    // statements
}
```

Example:

```js
for (let i = 0; i < 5; i++) {
    console.log(i);
}
```

# How the Engine Executes a `for` Loop

Let's simulate:

```js
for (let i = 0; i < 3; i++) {
    console.log(i);
}
```

---

## Step 1 — Initialization

```js
let i = 0;
```

Memory:

```text
+---------+
| i | 0   |
+---------+
```

Runs **once**.

---

## Step 2 — Condition

```text
0 < 3

↓

true
```

---

## Step 3 — Body

```js
console.log(i);
```

Output:

```text
0
```

---

## Step 4 — Update

```js
i++;
```

Memory:

```text
i → 1
```

---

## Step 5

The Program Counter jumps back to the condition.

```text
1 < 3

↓

true
```

Repeat.

---

## Final Check

Eventually:

```text
i → 3
```

Condition:

```text
3 < 3

↓

false
```

Exit.

# Example 1

Print numbers from 1 to 5.

```js
for (let i = 1; i <= 5; i++) {
    console.log(i);
}
```

Output:

```text
1
2
3
4
5
```

---

## Line-by-Line Explanation

```js
let i = 1;
```

Create loop variable.

---

```js
i <= 5
```

Continue while `i` is less than or equal to `5`.

---

```js
console.log(i);
```

Print current value.

---

```js
i++;
```

Move to the next number.

---

# Example 2

Count backwards.

```js
for (let i = 5; i >= 1; i--) {
    console.log(i);
}
```

Output:

```text
5
4
3
2
1
```

Notice:

We changed:

- Condition
- Update

The loop structure stayed exactly the same.

---

# Example 3

Skip numbers.

```js
for (let i = 0; i <= 20; i += 5) {
    console.log(i);
}
```

Output:

```text
0
5
10
15
20
```

Instead of adding **1**, we add **5**.

# Common Mistakes

| Mistake | Why it Happens | Fix |
|----------|----------------|-----|
| Forgetting `i++` | The loop variable never changes | Always ensure the update moves toward the stopping condition |
| Using the wrong comparison operator | The loop executes too many or too few times | Check whether you need `<`, `<=`, `>`, or `>=` |
| Starting from the wrong value | Initialization is incorrect | Decide the first value before writing the loop |
| Modifying `i` inside the body unexpectedly | The loop becomes difficult to reason about | Keep updates in the update expression whenever possible |

---

# Performance

If the loop runs `n` times:

| Operation | Time | Space |
|-----------|------|-------|
| `for` loop | O(n) | O(1) |

Each iteration performs constant work unless the body contains more expensive operations.

---

# Best Practices

- Use meaningful variable names when appropriate (`index`, `row`, `column`).
- Keep initialization, condition, and update simple.
- Avoid changing the loop variable inside the body unless necessary.
- Trace the first few iterations manually when debugging.

> **Tip**
>
> Read a `for` loop in this order:
>
> 1. Where does it start?
> 2. When does it stop?
> 3. How does it move?

# Summary

The `for` loop combines initialization, condition checking, updating, and repeated execution into a single, readable structure. Internally, it behaves much like a `while` loop, but it groups the loop's control logic in one place, making it easier to read and maintain when the number of iterations is known.

---

# Key Takeaways

- A `for` loop is ideal when you know how iteration should start, stop, and progress.
- The four essential parts are initialization, condition, body, and update.
- The initialization runs once.
- The condition is checked before every iteration.
- The update runs after every successful iteration.
- The Program Counter jumps back to the condition after each update.

---

# Glossary

| Term | Definition |
|------|------------|
| `for` Loop | A control structure for repeated execution with initialization, condition, and update. |
| Iteration | One complete execution of the loop body. |
| Initialization | The one-time setup before the loop begins. |
| Update Expression | The operation that changes the loop variable after each iteration. |
