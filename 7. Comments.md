# Lesson 7: Comments

## Phase 1: Programming Fundamentals

## Learning Objectives

By the end of this lesson, you will be able to:

- Explain what comments are.
- Write single-line and multi-line comments.
- Know when comments are helpful.
- Avoid unnecessary comments.
- Improve code readability with meaningful documentation.

---

# What Are Comments?

Comments are notes for humans. JavaScript ignores them when executing code.

## Single-line Comments

```javascript
// This is a single-line comment
console.log("Hello!");
```

## Multi-line Comments

```javascript
/*
This is
a multi-line
comment.
*/
console.log("Hello!");
```

## Why Use Comments?

- Explain complex logic.
- Document assumptions.
- Leave useful reminders.
- Describe algorithms.

## Avoid

```javascript
// Print Hello
console.log("Hello");
```

Instead, write self-explanatory code and comment only when necessary.

## Best Practices

- Explain *why*, not just *what*.
- Keep comments accurate.
- Remove outdated comments.
- Prefer readable code over excessive comments.

## Mini Quiz

1. What is a comment?
2. Does JavaScript execute comments?
3. Which symbol starts a single-line comment?
4. How do you write a multi-line comment?
5. Why should comments stay up to date?

## Practice Mode

### 10 Practice Questions

1. Define a comment.
2. Why are comments useful?
3. What does `//` mean?
4. What do `/*` and `*/` do?
5. Should every line have a comment?
6. Why avoid outdated comments?
7. Should comments explain "why" or "what"?
8. Will comments affect output?
9. Give an example of a useful comment.
10. How do comments improve teamwork?

### 5 Coding Exercises

1. Add a single-line comment.
2. Add a multi-line comment.
3. Comment three program steps.
4. Replace an unnecessary comment with a better variable name.
5. Write an algorithm as comments, then one `console.log()`.

### Real-world Challenge

Create a welcome program that includes:
- A multi-line header comment.
- Three single-line comments.
- A welcome message.

## Summary

- Comments are ignored by JavaScript.
- `//` creates single-line comments.
- `/* ... */` creates multi-line comments.
- Good comments explain intent.
- Readable code reduces the need for comments.
