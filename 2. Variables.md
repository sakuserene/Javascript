# Lesson 2: Variables

## Phase 1: Programming Fundamentals

## Learning Objectives

By the end of this lesson you will be able to:

- Explain what a variable is.
- Understand why variables are useful.
- Declare variables with `let` and `const`.
- Assign and read values.
- Choose meaningful variable names.
- Recognize common beginner mistakes.

---

## What Is a Variable?

A **variable** is a named place in memory that stores a value.

Think of a variable as a **labeled storage box**.

- The label is the variable name.
- The contents are the value.

Instead of remembering a value yourself, you ask JavaScript to remember it.

### Real-world analogy

Imagine a kitchen.

Instead of saying:

> "Use the flour in the blue container on the second shelf..."

You simply label it:

> **Flour**

Variables work the same way.

---

## Visual Mental Model

```text
score ─────► 95
name  ─────► "Alice"
```

The name points to a stored value.

---

## Declaring Variables

### Using `let`

```javascript
let age = 25;
console.log(age);
```

Output:

```text
25
```

`let` creates a variable whose value can change later.

### Using `const`

```javascript
const country = "Kenya";
console.log(country);
```

`const` creates a variable that cannot be reassigned.

---

## Updating Variables

```javascript
let score = 10;

score = 20;

console.log(score);
```

Output:

```text
20
```

This is called **reassignment**.

---

## Why Use Variables?

Without variables:

```javascript
console.log(5 + 5);
console.log(5 + 5);
```

With variables:

```javascript
const price = 5;

console.log(price + price);
```

Variables make code easier to understand and update.

---

## Naming Variables

Good names:

```javascript
let firstName;
let totalPrice;
let isLoggedIn;
```

Poor names:

```javascript
let x;
let abc;
let thing;
```

Choose names that explain their purpose.

---

## Common Beginner Mistakes

1. Reassigning a `const`.

```javascript
const age = 20;
age = 21; // Error
```

2. Using a variable before declaring it.

3. Choosing unclear names.

---

## Best Practices

- Prefer `const` by default.
- Use `let` only when the value will change.
- Use descriptive camelCase names.
- Keep one purpose per variable.

---

## Mini Quiz

1. What is a variable?
2. When should you use `let`?
3. When should you use `const`?
4. What is reassignment?
5. Why are meaningful names important?

---

# Practice Mode

## 10 Practice Questions

1. Define a variable in your own words.
2. Why not hard-code repeated values?
3. What does `let` do?
4. What does `const` do?
5. Can a `const` be reassigned?
6. Which is better: `userAge` or `x`?
7. Why?
8. What is camelCase?
9. Predict:

```javascript
let n = 3;
n = 8;
console.log(n);
```

10. Predict:

```javascript
const city = "Nairobi";
console.log(city);
```

---

## Coding Exercises

1. Create a `const` named `firstName` and print it.
2. Create a `let` named `score`, assign 0, then change it to 100.
3. Create variables for your favorite food and color, then print both.
4. Fix this code:

```javascript
const age = 18;
age = 19;
```

5. Create three descriptive variables representing a simple online store (`productName`, `price`, `quantity`) and print each.

---

## Real-world Challenge

Write a small JavaScript program representing a student.

Store:

- Name
- Age
- School
- Favorite subject

Print each value using `console.log()`.

---

## Summary

You learned:

- Variables store values.
- `let` allows reassignment.
- `const` prevents reassignment.
- Good names improve readability.
- Variables make programs flexible.
